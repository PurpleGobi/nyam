# 식당 기록 DB 작동 로직

---

## 핵심 구조: 3개 테이블의 역할 분리

```
restaurants   →  장소 정보 (이름, 주소, 좌표, 카테고리 등)
records       →  개인의 방문 기록 (평가, 한줄평, 사진, 가격 등)
bubble_shares →  버블에 공유된 기록 연결
```

**한 줄 요약**: `restaurants`는 식당 자체, `records`는 사람들의 기록, `bubble_shares`는 기록의 공유 경로.

---

## 1. restaurants — 식당은 딱 1열

하나의 식당은 `restaurants` 테이블에 **딱 1개 행**만 존재한다.

```
restaurants 테이블
┌──────┬──────────┬──────────┬────────┬───────┐
│  id  │   name   │ address  │  area  │ genre │
├──────┼──────────┼──────────┼────────┼───────┤
│ R001 │ 을지로골목집 │ 을지로3가... │ 을지로 │ 한식  │
│ R002 │ 트라토리아  │ 이태원...   │ 이태원 │ 양식  │
└──────┴──────────┴──────────┴────────┴───────┘
```

- 누가 몇 번 방문하든 식당 행은 늘어나지 않음
- 식당 자체의 고정 정보만 저장 (이름, 주소, 좌표, 카테고리, 가격대 등)

---

## 2. records — 개인 기록은 사람 × 방문 수만큼 생성

A, B, C 세 명이 "을지로골목집"에 방문하면:

```
records 테이블
┌──────┬─────────┬───────────┬───────┬───────┬──────┬──────────┬────────┐
│  id  │ user_id │ target_id │ axis_x│ axis_y│ scene│ comment  │ price  │
├──────┼─────────┼───────────┼───────┼───────┼──────┼──────────┼────────┤
│ rc01 │   A     │   R001    │  85   │  72   │ 회식 │ 삼겹살 미침 │ 25000 │
│ rc02 │   B     │   R001    │  70   │  90   │ 데이트│ 분위기 좋음│ 30000 │
│ rc03 │   C     │   R001    │  60   │  65   │ 혼밥 │ 가성비 굿  │ 12000 │
└──────┴─────────┴───────────┴───────┴───────┴──────┴──────────┴────────┘
```

- 같은 식당(R001)이지만 **사람마다 별도 행**
- `user_id`로 "누구의 기록인지", `target_id`로 "어떤 식당인지" 연결
- 같은 사람이 재방문하면 행이 또 추가됨 (A가 2번 가면 → rc01, rc04)

---

## 3. 상세 페이지 — 같은 식당, 다른 화면

같은 "을지로골목집" 상세 페이지라도 **보는 사람마다 내용이 다르다**.

### A가 보는 상세 페이지

```
┌─────────────────────────────────────┐
│ 🏠 을지로골목집                      │ ← restaurants 테이블 (R001)
│ 을지로3가 · 한식 · ₩₩               │
├─────────────────────────────────────┤
│ 내 기록 (2회 방문)                   │ ← records WHERE user_id=A AND target_id=R001
│                                     │
│ 📍 1차: 85/72 "삼겹살 미침" (회식)    │
│ 📍 2차: 80/78 "역시 맛있다" (친구)    │
│                                     │
│ 내 평균 만족도: 79                   │ ← 프론트에서 계산
│ 사분면에 점 2개 표시                  │
└─────────────────────────────────────┘
```

### C가 보는 상세 페이지

```
┌─────────────────────────────────────┐
│ 🏠 을지로골목집                      │ ← 같은 restaurants 데이터
│ 을지로3가 · 한식 · ₩₩               │
├─────────────────────────────────────┤
│ 내 기록 (1회 방문)                   │ ← records WHERE user_id=C AND target_id=R001
│                                     │
│ 📍 1차: 60/65 "가성비 굿" (혼밥)     │
│                                     │
│ 내 평균 만족도: 63                   │
│ 사분면에 점 1개 표시                  │
└─────────────────────────────────────┘
```

### D (방문 안 한 사람)가 보는 상세 페이지

```
┌─────────────────────────────────────┐
│ 🏠 을지로골목집                      │ ← 식당 기본 정보만
│ 을지로3가 · 한식 · ₩₩               │
├─────────────────────────────────────┤
│                                     │
│       아직 기록이 없어요              │ ← 빈 상태 + CTA
│     [ 첫 기록 남기기 ]               │
│                                     │
└─────────────────────────────────────┘
```

## 4. 버블 공유 — bubble_shares로 연결

A가 자신의 기록(rc01)을 "맛잘알 버블"에 공유하면:

```
bubble_shares 테이블
┌──────┬───────────┬───────────┐
│  id  │ record_id │ bubble_id │
├──────┼───────────┼───────────┤
│ bs01 │   rc01    │  BUB-01   │  ← A의 기록 → 맛잘알 버블
│ bs02 │   rc02    │  BUB-01   │  ← B의 기록 → 맛잘알 버블
└──────┴───────────┴───────────┘
```

이제 같은 버블 멤버인 C가 "을지로골목집" 상세 페이지를 열면:

```
┌─────────────────────────────────────┐
│ 🏠 을지로골목집                      │
├─────────────────── 내 기록 ──────────┤
│ 📍 60/65 "가성비 굿" (혼밥)          │ ← C 본인의 records
├──────────── 맛잘알 버블 기록 ─────────┤
│ 👤 A: 85/72 "삼겹살 미침"           │ ← bubble_shares → records 조인
│ 👤 B: 70/90 "분위기 좋음"           │
│                                     │
│ 사분면에 점 3개 (내 것 + 버블 기록)    │
└─────────────────────────────────────┘
```

## 5. 데이터 흐름 요약

```
[기록 생성]
  A가 을지로골목집 방문 → records에 1행 추가 (user_id=A, target_id=R001)
  사진 첨부 → record_photos에 N행 추가

[상세 페이지 조회]
  SELECT * FROM restaurants WHERE id = R001          → 식당 기본 정보
  SELECT * FROM records WHERE user_id=나 AND target_id=R001  → 내 기록들

[버블 공유]
  A가 버블에 공유 → bubble_shares에 1행 추가 (record_id=rc01, bubble_id=BUB-01)

[버블 기록 조회]
  SELECT r.* FROM records r
  JOIN bubble_shares bs ON bs.record_id = r.id
  JOIN bubble_members bm ON bm.bubble_id = bs.bubble_id
  WHERE bm.user_id = 나 AND r.target_id = R001      → 내 버블 멤버들의 기록
```

---

## 6. 핵심 포인트

| 개념 | 설명 |
|------|------|
| restaurants는 공유 자원 | 모든 사용자가 같은 식당 행을 참조 |
| records는 개인 소유 | 내 기록은 나만 기본 열람, RLS로 보호 |
| 상세 페이지는 합성 뷰 | 식당 정보 + 내 기록 + 버블 공유 기록을 조합해서 렌더링 |
| bubble_shares는 다리 | records와 bubbles를 연결하는 중간 테이블 |
| 같은 페이지, 다른 내용 | 로그인한 사용자에 따라 상세 페이지 내용이 완전히 달라짐 |
