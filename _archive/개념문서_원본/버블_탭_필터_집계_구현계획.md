# 버블 탭 필터 시스템 + 집계 데이터 구현 계획

## Context

홈 3탭(식당/와인/버블) 재구성 후, 버블 탭에도 식당/와인 탭과 동일한 필터 시스템이 필요하다. 현재 버블 탭은 카드 목록만 나열. 추가로, 버블별 멤버들의 축별(지역/장르/산지/품종) 전문성 데이터를 산출하여 버블 카드에 표시하고, 필터/탐색에도 활용해야 한다.

현재 상태:
- `xp_totals` 테이블에 개인별 축별 레벨 데이터 존재 (area, genre, wine_variety, wine_region)
- `bubble_members`에 멤버 통계 존재 (avg_satisfaction, member_unique_target_count 등)
- 버블별 축별 멤버 평균 레벨은 **미구현** — JOIN이 필요
- `BUBBLE_FILTER_ATTRIBUTES` 6개 이미 정의 (role, focus_type, area, join_policy, member_count, activity)
- `BubbleDiscoverSheet`에 추천/근처/인기/새로운 버블 데이터 존재

DB 원칙 (DB_v4 참조): Target 중심 구조. 불필요한 가상 row 없이 실제 데이터만 사용.

## Phase 1: 버블 전문성 집계 (DB + Domain)

### 1-1. DB View 생성

`supabase/migrations/049_bubble_expertise_view.sql`

```sql
CREATE VIEW bubble_expertise AS
SELECT
  bm.bubble_id,
  xt.axis_type,
  xt.axis_value,
  COUNT(DISTINCT bm.user_id) AS member_count,
  AVG(xt.level)::INT AS avg_level,
  MAX(xt.level) AS max_level,
  SUM(xt.total_xp) AS total_xp
FROM bubble_members bm
JOIN xp_totals xt ON xt.user_id = bm.user_id
WHERE bm.status = 'active'
  AND bm.role IN ('owner', 'admin', 'member')
GROUP BY bm.bubble_id, xt.axis_type, xt.axis_value;
```

RLS: view는 underlying 테이블의 RLS를 따름 (bubble_members + xp_totals).

### 1-2. Domain 엔티티

`src/domain/entities/bubble.ts`에 추가:

```typescript
export interface BubbleExpertise {
  axisType: 'area' | 'genre' | 'wine_variety' | 'wine_region'
  axisValue: string
  memberCount: number
  avgLevel: number
  maxLevel: number
}
```

### 1-3. Repository 인터페이스

`src/domain/repositories/bubble-repository.ts`에 추가:

```typescript
getExpertise(bubbleId: string): Promise<BubbleExpertise[]>
getExpertiseForBubbles(bubbleIds: string[]): Promise<Map<string, BubbleExpertise[]>>
```

### 1-4. Infrastructure 구현

`src/infrastructure/repositories/supabase-bubble-repository.ts`에 추가:
- `bubble_expertise` view 조회
- batch 조회 (여러 버블 한번에)

## Phase 3: 버블 탭 필터 시스템

### 3-1. 필터 속성 정의

`src/domain/entities/filter-config.ts`에 `HOME_BUBBLE_FILTER_ATTRIBUTES` 추가:

| 필터 | 타입 | 값 | 설명 |
|------|------|-----|------|
| **소속** | select | 내 버블 / 공개 전체 | 기본: 내 버블. 공개로 전환하면 탐색 모드 |
| **집중 유형** | select | 전체 / 식당 / 와인 | bubble.focusType |
| **역할** | select | 오너 / 멤버 | 내 버블일 때만 유효 |
| **전문 지역** | select | 강남, 을지로, ... | expertise에서 avg_level 기준 필터 |
| **전문 장르** | select | 일식, 한식, ... | expertise에서 avg_level 기준 필터 |
| **전문 산지** | select | France, Italy, ... | 와인 버블 탐색용 |
| **전문 품종** | select | Cabernet Sauvignon, ... | 와인 버블 탐색용 |
| **멤버 수** | threshold | 5+, 10+, 20+, 50+ | bubble.memberCount |
| **활동** | threshold | 24시간 / 1주 / 1개월 | bubble.lastActivityAt |

"전문 지역/장르/산지/품종" 필터 동작:
- 선택한 축에서 avg_level이 1 이상인 버블만 표시
- 정렬: 해당 축의 avg_level 높은 순

### 3-2. Home Container 버블 탭 UI

식당/와인 탭과 동일한 패턴:
- `ConditionFilterBar` + `AdvancedFilterSheet` 재사용
- `SortDropdown` 재사용 (정렬: 최근활동순 / 멤버많은순 / 기록많은순 / 이름순)
- 검색바 재사용 (버블 이름/설명/지역 검색)

### 3-3. 공개 버블 탐색

"소속" 필터에서 "공개 전체" 선택 시:
- 기존 `BubbleDiscoverSheet`의 데이터 소스 재사용 (`useBubbleDiscover`)
- 추천/근처/인기/새로운 대신 필터로 탐색
- `bubbleRepo.findPublic()` 활용
