# 와인 검색 로직

## 진입점 3가지 (모두 /add?type=wine)

| 경로 | 트리거 | 동작 |
|------|--------|------|
| 카메라 촬영 | 라벨 촬영 | Phase 1 → Phase 2 |
| 앨범에서 추가 | 갤러리 선택 | Phase 1 → Phase 2 |
| 이름으로 검색 | 검색창 입력 | 텍스트 검색만 (사진 없음) |

---

## Phase 1: 리사이즈 + 즉시 업로드 (카메라/앨범 공통)

```
유저가 사진 촬영 또는 앨범 선택
  │
  ├─ base64 → File 변환
  ├─ imageService.resizeImage() — 800px width, WebP, quality 0.7
  ├─ imageService.uploadImage(userId, 'pending', blob, uuid)
  │   → Storage: record-photos/{userId}/pending/{uuid}.webp
  ├─ 리사이즈된 blob → base64 변환 (Phase 2 검색용)
  └─ 기록 미완료 시 cleanup: unmount → imageService.deleteImage(tempUrl)
```

## Phase 2: AI 검색 (Gemini Vision)

```
리사이즈된 base64 (WebP, ~100-300KB)
  │
  ├─ POST /api/records/identify (targetType: 'wine')
  ├─ Gemini Vision (gemini-3-flash-preview) 호출
  │   → WINE_LABEL_PROMPT: 25개 필드 JSON 추출
  │
  ├─ upsertWineFromAI() — 3단계 퍼지 매칭으로 중복 체크
  │   ├─ 1단계: 정확한 이름 매칭 (+ vintage)
  │   ├─ 2단계: 핵심 단어 포함 매칭 (2개 이상 단어 매칭)
  │   ├─ 3단계: 생산자 + 빈티지 매칭
  │   └─ 모두 실패 → 새 와인 INSERT (AI 데이터 전체 저장)
  │
  ├─ 후보 있음 → 바로 record 페이지 이동
  └─ 인식 실패 → search 화면으로 폴백
```

## 이름 검색 (텍스트, 사진 없음)

```
유저가 검색창에 와인 이름 입력
  │
  ├─ [1차] DB 검색 — GET /api/wines/search?q=검색어
  │   → wines.name/producer ILIKE, 기록 있는 와인 우선 정렬
  │
  ├─ [2차] AI 검색 — POST /api/wines/search-ai
  │   → Gemini Text: WINE_SEARCH_PROMPT (최대 5개 후보)
  │
  ├─ 유저가 DB 결과 선택 → 바로 기록 폼
  └─ 유저가 AI 후보 선택
      → POST /api/wines/detail-ai → 25필드 상세 + upsert → 기록 폼
```

## 산지 4단계 Cascade

```
country → region → sub_region → appellation

예시:
  France → Burgundy → Côte de Nuits → Gevrey-Chambertin
  USA → California → Napa Valley → Rutherford
  Australia → South Australia → Barossa → Barossa Valley
  South Africa → Cape South Coast → Walker Bay → Hemel-en-Aarde Wards
```

WSET Level 3 공식 Specification 기준. 상세 목록: `development_docs/systems/DATA_MODEL.md` 참조.

- 15개국, 73 region, 198 sub_region
- 4단계(appellation)는 Burgundy, California, South Australia, Cape South Coast 4곳만 해당
- 코드 상수: `src/shared/constants/wine-meta.ts` (WINE_REGIONS, WINE_SUB_REGIONS, WINE_APPELLATIONS)

## 필드-제약조건 요약 (Gemini 프롬프트 기준)

```
wine_type:          ENUM 7종 — red | white | rose | sparkling | orange | fortified | dessert
country:            15개국 (France, Germany, Austria, Hungary, Greece, Italy, Spain, Portugal, USA, Canada, Chile, Argentina, South Africa, Australia, New Zealand)
region:             WINE_REGIONS[country] cascade
sub_region:         WINE_SUB_REGIONS[region] cascade
appellation:        WINE_APPELLATIONS[sub_region] cascade (해당 시)
variety:            WINE_VARIETIES_BY_TYPE[wine_type] cascade
body_level:         정수 1~5 (1=Light, 2=Medium-, 3=Medium, 4=Medium+, 5=Full)
acidity_level:      정수 1~3 (1=낮음, 2=보통, 3=높음)
sweetness_level:    정수 1~3 (1=Dry, 2=Medium, 3=Sweet)
grape_varieties:    [{name: string, pct: 0~100}] — pct 합계 = 100
abv:                소수점 1자리 (예: 13.5)
vivino_rating:      소수점 1자리 1.0~5.0
critic_scores:      {RP?: 50~100, WS?: 50~100, JR?: 12~20, JH?: 50~100}
reference_price:    정수, KRW (예: 45000)
vintage:            정수 연도 (예: 2020)
drinking_window:    정수 연도 (예: 2024, 2030)
```

## 모델 및 설정

- 모델: `gemini-3-flash-preview`
- temperature: 0 (결정론적 응답)
- Vision maxOutputTokens: 2048
- Text maxOutputTokens: 4096
- 이미지 리사이즈: 800px width, WebP, quality 0.7
- mime_type: base64 헤더에서 자동 감지 (WebP/PNG/JPEG)
