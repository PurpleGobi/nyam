## 현재 Discover 파이프라인 구조

```
사용자 입력 (자유텍스트 + 씬 + 장소 + 장르)
    │
    ▼
Step 1: LLM 랭킹 엔진 (Gemini)
    │  - 씬별 6축 가중치 프롬프트
    │  - 자유입력 쿼리가 최우선 검색 의도
    │  - 8~10개 후보 + 점수(100점) + confidence + category
    ▼
Step 2: 카카오맵 실존 검증
    │  - 병렬 키워드 검색 → fuzzy name matching
    │  - 검증 실패 시 fallback 키워드 검색 보충
    ▼
Step 3: 내부 DB + 사용자 DNA 로딩
    │  - 방문 이력, 블랙리스트, taste_dna, style_dna
    ▼
Step 4: LLM 기반 + DNA 매칭 보너스
    │  - LLM totalScore가 베이스 (0~100)
    │  - DNA 일치도 → ±15점 보너스/패널티
    │  - finalScore = clamp(llmScore + dnaBonus, 0, 100)
    ▼
Step 5: Top 5 선별 + 결과 패키징
```

### DNA 스코어링 상세

```
DNA overall (0~100) → matchRatio = (overall - 50) / 50  → -1.0 ~ +1.0
dnaBonus = round(matchRatio * 15)                        → -15 ~ +15
finalScore = clamp(llmScore + dnaBonus, 0, 100)
```

- DNA overall 50 = 중립 (보정 없음)
- DNA overall 80 = +9점 보너스
- DNA overall 20 = -9점 패널티
