## 1. 전체 화면 맵

```
App
├── 온보딩 (/onboarding)
│   ├── 가입 (소셜 로그인)
│   ├── 동네 선택
│   └── 가본 곳 체크
│
├── 탭 네비게이션 (하단 5탭)
│   ├── [홈] (/)
│   │   ├── 내 기록 타임라인
│   │   ├── 미평가 넛지 카드
│   │   ├── "오늘 뭐 먹지?" (접이식 추천)
│   │   ├── "와인 고르기" (접이식 추천)
│   │   └── 버블 피드 (P2)
│   │
│   ├── [탐색] (/discover)
│   │   ├── 지역별 식당/와인
│   │   ├── 권위 추천 (미슐랭/블루리본)
│   │   └── 취향 유사 추천 (P2+)
│   │
│   ├── [+ 기록] (모달/시트)
│   │   ├── 검색으로 등록
│   │   │   ├── 식당 검색 → 선택 → 평가
│   │   │   └── 와인 검색 → 선택 → 빈티지 → 평가
│   │   └── 사진으로 등록
│   │       ├── 음식 사진 → EXIF+AI → 식당 매칭 → 평가
│   │       └── 와인 라벨 → OCR → 와인 매칭 → 평가
│   │
│   ├── [버블] (/bubbles) — P2
│   │   ├── 버블 목록
│   │   ├── 버블 상세 (/bubbles/[id])
│   │   │   ├── 피드 탭
│   │   │   ├── 식당 목록 탭
│   │   │   ├── 와인 목록 탭
│   │   │   └── 통계 탭
│   │   └── 버블 생성
│   │
│   └── [나] (/profile)
│       ├── 프로필 (레벨, 뱃지, 사분면)
│       ├── 내 기록 전체
│       ├── 위시리스트
│       ├── 뱃지 컬렉션
│       └── 설정
│
├── 상세 페이지
│   ├── 식당 상세 (/restaurants/[id])
│   └── 와인 상세 (/wines/[id])
│
├── 기록 상세 (/records/[id])
│
└── 알림 (/notifications)
```

## 2. 핵심 사용자 플로우

### Flow A: 기록하기 (핵심 루프)
```
+ 버튼 → 검색/사진 선택 → 대상 특정 → 사분면 평가 → 상황 태그 → (선택: 한줄평/사진/메뉴) → 저장
```

### Flow B: 재방문 기록
```
식당 상세 → "재방문 기록" → 사분면 (이전 위치 반투명 표시) → 상황 태그 → 저장
```

### Flow C: 추천 → 기록
```
홈 "오늘 뭐 먹지?" → 상황 필터 → 식당 선택 → 상세 → 기록 or 찜
```

### Flow D: 사진 넛지 → 기록
```
앱 진입 → 갤러리 사진 감지 카드 → "기록하기" → EXIF+AI 매칭 → 사분면 → 저장
```

## 4. 구현 순서 (스프린트 계획 — P1+P2 통합)

> P2 테이블은 S1에서 전부 생성. P2 기능은 S7~S8에서 구현.
> 온보딩은 모든 기능 완성 후 마지막에 연결.

```
S1 ─── S2 ─── S3 ─┬─ S4
 (기반)  (코어)  (검색) │ (상세P1)
                    │
                    S5 ─── S6 ─── S7 ─── S8 ─── S9
                   (홈+추천) (XP+프로필) (버블) (통합) (온보딩+마무리)
```

| Sprint | 문서 | 산출물 | 선행 | 비고 |
|--------|------|--------|------|------|
| **S1** | DATA_MODEL + AUTH + DESIGN_SYSTEM | **P1+P2 전체** DB 스키마 (버블·소셜 테이블 포함), RLS, 인증, 디자인 토큰 | 없음 | P2 테이블도 이때 생성 → 나중에 마이그레이션 고통 방지 |
| **S2** | RATING_ENGINE + RECORD_FLOW | 사분면 UI, 아로마 팔레트, 제스처, 기록 저장 플로우 | S1 | 핵심 UX — 여기서 사분면 완성도가 앱 품질 결정 |
| **S3** | SEARCH_REGISTER | 검색 (Nyam DB → 외부 API), 식당/와인 등록, 사진 등록 (EXIF+AI) | S1, S2 | 검색→선택→S2 평가로 연결되는 풀 플로우 완성 |
| **S4** | RESTAURANT_DETAIL + WINE_DETAIL | 상세 페이지 L1~L8 (히어로, 평점, 내 기록, 사분면 분포, 실용 정보, 와인 링크) | S1, S2 | P2 소셜 레이어(L9)는 S8에서 추가 |
| **S5** | HOME + RECOMMENDATION | 홈 타임라인, 넛지 카드, 추천 알고리즘 7종 (재방문/상황/사분면/위시/권위/와인/지오펜스), Discover | S1~S4 | 추천은 기록 수 기반 점진 해금 |
| **S6** | XP_SYSTEM + PROFILE + SETTINGS | XP 계산, 레벨/뱃지, 프로필 화면, 위시리스트, 설정 전체 | S2 | 뱃지 소급 적용 포함 |
| **S7** | BUBBLE | 버블 생성/가입/초대, 공유 플로우, 피드(크로노), 댓글/리액션, 버블 통계, 알림, 역할/권한 | S1, S6 | P2 코어 — 버블 시스템 전체를 한 스프린트에 |
| **S8** | BUBBLE 통합 + FOLLOW | 상세 페이지 L9 (버블 멤버 기록), 홈 버블 피드, 기록→버블 공유, 팔로우, 공개 프로필 | S4, S5, S7 | 기존 P1 페이지에 P2 레이어 통합 |
| **S9** | ONBOARDING + 마무리 | 온보딩 풀 플로우 (가입→동네→식당→와인→홈), 넛지 시스템, 전체 플로우 검증, 빌드 | 전체 | 모든 기능 완성 후 온보딩 연결 |

## 5. 하단 탭 구성

| 탭 | 아이콘 | Phase 1 | Phase 2 |
|----|--------|---------|---------|

| 탐색 | Compass | 지역/권위 추천 | + 취향 유사 |
