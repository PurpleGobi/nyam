# Nyam v2 Information Architecture

> 화면 맵, 네비게이션 구조, 사용자 플로우, 화면 간 의존성
> **SSOT**: `prototype/*.html` 목업 기반, `pages/*.md` 문서와 동기화

---

## 1. 전체 화면 맵

```
App
├── 온보딩 (/onboarding)
│   ├── 인트로 (환영 메시지)
│   ├── Step 1 — 맛집 등록 (지역 선택 → 시드 식당 방문 표시)
│   ├── Step 2 — 버블 생성 (가족/친구/동료 템플릿)
│   └── Step 3 — 버블 탐색 (시드 버블 둘러보기 → 보너스 XP → 홈 이동)
│
├── 로그인 (/auth/login)
│   ├── nyam 로고 + 태그라인
│   ├── OAuth 버튼 (Google, Kakao, Naver, Apple)
│   └── 이용약관/개인정보처리방침 링크
│
├── 홈 (/) — 앱 메인 허브
│   ├── 앱 헤더 (nyam 로고 · bubbles · 🔔알림 · 레벨바 · 아바타 드롭다운)
│   │   ├── bubbles 텍스트 버튼 → /bubbles
│   │   ├── 🔔 → 알림 드롭다운 (인라인, 별도 페이지 아님)
│   │   ├── 레벨바 (레벨 뱃지 + XP 트랙)
│   │   └── 아바타 → 드롭다운 (프로필, 설정)
│   │
│   ├── AI 인사 (조건부, 5초 자동 소멸)
│   │
│   ├── 콘텐츠 탭 [식당 | 와인]
│   │   ├── 뷰 순환 버튼 (카드 → 리스트 → 캘린더)
│   │   ├── 지도 뷰 버튼 (식당만)
│   │   ├── 소팅 드롭다운
│   │   └── 검색 아이콘 → /discover 이동
│   │
│   ├── 조건 필터 바
│   │   ├── 필터 칩 (속성:값 쌍, 드래그 스크롤)
│   │   ├── + 추가 버튼 (속성 → 값 선택 팝오버)
│   │   ├── 고급 필터 시트
│   │   └── 인라인 페이저 (◄ 1/5 ►)
│   │
│   ├── 식당 탭
│   │   ├── 카드 뷰 (사진, 점수, 장르, 지역, 사분면)
│   │   ├── 리스트 뷰 (순위 뱃지, 썸네일, 이름, 점수, 방문수)
│   │   ├── 캘린더 뷰 (월별 — 날짜별 사진 도트 + 선택 시 상세)
│   │   ├── 지도 뷰 (카카오맵 — 방문 식당 커스텀 핀)
│   │   └── 통계 패널 (접이식 토글 — 세계지도, 장르 차트, 점수 분포, 월별 트렌드, 씬 차트)
│   │
│   ├── 와인 탭
│   │   ├── 카드/리스트/캘린더 뷰 (식당과 동일 구조)
│   │   └── 통계 패널 (산지 지도, 품종 차트, 점수 분포, 월별 트렌드, 와인 타입 차트)
│   │
│   └── FAB (+) → /add 페이지 이동 (type=restaurant|wine)
│
├── 추가 플로우 (/add) — 기록 대상 선택
│   ├── 카메라 캡처/앨범 (AI 인식 → 후보 선택)
│   ├── 검색 폴백 (이름 검색 + 근처 식당 GPS)
│   ├── 신규 등록 옵션
│   └── 성공 화면 (상세 추가 / 한 개 더 / 홈)
│
├── 검색 (/search)
│   ├── 검색바 (타입 토글: 식당|와인)
│   ├── 내부 DB + 외부 API 결과
│   ├── 근처 식당 (GPS 기반, 장르 필터, 반경 조절)
│   ├── 최근 검색 히스토리
│   └── 선택 → /record로 이동
│
├── 등록 (/register)
│   ├── 식당 등록 폼 (이름, 지역, 장르, 주소, GPS, 전화, 카카오맵 URL)
│   └── 와인 등록 폼 (이름, 생산자, 빈티지, 타입, 산지, 품종, 분류 등)
│
├── 기록 작성/수정 (/record)
│   ├── 사진 피커 (추가/삭제/순서변경/공개전환)
│   ├── 식당 기록 폼 (만족도, 사분면, 씬태그 6종, 방문일, 동행자, 코멘트, 가격대 등)
│   ├── 와인 기록 폼 (만족도, 사분면, 아로마 16섹터 3링, 품질 평가 BLIC, 코멘트 등)
│   ├── 버블 자동 동기화
│   └── 수정 모드 (기존 기록 프리필 + 삭제 옵션)
│
├── 탐색 (/discover) — AI 기반 식당 탐색
│   ├── 지역 + 장르 선택
│   ├── 식당 검색 결과
│   ├── AI 랭킹 (Top 30, 점수 브레이크다운)
│   ├── AI 추천 (자연어 쿼리)
│   └── AI 상세 분석 (한줄 평가, 맛, 분위기, 꿀팁, 가격대, 추천 메뉴)
│
├── 식당 상세 (/restaurants/[id])
│   ├── L1: 히어로 (사진 캐러셀, 찜/공유)
│   ├── L2: 기본 정보 (이름, 가격대, 국가›도시›지역›구, 장르 대›소)
│   │   └── 축 레벨 뱃지 (지역/장르별 경험치 레벨)
│   ├── L3: 점수 카드 2칸 (내 점수 · 버블 점수 — 버블 확장 패널)
│   ├── L4: 뱃지 행 (미슐랭, 블루리본, TV 출연)
│   ├── L5: 포지션 맵 (가격 × 분위기 사분면 — 평균/최근 모드 전환)
│   ├── L6: 내 기록 타임라인 (방문별 날짜, 사진, 코멘트, 점수, 연결 와인)
│   ├── L7: 버블 기록 (버블 필터 칩 + 멤버별 기록 카드)
│   ├── L8: 실용 정보 (주소+지도링크, 영업시간, 전화, 메뉴)
│   ├── DetailFab (좌: 뒤로, 우: 기록 추가)
│   └── FabActions (수정 · 공유 · 삭제 — 기록 있을 때만)
│
├── 와인 상세 (/wines/[id])
│   ├── L1: 히어로 (사진 캐러셀, 라벨 이미지, 찜/공유)
│   ├── L2: 기본 정보
│   │   ├── 와인명 + 타입칩 + 빈티지 + 생산자
│   │   ├── 적정가 + 가격 리뷰 (추천/비추천/대안)
│   │   ├── 분류/Vivino/RP/WS 등급 뱃지
│   │   ├── 산지 (국가›지역›서브지역) + 축 레벨 뱃지
│   │   ├── 품종 칩 (비율 포함) + 축 레벨 뱃지
│   │   ├── 바디 | 산미 | 당도 | 도수
│   │   ├── 서빙 온도, 디캔팅, 음용 적기
│   │   ├── 음식 페어링 태그
│   │   └── 테이스팅 노트
│   ├── L3: 점수 카드 2칸 (내 점수 · 버블 점수)
│   ├── L4: 내 기록 (사분면 + 아로마 휠 + 품질 평가 BLIC)
│   ├── L5: 기록 히스토리 (날짜, 사진, 점수, 코멘트, 연결 식당)
│   ├── DetailFab (좌: 뒤로, 우: 기록 추가)
│   └── FabActions (수정 · 공유 · 삭제 — 기록 있을 때만)
│
├── 프로필 (/profile)
│   ├── 프로필 헤더 (아바타, 닉네임, 맛 정체성 태그, 기록 수)
│   ├── Wrapped 공유 버튼 → /profile/wrapped
│   ├── 레벨/XP 카드 (총 레벨 + XP 바)
│   ├── 활동 개요 그리드
│   ├── 활동 히트맵 (총 기록, 연속, 활동 월)
│   ├── 최근 XP 리스트
│   ├── 통계 탭 [식당 | 와인]
│   │   ├── 식당: 지역 / 장르 서브탭
│   │   └── 와인: 산지 / 품종 서브탭
│   ├── 경험 레벨 (카테고리별 필터 + 상세 시트)
│   └── 레벨 상세 시트 (고유 수, 총 기록, 재방문, XP 분해, 마일스톤)
│
├── Wrapped (/profile/wrapped)
│   ├── 카테고리 필터 (전체/식당/와인)
│   ├── 프라이버시 게이지 (최소/보통/공개)
│   ├── 디테일 게이지 (심플/보통/상세)
│   ├── Wrapped 카드 미리보기
│   └── 저장 (PNG) / 공유
│
├── 버블러 프로필 (/users/[id])
│   ├── 히어로 (아바타, 닉네임, 레벨, 맛태그, 팔로워/팔로잉 수)
│   ├── 버블 컨텍스트 카드 (공통 버블, 취향 매칭도 %)
│   ├── 스티키 탭 (식당/와인)
│   ├── 취향 프로필 (카테고리 분석, 점수 경향)
│   ├── 추천 픽 (Top 추천 — 맞팔 전용)
│   ├── 최근 기록 (맞팔 전용)
│   ├── 활동 히트맵
│   └── 프라이버시 제어 (none / follow+ / mutual 단계별 공개)
│
├── 버블 (/bubbles)
│   ├── 버블 목록 (/bubbles)
│   │   ├── HOT 버블 스트립 (주간 활동 기반)
│   │   ├── 스티키 탭: 버블 | 버블러
│   │   ├── 소팅 (최근 활동, 멤버 수, 기록 수, 이름)
│   │   ├── 검색 + 조건 필터 칩 + 고급 필터
│   │   ├── 페이지네이션
│   │   └── FAB (+) → /bubbles/create
│   │
│   ├── 버블 상세 (/bubbles/[id])
│   │   ├── 스티키 헤더 (아이콘, 이름, 설명, 멤버 수)
│   │   ├── 스티키 탭: 식당 | 와인 (식당/와인 컬러 적용)
│   │   ├── 아이콘 토글 (트로피=랭킹, 유저=멤버, 지도, 소팅, 검색)
│   │   ├── 리스트 뷰 (기본 — 타겟별 집계, 평균 점수, 멤버 수, 매칭도, 멤버 도트)
│   │   ├── 랭킹 뷰 (트로피 토글 — 포디움 Top 3 + 순위 리스트)

│   │
│   ├── 버블 생성 (/bubbles/create)
│   │   ├── 기본 정보 (이름, 설명, 아이콘+컬러)
│   │   ├── 공개 범위 (private/public)

│   ├── 프라이버시
│   │   ├── 프로필 공개 범위 (public/bubble_only/private)
│   │   ├── 기록 공개 범위 (shared_only/all)
│   │   ├── 공개 항목 토글 (점수, 코멘트, 사진, 레벨, 사분면, 버블, 가격)
│   │   ├── 버블 멤버 기본값
│   │   └── 버블별 오버라이드
│   ├── 알림 (푸시, 레벨업, 버블 가입, 팔로우, 방해금지 시간대)

│   │   ├── 기본 정렬, 기록 입력 방식, 버블 공유 정책, 온도 단위, 시간대
│   │   └── 수동 공유 정리
│   ├── 데이터 (내보내기, 가져오기, 캐시 삭제)
│   ├── 정보 (이용약관, 개인정보처리방침, 버전)

│
└── 알림 (인라인 드롭다운 — 별도 페이지 아님)
```

## 2. 핵심 사용자 플로우

### Flow A: 식당 기록하기 (핵심 루프)
```
FAB (+) → /add?type=restaurant
  → 카메라 촬영/앨범 → AI 인식 → 후보 선택
     또는 검색 → 대상 선택
     또는 신규 등록 → /register
  → /record?type=restaurant&targetId={id}
  → 사진 피커 + 기록 폼 (만족도, 사분면, 씬태그 6종, 방문일, 동행자, 코멘트, 가격대 등)
  → [저장] → 버블 자동 동기화 → XP 적립
```

### Flow B: 와인 기록하기
```
FAB (+, 와인 탭) → /add?type=wine
  → 라벨 촬영(OCR+AI) / 검색 → 대상 선택
  → /record?type=wine&targetId={id}
  → 사진 피커 + 기록 폼 (만족도, 사분면(산미×바디), 아로마 16섹터 3링, 품질 평가 BLIC, 코멘트 등)
  → [저장] → 버블 자동 동기화 → XP 적립
```

### Flow C: 재방문/재음용 기록
```
상세 페이지 DetailFab (+) → /record?type=restaurant&targetId={id}&from=detail
  → 사분면 (이전 기록 참조점 표시) → 기록 폼 → 저장
```

### Flow D: 탐색 → 기록
```
홈 검색 아이콘 → /discover → 지역+장르 선택 → AI 랭킹/추천 → 식당 상세 → 기록 or 찜
```

### Flow E: 넛지 → 기록 완성
```
앱 진입 → AI 인사 (조건부, 5초 자동 소멸) → 클릭 시 식당 상세로 이동
```

### Flow F: 버블 공유 플로우
```
상세 페이지 → FabActions "공유" → 버블 선택 시트 → 공유 완료
기록 저장 시 → 자동 동기화 (버블 공유 규칙에 따라)
```

### Flow G: 온보딩 플로우
```
로그인 (Google/Kakao/Naver/Apple) → 인트로 → Step 1: 지역 선택 + 맛집 등록
  → Step 2: 버블 생성 (템플릿) → Step 3: 시드 버블 탐색 → 보너스 XP → 홈
```

## 3. 네비게이션 구조

### 3-1. 글로벌 헤더 (모든 페이지 공통)

```
nyam                    bubbles  🔔  [Lv]  [@]
```

| 요소 | 기능 |
|------|------|
| nyam | 로고 (Comfortaa, 그라디언트 #FF6038→#8B7396), 홈 링크 |
| bubbles | 텍스트 버튼 (Comfortaa, --brand 컬러), `/bubbles` 이동 |
| 🔔 | 알림 드롭다운 토글 (읽지 않은 알림 시 빨간 도트 뱃지) |
| 레벨바 | 레벨 뱃지 (20px 원) + XP 트랙 (32x4px 프로그레스바) |
| 아바타 | 드롭다운 메뉴 (프로필, 설정) |

### 3-2. 홈 콘텐츠 탭 바

```
[식당 | 와인]                    [⊞뷰] [📍지도] [↕정렬] [🔍→discover]
```

| 요소 | 기능 |
|------|------|
| 식당 / 와인 | 도메인 전환 (밑줄: 식당=--accent-food, 와인=--accent-wine) |
| 뷰 순환 | 카드 → 리스트 → 캘린더 순환 (3종) |
| 지도 | 방문 식당 지도 뷰 토글 (식당 탭에서만 표시) |
| 정렬 | 드롭다운 (최신순, 점수 높은순, 점수 낮은순, 이름순, 방문 많은순) |
| 검색 | /discover 페이지로 이동 |

### 3-3. 조건 필터 바

```
[칩1 ✕] [칩2 ✕] ... [+]               [◄ 1/5 ►]
```

| 요소 | 기능 |
|------|------|
| 필터 칩 | 속성:값 쌍, 클릭 편집, X 제거, 계단식 선택 지원 |
| + 추가 | 속성 선택 → 값 선택 팝오버, 고급 필터 옵션 |
| 인라인 페이저 | 총 페이지 > 1일 때 표시 (카드 5개, 리스트 10개, 지도 10개/페이지) |

### 3-4. FAB (Floating Action Button)

| FAB | 위치 | 기능 |
|-----|------|------|
| FAB (+) 추가 | 우하단, 56x56px | /add 이동 (탭에 따라 type=restaurant\|wine) |
| DetailFab 뒤로 | 좌하단 (상세 페이지) | router.back() |
| DetailFab 추가 | 우하단 (상세 페이지) | /record 이동 (대상 프리셋) |
| FabActions | 우하단 바 (기록 있을 때) | 수정 · 공유 · 삭제 |
| FabBack | 좌하단 (기록 페이지) | 이전 페이지 |

### 3-5. 페이지 간 이동 맵

```
홈 (/)
├── 카드/리스트 탭 → 식당 상세 (/restaurants/[id])
├── 카드/리스트 탭 → 와인 상세 (/wines/[id])
├── FAB (+) → 추가 플로우 (/add)
├── 🔍 검색 아이콘 → 탐색 (/discover)
├── bubbles → 버블 목록 (/bubbles)
├── 🔔 → 알림 드롭다운
├── 아바타 → 프로필 (/profile)
└── 아바타 → 설정 (/settings)

추가 플로우 (/add)
├── 카메라/AI → 대상 선택 → 기록 (/record)
├── 검색 → /search → 대상 선택 → 기록 (/record)
├── 신규 등록 → /register → 기록 (/record)
└── 성공 화면 → 홈 / 한 개 더 추가

식당 상세 (/restaurants/[id])
├── DetailFab (←) → 이전 페이지
├── DetailFab (+) → 기록 (/record?type=restaurant&targetId={id}&from=detail)
├── FabActions → 수정 (/record?edit={recordId}) · 공유 · 삭제
├── 와인 링크 → 와인 상세 (/wines/[id])
└── 사분면 참조점 → 다른 식당/와인 상세

와인 상세 (/wines/[id])
├── DetailFab (←) → 이전 페이지
├── DetailFab (+) → 기록 (/record?type=wine&targetId={id}&from=detail)
├── FabActions → 수정 (/record?edit={recordId}) · 공유 · 삭제
├── 식당 링크 → 식당 상세 (/restaurants/[id])
└── 사분면 참조점 → 다른 식당/와인 상세

프로필 (/profile)
├── Wrapped 공유 → /profile/wrapped
└── 경험 레벨 → 상세 시트

버블 목록 (/bubbles)
├── 버블 카드 → 버블 상세 (/bubbles/[id])
├── FAB (+) → 버블 생성 (/bubbles/create)
└── 버블러 탭 → 버블러 프로필 (/users/[id])

버블 상세 (/bubbles/[id])
├── 리스트 카드 → 식당/와인 상세 (버블 모드: ?bubble={id})
├── 멤버 → 버블러 프로필 (/users/[id])
├── 설정 → /bubbles/[id]/settings
└── 트로피/유저 아이콘 → 랭킹/멤버 뷰 전환

탐색 (/discover)
├── 식당 결과 → 식당 상세
├── AI 추천 → 식당 상세
└── 찜 → 위시리스트 추가
```

## 5. 구현 순서 (스프린트 계획 — P1+P2 통합)

> P2 테이블은 S1에서 전부 생성. P2 기능은 S7~S8에서 구현.
> 온보딩은 모든 기능 완성 후 마지막에 연결.
> S10은 지도 고도화 (드릴다운 SVG).

```
S1 ─── S2 ─── S3 ─┬─ S4
 (기반)  (코어)  (검색) │ (상세P1)
                    │
                    S5 ─── S6 ─── S7 ─── S8 ─── S9 ─── S10
                   (홈+탐색) (XP+프로필) (버블) (통합) (온보딩) (지도)
```

| Sprint | 문서 | 산출물 | 선행 | 비고 |
|--------|------|--------|------|------|
| **S1** | DATA_MODEL + AUTH + DESIGN_SYSTEM | **P1+P2 전체** DB 스키마 (버블·소셜 테이블 포함), RLS, 인증, 디자인 토큰 | 없음 | P2 테이블도 이때 생성 → 나중에 마이그레이션 고통 방지 |
| **S2** | RATING_ENGINE + RECORD_FLOW | 사분면 UI, 아로마 16섹터 3링, 품질 평가 BLIC, 제스처, 기록 저장 플로우 | S1 | 핵심 UX — 여기서 사분면 완성도가 앱 품질 결정 |
| **S3** | SEARCH_REGISTER | 검색 (Nyam DB → 외부 API), 식당/와인 등록, 사진 등록 (EXIF+AI) | S1, S2 | 검색→선택→S2 평가로 연결되는 풀 플로우 완성 |
| **S4** | RESTAURANT_DETAIL + WINE_DETAIL | 상세 페이지 L1~L8 (히어로, 평점, 내 기록, 사분면 분포, 실용 정보) | S1, S2 | P2 소셜 레이어(버블 기록)는 S8에서 추가 |
| **S5** | HOME + DISCOVER | 홈 식당/와인 탭 구조, AI 인사, 조건 필터/소팅/뷰 전환, 탐색(Discover) | S1~S4 | 추천은 기록 수 기반 점진 해금 |
| **S6** | XP_SYSTEM + PROFILE + SETTINGS | XP 계산, 레벨, Wrapped, 프로필, 설정, 알림 | S2 | |
| **S7** | BUBBLE | 버블 생성/가입/초대, 피드(리스트), 랭킹, 멤버, 역할/권한, 버블 설정 | S1, S6 | P2 코어 — 버블 시스템 전체를 한 스프린트에 |
| **S8** | BUBBLE 통합 + FOLLOW | 상세 페이지 버블 기록, 홈 팔로잉 모드, 버블 자동 동기화, 팔로우, 버블러 프로필 | S4, S5, S7 | 기존 P1 페이지에 P2 레이어 통합 |
| **S9** | ONBOARDING + 마무리 | 온보딩 풀 플로우 (로그인→맛집등록→버블생성→탐색→홈), 넛지 정교화, 전체 플로우 검증 | 전체 | 모든 기능 완성 후 온보딩 연결 |
| **S10** | MAPS | 식당 드릴다운 지도, 와인 산지 3단계 드릴다운 SVG 지도 | S6 | 프로필/홈 통계 지도 고도화 |
