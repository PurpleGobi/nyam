# Dependency Map

> 의존성 기반 병렬 실행 전략. MASTER_TRACKER의 웨이브 배정 근거.

---

## 스프린트 의존성 그래프

```
S1 (Foundation)
 │
 └─→ S2 (Core Recording)
      │
      ├─→ S3 (Search)         ─┐
      ├─→ S4.1~4.4 (Detail)   ─┼─ Wave 3 (3 병렬)
      └─→ S6 (XP/Profile)     ─┘
           │                        │
           └─→ S7 (Bubble) ────┐   │
                                │   │
           S3 완료 → S4.5~4.6 ─┤   │
                    S4 완료 ──→ S5  │  Wave 4 (2 병렬)
                                │   │
                                S5 + S7 완료
                                    │
                                    └─→ S8 (Integration)  Wave 5
                                         │
                                         └─→ S9 (Onboarding)  Wave 6
```

---

## 웨이브 전환 조건

| 전환 | 조건 | 가동 세션 |
|------|------|----------|
| → Wave 1 | 없음 | 1 |
| → Wave 2 | S1 완료 | 1 |
| → Wave 3 | S2 완료 | **최대 3** (S3 + S4 + S6) |
| → Wave 4 | S3 또는 S6 완료 (유동 시작) | **최대 2** (S5트랙 + S7트랙) |
| → Wave 5 | S5 **그리고** S7 완료 | 1 |
| → Wave 6 | S8 완료 | 1 |

> Wave 4는 고정 시점이 아니라 **조건 충족 즉시** 시작:
> - S6 먼저 끝남 → S7 즉시 시작 (Wave 3 도중)
> - S3 먼저 끝남 → S4.5 즉시 시작 → S4 완료 → S5 시작

## 병렬 세션 운용 타임라인 (최적 시나리오)

```
시간 →  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓

세션A:  [  S1  ][  S2  ][ S3          ][ S4.5 ][  S5        ][ S8   ][ S9  ]
세션B:                   [ S4(4.1~4.4)]  (대기)  [ S4.5 지원 불가 → 종료 ]
세션C:                   [ S6     ][ S7                      ]
                                                              ↑
                          ←Wave3→ ←────Wave4─────→ ←W5→ ←W6→
```

> 세션 C가 S6을 끝내면 즉시 S7 착수 (Wave 3↔4 경계 무시).
> 세션 B가 S4.4까지 끝나고 S3 미완이면 → 가용 작업 없음 → **세션 종료** (S7은 세션 C가 담당, 1스프린트 1세션 원칙).

## 크리티컬 패스 (두 갈래)

```

경로 B (소셜):
S2.9 → S6.1 → S6.2 → S7.1 → S7.3 → S8.1 → S8.6 → S9.3
```
