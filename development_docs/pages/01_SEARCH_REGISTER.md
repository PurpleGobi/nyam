# SEARCH_REGISTER — 검색 & 등록

> depends_on: DATA_MODEL, RATING_ENGINE, DESIGN_SYSTEM
> affects: RECORD_FLOW
> prototype: `prototype/01_home.html` (screen-add-restaurant, screen-add-restaurant-search, screen-add-wine, screen-add-wine-search, screen-add-wine-confirm, screen-nudge-photo-picker)

**이 문서의 범위**: 식당/와인을 **찾고 특정**하는 과정 (카메라 화면, 검색 화면, 확인 화면, 넛지 사진 선택).
기록 화면(screen-rest-record, screen-wine-record), 성공 화면(screen-add-success), 저장 시퀀스는 → **RECORD_FLOW** 참조.

---

## 1. 진입점

| 경로 | 동작 | 화면 |
|------|------|------|
| 홈 FAB (+) | 현재 탭(식당/와인)에 따라 카메라 화면 진입 | 식당탭 → `screen-add-restaurant` / 와인탭 → `screen-add-wine` |
| 홈 넛지 스트립 "기록" | AI 감지된 사진 → 사진 선택 | `screen-nudge-photo-picker` |
| 상세 페이지 FAB (+) | 검색/특정 스킵, 바로 기록 화면 | → RECORD_FLOW §2 |
| 추천 카드 "다녀왔어요" | 추천 → 기록 전환 | → RECORD_FLOW §2 |

## 2. 식당 추가 — 카메라 화면 (`screen-add-restaurant`)

### 레이아웃
```
┌──────────────────────────────┐
│  ← 식당 추가              ✕  │
├──────────────────────────────┤
│                              │
│     ┌────────────────┐       │
│     │   🍴 음식 사진을  │       │
│     │   촬영하세요     │       │
│     └────────────────┘       │
│  음식 또는 식당 사진을 촬영하면  │
│  자동으로 인식합니다            │
│                              │
│         [📷 촬영]            │
│                              │
│  [앨범에서 추가] [목록에서 추가] │
└──────────────────────────────┘
```

### 진입 경로 (3가지)
| 버튼 | 동작 |
|------|------|
| 촬영 (📷) | 카메라 → AI 인식(GPS+Vision) → 식당 매칭 → 기록 화면 (RECORD_FLOW §3) |
| 앨범에서 추가 | 갤러리 선택 → AI 인식 → 기록 화면 (RECORD_FLOW §3) |
| 목록에서 추가 | `screen-add-restaurant-search` 이동 |

### AI 인식 플로우 (촬영/앨범)
```
사진 → EXIF 추출 (GPS, 날짜) → Vision AI (음식 판별, 장르 추정)
  ├─ 확실한 매칭 (1곳): 바로 기록 화면 진입
  ├─ 여러 후보: 후보 목록 표시 → 사용자 선택 → 기록 화면
  └─ GPS 없음: 검색으로 폴백
→ 식당 확정 → 기록 화면 (사진 자동 첨부, status: 'rated')
```

### GPS + AI 교차 매칭
```
GPS 100m 내: 스시코우지(일식), 교대짬뽕(중식), 스타벅스(카페)
AI 분석: "일식/초밥류"
→ 스시코우지 = 최우선 → 바로 기록 화면 진입
```

## 3. 식당 검색 화면 (`screen-add-restaurant-search`)

### 레이아웃
```
┌──────────────────────────────┐
│  ← 식당 검색              ✕  │
├──────────────────────────────┤
│ 🔍 식당 이름 검색...          │
├──────────────────────────────┤
│ 📍 근처 식당                  │  ← 기본 표시 (검색 전)
│  🍴 스시코우지  일식·을지로     │
│     120m              기록있음 │
│  🍴 몽탄  한식·을지로          │
│     350m                     │
│  🍴 을지로 골목집  한식·을지로3가│
│     500m                     │
│  🍴 비스트로 드 욘트빌  프렌치·을지로│
│     650m                     │
│  🍴 리스토란테 에오  이탈리안·청담│
│     1.2km                    │
│  ──────────────────────────  │
│  ⊕ 목록에 없나요? 직접 등록하기 │
└──────────────────────────────┘
```

### 상태 전환
- **검색 전**: 근처 식당 목록 표시 (GPS 기반, 거리순)
- **검색 중 (2자 이상 입력)**: 자동완성 결과로 전환 (근처 목록 숨김)

### 자동완성 순서
1. Nyam 캐시 DB (이미 등록된 식당 — "기록 있음" 뱃지)
2. 사용자 활동 지역 내 결과 우선
3. 외부 API (카카오+네이버+구글 동시)

### 검색 결과 항목
| 항목 | 표시 |
|------|------|
| 가게명 | 볼드 텍스트 |
| 장르 · 위치 | 서브 텍스트 (예: "일식 · 교대역") |
| 기록 여부 | "기록 있음" 뱃지 (accent-food 컬러, 있을 때만) |

### 근처 식당 항목
| 항목 | 표시 |
|------|------|
| 아이콘 | utensils (18px) |
| 가게명 | 볼드 텍스트 |
| 장르 · 위치 | 서브 텍스트 |
| 거리 | "120m", "1.2km" 등 |
| 기록 여부 | "기록 있음" (있을 때만) |

### fuzzy matching
- 띄어쓰기 무시 ("스시 코우지" = "스시코우지")
- 점/역/호 접미사 무시
- 한영 혼용 허용 ("sushi" → "스시")

### 선택 시 동작
| 상태 | 동작 |
|------|------|
| 새 식당 | 성공 화면 (`screen-add-success`). `status: 'checked'`로 저장 |
| "기록 있음" 식당 | 토스트 "다녀온 적 있어요 — 상세페이지로 이동합니다" → 상세 페이지 |

> 검색/목록 경로는 `checked` 상태만 생성. 사분면 평가 등 풍성화는 상세페이지에서 진행 (RECORD_FLOW §8).

### 직접 등록 (목록에 없을 때)
```
"⊕ 목록에 없나요? 직접 등록하기" 버튼 → 성공 화면
→ 가게명 (필수) + 주소/위치 (선택) + 장르 (선택)
```

## 4. 와인 추가 — 카메라 화면 (`screen-add-wine`)

### 레이아웃
```
┌──────────────────────────────┐
│  ← 와인 추가              ✕  │
├──────────────────────────────┤
│                              │
│     ┌────────────────┐       │
│     │   🍷 라벨을     │       │
│     │   맞춰주세요    │       │
│     └────────────────┘       │
│  와인 라벨을 촬영하면          │
│  자동으로 인식합니다           │
│                              │
│         [📷 촬영]            │
│                              │
│  [앨범] [이름검색] [진열장] [영수증] │
└──────────────────────────────┘
```

### 진입 경로 (5가지)
| 버튼 | 동작 | `records.camera_mode` |
|------|------|----------------------|
| 촬영 (📷) | 라벨 촬영 → OCR → 확인 화면 or 기록 화면 | `individual` |
| 앨범에서 추가 | 갤러리 → OCR → 확인 화면 or 기록 화면 | `individual` |
| 이름으로 검색 | `screen-add-wine-search` 이동 | — |
| 진열장 | 여러 병 촬영 → 목록 인식 | `shelf` |
| 영수증 | 영수증 촬영 → 구매 목록 인식 | `receipt` |

### OCR 인식 데이터 (`records.ocr_data`)
| 모드 | JSONB 구조 |
|------|-----------|
| `individual` | `{"wine_name":"...", "vintage":"...", "producer":"..."}` |
| `shelf` | `{"wines":[{"name":"...", "price":...}, ...]}` |
| `receipt` | `{"items":[{"name":"...", "price":..., "qty":1}], "total":...}` |

### AI 인식 후 분기
```
라벨 촬영/앨범
    ↓
OCR (와인명, 생산자, 빈티지, 산지) → 와인 DB 매칭
    ↓
┌─ 확실한 매칭 → 바로 기록 화면 (RECORD_FLOW §4)
│
└─ 후보 매칭 → 확인 화면 (screen-add-wine-confirm)
```

## 6. 와인 검색 화면 (`screen-add-wine-search`)

### 레이아웃
```
┌──────────────────────────────┐
│  ← 와인 검색              ✕  │
├──────────────────────────────┤
│ 🔍 와인 이름을 입력하세요      │
├──────────────────────────────┤
│                              │  ← 검색 전: 힌트 텍스트
│  와인 이름이나 생산자를 입력하세요│  "와인 이름이나 생산자를 입력하세요"
│                              │
│  (검색 후)                    │
│  Chateau Margaux 2018        │
│  Red · Bordeaux, France      │
│  ──────────────────────────  │
│  Chateau Margaux 2015        │
│  Red · Bordeaux   기록 있음   │
│  ──────────────────────────  │
│  Chateau Margaux 2010        │
│  Red · Bordeaux, France      │
└──────────────────────────────┘
```

### 상태 전환
- **검색 전**: 힌트 텍스트 "와인 이름이나 생산자를 입력하세요"
- **검색 중 (2자 이상 입력)**: 자동완성 결과 표시 (힌트 숨김)

### 검색 결과 항목
| 항목 | 표시 |
|------|------|
| 와인명 + 빈티지 | 볼드 텍스트 (예: "Chateau Margaux 2018") |
| 타입 · 산지 | 서브 텍스트 (예: "Red · Bordeaux, France") |
| 기록 여부 | "기록 있음" 뱃지 (accent-wine 컬러, 있을 때만) |

### 자동완성 순서
1. Nyam 캐시 DB (이미 등록된 와인)
2. 와인 DB API

### 선택 시 동작
| 상태 | 동작 |
|------|------|
| 새 와인 | 성공 화면 (`screen-add-success`). `status: 'checked'`로 저장 |
| "기록 있음" 와인 | 토스트 "이미 기록한 와인이에요 — 상세페이지로 이동합니다" → 상세 페이지 |

> 검색 경로는 `checked` 상태만 생성. 사분면·아로마 등 풍성화는 상세페이지에서 진행 (RECORD_FLOW §8).

### 빈티지 처리
- 같은 와인 + 다른 빈티지 = 다른 기록
- "빈티지 모름" 선택 가능
- 최근 연도 기본 선택

## 7. 넛지 → 사진 선택 (`screen-nudge-photo-picker`)

홈 넛지 스트립 "기록" 탭 시 진입하는 사진 선택 화면.

### 레이아웃
```
┌──────────────────────────────┐
│  ← 사진 선택              ✕  │
├──────────────────────────────┤
│ ✨ 감지된 사진을 선택하세요     │
│ [최근 항목] [음식] [스크린샷] [즐겨찾기] │
├──────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐   │
│  │ 🍴  │ │ 🍴  │ │     │   │
│  │✨감지│ │✨감지│ │     │   │
│  │ ✓   │ │     │ │     │   │
│  └─────┘ └─────┘ └─────┘   │
│  ┌─────┐ ┌─────┐ ┌─────┐   │
│  │     │ │     │ │ 🍷  │   │
│  │     │ │     │ │     │   │
│  └─────┘ └─────┘ └─────┘   │
├──────────────────────────────┤
│  [ 사진 N장 선택 — 다음 ]     │
└──────────────────────────────┘
```

### 사진 그리드
- AI 감지된 사진: `✨ 감지` 뱃지 + accent 컬러 배경
  - 음식 사진: accent-food 배경 + utensils 아이콘
  - 와인 사진: accent-wine 배경 + wine 아이콘
- 일반 사진: 회색 배경
- 탭하여 선택/해제 (복수 선택 가능), ✓ 체크 표시

### 하단 버튼
- 0장 선택: "사진을 선택하세요" (비활성)
- N장 선택: "사진 N장 선택 — 다음" (활성)

### "다음" 동작
- 선택된 사진 → AI 인식 → 사진 유형에 따라 분기:
  - 음식 사진 → `screen-rest-record` (식당 기록 화면)
  - 와인 사진 → `screen-wine-record` (와인 기록 화면)
- → 기록 화면 상세는 RECORD_FLOW §3, §4 참조

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 이미 기록한 식당/와인 선택 | 토스트 알림 + 상세 페이지 이동 |
| 동명 식당 | 주소/지도로 구분 |
| 프랜차이즈 | 지점별 별도 식당 |
| 폐업 식당 | 기록 아카이브 유지, 신규 등록 경고 |
| 오타/불완전 입력 | fuzzy matching |
| 외부 API 장애 | 캐시 DB만 검색, 없으면 직접 등록 |
| 음식 사진 아님 | "음식 사진을 선택해주세요" |
| EXIF 없음 | 검색으로 폴백 |
| EXIF 날짜 1개월+ | "N개월 전 사진이네요. 맞나요?" |
| 와인 라벨 인식 실패 | "와인을 찾지 못했어요" → [이름으로 검색] or [직접 등록] |

## 9. 캐싱 정책

| 항목 | 정책 |
|------|------|
| 최초 캐싱 | 사용자 첫 등록 시 외부 API → Nyam DB 저장 |
| 갱신 주기 | 2주 |
| 범위 | 사용자가 등록한 식당/와인만 (전체 크롤링 안 함) |
| 폐업 감지 | 갱신 시 외부 DB 삭제 확인 → 알림 |
