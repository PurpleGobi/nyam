## 1. 프로필 화면 — 전체 구조

프로필 = **미식 정체성 + 경험치/레벨 + 세부 통계**. 3개 영역으로 구성된 싱글 스크롤 페이지.

┌──────────────────────────────────────┐
│ [nyam 로고]   [bubbles] [🔔] [아바타] │  메인 헤더 (공통)
├──────────────────────────────────────┤
│ [아바타+Lv.7]   │ ┌─ 미식 정체성 카드 ─┐ │  ← §2. 2컬럼 헤더
│ 김맛잘알         │ │ "을지로 골목을…"    │ │
│ @matjal_kim      │ │ [태그] [태그]       │ │
│                  │ │ ✨ 127개 기반 [공유] │ │

├──────────────────────────────────────┤
│ ── 활동 요약 ──                       │  ← §3
│ ┌─ 총 레벨 카드 ─────────────────┐   │

## 2. 프로필 헤더 (2컬럼)

2컬럼 레이아웃: 좌측에 아바타+이름, 우측에 미식 정체성 카드.

### 좌측 컬럼

| 요소 | 설명 |
|------|------|
| 아바타 | 64px 원형, 그라디언트 배경, 이니셜 표시 |
| 레벨 뱃지 | 아바타 우하단, `Lv.N` 표시 (보라색 배경) |
| 닉네임 | 15px bold, 중앙 정렬 |
| 핸들 | `@handle_name`, 11px hint 색상 |

### 우측 컬럼 — 미식 정체성 카드

- 기록 데이터 기반 자동 생성 텍스트 + 태그
- 주력 동네 + 선호 분위기 + 와인 취향 요약 (이탤릭)
- 텍스트 최대 3줄 clamp (`-webkit-line-clamp: 3`), 초과 시 말줄임
- **강조 키워드**는 primary 색상 bold 처리
- 태그 목록: pill 형태, `rgba(primary, 0.1)` 배경
- 하단: 기록 개수 표시 (`✨ 127개 기록 기반`) + **공유 버튼** → Wrapped 공유 화면으로 이동
- Phase 1: 규칙 기반 템플릿, Phase 3: AI 생성

## 3. 활동 요약 섹션

프로필 헤더 아래, 세부 통계 탭 위에 위치하는 통합 활동 개요.

### 3-1. 총 레벨 카드

```
┌─ 그라디언트 배경 (bg-card → primary-light) ─┐
│ [🏆 아이콘]  총 레벨 Lv.7                   │
│              1,780 XP · 다음 레벨까지 120 XP │
│ [══════════════ 진행바 74% ═══════      ]   │
└──────────────────────────────────────────────┘
```

- 아이콘: award (primary 배경, 흰색)
- XP 수치 + 다음 레벨까지 남은 XP 표시
- 진행 바: 6px, primary 색상

### 3-2. 개요 그리드 (2×2)

| 카드 | 값 | 색상 | 트렌드 |
|------|-----|------|--------|
| 식당 방문 | 카운트 | primary | `+N 이번 달` |
| 와인 시음 | 카운트 | wine | `+N 이번 달` |
| 평균 점수 | 소수점 1자리 | text | `-` or `±N` |
| 이번 달 XP | `+N` | positive | `활발!` 등 |

- 각 카드: 12px 라운드, border, 중앙 정렬
- 트렌드: `trend-up` (positive 색상) 또는 `trend-flat` (hint 색상)

### 3-3. 활동 기록 (히트맵)

GitHub 스타일 활동 히트맵.

| 요소 | 설명 |
|------|------|
| 상단 통계 3개 | 총 기록 · 연속 기록(streak, caution 색상 강조) · 활동 기간 |
| 히트맵 그리드 | 13열 × 7행 (91셀), 셀 크기 aspect-ratio: 1 |
| 셀 강도 | l1~l4 (accent-social 25%~100%) |
| 하단 월 라벨 | 최근 3개월 표시 |

### 3-4. 최근 경험치

- 최근 XP 획득 내역 4~5개 (DB: `xp_histories`)
- 각 항목 구성: `[아이콘] 장소명 (유형) | 축 · 기록 품질 태그 · 시간 | +XP`
  - 기록 품질 태그: 사진+한줄평+메뉴, 향팔레트+심화, 사진 등 (XP 산출 근거)
  - 마일스톤 항목: `장소명` 대신 달성 내용, `축` 대신 "마일스톤 보너스"

| 아이콘 | 색상 | 의미 |
|--------|------|------|
| sparkles | primary-light/primary | 새 장소 |
| wine | wine-light/wine | 와인 기록 |
| repeat | green-light/positive | 재방문 |
| milestone | gold-light/caution | 마일스톤 |

## 5. 식당 통계 패널

### 5-1. 요약 카드 (3열)

| 카드 | 값 | 트렌드 |
|------|-----|--------|
| 총 방문 | 카운트 | `+N 이번 달` |
| 평균 점수 | 소수점 1자리 | `-` or delta |
| 방문 지역 | 카운트 | `+N 신규` |

### 5-2. 경험치 & 레벨 (식당)

- 섹션 타이틀: "경험치 & 레벨"
- **미니탭**으로 세부 축 전환:
  - **지역 탭**: 을지로, 광화문, 강남, 성수…
  - **장르 탭**: 일식, 한식, 이탈리안, 프렌치…
- DB: `user_experiences` — `axis_type='area'`/`'genre'`, 각 축별 XP/레벨

#### 레벨 리스트 아이템 구성

```
[Lv뱃지] 축 이름  방문 수    현재XP/다음레벨XP
         [═══ 진행바 ══       ]
```

- Lv 뱃지: 26px 정사각 라운드, 레벨 색상 적용
- 방문 수: hint 색상, 작은 글씨 (`42곳`, `28곳`)
- 진행바: 4px, 해당 레벨 색상
- 낮은 레벨 항목: `opacity: 0.55` 처리
- 리스트 영역: `max-height: 340px`, 초과 시 내부 스크롤 (`.lv-list-scroll`)
- **탭 시**: 레벨 상세 바텀시트 오픈 (§8 참조)

#### 레벨 색상 체계

| 레벨 | 색상 | CSS 값 |
|------|------|--------|
| 1~3 | green | `#7EAE8B` |
| 4~5 | blue | `#7A9BAE` |
| 6~7 | purple | `#8B7396` |
| 8~9 | primary (orange) | `#C17B5E` |
| 10 | gold | `#C9A96E` |
| 미활성/낮은 레벨 | border | `#D4CFC8` |

### 5-3. 가본 식당 지도

SVG 세계 지도에 방문 식당을 도트로 표시.

| 요소 | 설명 |
|------|------|
| 섹션 헤더 | "가본 식당 지도" + 서브타이틀 `N개국 N곳` (예: "5개국 12곳") |
| 지도 | 대륙 윤곽선 SVG, bg-page/border 색상 |
| 도시 마커 | 원형, primary 색상, 방문 수에 비례한 크기 |
| 큰 마커 (6곳+) | 내부에 숫자 표시 (흰색) |
| 범례 | `● 1~2곳` `● 3~5곳` `● 6곳+` (opacity 차등) |
| 데이터 소스 | `restaurants.country` + `restaurants.city`로 그룹핑 |

### 5-4. 장르 수평 바 차트

- 가로 막대형 차트 (primary 색상 계열)
- 방문 수 기준 정렬 (내림차순)
- 각 행: `장르명 | [══ 바 ══] | 숫자`
- opacity 차등으로 비중 표현 (1위=1.0, 이하 점차 감소)

### 5-5. 점수 분포

세로 바 차트 (6구간).

| 구간 | 표시 |
|------|------|
| ~49, 50s, 60s | 방문 적으면 최소 높이 바 |
| 70s, 80s, 90s | 비율에 따른 높이, 내부 숫자 표시 |

- primary 색상, opacity 구간별 차등 (0.4~1.0)
- 바 상단에 카운트 숫자 (흰색, 바 높이 충분할 때)

### 5-6. 월별 소비

- 세로 바 차트, 최근 6개월
- 바 높이 = **금액** 기준
- 상단: 총액 표시 (`총 186만`)
- 각 바 아래: 월 라벨 + 금액 pill
- 최고 금액 월: primary-light 배경 + primary 글씨 강조
- 데이터 소스: `records.total_price` (1인 결제 금액) 월별 SUM

### 5-7. 상황별 방문

- 수평 바 차트 (장르와 동일 패턴)
- 데이터 소스: `records.scene` 기준 집계
- 각 상황별 DB 값 → 한국어 라벨 + 고유 색상:

## 6. 와인 통계 패널

### 6-1. 요약 카드 (3열)

| 카드 | 값 | 트렌드 |
|------|-----|--------|
| 시음 수 | 카운트 | `+N 이번 달` |
| 평균 점수 | 소수점 1자리 | `-` or delta |
| 셀러 보유 | 카운트 | `+N 신규` |

### 6-2. 경험치 & 레벨 (와인)

- 섹션 타이틀: "경험치 & 레벨"
- **미니탭**으로 세부 축 전환:
  - **산지 탭**: 보르도, 부르고뉴, 나파 밸리, 론 밸리…
  - **품종 탭**: 카베르네 소비뇽, 피노 누아, 샤르도네…
- DB: `user_experiences` — `axis_type='wine_region'`/`'wine_variety'`, 각 축별 XP/레벨
- 미니탭 active 색상: wine (`.mini-tab-row.wine .mini-tab.active`)
- 리스트 아이템 구성: 식당과 동일 패턴 (단위만 `병`으로 변경, 동일 스크롤 제약)

### 6-3. 산지 지도 — 3단계 드릴다운

인터랙티브 와인 산지 지도. 세계 → 국가 → 산지 3단계 확대.

#### Level 0: 세계 지도
```
┌─ 다크 배경 (#1a1520) ──────────────────┐
│ 산지                    탭하여 확대      │
│                                         │
│ [대륙 실루엣들 — 다크 톤]              │
│                                         │
│    ④       ③        ②   ①             │
│   FR      CL       AU  NZ  US          │
│ [와인점] [와인점]  [와인점] [와인점]   │
│                                         │
│ ● 레드  ● 화이트  ● 로제  ● 스파클링   │
└─────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 배경 | 다크 테마 (`#1a1520`) |
| 국가 마커 | wine 색상 원형, 와인 수에 비례한 크기, 내부 숫자+국가코드 |
| 와인 도트 | 마커 주변에 색상별 작은 원: 레드(`#722F37`), 화이트(`#D4C98A`), 로제(`#E8A0B0`), 스파클링(`#C8D8A0`) |
| 미탐험 국가 | 점선 원 + 국가코드 (dim 색상) |
| 하이라이트 | 탐험한 국가는 반투명 wine 색상 fill |
| 탭 인터랙션 | 국가 마커 클릭 → Level 1로 전환 |
| 범례 | 하단 — 와인 타입별 색상 표시 |

#### Level 1: 국가 지도 (예: 프랑스)
```
┌─ [← 뒤로] 프랑스 — 4병 ────────────────┐
│                                          │
│ [프랑스 윤곽선]                          │
│   Champagne ●●                          │
│          Burgundy ●                      │
│   Loire(미탐험)  Alsace(미탐험)         │
│          Rhone(미탐험)                   │
│   Bordeaux ←── 탭하여 확대 →            │
│          Languedoc(미탐험)               │
│   Provence ●                             │
│                                          │
│ ● 마신 와인  --- 미탐험  █ 확대 가능     │
└──────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 탐험 산지 | 반투명 와인 타입별 fill + 와인 도트 |
| 미탐험 산지 | 점선 윤곽 + dim 텍스트 |
| 확대 가능 산지 | wine 색상 두꺼운 border, "탭하여 확대 →" 안내 |
| 뒤로 버튼 | Level 0으로 복귀 |

#### Level 2: 산지 상세 (예: 보르도)
```
┌─ [← 뒤로] Bordeaux 보르도 ──────────────┐
│ 아직 마신 와인이 없어요                   │
│                                          │
│ [보르도 윤곽 + 지롱드 강]               │
│   Medoc                                  │
│     Pauillac / St-Julien                 │
│     Margaux / St-Estephe                 │
│   St-Emilion / Pomerol                   │
│   Graves / Pessac-Leognan                │
│   Sauternes                              │
│   Entre-Deux-Mers                        │
│                                          │
│ ┌─ 보르도 와인 기록하기 → ─────────────┐ │  ← CTA
│ └──────────────────────────────────────┘ │
│ 지역을 탭하면 해당 산지 와인을 볼 수 있어요 │
└──────────────────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 서브 산지 | 반투명 fill + 서브 AOC 텍스트 |
| CTA | 와인 기록하기 버튼 (wine 색상 dashed border) |
| 빈 상태 | "아직 마신 와인이 없어요" 메시지 |
| 산지 탭 | 해당 산지 와인 리스트 필터 연동 |

### 6-4. 품종 수평 바 차트

- **껍질 두께 순** (얇음→두꺼움) 으로 정렬: 뮈스카 → 리슬링 → … → 카베르네 소비뇽 → 타나
- wine 색상 계열, opacity 차등
- **토글 스위치**: "마신 품종만" ↔ "전체 품종"
  - ON: 마신 적 없는 품종 숨김 (`display: none`)
  - OFF: 전체 표시, 미시음 품종은 `opacity: 0.35` + `—` 표시
- 각 행 클릭 시 해당 품종 와인 필터 연동

### 6-5. 점수 분포

- 식당과 동일 패턴, wine 색상 적용
- 6구간: ~49, 50s, 60s, 70s, 80s, 90s
- 바 탭 시 해당 점수대 와인 필터 연동

### 6-6. 월별 소비

- **식당 차트와 다른 패턴**: 바 높이 = **병 수** 기준 (식당은 금액 기준)
- 바 상단에 병 수 카운트 표시 (숫자가 바 밖 또는 안에)
- 바 하단: 월 라벨 + **금액 pill** 별도 표시
- wine 색상, 총액: wine-light 배경 pill (`총 124만`)
- 최고 금액 월: wine-light+wine 강조
- 바 탭 시 해당 월 와인 필터 연동
- 데이터 소스: `records.purchase_price` (와인 구매 단가)

### 6-7. 타입별 분포

- 수평 바 차트 (와인 전용)
- 타입별 고유 색상:
  - 레드: `#8B2252`
  - 화이트: `#C9A96E`
  - 로제: `#D4A0A0`
  - (스파클링: 목업에 미포함, 데이터 있을 시 추가)

## 7. Wrapped 공유 화면

Spotify Wrapped 스타일의 출판 가능한 카드. 게이지 두 개로 노출 정보를 실시간 조절.

```
┌──────────────────────────────────────┐
│ ← 프로필              [💾 저장]      │
│                   [bubbles] [🔔] [아바타]│
│                                       │
│ [전체] [식당] [와인]   ← 카테고리 필터 │
│                                       │
│ 👁 개인정보  ○───●───○               │
│              최소  보통  공개           │
│                                       │
│ 📋 디테일    ○───●───○               │
│              심플  보통  상세           │
│                                       │
│ ┌─── 미리보기 카드 ──────────────┐    │
│ │  (카테고리별 다른 카드)        │    │
│ └───────────────────────────────┘    │
└──────────────────────────────────────┘
```

### 카테고리별 카드 디자인

| 카테고리 | 그라디언트 | 연도/라벨 | 특징 |
|----------|-----------|-----------|------|
| **전체** | `#3D3833 → #5A4E44 → #C17B5E` | `2026` | 식당+와인 통합 |
| **식당** | `#3D3833 → #5A4E44 → #FF6038` | 🍽 식당 | 식당 전용 태그/레벨/최애 |
| **와인** | `#2A2438 → #4A3D5E → #8B7396` | 🍷 와인 | 와인 전용 태그/레벨/최애 |

### 카드 구성 요소

1. **헤더**: nyam 브랜드 + 연도/카테고리
2. **아이덴티티**: 아바타 + 닉네임 + 한줄 소개 (이탤릭)
3. **숫자 통계**: 카테고리별 3개 (border-top/bottom 구분선)
4. **태그**: pill 형태, 반투명 배경
5. **레벨**: 레벨 pill (lv-purple/lv-blue/lv-green)
6. **올해 최애**: 순위 + 이름 + 메타 + 점수
7. **활동 버블**: 버블 칩 리스트
8. **푸터**: nyam + "나만의 미식/맛집/와인 여정"

### 게이지 조합 매트릭스

| | 개인정보: 최소 | 개인정보: 보통 | 개인정보: 공개 |
|---|---|---|---|
| **디테일: 심플** | 익명 아바타 + 태그만 | 닉네임 + 숫자 + 태그 | + 버블 표시 |
| **디테일: 보통** | + 레벨 표시 | + 레벨 표시 | + 레벨 + 버블 |
| **디테일: 상세** | + 레벨 + 최애 | + 레벨 + 최애 | 전체 공개 |

#### 게이지별 요소 노출 로직

| 요소 | 개인정보 조건 | 디테일 조건 |
|------|-------------|------------|
| 아바타 | ≥ 보통: 실제 아바타 / 최소: 익명 | — |
| 닉네임 | ≥ 보통: 실제 닉네임 / 최소: "미식 탐험가" | — |
| 숫자 통계 | ≥ 보통 | — |
| 태그 | 항상 | — |
| 레벨 | — | ≥ 보통 |
| 올해 최애 | — | = 상세 |
| 버블 | = 공개 | — |

### 카테고리 필터 상세

- **전체**: 식당 + 와인 통합 정보
  - 통계: 기록/식당/와인, 태그: 통합, 레벨: 동네 레벨, 최애: 식당+와인 혼합
- **식당**: 식당 전용
  - 통계: 식당/방문/동네, 태그: 식당 태그, 레벨: 지역 레벨, 최애: 식당만
- **와인**: 와인 전용
  - 통계: 와인/시음/품종, 태그: 와인 태그 (wine 색상), 레벨: 품종 레벨, 최애: 와인만

### 저장/공유

- 카드를 이미지로 렌더링 → 저장 또는 OS 공유 시트
- 반응형 레이아웃: 게이지에 따라 카드 높이가 자동 조절

## 9. 쿼리 전략

프로필 페이지 로드 시 **총 5~6 HTTP 요청**으로 최적화:

| 쿼리 | 테이블 | 방식 | 비고 |
|------|--------|------|------|
| 유저 프로필 | `users` | `.select()` | nickname, avatar, handle, taste_summary, total_xp, record_count 등 |
| 전체 XP 축 | `user_experiences` | `.select().eq('user_id').in('axis_type', [...])` | **5개 axis_type을 IN 절 1회로 조회**, 클라이언트에서 그룹핑 |
| 최근 XP | `xp_histories` | `.select().eq('user_id').order('created_at', desc).limit(5)` | 최근 경험치 |
| 마일스톤 | `user_milestones` + `milestones` | `.select('*, milestones(*)').eq('user_id')` | PostgREST 임베딩 |
| 기록 통계 | `records` | 집계 쿼리 또는 materialized view | 장르별, 점수 분포, 월별 소비, 상황별 등 |
| 활동 히트맵 | `records` | `.select('created_at').eq('user_id')` | 날짜별 기록 수로 히트맵 생성 |

- `level_thresholds`는 소규모 시드 테이블(~100행) → 앱 초기화 시 1회 로드 후 인메모리 캐시
- `user_experiences`의 UNIQUE(user_id, axis_type, axis_value) 인덱스가 IN 절 쿼리 커버
- 통계 데이터가 무거울 경우: Supabase Edge Function으로 집계 → 캐시 고려
