## 2. 섹션별 상세

### Layer 1: 히어로 (사진 캐러셀)
- 높이: h-56 (224px)
- 사진 없을 때: `bg-neutral-100` + 장르 아이콘
- 좌우 스와이프, dot indicator
- **좋아요/공유 버튼**: 사진 우하단, 배경 없는 흰색 아이콘 (`rgba(255,255,255,0.85)`)

### Layer 2: 정보 (일관된 레이아웃)

**기록 유무에 관계없이 항상 동일한 구조.**

- **이름**: 21px, weight 800
- **메타**: 장르 · 지역 · 가격대 (12px, 한 줄)

### Layer 3: 점수 표시 (3가지 모드)

상세페이지는 3가지 상태에 따라 점수 영역이 달라진다:

| 모드 | 점수 영역 | 조건 |
|------|----------|------|
| **내 기록** | 내 점수 (크게) + 버블 칩 (버블별 점수) + 범위 바 | 내가 방문/시음한 식당/와인 |
| **추천** | nyam 점수 (크게) + "웹+명성 기반" 설명 (클릭→산출식) | 안 가본 곳, 버블 기록 없음 |
| **버블 리뷰** | nyam (작게/흐리게) + 버블 평균 (크게) + 버블 칩 + 범위 바 | 안 가본 곳, 버블 기록 있음 |

#### nyam 점수 산출 로직 (식당)

| 요소 | 가중치 |
|------|--------|
| 네이버 (N) | 40% |
| 카카오 (K) | 30% |
| 구글 (G) | 30% |
| → 가중평균 × 20 | = 100점 환산 |
| 명성 보너스 | 미슐랭 +8, 블루리본 +5, TV +3 (상한 15) |
| 최종 | 웹점수 × 0.82 + 명성 × 1.15 |

- N/K/G 점수는 별도 칩으로 표시하지 않음 → nyam 점수 클릭 시 산출식 팝업으로 확인
- 명성 뱃지(미슐랭, 블루리본, TV)는 별도 행에 표시

#### 버블 점수 표시

- 메인: 버블 평균 점수 (큰 숫자)
- 칩: 버블별 개별 점수 (최고=초록 하이라이트, 최저=빨강)
- 범위 바: min–max + avg
- "전체 N명 점수 보기" 확장 버튼

### 뱃지 행
- 미슐랭/블루리본 등 권위 뱃지 (있을 때만 표시, 없어도 레이아웃 영향 없음)

### FAB (+)
- 우하단 52px 플로팅 버튼 (`--primary` 색상)
- 기록 추가 진입점 (기존 "기록 남기기" 액션 바 대체)
- 항상 같은 위치에 표시

### Layer 5: 나의 기록
**기록 있음**: 방문 N회 · 최근 날짜 → 타임라인 (최신순)
- 각 카드: `● 날짜 — 상황칩 | 점수 · 한줄평 | [사진]`
- 탭 → 기록 상세

**기록 없음**: "아직 방문 기록이 없어요" + "우하단 + 버튼으로 첫 기록을 남겨보세요"

**찜만 한 상태**: "찜한 식당이에요. 방문하셨나요?"

### Layer 6: 사분면 분포
- 기록 2개 이상일 때만 표시
- 모든 기록을 2D 좌표 위에 점으로 표시
- **점 fill 색상** = 만족도 기반 (DESIGN_SYSTEM 만족도 색상 매핑: 1~30 레드 → 86~100 블루)
- **점 테두리 색상** = 상황 태그 색상
- **점 크기** = 만족도 (1~100 → 12~54px)
- 내 기록: 불투명, 타인/버블(P2): 반투명(30%)
- 인기 메뉴: 태그 빈도 순 표시

### Layer 7: 실용 정보
- 주소 + 미니 지도 (h-32, 탭→외부 지도 앱)
- 영업 상태: `text-green-600` (영업중) / `text-neutral-400` (종료)
- 전화: `tel:` 링크
- 메뉴판: 접이식

### Layer 8: 식당-와인 연결
- 이 식당에서 와인 기록 있을 때만
- 가로 스크롤 카드: 라벨 썸네일 + 와인명 + 점수

### Layer 9: 버블 기록 (Phase 2)
- Phase 1: 숨김
- Phase 2: 버블별 기록, 좋아요/댓글

## 4. 인터랙션

- 스크롤 < 히어로: 투명 헤더
- 스크롤 >= 히어로: 고정 헤더 + 식당명 (backdrop-blur)
- 기록 CTA: 스크롤 시 하단 floating
- 접이식: 200ms ease-in-out
