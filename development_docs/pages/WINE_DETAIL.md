# WINE_DETAIL — 와인 상세 페이지

> depends_on: DATA_MODEL, RATING_ENGINE, DESIGN_SYSTEM, RECOMMENDATION
> route: /wines/[id]

---

## 1. 식당 상세와의 차이점

| 요소 | 식당 | 와인 |
|------|------|------|
| 히어로 | 사진 캐러셀 (전폭, 가로) | 라벨 이미지 + 텍스트 (좌우 배치) |
| 컬러 | `--primary` #C17B5E | `--wine` #8B7396 |
| nyam 점수 소스 | 웹+명성 (N/K/G + 미슐랭/블루리본) | Vivino+WS+명성 |
| 사분면 축 | 가격×분위기, 색=상황 | 산미×바디, 색=향팔레트 |
| 실용 정보 | 주소/지도/전화/영업/메뉴 | 품종/산지/알코올/온도/디캔팅 (테이블) |
| 연결 | 여기서 마신 와인 | 이 와인을 마신 식당 |
| 추가 섹션 | — | 음식 페어링 태그 |
| 좋아요/공유 | 히어로 사진 우하단 (공통) | 히어로 사진 우하단 (공통) |
| 점수 카드 | 3슬롯 (공통) | 3슬롯 (공통) |
| FAB | `--primary` (공통) | `--wine` (공통) |

## 3. 와인 전용 디자인 토큰

```css
--wine-primary: #6B4C8A;
--wine-primary-light: #F8F5FA;
--wine-tag-bg: #EDE6F5;
--wine-accent: #8B5CF6;
```

### 와인 타입 칩
- 레드: `bg-red-50 text-red-700`
- 화이트: `bg-amber-50 text-amber-700`
- 로제: `bg-pink-50 text-pink-700`
- 스파클링: `bg-sky-50 text-sky-700`
- 오렌지(orange): `bg-orange-50 text-orange-700`
- 주정강화(fortified): `bg-yellow-50 text-yellow-700`
- 디저트(dessert): `bg-amber-100 text-amber-800`

### 사분면 점 색상
- 식당: 만족도 기반 (레드→블루 그라데이션)
- **와인: 향 팔레트 중심 색상** (블랙베리=#3d0a2e, 라임=#2d7a1e 등)

## 4. 기록 타임라인 (식당과 차이)

각 기록에 **식당 연결** 표시:
```
● 2026.03.10 — 페어링
  90점 · "스테이크와 환상 궁합"
  📍 레스토랑 A (탭→식당 상세)
```

## 5. 빈 상태

**점수 카드 3슬롯은 항상 표시, 데이터 없으면 `—`** (레이아웃 불변).

| 섹션 | 빈 상태 | 비고 |
|------|---------|------|
| 점수 카드 | 3슬롯 유지, 값만 `—` 표시 | 레이아웃 불변 |
| 기록 | "아직 기록이 없어요" | FAB로 기록 추가 |
| 사분면 | 섹션 숨김 | 기록 2개+ 시 표시 |
| 페어링/와인정보 | DB 데이터이므로 기록 없어도 표시 | 즉시 유용 |
| 식당 연결 | 섹션 숨김 | — |
| 버블 기록 | "아직 버블 멤버의 기록이 없어요" | — |
