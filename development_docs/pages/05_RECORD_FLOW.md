# RECORD_FLOW — 기록 플로우

> depends_on: RATING_ENGINE, SEARCH_REGISTER, DATA_MODEL, DESIGN_SYSTEM, XP_SYSTEM
> affects: HOME, RESTAURANT_DETAIL, WINE_DETAIL, RECORD_DETAIL
> prototype: `prototype/01_home.html` (screen-add-restaurant, screen-add-wine, screen-rest-record, screen-wine-record, screen-add-success)

---

## 1. 핵심 원칙

**기록은 카메라 촬영 → AI 인식 → 통합 기록 화면 → 저장의 단일 플로우다.**

| 경로 | 설명 | 결과 |
|------|------|------|
| **카메라 경로** | 촬영/앨범 → AI 인식 → 통합 기록 화면 (AI pre-fill) | `status: 'rated'` (사분면·태그·코멘트 등 포함) |
| **검색/목록 경로** | 이름 검색 or GPS 목록 선택 | `status: 'checked'` (최소 데이터만, 풍성화는 상세페이지에서) |
| **상세 FAB 경로** | 식당/와인 상세 페이지 FAB → 바로 통합 기록 화면 | `status: 'rated'` (대상 선택 스킵) |

**카메라 경로가 Primary이며, AI가 최대한 자동 입력한다.**
검색/목록은 폴백이고, 풍성화는 상세페이지에서 언제든 가능.

---

## 2. 진입점

### 홈 FAB (+) → 현재 탭 기반 직접 진입

```
홈 [+] 탭
    ↓
현재 탭 판별 (currentHomeTab)
    ↓
┌─ 식당 탭 ──→ screen-add-restaurant (카메라 촬영)
│
└─ 와인 탭 ──→ screen-add-wine (라벨 촬영)
```

- 바텀시트 없이 현재 활성 탭에 따라 바로 해당 기록 플로우 진입
- FAB 스펙: 44×44px, glassmorphism `rgba(248,246,243,0.88)` + blur(12px), z-index 85 (DESIGN_SYSTEM 참조)

### 상세 페이지 FAB (+)

| 진입 | 동작 |
|------|------|
| 식당 상세 FAB | 식당 선택 스킵 → 바로 식당 기록 화면 (screen-rest-record) |
| 와인 상세 FAB | 와인 선택 스킵 → 바로 와인 기록 화면 (screen-wine-record) |
| 기록 있을 때 | "새 방문 추가" (이전 기록 참조점 표시) |
| 미방문 상태 | "첫 기록 남기기" |

### 기타 진입점

| 진입 | 경로 |
|------|------|
| 추천 카드 "다녀왔어요" | 대상 특정 완료 → 기록 화면 |
| 넛지 스트립 "기록하기" | 사진 감지 기반 → 기록 화면 |
| 재방문/재시음 | 상세 → FAB → 기록 화면 (이전 평가 반투명 참조) |

#### 사분면 인터랙션

| 요소 | 스펙 |
|------|------|
| 참조 점 (기존 기록) | 반투명, 점수 표시, 이름 라벨, 포인터 이벤트 없음 |
| 현재 기록 점 | 불투명, 좌우 드래그로 위치 이동 (가격×분위기) |
| 만족도 조절 | 현재 점 위에서 상하 드래그 → 원 크기(=만족도) 변경 |
| 원 크기 | 만족도 → 점 지름 (12~54px, DESIGN_SYSTEM §8 참조) |
| 수치 표시 | 사분면 아래 칩: 가격/분위기/만족도, 실시간 갱신 |
| 점 색상 | 만족도 기반 5단계 게이지 색상: `--gauge-1`(#C4B5A8) ~ `--gauge-5`(#7A9BAE) (DESIGN_SYSTEM §1 참조) |

#### 와인 연결

```
[⊕ 와인 추가] 탭
    ↓
인라인 검색 입력 (wineSearchArea)
    ↓
┌─ 내 기록에 있는 와인 ──→ 바로 연결 + "내 기록" 뱃지
│
├─ DB에 있지만 내 기록 없음 ──→ 연결
│
└─ 검색 결과 없음 ──→ "새 와인으로 등록" → 와인 추가 플로우 진입
                      완료 후 자동 연결 + 식당 기록으로 복귀
```

| 상태 | 동작 |
|------|------|
| 등록된 와인 (내 기록 있음) | 탭 → 즉시 연결, "내 기록" 뱃지 (wine-light 배경) |
| 등록된 와인 (내 기록 없음) | 탭 → 연결 |
| 미등록 와인 | "새 와인으로 등록" → 와인 추가 플로우 → 완료 후 자동 연결 |
| 연결 해제 | ✕ 버튼으로 연결 제거 |

| 카테고리 | 예시 음식 |
|---------|----------|
| 적색육 | 스테이크 · 양갈비 · 오리 · 사슴 |
| 백색육 | 닭 · 돼지 · 송아지 · 토끼 |
| 어패류 | 생선 · 갑각류 · 조개 · 굴 · 초밥 |
| 치즈·유제품 | 숙성 · 블루 · 브리 · 크림소스 |
| 채소·곡물 | 버섯 · 트러플 · 리조또 · 파스타 |
| 매운·발효 | 커리 · 마라 · 김치 · 된장 |
| 디저트·과일 | 다크초콜릿 · 타르트 · 건과일 |
| 샤퀴트리·견과 | 하몽 · 살라미 · 아몬드 · 올리브 |

#### 아로마 휠 상세

| 링 | 섹터 수 | 항목 |
|----|---------|------|
| Ring 1 (1차향, 외곽) | 8 | 시트러스, 사과/배, 열대과일, 핵과, 붉은베리, 검은베리, 꽃, 흰꽃 |

- 각 섹터 탭 → `active` 토글

## 5. 이미 있는 항목 처리

```
식당/와인 특정
    ↓
DB에서 사용자 기록 조회
    ↓
┌─ 기록 없음 → 저장 → "추가됐어요!" 성공 화면
│
├─ 찜만 있음 → "찜해둔 곳이에요! 방문하셨나요?" → 상세페이지
│                                                 (기록 추가 유도)
│
└─ 기록 있음 → "이전에 기록한 곳이에요" → 상세페이지
                                         (재방문/재음용 기록 유도)
```

## 6. 재방문/재시음 기록

```
상세 → FAB (+)
→ 식당/와인 정보 이미 있음 (특정 스킵)
→ 통합 기록 화면 진입
→ 사분면: 이전 평가 위치를 반투명 참조점으로 표시
→ 새 위치 찍기 (비교 가능)
→ 상황 태그 (이전과 다를 수 있음)
→ 저장 (독립된 새 기록)
```

## 7. 저장 후 성공 화면 (screen-add-success)

```
┌──────────────────────────────┐
│                              │
│         ✓                    │  add-success-check (food/wine 테마)
│    추가되었습니다!             │  add-success-title
│                              │
│   몽탄 · 한식 · 을지로        │  add-success-sub (대상명·메타)
│                              │
│ [내용 추가하기]               │  Primary 버튼 → 상세페이지 (풍성화 유도)
│ [한 곳 더 추가]               │  Ghost 버튼 → restartAdd()
│ [홈으로]                     │  Text 버튼 → exitRecord()
│                              │
└──────────────────────────────┘
```

- **검색/목록 경로**: 이 화면 표시 (추가만 완료)
- **카메라 경로**: 기록 화면에서 저장 완료 후에도 이 화면 표시
- food/wine 테마에 따라 체크 아이콘과 Primary 버튼 색상 변경

## 8. 풍성화 (기록 보강)

검색/목록 경로로 `status: 'checked'`만 된 기록은 상세페이지에서 풍성화 가능.

### 진입점

| 진입 | 경로 |
|------|------|
| 성공 화면 | "내용 추가하기" 버튼 → 상세페이지 |
| 홈 카드 | 미완성 기록 카드 탭 → 상세페이지 |
| 상세 FAB | 식당/와인 상세 FAB (+) → 통합 기록 화면 |
| 넛지 | "기록 완성하기" → 상세페이지 |

풍성화 시 기록 화면에서 사분면·상황·코멘트 등을 채우면 `status: 'rated'`로 UPDATE.

## 9. 데이터 저장

```typescript
// 카메라 경로: AI pre-fill → 사용자 수정 → rated로 저장
{
  target_type: 'restaurant' | 'wine',
  target_id: string,           // UUID (restaurants.id 또는 wines.id)
  status: 'rated',
  satisfaction: number,        // 1~100 (사분면 점 크기 = 만족도)
  axis_x: number,              // DECIMAL(5,2) 0~100. 가격%(식당) / 산미%(와인)
  axis_y: number,              // DECIMAL(5,2) 0~100. 분위기%(식당) / 바디%(와인)
  scene?: string,              // VARCHAR(20) 단일 값. 식당: solo|romantic|friends|family|business|drinks / 와인: solo|gathering|pairing|gift|tasting|decanting (와인은 기록 플로우에서 미수집 — 풍성화 시 추가)
  comment?: string,            // VARCHAR(200) 한줄 코멘트
  companions?: string[],       // TEXT[] 동반자 이름 (⚠️ 비공개 — 본인만 열람)
  companion_count?: number,    // INT 1|2|3|4|5+ (필터/통계용, companions와 별도)
  total_price?: number,        // INT 식당 1인 가격 (₩)
  // 사진은 record_photos 테이블에 별도 저장
  // 와인 전용
  aroma_regions?: object,      // JSONB 아로마 휠 선택 좌표
  aroma_labels?: string[],     // TEXT[] 선택된 아로마 이름
  complexity?: number,         // INT 0~100 복합성
  finish?: number,             // DECIMAL(5,2) 0~100 여운
  balance?: number,            // DECIMAL(5,2) 0~100 균형
  pairing_categories?: string[], // TEXT[] WSET 페어링 카테고리 (예: ['red_meat','cheese'])
  purchase_price?: number,     // INT 와인 병 가격 (₩)
  linked_restaurant_id?: string, // UUID 와인→식당 연결
  linked_wine_id?: string,     // UUID 식당→와인 연결
}

// 검색/목록 경로: 최소 데이터만
{
  target_type: 'restaurant' | 'wine',
  target_id: string,
  status: 'checked',           // 방문/시음 확인만 된 상태
  created_at: timestamp,
  // 풍성화 시 채워질: satisfaction, axis_x/y, scene, comment 등
}
```

### 저장 시퀀스

```typescript
1. records INSERT (status='rated' or 'checked')
2. record_photos INSERT (사진 있으면)
3. wishlists UPDATE (찜 있었으면 is_visited=true)
4. XP 적립 (xp_histories INSERT + user_experiences UPDATE) — rated일 때만
5. 레벨 체크 → 레벨업 알림 (있으면)
// 와인/식당 연결은 records 테이블 필드로 저장 (별도 테이블 없음)
// - 식당 기록 + 와인 연결: records.linked_wine_id
// - 와인 기록 + 식당 연결: records.linked_restaurant_id
```

## 12. Phase 구분

### Phase 1
- 카메라 촬영 → AI 인식 → 통합 기록 화면 → 저장 (전체 플로우)

- 식당 기록: 사분면 + 상황태그 + 코멘트 + 동반자 + 가격 + 와인 연결
- 와인 기록: 사분면 + 아로마 + 구조평가 + 페어링 + 코멘트 + 가격 + 식당 연결
- 재방문/재시음, 수정 기능

### Phase 2
- 버블에 공유 기능 (성공 화면 + 상세페이지)
