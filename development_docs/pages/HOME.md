## 2. 공통 요소

### 2-1. 앱 헤더 (sticky)

```
┌──────────────────────────────┐
│ nyam         ◉  🔔 [J]     │
└──────────────────────────────┘
```

| 요소 | 스펙 |
|------|------|
| 브랜드 로고 | `nyam` 텍스트, 좌측 정렬 |
| 버블 아이콘 | `circle-dot` lucide 아이콘, `icon-btn` (34×34), `/bubbles` 페이지 진입점 |
| 알림 벨 | `icon-btn` (34×34), 알림 있으면 빨간 뱃지 dot |
| 아바타 | 이니셜 원형, 탭 → 프로필/설정 |
| 동작 | sticky top:0, 하단 border |

### 2-2. AI 인사

```
┌──────────────────────────────┐
│ 점심 메뉴 고민 중이세요?       │
│ 이번 주 광화문 쪽을 자주       │
│ 가셨네요 — 오늘은 새로운 데    │
│ 어때요?                      │
│ ● nyam AI · 나의 기록 기반    │
└──────────────────────────────┘
```

| 속성 | 값 |
|------|-----|
| 위치 | 헤더 아래, 넛지 위 |
| 소멸 | **5초 후 자동** (opacity + max-height fade out) |
| 재등장 | 세션 내 1회만 |
| 탭 동작 | `data-restaurant-id` 있으면 해당 식당 상세로 이동, 없으면 무반응 |
| 서브텍스트 | `● nyam AI · 나의 기록 기반` |

#### 시간대별 멘트 풀

| 시간대 | 예시 | 연결 식당 |
|--------|------|-----------|
| 아침 (6~11시) | "어제 스시코우지는 어떠셨어요?" | 스시코우지 |
| 점심 (11~15시) | "이번 주 광화문 쪽을 자주 가셨네요" | 없음 또는 추천 식당 |
| 저녁 (15~21시) | "데이트라면 스시코우지 다시 가셔도" | 스시코우지 |
| 밤 (21~6시) | "이번 주 기록 3건 — 꾸준히 잘 하고 계세요" | 없음 |

- 사용자 기록 기반 멘트 생성 (식당명, 점수, 방문 빈도, 지역 패턴 반영)

### 2-3. 넛지 스트립

최우선순위 1개만 노출. 한 줄 스트립 형태.

```
┌──────────────────────────────┐
│ 📷  사진 감지 · 3/19 12:34   │
│                 [기록] [✕]   │
└──────────────────────────────┘
```

| 요소 | 스펙 |
|------|------|
| 아이콘 | 좌측 원형 (카메라 아이콘, 흰색 stroke) |
| 텍스트 | `<strong>유형</strong> · 날짜/시간` |
| 액션 버튼 | `[기록]` → 기록 플로우 진입 |
| 닫기 | `[✕]` → dismissNudge() |

#### 넛지 우선순위

| 순위 | 유형 | 내용 |
|------|------|------|
| 1 | 사진 감지 | 최근 사진에서 음식/와인 감지 → 기록 유도 |
| 2 | 미평가 기록 | 등록만 하고 평가 안 한 항목 |
| 3 | 식사 후 시간대 | 식사 시간대 이후 기록 유도 |

### 2-4. 콘텐츠 탭

```
[식당]  [와인]        [간단|상세] [아이콘]
  ↑ active              ↑          ↑
  accent 밑줄          보기 토글   식당: 필터
                                   와인: 목록
```

| 요소 | 스펙 |
|------|------|
| 탭 | `식당` / `와인` — 2개 |
| 활성 표시 | 하단 accent 밑줄 (식당: `--primary` #C17B5E, 와인: `--wine` #8B7396) |
| 보기 토글 | `간단` / `상세` pill 토글 (DESIGN_SYSTEM.md "보기 토글" 참조) |
| 우측 아이콘 — 식당 | 필터 아이콘 (filterToggle) → 상황 필터 드로어 토글 |
| 우측 아이콘 — 와인 | 목록 아이콘 (wineListBtn) → /collection (와인 탭 활성, COLLECTION.md 참조) |
| 탭 전환 시 | 우측 아이콘도 같이 교체 (식당↔와인), 보기 모드는 유지 |

### 2-4-1. 보기 토글 (간단 / 상세)

식당·와인 탭 공통으로 적용되는 보기 형식 전환.

| 속성 | 값 |
|------|-----|
| 위치 | 콘텐츠 탭 바 우측, 필터/목록 아이콘 왼쪽 |
| 컴포넌트 | `view-toggle` pill (DESIGN_SYSTEM.md 참조) |
| 기본값 | `상세` (기존 카드 뷰) |
| 상태 공유 | 식당↔와인 탭 전환해도 보기 모드 유지 |

#### 간단 보기

컴팩트 리스트 형태로 이름 + 기본 정보 + 점수만 표시. 한눈에 많은 항목 비교 가능.

```
┌──────────────────────────────┐
│ 1  [썸네일] 스시코우지          92 │
│            일식 · 광화문 · 미슐랭★  │
│───────────────────────────────│
│ 2  [썸네일] 미진               88 │
│            한식 · 광화문 · 냉면    │
│───────────────────────────────│
│ 3  [썸네일] 을지로 골뱅이        85 │
│            한식 · 을지로         │
│───────────────────────────────│
│ ...최대 10개                    │
└──────────────────────────────┘
```

| 요소 | 스펙 |
|------|------|
| 컴포넌트 | `compact-item` (DESIGN_SYSTEM.md "컴팩트 리스트 아이템" 참조) |
| 최대 표시 | 10개 |
| 순위 표시 | 좌측 숫자 (1~3위: accent 색상, 4위~: hint 색상) |
| 썸네일 | 40×40, `border-radius: 10px` |
| 점수 | 우측 정렬, 18px bold (식당: `--primary`, 와인: `--wine`) |
| 미평가 | 점수 자리에 `—` 표시 |
| 정렬 행 | 간단 보기에서는 숨김 (정렬은 점수순 고정) |
| 통계 섹션 | 간단 보기에서는 숨김 |

#### 상세 보기

기존 카드 뷰 그대로 (3-4. 플레이스 카드 / 4-3. 와인 카드 참조).

### 2-5. FAB (+) — 추가 버튼

```
                                     [+]
```

| 속성 | 값 |
|------|-----|
| 위치 | 화면 우하단 고정 |
| 크기 | 56×56 원형 |
| 색상 | 식당 탭: `--primary`, 와인 탭: `--wine` (현재 탭에 따라 자동 전환) |
| 탭 동작 | 현재 탭에 따라 바로 기록 플로우 진입 (바텀시트 없음) |

#### FAB 동작 (탭별 분기)

| 현재 탭 | 동작 | 다음 화면 |
|---------|------|-----------|
| 식당 | 식당 방문 기록 추가 | **카메라 촬영** (음식/식당 사진 인식) + 앨범에서 추가 / 목록에서 추가 (폴백) |
| 와인 | 와인 시음 기록 추가 | **카메라 촬영** (라벨 인식) + 앨범에서 추가 / 이름으로 검색 (폴백) |

- 식당/와인 특정 완료 → 저장 (Phase 1 끝)
- 이미 기록한 항목 → 토스트 알림 + 상세페이지 이동
- 풍성화(점수, 한줄평 등)는 상세페이지에서 진행 (RECORD_FLOW.md Phase 2)

## 3. 식당 탭

### 3-1. 상황 필터 드로어

필터 아이콘 탭 시 토글되는 칩 행.

```
[전체] [혼밥] [데이트] [회식] [가족] [친구]
```

| 속성 | 값 |
|------|-----|
| 기본 | 접힘 (hidden) |
| 칩 | 가로 스크롤, 단일 선택 |
| active 칩 | accent 배경, 흰색 텍스트 |
| 기본 선택 | `전체` |

### 3-2. 서브 네비게이션

```
방문(8) | 찜(3) | 추천(12)
```

| 탭 | 설명 | 카운트 |
|----|------|--------|
| 방문 | 내가 가본 식당 (평가 완료 + 미평가) | 실제 방문 수 |
| 찜 | 찜해둔 식당 | 찜 수 |
| 추천 | AI/권위/버블 추천 | 추천 수 |

#### 추천 서브탭 카운트 포함 범위

추천 카운트는 다음 추천 타입의 합산:
- 재방문 추천 (내 기록 기반)
- 상황별 추천 (상황 태그 기반)
- 사분면 유사 추천 (같은 포지션 식당)
- 찜 리마인드 (오래된 찜 항목)
- 권위 추천 (미슐랭/블루리본 등)
- 버블 추천 (버블 멤버 높은 점수)

> 7가지 추천 타입의 전체 목록은 RECOMMENDATION.md 참조

- active 탭: bold + accent 색상 밑줄
- 카운트: 탭명 옆 `nav-count` 뱃지

### 3-3. 정렬 행

```
5곳                    최신순 ▾
```

| 요소 | 스펙 |
|------|------|
| 좌측 | 현재 필터 결과 수 (`filterSubLabel`) |
| 우측 | 정렬 버튼, 탭하면 순환: 최신순 → 점수순 → 가나다순 |

### 3-4. 플레이스 카드

```
┌──────────────────────────────┐
│ ┌─────┐                      │
│ │사진  │ 미진                  │
│ │     │ 한식 · 광화문 · 냉면    │
│ │[블루 │ [맛][분][가][재]  88   │
│ │리본] │ 🏷나 88 · 혼밥 3/17    │
│ └─────┘ 🏷버블 김영수 93       │
│         🏷웹 N4.4 K4.1 블루리본 │
│         ♡ 4  💬 2             │
└──────────────────────────────┘
```

#### 카드 구성 요소

| 영역 | 요소 | 스펙 |
|------|------|------|
| 좌측 | 사진 컬럼 | gradient 배경 (사진 대체), 약 35% 폭 |
| 좌측 | 뱃지 | 좌하단 오버레이. 블루리본(파랑), 미슐랭(금), TV출연(주황) |
| 우측 | 식당명 | `place-name`, bold |
| 우측 | 메타 | `장르 · 지역 · 시그니처` |
| 우측 | 사분면 미니 | 4칸 컬러 블록 (맛/분/가/재) + 총점 |
| 우측 | 3소스 | 나/버블/웹 태그별 정보 행 |
| 우측 | 인게이지먼트 | ♡ 수 (버블 내 좋아요 합산), 💬 수 (버블 내 댓글 합산) |

#### 3소스 시스템

| 소스 태그 | 색상 | 표시 내용 |
|-----------|------|-----------|
| `나` (me) | accent | 내 점수 · 상황태그 날짜 |
| `버블` (bubble) | purple | 버블 멤버 이름 + 점수 + 한줄평 |
| `웹` (web) | neutral | N(네이버) K(카카오) G(구글) 점수, 수상 이력 |

- 3소스 모두 있는 카드 (예: 미진): 나 + 버블 + 웹
- 2소스 카드 (예: 스시코우지): 나 + 웹
- 버블+웹만 (예: 광화문국밥): 내 방문 없음, 총점 `—` 표시
- 미평가 카드 (예: 토속촌): 웹 소스만, `평가하기 →` 링크

#### 뱃지 종류

| 뱃지 | 스타일 | 예시 |
|------|--------|------|
| 블루리본 | `place-badge blue` | 블루리본 |
| 미슐랭 | `place-badge michelin` | 미슐랭 ★ |
| TV 출연 | `place-badge tv` | 흑백요리사 |

│ ┌─────┐                      │

## 4. 와인 탭

와인 탭 전환 시 우측 아이콘이 필터 → 목록 아이콘으로 변경.

### 4-1. 서브 네비게이션

```
시음(12) | 찜(8) | 셀러(5)
```

| 탭 | 설명 | 대응 와인 목록 분류 |
|----|------|---------------------|
| 시음 | 마신 와인 (시음 기록) | 마신 와인 |
| 찜 | 관심 와인 | 관심 와인 |
| 셀러 | 보유 와인 (셀러/구매) | 보유 와인 |

### 4-2. 카메라 3모드

```
📷 촬영으로 확인하세요

┌──────────┬──────────┬──────────┐
│ 🍷 개별   │ ⊞ 진열장  │ 📄 영수증  │
└──────────┴──────────┴──────────┘
```

| 모드 | 아이콘 | 설명 | 결과 |
|------|--------|------|------|
| 개별 | 와인병 | 1병 라벨 촬영 | 해당 와인 상세 정보 |
| 진열장 | 4분할 그리드 | 여러 병 촬영 | 가격 포함 리스트 |
| 영수증 | 문서 | 영수증 촬영 | 구매 와인/가격 정리 → 보유 와인 등록 |

- 3등분 버튼, `--wine` 색상 아이콘
- 구분선으로 분리

### 4-4. 산지 지도 (드릴다운)

3단계 계층 구조: **세계 → 국가 → 산지**

#### Level 0: 세계지도

|------|------|

| 대륙 | 실루엣 path, `#2a2030` 채움 |

### 4-6. 점수 분포

6등분 막대 차트, 병수 기준.

```
              ┌─┐
         ┌─┐  │5│ ┌─┐
         │ │  │ │ │6│
    ─ ─  │ │  │ │ │ │
   ~49 50s 60s 70s 80s 90s
```

| 속성 | 값 |
|------|-----|
| 구간 | ~49, 50s, 60s, 70s, 80s, 90s |
| 높이 | 병수 비례 (max 기준 100%) |
| 0병 구간 | 최소 높이 2px, `--border` 색상 |
| 1병 이상 | `--wine` 색상, opacity 비례 |
| 숫자 | 짧은 바: 상단 외부, 긴 바: 내부 흰색 |
| 탭 동작 | 구간 탭 → 해당 점수대 와인 필터 |

### 4-7. 월별 소비

6개월 막대 차트, 병수 기준 + 금액 스티커.

```
                   ┌─┐
              ┌─┐  │ │
              │3│  │4│
   ┌─┐  ┌─┐  │ │  │ │
   1  2  1  │ │  1  │ │
  10월 11월 12월 1월 2월 3월
  8만 22만 12만 32만 14만 36만

              총 124만
```

| 속성 | 값 |
|------|-----|
| 범위 | 최근 6개월 |
| 막대 | 병수 비례 높이, `--wine` 색상 |
| 금액 | 각 월 하단 작은 뱃지 (`8만`, `22만` 등) |
| 총액 | 섹션 타이틀 옆 `총 124만` 뱃지 (`--wine-light` 배경) |
| 탭 동작 | 월 막대 탭 → 해당 월 와인 필터 |

## 5. 와인 소셜 (버블 연동)

와인 카드에 버블 멤버 정보를 조용히 표시하는 구조.

### 5-1. 와인 카드 내 버블 행

```
👤박 👤김   박소연 96 · 김영수 88
```

| 요소 | 스펙 |
|------|------|
| 아바타 | `mini-av` — 이니셜 원형 (gradient 배경) |
| 텍스트 | `멤버명 점수 · 멤버명 점수` 또는 `멤버명 점수 · "한줄평"` |
| 최대 표시 | 2명 (점수 높은 순), 3명 이상이면 `+N` 표시 |
| 우선순위 | 점수 높은 멤버 우선, 동점이면 최근 기록 우선 |
| 위치 | 사분면+총점 아래, 날짜 위 |
| 표시 조건 | 같은 와인을 등록한 버블 멤버가 있을 때만 |

### 5-2. 동작 원칙

- 내가 와인 등록 시 → 버블 내 같은 와인 등록자가 있으면 나에게 1회 알림
- 상대방 → 자기 와인 목록에서 카운트 +1 (별도 알림 없음)
- 카드 내 버블 행 탭 → 버블 멤버들의 점수/시음기/가격 확인
- 익명 옵션 가능

| 상태 | 식당 탭 | 와인 탭 |
|------|---------|---------|
