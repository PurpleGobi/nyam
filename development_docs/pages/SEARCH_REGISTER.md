# SEARCH_REGISTER — 검색 & 등록

> depends_on: DATA_MODEL, RATING_ENGINE, DESIGN_SYSTEM

---

## 1. 진입점

| 경로 | 설명 |
|------|------|
| 홈 FAB (+) → 검색/사진 | 가장 일반적 |
| 상세 페이지 "재방문 기록" | 재방문 (검색 스킵) |
| 추천 카드 "다녀왔어요" | 추천→기록 전환 |

---

## 2. 검색 등록 — 식당

### 플로우
```
입력 → 자동완성 (Nyam캐시 → 외부API) → fuzzy merge → 후보 리스트 → 선택 → 평가
```

### 검색 UI
```
┌──────────────────────────────┐
│ 🔍 식당 이름을 입력하세요       │
├──────────────────────────────┤
│  스시코우지                    │  자동완성 후보
│  일식 · 교대역 · ★4.6         │
│  ──────────────────────────  │
│  스시코우지 을지로점            │
│  일식 · 을지로3가 · ★4.3      │
│                              │
│  결과가 없나요? [직접 등록]     │
└──────────────────────────────┘
```

### 자동완성 순서
1. Nyam 캐시 DB (이미 등록된 식당)
2. 사용자 활동 지역 내 결과 우선
3. 외부 API (카카오+네이버+구글 동시)

### fuzzy matching
- 띄어쓰기 무시 ("스시 코우지" = "스시코우지")
- 점/역/호 접미사 무시
- 한영 혼용 허용 ("sushi" → "스시")

### 검색 결과 카드 정보
| 항목 | 소스 |
|------|------|
| 가게명 | 교차 취합 |
| 주소 (간략) | 주요 랜드마크 기준 |
| 장르 | 카카오 카테고리 |
| 대표 사진 | 카카오/구글 |
| 외부 평점 | 최고 1개 표시 |
| 권위 인증 | 미슐랭/블루리본 뱃지 |
| 거리 | GPS 사용 시 |

### 직접 등록 (검색 결과 없을 때)
```
가게명 (필수) → 주소/위치 (선택, 핀 드롭 or 텍스트) → 장르 (선택) → 평가 플로우
```

## 3. 검색 등록 — 와인

### 플로우
```
입력 → 자동완성 (Nyam캐시 → 와인DB API) → 후보 리스트 → 선택 + 빈티지 확인 → 평가
```

### 와인 결과 카드 정보
| 항목 | 소스 |
|------|------|
| 와인명 | 와인 DB |
| 생산자 | 와인 DB |
| 산지 | 와인 DB |
| 품종 | 와인 DB |
| 타입 | 칩 (레드/화이트/로제/스파클링/오렌지/주정강화/디저트) |
| 빈티지 | 사용자 선택/확인 (연도 드롭다운) |
| 라벨 사진 | 와인 DB |

### 빈티지 처리
- 같은 와인 + 다른 빈티지 = 다른 기록
- "빈티지 모름" 선택 가능
- 최근 연도 기본 선택

---

## 4. 사진 등록 — 식당

### 플로우
```
사진 선택/촬영 → EXIF 추출 (GPS, 날짜) → Vision AI 분석 (음식 판별, 장르 추정)
  ├─ GPS 있음: 반경 100m 식당 검색 → AI 장르 교차 → "여기서 드셨나요?" 제안
  └─ GPS 없음: 검색으로 폴백 ("어디서 드셨나요?")
→ 식당 확정 → 날짜 확인 (EXIF or 오늘) → 평가 (사진 자동 첨부)
```

### GPS + AI 교차 매칭
```
GPS 100m 내: 스시코우지(일식), 교대짬뽕(중식), 스타벅스(카페)
AI 분석: "일식/초밥류"
→ 스시코우지 = 최우선 → "스시코우지에서 드셨나요?" [예] [아니요]
```

## 5. 사진 등록 — 와인

### 플로우
```
라벨 사진 → OCR (와인명, 생산자, 빈티지, 산지) → 와인 DB 검색 → 매칭 → 빈티지 확인 → 평가
```

### 라벨 인식 실패 시
```
"와인을 찾지 못했어요" → [이름으로 검색] or [직접 등록]
```

---

## 6. 등록 후 → 평가 연결

### 식당
```
식당 확정 → 자동 채움 (가게명, 주소, 장르, 사진) → RATING_ENGINE 진입
→ 사분면(가격×분위기) → 만족도 → 상황 태그 → (선택) 확장 평가 → 저장
```

### 와인
```
와인 확정 + 빈티지 → 자동 채움 (와인명, 생산자, 산지, 품종) → RATING_ENGINE 와인모드
→ 사분면(산미×바디) → 향 팔레트 → (선택) 복합성/여운/균형 → 만족도 → 상황 태그 → 저장
```

---

## 7. 재방문/재음용 등록

```
상세 페이지 "재방문 기록" 탭
→ 검색 단계 스킵 (식당/와인 정보 이미 있음)
→ 날짜 확인 (기본: 오늘)
→ 사분면 (이전 평가 위치 반투명 표시: "지난번에는 여기였어요")
→ 새 위치 찍기 → 상황 태그 → 저장 (독립된 새 기록)
```

## 8. 예외 처리

| 상황 | 처리 |
|------|------|
| 동명 식당 | 주소/지도로 구분 |
| 프랜차이즈 | 지점별 별도 식당 |
| 폐업 식당 | 기록 아카이브 유지, 신규 등록 경고 |
| 오타/불완전 입력 | fuzzy matching |
| 외부 API 장애 | 캐시 DB만 검색, 없으면 직접 등록 |
| 음식 사진 아님 | "음식 사진을 선택해주세요" |
| EXIF 없음 | 검색으로 폴백 |
| EXIF 날짜 1개월+ | "N개월 전 사진이네요. 맞나요?" |

## 9. 캐싱 정책

| 항목 | 정책 |
|------|------|
| 최초 캐싱 | 사용자 첫 등록 시 외부 API → Nyam DB 저장 |
| 갱신 주기 | 2주 |
| 범위 | 사용자가 등록한 식당/와인만 (전체 크롤링 안 함) |
| 폐업 감지 | 갱신 시 외부 DB 삭제 확인 → 알림 |
