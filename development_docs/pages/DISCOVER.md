# DISCOVER — 탐색 화면

> depends_on: DATA_MODEL, RECOMMENDATION, DESIGN_SYSTEM, ONBOARDING (동네 선택)
> affects: RESTAURANT_DETAIL, WINE_DETAIL, HOME

---

## 1. 화면 역할

- **홈과의 차이**: 홈은 **내 기록 + 개인 추천**, 탐색은 **새로운 곳 발견**
- 사용자가 아직 가보지 않은 식당/와인을 지역, 장르, 권위 기준으로 브라우징
- Phase 1: 지역별 + 권위 추천 + 장르 필터
- Phase 2: 취향 유사 추천 (Collaborative Filtering) 섹션 추가

## 3. 도메인 탭

- 상단 밑줄 탭: **식당** / **와인**
- 식당 활성: `--restaurant-primary` 밑줄
- 와인 활성: `--wine-primary` 밑줄
- DESIGN_SYSTEM 필터 탭 스타일 적용 (밑줄 슬라이드 200ms ease)
- 탭 전환 시 아래 필터 칩 + 목록 전체 교체

---

## 4. 필터 칩

### 4-1. 지역 칩 (식당 탭)

- 온보딩에서 선택한 동네 목록 기반
- 첫 번째 칩: **전체** (기본 선택)
- 나머지: 사용자 선택 동네 (ex. 강남, 홍대, 성수, 광화문)
- 가로 스크롤, 단일 선택
- 스타일: pill `rounded-full px-3 py-1.5 text-sm`
  - 비활성: `bg-neutral-100 text-neutral-600`
  - 활성 (식당): `bg-restaurant-primary text-white`
  - 활성 (와인): `bg-wine-primary text-white`

### 4-2. 장르 칩 (식당 탭)

- [전체] [한식] [일식] [양식] [중식] [아시안] [카페] [바/펍]
- 가로 스크롤, 단일 선택
- 지역 + 장르 조합 필터 (AND 조건)

### 4-3. 와인 탭 필터

- 지역 칩 → **산지** 칩으로 교체: [전체] [프랑스] [이탈리아] [미국] [스페인] [호주] [칠레] →
- 장르 칩 → **타입** 칩으로 교체: [전체] [레드] [화이트] [스파클링] [로제] [내추럴]
- 타입 칩 색상: DESIGN_SYSTEM 와인 타입 칩 색상 적용

## 5. 권위 추천 섹션

### 데이터 소스
- `/api/recommend/authority` (RECOMMENDATION 2-5)
- 미슐랭 / 블루리본 뱃지 보유 식당/와인
- 외부 평점 상위 (N >= 4.3 AND K >= 4.0 AND G >= 4.2)

### 카드 디자인 (가로 스크롤)
```
┌─────────────────┐
│ [대표 사진]       │  160x120, rounded-t-lg
├─────────────────┤
│ 스시코우지         │  H2, semibold
│ 일식 · 강남        │  Small, neutral-600
│ ⭐ 미슐랭 3스타    │  뱃지 칩
│ N4.5  K4.3        │  외부 평점 (Caption)
│ Nyam 92점         │  있을 때만 표시
└─────────────────┘
```

- 카드 너비: 160px 고정
- 최대 10개, 가로 스크롤
- 뱃지 우선순위: 미슐랭 > 블루리본 > 외부 평점 상위
- 와인 탭: "권위 추천" → 유명 와이너리/평론가 평점 기준 (WS, RP 등)

### 권위 뱃지 표시
| 뱃지 | 표시 |
|------|------|
| 미슐랭 1~3 | ⭐ 미슐랭 {n}스타 |
| 블루리본 | 🔵 블루리본 |
| 외부 평점 상위 | 🏆 평점 상위 |

## 6. 지역 인기 섹션

### 데이터 소스
- 선택된 지역의 식당/와인을 복합 점수 기준 정렬
- 복합 점수:
  ```
  score = (외부 평점 평균 정규화) * 0.4
        + (Nyam 사용자 평균 만족도 / 100) * 0.3
        + (Nyam 기록 수 로그) * 0.2
        + (권위 뱃지 보유 여부) * 0.1
  ```
- Nyam 기록이 없는 식당도 외부 평점 기준으로 노출

### 카드 디자인 (세로 리스트)
```
┌──────────────────────────────┐
│ [사진] 식당명                  │  사진 64x64, rounded-lg
│        장르 · 지역             │  Small, neutral-600
│        N4.2  K4.0             │  외부 평점 (Caption)
│        Nyam 88점  (12명 평가)  │  있을 때만
│        ⭐ 미슐랭 1스타          │  뱃지 (있을 때만)
└──────────────────────────────┘
```

- 무한 스크롤 (20개씩 페이지네이션)
- 탭 → 식당/와인 상세 페이지 이동

### 와인 탭 카드
```
┌──────────────────────────────┐
│ [라벨] 와인명                  │  라벨 이미지 48x64
│        타입 · 산지 · 품종       │
│        WS 92  RP 94           │  평론가 점수
│        Nyam 89점  (5명 평가)   │
└──────────────────────────────┘
```

## 7. 취향 유사 추천 (Phase 2)

### 데이터 소스
- RECOMMENDATION 3절 "취향 유사도 (CF)" 알고리즘
- 최소 조건: 전체 사용자 1000+, 개인 기록 20+ (RECOMMENDATION 5절 콜드스타트 기준)

### 화면 구성
```
── 🎯 나와 취향이 비슷한 사람들의 추천 ──

┌────────┐ ┌────────┐ ┌────────┐
│ [사진]  │ │ [사진]  │ │ [사진]  │  가로 스크롤
│ 여문식당 │ │ 라미티에│ │ 밀탑    │
│ 한식·성수│ │ 프렌치  │ │ 면·을지 │
│ 유사88% │ │ 유사85%│ │ 유사82%│
│ Nyam 91 │ │ Nyam 88│ │ Nyam 86│
└────────┘ └────────┘ └────────┘
```

- "유사 N%": 취향 매칭 점수 표시
- Phase 2 이전: 이 섹션 자체가 노출되지 않음

## 9. API 구조

### Phase 1
```
GET /api/discover/authority
  ?domain=restaurant|wine
  &area=강남
  &limit=10
  → 권위 추천 (미슐랭/블루리본/외부 평점 상위)

GET /api/discover/list
  ?domain=restaurant|wine
  &area=강남
  &genre=일식
  &sort=score
  &page=1
  &limit=20
  → 지역 인기 목록 (복합 점수 정렬, 페이지네이션)
```

### Phase 2 추가
```
GET /api/discover/similar-taste
  ?domain=restaurant|wine
  &limit=10
  → 취향 유사 추천 (CF 기반)
```

### 캐싱 전략
- 권위 추천: 24시간 TTL (변경 빈도 낮음)
- 지역 인기: 1시간 TTL
- 취향 유사: 6시간 TTL (CF 재계산 주기)
- 지역 변경 시 해당 지역 캐시 무효화

| 기록 20+ | 동일 | Nyam 중심 | 활성화 |

## 11. 인터랙션

### 스크롤 동작
- 도메인 탭: 상단 고정 (sticky)
- 필터 칩: 도메인 탭 아래 sticky (스크롤 시 상단 고정)
- 아래 콘텐츠만 스크롤

### 카드 탭
- 식당 카드 탭 → `/restaurants/[id]` (식당 상세)
- 와인 카드 탭 → `/wines/[id]` (와인 상세)

### 필터 변경
- 칩 탭 시 즉시 목록 갱신
- 목록 갱신 중 이전 데이터 유지 + 스켈레톤 오버레이 (로딩 상태)
- 스크롤 위치 초기화

### Pull-to-Refresh
- 당겨서 새로고침 지원
- 캐시 무효화 후 데이터 재요청
