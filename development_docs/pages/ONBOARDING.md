## 0. 온보딩 공통 UI 규칙

→ 상세 스타일은 DESIGN_SYSTEM.md "온보딩 네비게이션" / "온보딩 헤더 패턴" / "필터 탭" 참조

### 화면 구조 (식당 온보딩 — 와인은 Section 5 별도 레이아웃)
```
┌──────────────────────────────┐
│  큰 타이틀 (22px, bold, 2줄)  │  ← 스크롤 시 올라감
│  서브텍스트 (14px)            │
├──────────────────────────────┤
│  전체  광화문  을지로          │  ← sticky 밑줄 탭 필터
├──────────────────────────────┤
│  [카드] [카드] [카드] ...     │  ← 스크롤 영역
│                              │
│  [← 이전]         [다음 →]   │  ← 양쪽 pill 버튼, 하단 고정
└──────────────────────────────┘
```

### 네비게이션 버튼
| 요소 | 스타일 |
|------|--------|
| **이전** | 좌하단 고정, 고스트 텍스트 `← 이전` (배경/보더 없음), `font-weight:600`, `color:var(--muted-foreground)` |
| **다음** | 우하단 floating pill, 식당=`var(--restaurant-primary)` / 와인=purple, `font-weight:700`, shadow |
| **마지막 Step** | "시작하기 →" |
| **로그인** | 이전 버튼 없음 |

### 하지 않는 것
- ❌ step dots / 진행 인디케이터 — 공간 낭비
- ❌ floating 통계 — 불필요한 정보
- ❌ 상단 좌측 뒤로가기 원형 버튼 — 하단 고스트 텍스트로 통일
- ❌ Step별 다른 네비게이션 패턴 — 전체 일관성 유지

## 1. 플로우

```
Step 1: 가입
  소셜 로그인 (카카오/구글/애플) → 닉네임 자동 설정

Step 2: 인트로 — "뭘 기록할까요?"
  앱 소개 + 기록 유형 선택 (맛집 / 와인 토글)
  선택에 따라 플로우 분기:
  - 둘 다 → 동네 → 식당 → 와인 → 홈
  - 맛집만 → 동네 → 식당 → 홈
  - 와인만 → 와인 → 홈 (동네 선택 스킵)

Step 3: 동네 선택 (맛집 선택 시에만)
  전국 지도 → 도시 탭 → 상권 버블 선택 (중복 가능)

Step 4: 식당 온보딩 (맛집 선택 시에만)
  선택한 동네의 대표 맛집 표시 (평점 상위 + 미슐랭/블루리본)
  각 식당에 [가봤음] / [가보고싶음] 선택
  - 가봤음 → 버튼 행이 만족도 게이지로 교체 → records에 status='rated'
  - 가보고싶음 → wishlists에 추가
  → "식당은 건너뛰기" 텍스트 링크

Step 5: 와인 온보딩 (사진 인식 기반) ← 건너뛰기 가능
  "혹시 와인도 즐기시나요?"
  사진 인식 또는 이름 검색으로 와인 추가
  - 맞아요 → 버튼 행이 만족도 게이지로 교체 → records
  - 다른 와인이에요 → 재검색 → 바텀시트 대안 제시
  → "와인은 건너뛰기" 텍스트 링크

Step 6: 홈 진입
  → 기록 + 위시리스트가 즉시 채워진 상태 (또는 빈 상태도 OK)
```

## 2. 핵심 결정

### 온보딩에서 하는 것
| 항목 | 방식 |
|------|------|
| 앱 소개 | 인트로 화면 — 뭐하는 앱인지, 온보딩 미리보기 |
| 동네 선택 | 전국 지도 → 상권 버블 (중복, 다도시 지원) |
| 식당 체험 | 가봤음(+점수) / 가보고싶음(찜) — **건너뛰기 가능** |
| 와인 체험 | 사진 인식 / 검색 기반 — **건너뛰기 가능** |

### 온보딩에서 안 하는 것
| 항목 | 이유 |
|------|------|
| 취향 카드 (2지선다) | 기록 데이터에서 자연스럽게 파악 |
| 사분면 평가 | 온보딩에서는 만족도 게이지만. 사분면은 본 기록 플로우에서 |
| 지역 인증 | 나중에 넛지로 유도 |
| 프로필 설정 | 나중에 변경 가능 |

## 2.5. 인트로 화면 — 뭘 기록할까요?

가입 직후, 온보딩 시작 전. 앱 소개 + **기록 유형 선택**.
선택에 따라 온보딩 플로우가 분기됨.

```
┌──────────────────────────────┐
│ 🍽️ 나만의 맛집·와인           │  ← 가치 스토리 3단계
│     컬렉션을 만드세요          │     (아이콘 + 헤드라인 + 설명)
│     가본 곳, 마셔본 와인을 기록 │
│                              │
│ ✨ 기록이 쌓이면              │
│     추천을 받으세요            │
│     상황별 맛집, 어울리는 와인  │
│                              │
│ 💬 버블에서                   │
│     은밀하게 공유하세요        │
│     친구·동료와 소모임을 만들어 │
│     숨은 맛집과 최애 와인을 나눠요 │
│ ────────────────────────────│
│  우선 뭘 기록해볼까요?         │
│                              │
│  ┌──────────────────────[✓]┐│  ← 탭하면 on/off 토글
│  │🍽️ 맛집 기록               ││      선택 시: 테마색 보더+배경
│  │  가본 식당에 점수를 매기고  ││      해제 시: 회색 보더, 체크 해제
│  │  [동네 선택] → [가본 곳 체크]││
│  └──────────────────────────┘│
│  ┌──────────────────────[✓]┐│
│  │🍷 와인 기록               ││
│  │  마셔본 와인을 사진/검색으로 ││
│  │  [사진 인식] 또는 [이름 검색]││
│  └──────────────────────────┘│
│                              │
│  나중에 언제든 추가할 수 있어요  │
│                              │
│       [시작할게요 →]          │
└──────────────────────────────┘
```

### 기록 유형 선택 → 플로우 분기
| 선택 | 온보딩 플로우 |
|------|-------------|
| 🍽️ + 🍷 둘 다 | 동네 선택 → 식당 → 와인 → 홈 |
| 🍽️ 맛집만 | 동네 선택 → 식당 → 홈 |
| 🍷 와인만 | 와인 → 홈 (동네 선택 불필요) |
| 아무것도 안 선택 | CTA 비활성 ("하나 이상 선택해주세요") |

### 카드 토글 상태
| 상태 | 보더 | 배경 | 체크 |
|------|------|------|------|
| 선택 (식당) | `var(--primary)` | `var(--primary-light)` | coral ✓ |
| 선택 (와인) | `var(--wine)` | `var(--wine-light)` | purple ✓ |
| 해제 | `var(--border)` | `var(--bg-card)` | 회색 빈 박스 |

### 규칙
- 기본값: 둘 다 선택 (초기 상태)
- 최소 1개 선택 필수 (0개 시 CTA 비활성)
- 이전 버튼 없음 (가입 직후)
- "나중에 언제든 추가할 수 있어요" — 부담 줄이기

## 3. 동네 선택 UI — 전국 지도 + 상권 버블

2단계 줌: **전국 지도 → 도시 상권 지도**

```
┌──────────────────────────────┐
│  어디서 주로 드시나요?          │
│  지도에서 동네를 탭하세요       │
│  📍 현재 위치: 광화문 근처      │
│                              │
│  ┌────────────────────────┐  │
│  │    [전국 지도]           │  │  Level 1: 도시 버블
│  │   (경기) (인천)          │  │  도시를 탭하면 확대
│  │       (서울●)            │  │  현재 위치 도시 강조
│  │     (대전)               │  │
│  │   (광주)  (대구)         │  │
│  │          (부산)          │  │
│  │     (제주)               │  │
│  └────────────────────────┘  │
│                              │
│  ┌────────────────────────┐  │
│  │  [← 전국]  서울 상권     │  │  Level 2: 상권 버블
│  │   (연남)(홍대)           │  │  탭하면 선택/해제
│  │  (신촌) (광화문)(을지로)  │  │  중복 선택 가능
│  │    ──── 한 강 ────      │  │
│  │   (여의도) (강남)(잠실)   │  │  ← 전국 버튼으로 돌아가기
│  └────────────────────────┘  │
│                              │
│  선택: [서울·광화문 ×] [부산·해운대 ×] [제주·서귀포 ×]  │
│                              │
│              식당 입력 건너뛰기 │  ← 우하단 고스트 텍스트
│  ← 이전              [다음 →] │
└──────────────────────────────┘
```

### 식당 입력 건너뛰기
- 동네 선택 화면 + 식당 온보딩 화면 모두에 표시
- 동네 선택 화면에서 탭: 동네+식당을 통째로 건너뜀
- 와인이 인트로에서 선택되어 있으면 → 와인 온보딩으로, 아니면 → 홈으로

**핵심 플로우**:
1. 전국 지도에서 서울 탭 → 서울 상권 확대
2. 광화문 탭 (선택) → ← 전국 탭 → 부산 탭
3. 해운대 탭 (선택) → ← 전국 → 제주 탭
4. 서귀포 탭 (선택) → 하단에 "서울·광화문, 부산·해운대, 제주·서귀포" 표시
5. [다음 →]

**도시별 상권 데이터**: 서울(16개), 부산(6개), 대구(3개), 제주(6개) 등 주요 도시
**상권 미등록 도시**: "상권 데이터 준비 중 — 직접 입력으로 추가 가능"

**위치 권한 허용 시**: 현재 위치 도시 강조 (pulse 애니메이션)
**위치 권한 거부 시**: 서울이 기본 강조

## 4. 식당 온보딩 UI

```
┌──────────────────────────────┐
│  선택한 동네의                 │  ← 큰 타이틀 (22px, 2줄)
│  대표 맛집이에요 🍽️            │
│  가봤거나 가보고 싶은 곳을 표시 │  ← 서브텍스트 (14px)
├──────────────────────────────┤
│  전체  광화문  을지로           │  ← sticky 밑줄 탭 (동네 필터)
├──────────────────────────────┤
│  ┌──────────────────────────┐│
│  │🍜 미진                    ││
│  │   한식 · 광화문 ⭐블루리본  ││
│  │   N 4.5 · K 4.2          ││
│  │ [🍽️ 가봤음] [💛 가보고싶음] ││  ← 초기 상태
│  │                           ││
│  │   가봤음 클릭 시 ↓ 버튼→게이지 교체 ││
│  │ [████████████░░░░] 78  [×]││  ← 버튼 행이 게이지로 교체
│  └──────────────────────────┘│
│  ...                         │
│                              │
│                식당은 건너뛰기 │  ← 우하단 고스트 텍스트
│  [← 이전]      [다음: 와인 →] │
└──────────────────────────────┘
```

### 건너뛰기
- "식당은 건너뛰기" 텍스트 링크 (우하단, 다음 버튼 위)
- 탭하면 식당 기록 없이 와인 온보딩으로 이동
- 스타일: `font-size:13px`, `color:var(--text-hint)`, 배경/보더 없음

### 동네 필터 탭
- 사용자가 Step 2에서 선택한 동네들이 탭으로 자동 생성
- "전체" 탭 항상 포함 (첫 번째, 기본 active)
- 예: 서울·광화문 + 서울·을지로 선택 → `[전체] [광화문] [을지로]`
- 탭 전환 시 해당 동네 식당만 필터링, 선택 상태(가봤음/가보고싶음/점수)는 유지
- 스타일: DESIGN_SYSTEM.md "필터 탭 (밑줄 스타일)" 참조

### 리스트 구성
선택 동네별 대표 식당 (미슐랭/블루리본 + 외부 평점 상위)

### 가봤음 선택 시
- 버튼 행이 만족도 게이지로 **교체** (아래에 추가가 아니라 같은 자리)
- 게이지 우측 × 버튼으로 취소 (버튼 행 복원)
- 좌우 드래그로 1~100 (색상 변화: 레드(1-30) → 앰버(31-50) → 옐로우(51-70) → 그린(71-85) → 블루(86-100) — DESIGN_SYSTEM 참조)
- `records`에 `status='rated'`, `satisfaction=N`으로 저장 (사분면/상황태그는 미입력)

### 가보고싶음 선택 시
- `wishlists`에 추가

## 5. 와인 온보딩 UI — 사진 인식 / 검색 기반 등록

식당 온보딩과 동일한 카드 리스트 패턴. 팝업/바텀시트 없음.
인식 결과가 카드로 리스트에 바로 추가되고, 식당과 같은 버튼→게이지 교체 패턴.

### 화면 구조
```
┌──────────────────────────────┐
│  혹시 와인도                  │  ← 큰 타이틀 (22px, bold, 보라색)
│  즐기시나요? 🍷               │
│  사진으로 찾거나 이름으로 검색   │  ← 서브텍스트 (14px)
├──────────────────────────────┤
│ [📷 사진으로 찾기] [🔍 이름으로 검색] │  ← sticky 버튼 2개
│   또는: [🔍 와인 이름...] [찾기] [취소]│  ← 검색 모드 시 인풋+찾기 버튼
├──────────────────────────────┤
│                              │
│  🍷 아직 등록한 와인이 없어요   │  ← 빈 상태
│                              │
│  ─── 사진 인식 후 ───         │
│  ┌──────────────────────────┐│
│  │🍷 몬테스 알파 까베르네      ││  ← 인식 결과 카드 (pending)
│  │   레드 · 칠레 · ★4.1       ││
│  │ [맞아요]  [다른 와인이에요]  ││  ← 버튼 행
│  └──────────────────────────┘│
│  ┌──────────────────────────┐│
│  │  🔍 다시 찾고 있어요...    ││  ← "다른 와인이에요" 후 (1초 로딩)
│  └──────────────────────────┘│  → 바텀시트로 대안 목록 제공
│  ┌──────────────────────────┐│
│  │🫧 모엣 & 샹동              ││  ← "맞아요" 누른 후 (confirmed)
│  │   샴페인 · 프랑스 · ★4.1   ││
│  │ [████████████░░░░] 75  [×]││  ← 게이지로 교체 (식당과 동일)
│  └──────────────────────────┘│
│                              │
│  [← 이전]       [시작하기 →]  │
└──────────────────────────────┘
```

### 사진 인식 플로우
```
[📷 사진으로 찾기] 탭
  ↓
갤러리에서 사진 1장 선택
  ↓
로딩 카드 표시 (리스트 내, 1초)
  ↓
인식 결과 카드가 리스트에 추가 (팝업 없음)
  ├─ 높은 확신 → 카드 1장 (pending): [맞아요] / [다른 와인이에요]
  ├─ 낮은 확신 → 후보 카드 2~3장 (pending): 각각 [맞아요] / [다른 와인이에요]
  └─ 실패 → 토스트 "인식이 어려워요 😅 검색해보세요"
```

### 카드 상태 전환
```
pending 카드: [맞아요] [다른 와인이에요]
  ├─ [맞아요] → 버튼 행이 만족도 게이지로 교체 (confirmed, 식당 "가봤음"과 동일)
  └─ [다른 와인이에요] → 카드에 "🔍 다시 찾고 있어요..." 로딩 (1초)
      └─ 바텀시트 올라옴 (대안 와인 목록 + [인풋][찾기] 검색)
      └─ 선택 → 검색중 카드가 confirmed 카드로 교체
      └─ 바텀시트 닫기 (선택 없이) → 검색중 카드 제거
```

### 검색 플로우 (바텀시트)
```
[🔍 이름으로 검색] 버튼 → 버튼 행이 [인풋] [찾기] [취소] 로 교체 (토글)
  └─ 이름 입력 → [찾기] 버튼 또는 엔터 → 바텀시트 올라옴 (검색 결과)
  └─ 결과 선택 → confirmed 카드로 추가 + sticky 버튼 행 복원
  └─ 취소 → 버튼 행 복원
```
검색은 사용자가 이미 와인을 알고 있으므로 pending 단계 없이 바로 게이지 상태로 추가.
바텀시트는 **검색 결과 선택** + **"다른 와인이에요" 대안 제시**에 사용 (사진 인식 결과에는 미사용).

### 바텀시트 구조
```
┌──────────────────────────────┐
│  ── 핸들 ──                  │
│  [🔍 와인 이름 검색...] [찾기]  │  ← 인풋 + 찾기 버튼 (엔터도 가능)
├──────────────────────────────┤
│  🍷 몬테스 알파 까베르네 소비뇽  │  ← 검색 결과 카드들
│     레드 · 칠레 · ★4.1        │
│  🥂 오이스터 베이 소비뇽 블랑    │
│     화이트 · 뉴질랜드 · ★3.9   │
│  ...                         │
└──────────────────────────────┘
```

### 인터랙션 규칙

| 항목 | 규칙 |
|------|------|
| 사진 선택 | 1장씩 처리, 반복 가능 |
| 카드 스타일 | 식당 ob-card와 동일 (보라색 테마만 차이) |
| 버튼→게이지 | "맞아요" 클릭 시 같은 자리에서 교체 (식당 "가봤음"과 동일) |
| 게이지 × 버튼 | 카드 삭제 (식당과 다름: 식당은 버튼 행 복원, 와인은 삭제) |
| 중복 | "이미 등록한 와인이에요" 토스트 |
| 건너뛰기 | 0개여도 [시작하기 →] 진행 가능 |
| 로딩 (사진 인식) | 🍷 "와인을 찾고 있어요..." (1초) |
| 로딩 (대안 검색) | 🔍 "다시 찾고 있어요..." (1초) |
| 초기 만족도 | 75 (기본값, 게이지 드래그로 변경) |
| 만족도 색상 | 보라색 그라데이션 (`#ddd6fe` → `#7c3aed`, 식당의 레드→블루와 다름) |
| 빈 상태 | 🍷 "아직 등록한 와인이 없어요" + "위 버튼으로 와인을 추가해보세요. 건너뛰어도 OK!" |
| 토스트 위치 | 하단 100px (내비게이션 위) |

### 데이터 저장
식당 온보딩과 동일:
- 등록 시 → `records`에 `status='rated'`, `satisfaction=N` (사분면/상황태그 NULL)
- XP 미적립 (0 XP)

### 기술 요구사항
| 항목 | 내용 |
|------|------|
| 와인 라벨 인식 | Vision API (GPT-4o / Claude vision) — 라벨 텍스트 추출 |
| 와인 DB 매칭 | 추출된 텍스트로 내부 와인 DB 검색 (퍼지 매칭) |
| 검색 API | 와인 이름 자동완성 (prefix match + fuzzy) |
| 와인 DB | 온보딩 시드 리스트 불필요. 전체 와인 DB 활용 |
| 이미지 미저장 | 온보딩 인식용 사진은 서버에 저장하지 않음 (인식 후 폐기) |

## 5.5. 온보딩 데이터 저장

### 식당 — 가봤음
```sql
INSERT INTO records (user_id, target_id, target_type, status, satisfaction)
VALUES (:user, :target_id, 'restaurant', 'rated', :score);
-- axis_x, axis_y, scene 은 NULL (나중에 기록 플로우에서 보충 가능)
```

### 식당 — 가보고싶음
```sql
INSERT INTO wishlists (user_id, target_id, target_type)
VALUES (:user, :target_id, 'restaurant');
```

### 와인 — 사진 인식/검색 등록
```sql
INSERT INTO records (user_id, target_id, target_type, status, satisfaction)
VALUES (:user, :wine_id, 'wine', 'rated', :score);
-- 사진 인식 또는 검색으로 매칭된 와인. 사분면/상황태그는 NULL
-- 와인 온보딩에서는 wishlists 생성 없음 (등록만)
```

### 온보딩 기록의 XP
- 온보딩 기록은 만족도만 있고 사분면/상황태그 없음
- **XP 미적립** (0 XP) — 본 기록 플로우에서 사분면 보충 시 XP 적립
- 온보딩 목적은 DB 씨딩이지 XP 적립이 아님

## 6. 넛지 시스템 (온보딩 이후 Week 1~2)

### 넛지 유형 & 우선순위

| 순위 | 유형 | 트리거 | 빈도 | 침습도 |
|------|------|--------|------|--------|
| 1 | 사진 감지 | 앱 진입 시 갤러리 음식 사진 | 매 진입 | 낮음 (앱 내) |
| 2 | 미평가 기록 | 앱 진입 시 | 1일 1회 | 낮음 (앱 내) |
| 3 | 식사 후 넛지 | 위치+시간 (식사 후 1~2시간) | 1일 1회 | 중간 (푸시) |
| 4 | 새 동네 | geofence 진입 | 동네당 1회 | 중간 (푸시) |
| 5 | 주간 리마인드 | 미사용 7일+ | 주 1회 | 중간 (푸시) |

### 넛지 피로 방지
- 하루 최대 푸시 1개
- 앱 내 카드 동시 1개만
- "건너뛰기" 3회 → 해당 넛지 2주 중단
- 23:00~08:00 푸시 없음

### 피로도 점수
```
푸시 전송: +3 | 앱 내 카드: +1 | 무시: +2 | "아니요": +1
기록 완료: -5 (리셋)
피로도 > 10 → 48시간 중단 | > 20 → 1주 중단 | 매일 -1 자연 감소
```
