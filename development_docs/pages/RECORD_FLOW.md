# RECORD_FLOW — 기록 플로우

> depends_on: RATING_ENGINE, SEARCH_REGISTER, DATA_MODEL, DESIGN_SYSTEM, XP_SYSTEM

---

## 1. 전체 플로우

```
등록 완료 (SEARCH_REGISTER)
  ↓
사분면 평가 (RATING_ENGINE)
  ↓ (식당: 가격×분위기, 와인: 산미×바디)
만족도 조절 (점 크기)
  ↓
상황 태그 선택
  ↓
[완료] or [더 추가하기]
  ↓
확장 평가 (선택):
  한줄평, 사진, 메뉴태그, 팁, 가격
  ↓
저장 → 성공 화면
  ↓
[홈으로] or [버블에 공유 (P2)] or [한 곳 더 기록]
```

---

## 2. 식당 기록 화면

### Step 1~4: 사분면 + 만족도
→ RATING_ENGINE 참조 (사분면 UI, 제스처, 애니메이션 전부)

### Step 5: 상황 태그
```
┌──────────────────────────────┐

```

```

│                              │
```

## 3. 와인 기록 화면

### Step 1: 사분면 (산미×바디)
→ RATING_ENGINE 참조

### Step 2: 향 팔레트
```
┌──────────────────────────────┐
│ 어떤 향이 느껴졌나요?          │
│ 손가락으로 영역을 칠해보세요    │
│                              │
│     [원형 향 팔레트]          │
│     드래그로 영역 칠하기       │
│     칠한 영역 = 와인의 향      │
│                              │
│ 감지된 향: 블랙베리, 자두      │
│                              │
│ [건너뛰기]     [다음 →]      │
└──────────────────────────────┘
```

### Step 3~4: 복합성, 여운/균형 (선택)
```
┌──────────────────────────────┐
│ 향의 복합성 (선택)             │
│ [1차 과일 ✓] [2차 효모] [3차 바닐라] │
│                              │
│ 여운: [──●──────] 42%        │
│ 균형: [────────●──] 85%      │
│                              │
│ [건너뛰기]     [다음 →]      │
└──────────────────────────────┘
```

### Step 5: 만족도 확정
- Step 3,4 입력 시: 자동 산출 숫자 → 사용자 상하 드래그로 조정
- Step 3,4 건너뛰기 시: 직접 입력 (식당과 동일)

### Step 6: 상황 태그
```
[혼술] [데이트] [모임] [페어링] [선물] [시음회]
```

### Step 7: 확장 (선택)
- 한줄평, 사진, 페어링 메모, 가격, 구매처/장소 (식당 연결 가능), 빈티지

---

## 4. 재방문 기록

```
상세 → "재방문 기록"
→ 식당/와인 정보 이미 있음 (검색 스킵)
→ 날짜 확인
→ 사분면: 이전 평가 위치를 반투명 점으로 표시
→ 새 위치 찍기 (비교 가능)
→ 상황 태그 (이전과 다를 수 있음)
→ 저장 (독립된 새 기록)
```

## 6. 데이터 저장

```typescript
// 저장 시 처리 순서
1. records 테이블에 INSERT (status='rated')
2. record_photos에 사진 INSERT (있으면)
3. XP 적립 (user_experience UPDATE)
4. 레벨 체크 → 레벨업 알림 (있으면)
5. 뱃지 조건 체크 → 달성 알림 (있으면)
6. wishlists 업데이트 (동일 target → visited=true)
```
