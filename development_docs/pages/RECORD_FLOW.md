# RECORD_FLOW — 기록 플로우

> depends_on: RATING_ENGINE, SEARCH_REGISTER, DATA_MODEL, DESIGN_SYSTEM, XP_SYSTEM

---

## 1. 핵심 원칙

**기록은 2단계로 나뉜다:**

| 단계 | 이름 | 핵심 | 마찰 |
|------|------|------|------|
| Phase 1 | **추가 (등록)** | 식당/와인을 특정하고 저장 | 최소 (탭 2~3번) |
| Phase 2 | **풍성화 (기록 보강)** | 점수, 한줄평, 사진, 태그 등 | 선택적, 언제든 |

**Phase 1만으로 끝내도 완전한 기록이다.**
풍성화는 상세페이지, 목록, 넛지 등 어디서든 자연스럽게 유도.

## 2. Phase 1: 추가 (등록)

### 진입: 홈 FAB (+) — 현재 탭에 따라 바로 분기

```
홈 [+] 탭
    ↓
┌─ 식당 탭 ──→ 카메라 촬영 → AI 인식 → 통합 기록 화면 → 저장
│
└─ 와인 탭 ──→ 라벨 촬영 → AI 인식 → 확인 → 저장
    ↓
┌─ 새 항목 ──→ 저장 → "추가됐어요!" → 끝 or 상세
│
└─ 이미 있음 ──→ "다녀온 적 있어요" 토스트 → 상세페이지 이동
                  (재방문/재음용 기록 유도)
```

### 식당 특정 (카메라 우선)

**Primary: 카메라 촬영 → AI 인식**

```
홈 [+] (식당 탭)
    ↓
┌──────────────────────────────┐
│ 식당 추가                 [✕] │
├──────────────────────────────┤
│                              │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │    카메라 뷰파인더     │    │  ← 음식/식당 사진 촬영 (primary)
│  │  음식 사진을 촬영하세요 │    │
│  │                      │    │
│  │        [◉]           │    │
│  └──────────────────────┘    │
│                              │
│  [📷 앨범에서 추가]           │  ← 폴백 1
│  [🔍 목록에서 추가]           │  ← 폴백 2 (GPS 목록 + 검색)
│                              │
└──────────────────────────────┘
```

#### AI 인식 후 분기

```
사진 촬영 (or 앨범)
    ↓
OCR + 사진 매칭 + GPS 위치 조합
    ↓
┌─ 확실한 매칭 (1곳) ──→ 바로 기록 화면으로 이동
│
└─ 여러 후보 ──→ 후보 목록 표시 → 사용자 선택 → 기록 화면
```

#### 통합 기록 화면 (AI 자동 입력)

```
┌──────────────────────────────┐
│ ← 기록                   [✕] │
├──────────────────────────────┤
│ ┌─────┐ 스시코우지            │  AI 인식 결과
│ │사진  │ 일식 · 을지로 · 120m  │
│ └─────┘ ✨ AI 자동 인식        │
├──────────────────────────────┤
│ 사분면 평가                   │  탭하여 위치 이동
│ ┌──────────────────────┐     │
│ │ 저렴 ──┼── 고가       │     │
│ │ 캐주얼 ──┼── 포멀     │     │
│ │      ●(87)            │     │
│ └──────────────────────┘     │
├──────────────────────────────┤
│ 상황          [AI 추천]       │
│ [데이트✓] [친구] [회식] ...   │  AI가 사진 분석으로 추천
├──────────────────────────────┤
│ 한줄 코멘트                   │
│ [________________________]   │
├──────────────────────────────┤
│ 동반자        [사진 인식]      │  사진에서 얼굴 인식 시 자동
│ 👤김영수  [✕]                 │
│ [+ 동반자 추가]               │
├──────────────────────────────┤
│ 가격 (1인)   [영수증 인식]     │  사진에서 금액 인식 시 자동
│ [85,000원]                   │
├──────────────────────────────┤
│          [저장]               │
└──────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 카메라 촬영 | 음식/식당 사진 → OCR + 이미지 매칭 + GPS 위치 조합 |
| 앨범에서 추가 | 기존 사진 선택 → 동일 AI 인식 |
| 목록에서 추가 | GPS 근처 식당 목록 + 이름 검색 (폴백) |
| AI 자동 입력 | 사분면 위치, 상황 태그, 동반자, 가격을 사진에서 추론 |
| 사분면 | 탭하여 위치 조정 가능 (AI 초기값 제공). 기존 기록 반투명 표시. 원 크기=만족도 |
| 상황 | AI 추천 (사진 분석: 인원수, 분위기 등), 사용자 수정 가능 |
| 동반자 | 사진 얼굴 인식 → 버블 멤버 매칭 시 자동 입력, 수동 추가/삭제 가능 |
| 가격 | 영수증/메뉴판 인식 시 자동 입력, 수동 수정 가능 |
| 와인 연결 | 같이 마신 와인 연결 (아래 참조) |

#### 같이 마신 와인 연결

```
[+ 와인 추가] 탭
    ↓
검색 입력
    ↓
┌─ 내 기록에 있는 와인 ──→ 바로 연결 (연결됨 뱃지)
│
├─ DB에 있지만 내 기록 없음 ──→ 연결 + 나중에 풍성화 유도
│
└─ 검색 결과 없음 ──→ "새 와인으로 등록" → 와인 추가 플로우 진입
                      완료 후 자동 연결 + 식당 기록으로 복귀
```

| 상태 | 동작 |
|------|------|
| 등록된 와인 (내 기록 있음) | 탭 → 즉시 연결, "내 기록" 뱃지 표시 |
| 등록된 와인 (내 기록 없음) | 탭 → 연결 + 와인 records에 linked로 INSERT |
| 미등록 와인 | "새 와인으로 등록" → 와인 추가 플로우 → 완료 후 자동 연결 |
| 연결 해제 | ✕ 버튼으로 연결 제거 |

```

```

| 페어링 | WSET 체계 분류 7종 (적색육/백색육/어패류/치즈·유제품/채소·곡물/매운·발효/디저트·과일) 복수 선택, 각 카테고리 아래 예시 음식 표시, 직접 입력 가능, AI가 품종 기반 추천 |
| 가격 | 영수증/메뉴판 인식 or 수동 입력 |
| 식당 연결 | 어디서 마셨는지 연결 (등록/미등록 분기는 식당 와인연결과 동일) |

### 이미 있는 항목 처리

```
식당/와인 특정
    ↓
DB에서 사용자 기록 조회
    ↓
┌─ 기록 없음 → 새로 저장 → "추가됐어요!" 토스트
│
├─ 찜만 있음 → "찜해둔 곳이에요! 방문하셨나요?" → 상세페이지
│                                                 (기록 추가 유도)
│
└─ 기록 있음 → "이전에 기록한 곳이에요" → 상세페이지
                                         (재방문/재음용 기록 유도)
```

### Phase 1 저장 데이터

```typescript
// Phase 1: 카메라 기반 — AI가 채울 수 있는 건 바로 저장
{
  target_type: 'restaurant' | 'wine',
  target_id: string,         // 식당/와인 DB ID
  status: 'rated',           // 카메라 플로우에서는 바로 rated
  created_at: timestamp,
  // 통합 기록 화면에서 저장 (AI 자동 + 사용자 수정)
  score: number,             // 사분면 기반 만족도
  axis_x: number,            // 가격(식당) / 산미(와인)
  axis_y: number,            // 분위기(식당) / 바디(와인)
  scene_tag: string,         // 상황 태그
  comment?: string,          // 한줄 코멘트
  companions?: string[],     // 동반자 (AI 얼굴 인식 or 수동)
  price?: number,            // 가격 (AI 영수증 인식 or 수동)
  photos: string[],          // 촬영 사진
}

// 목록에서 추가 (폴백) — 최소 데이터만
{
  target_type: 'restaurant' | 'wine',
  target_id: string,
  status: 'added',           // 추가만 된 상태
  created_at: timestamp,
  // Phase 2에서 채워질 필드들: score, comment, photos, tags...
}
```

## 3. Phase 2: 풍성화 (기록 보강)

### 진입점 (어디서든 자연스럽게)

| 진입 | 경로 |
|------|------|
| 상세페이지 | "내용 추가하기" 버튼 |
| 최근 기록 (홈) | 미완성 기록 카드 탭 |
| 내 컬렉션 목록 | "평가하기" 링크 |
| 넛지 스트립 | "기록 완성하기" 탭 |
| 재방문/재음용 | 상세페이지 "다시 기록하기" |

```
│  저렴  ──┼──  고가            │

│    ○을지로65    ●스시코우지87  │  ← 현재 기록 (불투명, 드래그 가능)
│           캐주얼   ○밍글스91  │  ← 기존 기록 (30% 투명)
└──────────────────────────────┘
  가격 65   분위기 65   만족도 87    ← 실시간 수치
  드래그: 위치 이동 · 더블탭: 원 크기 변경
```

| 요소 | 스펙 |
|------|------|
| 기존 기록 참조 점 | 30% 투명, 점수 표시, 이름 라벨, 포인터 이벤트 없음 |
| 현재 기록 점 | 불투명, 드래그로 위치 이동 (가격×분위기) |
| 원 크기 | 만족도 → 점 지름 (16~54px, DESIGN_SYSTEM §7 참조) |
| 크기 조절 | 더블탭으로 3단계 순환 (소→중→대) 또는 스크롤/핀치 |
| 수치 표시 | 사분면 아래에 가격/분위기/만족도 칩, 위치·크기 변경 시 실시간 갱신 |
| 점 색상 | 만족도 기반 게이지 색상 (DESIGN_SYSTEM §1 참조) |

### 풍성화 원칙

- **점진적 공개**: 사분면+태그만 먼저, "더 추가하기"로 확장
- **중간 저장 가능**: 어디서든 저장 가능, 나중에 다시 추가 가능
- **이전 기록 참조**: 재방문 시 이전 사분면 위치를 반투명 점으로 표시

## 4. 재방문/재음용 기록

```
상세 → "다시 기록하기"
→ 식당/와인 정보 이미 있음 (특정 스킵)
→ 날짜 확인
→ 사분면: 이전 평가 위치를 반투명 점으로 표시
→ 새 위치 찍기 (비교 가능)
→ 상황 태그 (이전과 다를 수 있음)
→ 저장 (독립된 새 기록)
```

## 7. 특정의 정확성 요구사항

**식당/와인 특정이 앱의 성패를 결정한다.**

| 항목 | 요구 |
|------|------|
| 식당 GPS 목록 | 실제 위치 기반 정확한 목록. 건물 내 여러 식당 구분 가능해야 함 |
| 식당 검색 | 외부 API (네이버/카카오) 연동. 부정확한 이름에도 매칭 |
| 와인 라벨 인식 | OCR + AI. 빈티지 포함 정확한 매칭. 실패 시 즉시 검색 폴백 |
| 와인 검색 | 외국어 입력 지원. 자동완성 필수. 빈티지 구분 |
| 중복 방지 | 같은 식당/와인 다른 표기 → 동일 항목으로 매칭 |
