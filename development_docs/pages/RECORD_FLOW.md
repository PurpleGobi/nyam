# RECORD_FLOW — 기록 플로우

> depends_on: RATING_ENGINE, SEARCH_REGISTER, DATA_MODEL, DESIGN_SYSTEM, XP_SYSTEM

---

## 1. 핵심 원칙

**기록은 2단계로 나뉜다:**

| 단계 | 이름 | 핵심 | 마찰 |
|------|------|------|------|
| Phase 1 | **추가 (등록)** | 식당/와인을 특정하고 저장 | 최소 (탭 2~3번) |
| Phase 2 | **풍성화 (기록 보강)** | 점수, 한줄평, 사진, 태그 등 | 선택적, 언제든 |

**Phase 1만으로 끝내도 완전한 기록이다.**
풍성화는 상세페이지, 목록, 넛지 등 어디서든 자연스럽게 유도.

---

## 2. Phase 1: 추가 (등록)

### 진입: 홈 FAB (+)

```
홈 [+] 탭
    ↓
┌─────────────────────────┐
│  무엇을 추가할까요?       │  바텀시트
│                          │
│  🍽 식당       🍷 와인    │  두 카드 택 1
│  (현재 탭에 따라 기본 선택) │
└─────────────────────────┘
    ↓
식당/와인 특정
    ↓
┌─ 새 항목 ──→ 저장 → "추가됐어요!" → 끝 or 상세
│
└─ 이미 있음 ──→ "다녀온 적 있어요" 토스트 → 상세페이지 이동
                  (재방문/재음용 기록 유도)
```

### 식당 특정 (현실적 방법)

**Primary: 위치 기반 목록**

```
┌──────────────────────────────┐
│ 식당 추가                 [✕] │
├──────────────────────────────┤
│ 🔍 식당 이름 검색...          │  ← 검색 폴백
├──────────────────────────────┤
│ 📍 근처 식당                  │  ← GPS 기반 (primary)
│                              │
│ ┌─────────────────────────┐  │
│ │ 스시코우지               │  │  거리순 정렬
│ │ 일식 · 120m              │  │
│ ├─────────────────────────┤  │
│ │ 몽탄                     │  │
│ │ 한식 · 350m              │  │
│ ├─────────────────────────┤  │
│ │ 비스트로 드 욘트빌        │  │
│ │ 프렌치 · 500m            │  │
│ └─────────────────────────┘  │
│                              │
│ 목록에 없나요?               │
│ [직접 등록하기]               │
└──────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 근처 식당 | GPS 기반, 반경 500m, 거리순 정렬 |
| 검색 | 이름/지역 입력, 외부 API (네이버/카카오) 연동 |
| 직접 등록 | DB에 없는 식당 수동 등록 (이름 + 위치 최소) |
| 탭 동작 | 식당 선택 → 즉시 저장 → 완료 |

### 와인 특정 (현실적 방법)

**Primary: 라벨 촬영**

```
┌──────────────────────────────┐
│ 와인 추가                 [✕] │
├──────────────────────────────┤
│                              │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │    카메라 뷰파인더     │    │  ← 라벨 촬영 (primary)
│  │    라벨을 맞춰주세요   │    │
│  │                      │    │
│  │        [◉]           │    │
│  └──────────────────────┘    │
│                              │
│  또는                        │
│  [🔍 이름으로 검색]           │  ← 검색 폴백
│  [📷 앨범에서 선택]           │
│                              │
└──────────────────────────────┘
            ↓ (촬영 후)
┌──────────────────────────────┐
│  이 와인이 맞나요?            │
│                              │
│  ┌───────┐ Château Margaux   │
│  │ 라벨   │ Red · Bordeaux    │
│  │ 이미지  │ 2018              │
│  └───────┘                   │
│                              │
│  [맞아요]     [다른 와인이에요] │
└──────────────────────────────┘
```

| 요소 | 설명 |
|------|------|
| 라벨 촬영 | OCR/AI 인식 → 와인 DB 매칭 → 확인 |
| 이름 검색 | 와인명/생산자 입력, 자동완성 |
| 앨범 선택 | 기존 사진에서 라벨 인식 |
| 확인 화면 | AI 매칭 결과 표시 → 맞으면 저장, 아니면 재검색 |
| 빈티지 | 인식 실패 시 수동 입력 유도 |

### 이미 있는 항목 처리

```
식당/와인 특정
    ↓
DB에서 사용자 기록 조회
    ↓
┌─ 기록 없음 → 새로 저장 → "추가됐어요!" 토스트
│
├─ 찜만 있음 → "찜해둔 곳이에요! 방문하셨나요?" → 상세페이지
│                                                 (기록 추가 유도)
│
└─ 기록 있음 → "이전에 기록한 곳이에요" → 상세페이지
                                         (재방문/재음용 기록 유도)
```

### Phase 1 저장 데이터

```typescript
// Phase 1에서 저장되는 최소 데이터
{
  target_type: 'restaurant' | 'wine',
  target_id: string,         // 식당/와인 DB ID
  status: 'added',           // 추가만 된 상태
  created_at: timestamp,
  // Phase 2에서 채워질 필드들: score, comment, photos, tags...
}
```

## 3. Phase 2: 풍성화 (기록 보강)

### 진입점 (어디서든 자연스럽게)

| 진입 | 경로 |
|------|------|
| 상세페이지 | "내용 추가하기" 버튼 |
| 최근 기록 (홈) | 미완성 기록 카드 탭 |
| 내 컬렉션 목록 | "평가하기" 링크 |
| 넛지 스트립 | "기록 완성하기" 탭 |
| 재방문/재음용 | 상세페이지 "다시 기록하기" |

### 풍성화 원칙

- **점진적 공개**: 사분면+태그만 먼저, "더 추가하기"로 확장
- **중간 저장 가능**: 어디서든 저장 가능, 나중에 다시 추가 가능
- **이전 기록 참조**: 재방문 시 이전 사분면 위치를 반투명 점으로 표시

## 4. 재방문/재음용 기록

```
상세 → "다시 기록하기"
→ 식당/와인 정보 이미 있음 (특정 스킵)
→ 날짜 확인
→ 사분면: 이전 평가 위치를 반투명 점으로 표시
→ 새 위치 찍기 (비교 가능)
→ 상황 태그 (이전과 다를 수 있음)
→ 저장 (독립된 새 기록)
```

## 6. 데이터 저장

```typescript
// Phase 1: 추가
1. records 테이블에 INSERT (status='added')
2. wishlists 업데이트 (있으면 visited=true)

// Phase 2: 풍성화 (이후 언제든)
3. records UPDATE (status='rated', score, axis_x, axis_y, ...)
4. record_photos INSERT (사진 있으면)
5. XP 적립 (user_experience UPDATE)
6. 레벨 체크 → 레벨업 알림 (있으면)
7. 뱃지 조건 체크 → 달성 알림 (있으면)
```

## 7. 특정의 정확성 요구사항

**식당/와인 특정이 앱의 성패를 결정한다.**

| 항목 | 요구 |
|------|------|
| 식당 GPS 목록 | 실제 위치 기반 정확한 목록. 건물 내 여러 식당 구분 가능해야 함 |
| 식당 검색 | 외부 API (네이버/카카오) 연동. 부정확한 이름에도 매칭 |
| 와인 라벨 인식 | OCR + AI. 빈티지 포함 정확한 매칭. 실패 시 즉시 검색 폴백 |
| 와인 검색 | 외국어 입력 지원. 자동완성 필수. 빈티지 구분 |
| 중복 방지 | 같은 식당/와인 다른 표기 → 동일 항목으로 매칭 |
