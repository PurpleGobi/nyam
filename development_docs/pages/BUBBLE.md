# BUBBLE — 버블 시스템

> depends_on: DATA_MODEL, AUTH, XP_SYSTEM, DESIGN_SYSTEM
> route: /bubbles, /bubbles/[id]

---

## 1. 핵심 원칙

- **기록이 먼저, 소셜은 그 다음** — 소셜 없이도 유용
- **신뢰가 네트워크의 핵심** — 경험치 기반 가입 조건으로 데이터 오염 방지
- **공유는 선택적** — 원하는 기록만 특정 버블에
- **피드는 시간순** — 알고리즘 피드 없음 (신뢰 보호)
- **버블은 가변적** — 멤버 탈퇴/제거로 데이터 오염 정화 가능

### 핵심 비전

nyam은 개인 기록용 앱이지만, **버블의 네트워크 기능이 신뢰할 만한 맛집/와인 추천을 이어가는 것이 핵심 가치**다.
홈 화면 추천에도 버블 내 공유 데이터가 내부적으로 반영된다 (→ RECOMMENDATION.md 참조).

## 2. 버블 생성

- **누구나 자유롭게, 무한정 생성 가능** (단톡방처럼)
- 1인 버블 가능 — 외부 공유용 큐레이션 채널로 활용
- 여러 명이 아니어도 됨

```
1. [+] 버블 만들기
2. 이름 (20자) + 설명 (100자, 선택)
3. 정체성: 모두 | 식당 | 와인
4. 노출도: private | members | public
5. 공개 수위: 평점만 | 평점+코멘트
6. 권한 설정: 댓글 허용, 외부 공유 허용
7. (members일 때) 가입 조건: 최소 경험치 레벨
8. 완료 → 버블 상세
```

## 3. 버블 속성

### 3-1. 노출도 (visibility)

| 값 | 검색 | 가입 방식 | 사용 예 |
|------|------|---------|--------|
| `private` | 불가 | 초대 링크/직접 초대만 | 직장 맛집, 가족 |
| `members` | 가능 | 가입 조건 충족 시 가입 (기본 조건: 경험치) | 동네 맛집 커뮤니티 |
| `public` | 가능 | 누구나 자유 가입 | 오픈 큐레이션 |

- `members` 버블의 가입 조건은 **경험치가 기본**이 됨 → 데이터 오염 방지의 핵심 메커니즘
- bubble owner가 최소 경험치 레벨을 설정

### 3-2. Owner 권한 설정 (AND 조건으로 조합)

| 설정 | 설명 | 기본값 |
|------|------|--------|
| `allow_comments` | 댓글 허용 여부 | true |
| `allow_external_share` | 외부 공유 허용 여부 | false |

### 3-3. 공개 수위 (exposure_level)

**버블에 공유된 기록이 비멤버에게 어떻게 보이는가** (→ SETTINGS.md §4-2 상세 매트릭스)

| 값 | 버블 멤버에게 | 비멤버에게 (상세 페이지 L9) |
|------|------------|-------------------------|
| `rating_only` | 점수 + 한줄평 + 사진 + 메뉴 전체 | 점수만 (아바타 + Lv + 만족도 숫자) |
| `rating_and_comment` | 점수 + 한줄평 + 사진 + 메뉴 전체 | 점수 + 한줄평 |

- `records.comment` 한줄평이 공유 텍스트 — 별도 "버블 전용 코멘트" 필드 없음
- 한줄평 없이 공유하면 점수만 표시
- 멤버에게는 공유 시점에 노출 동의한 것으로 간주

### 3-4. 버블 정체성 (identity)

| 값 | 의미 |
|------|------|
| `all` | 식당 + 와인 모두 |
| `restaurant` | 식당 중심 |
| `wine` | 와인 중심 |

- 정체성을 정한다고 **제한이 생기지는 않음** — 노출되는 라벨일 뿐
- 언제든 변경 가능

## 4. 버블 사용 시나리오 (중요도 순)

1. **지인 간 맛집/와인 경험 공유** — 직장 동료, 친구, 가족
2. **생소한 동네에서 믿을 만한 맛집 추천 받기** — members/public 버블 follow

### 4-1. 취향 유사도 매칭 (public/members 버블 탐색 시)

공개 버블을 탐색할 때, 내 기록과 버블 멤버들의 기록을 비교하여 **취향 유사도**를 표시한다.

#### 원리

- 내가 기록한 식당/와인과 **같은 대상을 리뷰한 버블**이 비교 대상
- 동일 대상의 **점수 오차 ±5% 이내**면 "취향 일치"로 판정
- 겹치는 대상이 많고 점수 오차가 작을수록 유사도 높음

#### 유사도 계산

```
겹치는 대상 수 = 내 기록 ∩ 버블 기록 (같은 식당/와인)
일치 수 = 겹치는 대상 중 점수 오차 ±5% 이내
유사도 = 일치 수 / 겹치는 대상 수 × 100%
```

#### 표시

```
┌─────────────────────────────────┐
│ 🔍 을지로 맛탐정 클럽            │
│ 멤버 23명 · 기록 156개           │
│ 📊 취향 유사도 78% (9/12곳 일치)  │  ← 내 기록과 겹치는 12곳 중 9곳 점수 유사
│ [가입 신청]                      │
└─────────────────────────────────┘
```

- **겹치는 대상이 3개 미만**이면 유사도 미표시 ("데이터 부족")
- members/public 버블에서만 표시 (private은 검색 불가이므로 해당 없음)
- 유사도순 정렬 옵션 제공

### 4-2. 버블 검색 + 필터

#### 검색바
| 속성 | 값 |
|------|-----|
| 위치 | "버블 탐색" 섹션 상단 |
| 입력 | 버블 이름, 지역, 키워드 실시간 필터링 |
| 필터 토글 | 우측 슬라이더 아이콘 → Notion 스타일 필터 패널 열기 |

#### 필터 속성 (7종)
| 필터 | 옵션 |
|------|------|
| 지역 | 을지로, 광화문, 강남, 홍대, 성수, 이태원, 종로, 전체 |
| 유형 | 양방향, 열람 전용, 전체 |
| 멤버 수 | 5명 이하, 5~10명, 10~20명, 20명+, 전체 |
| 취향 유사도 | 80%+, 60%+, 40%+, 전체 |
| 카테고리 | 맛집, 와인, 혼합, 전체 |
| 활동량 | 활발(주3+), 보통(주1+), 조용, 전체 |
| 공개 여부 | 공개, 비공개, 전체 |

→ Notion 스타일 필터 엔진 (DESIGN_SYSTEM.md 참조)
→ And/Or 접속사, 복합 조건 지원

#### 탐색 방법 (4종)
| 탭 | 설명 | 표시 정보 |
|----|------|----------|
| 추천 | 취향 유사도 기반 AI 추천 | 유사도 %, 겹치는 식당/와인 칩 |
| 근처 | 지역 기반 | 거리 표시 |
| 인기 | 활동량 순 | 이번 주 기록 수 |
| 새로운 | 최신 생성 | 생성일 |

## 5. 핵심 구동 원칙

### 5-1. 경험치 기반 데이터 신뢰

- `members` 버블의 **가입 조건 기본 = 경험치** → 데이터 오염 상당 부분 방지
- 문제 확인 시 **해당 멤버를 제거하면 오염 정화** 가능
- 데이터는 고정이 아니라 **개인별로 가변적** — 멤버 추가/제거에 따라 버블 데이터 동적 변화
- 개인이 **능동적으로 버블 탈퇴**함으로써도 오염 정화 가능

### 5-2. 오염 감지 (추후 구현)

- bubble owner에게 abuser/오염원을 **자동 경고**하는 기능 예정
- 이상 패턴 감지: 과도한 고평점, 특정 식당 집중 공유 등

### 5-3. 홈 추천과의 연동

- 홈 화면의 추천 식당/와인은 표면적으로 기본 노출 데이터이나
- **내부 로직에서 버블 내 공유 데이터 중 평이 좋은 것을 반영**
- `members`/`private` 버블의 경우 **버블 존재 자체가 노출되지 않으면서** 추천에만 영향

## 6. 공유 플로우

> `privacy_records = 'private'`이면 공유 버튼 비활성. 상세 규칙: SETTINGS.md §3~§4

```
방법 1: 기록 직후 → "버블에 공유하기" → 버블 선택 → 공유
방법 2: 기존 기록 → 공유 아이콘 → 버블 선택
방법 3: 필터 일괄 → "광화문 + 80점 이상" → 5개 한번에 공유
```

**공유 시 포함**: 식당/와인명, 사분면 좌표, 만족도, 상황, 한줄평, 사진, 메뉴, 팁, 날짜
**버블 내에서만 공개**: 가격 (프로필 방문자에게는 비공개)
**항상 비공개**: 동반자 정보 (나만 열람)
**공유 취소**: 언제든 철회 가능 (달린 댓글은 고아 상태로 유지)
**버블 간 격리**: 기록 A를 버블 X에 공유해도 버블 Y에는 보이지 않음

## 9. 버블 상세 페이지

#### 레이아웃
- 내부 페이지 헤더 (← 버블 + bubbles/알림/아바타) — DESIGN_SYSTEM §내부 페이지 헤더 참조
- 히어로: 버블 아이콘(크게) + 이름 + 설명 + 멤버 수 + 취향 유사도 배지
- 퀵 통계: 총 기록 | 평균 점수 | 이번 주 활동
- 멤버 아바타 가로 스크롤
- 하단 좌측: fab-back (뒤로가기)

#### 탭 (5개)
| 탭 | 내용 |
|----|------|
| 피드 | 멤버 기록 타임라인 (점수 + 한줄평 + 리액션) |
| 식당 | 버블 내 식당 랭킹 (정렬 가능) |
| 와인 | 버블 내 와인 랭킹 |
| 통계 | 공유 랭킹, 인기 식당, 장르 분포, 사분면 분포 |
| 겹치는 곳 | 나와 버블이 공통 방문한 식당/와인 + 점수 비교 (나 vs 버블 평균) |

## 11. 식당/와인 상세 — 소셜 레이어

상세페이지 Layer 9에 버블 기록 표시:
- 내 기록 아래에 배치
- 버블 필터 가능 (복수 소속 시)
- 각 기록: 아바타 + 닉네임 + 버블명 + 만족도 + 사분면좌표 + 한줄평
- 최대 5개 + "더보기"
- 사분면 차트에 버블 멤버 점도 반투명(30%)으로 표시

## 12. 경험치 & 소셜 신뢰

- 기록 옆에 **관련 레벨만** 표시 (을지로 기록 → 을지로 Lv.7)
- "경험자 순" 정렬 = 필터/정렬 옵션. 자동 가중치 아님
- **버블 활동은 경험치에 기여** (공유, 리액션, 댓글 등 → XP_SYSTEM.md 참조)
