# Nyam v2 PRD (Product Requirements Document)

> 최종 갱신: 2026-03-20 | Phase 1 구현 범위 기준

---

## 1. 제품 정의

**한 줄 요약**: 나의 식도락 여정을 기록하고, 분석하고, 신뢰할 수 있는 사람들과 공유하는 앱

**핵심 가치 3가지**:
1. **기록** — 3초 사분면 평가로 식당/와인 경험을 구조화
2. **분석** — 상황별(혼밥/데이트/회식) 취향 패턴 자동 발견
3. **신뢰** — 버블(폐쇄형 그룹)에서 아는 사람의 경험만 공유

**Nyam이 아닌 것**:
- 네이버/카카오 같은 공개 리뷰 플랫폼이 아님
- 캐치테이블 같은 예약 서비스가 아님
- 배달/요리 앱이 아님
- **범위: 식당 + 와인만** (배달, 요리, 카페 제외)

---

## 2. 차별점

| 기존 앱 | 방식 | Nyam |
|---------|------|------|
| 네이버/카카오 | 인기도/광고 기반 공개 리뷰 | 내 기록 기반 개인화 + 폐쇄형 신뢰 |
| Vivino | 와인만, CF 추천 | 식당+와인 통합, 상황 인식 |
| 인스타그램 | 비구조화된 사진 공유 | 구조화된 3차원 평가 (위치+크기+상황) |
| Letterboxd (영감) | 영화 다이어리+소셜 | 식도락 다이어리+소셜 (상황 태그 추가) |

**시그니처 기능**: 사분면 평가
- 2D 좌표(특성 2축) + 점 크기(만족도) + 상황 태그 = 3초에 4차원 데이터
- 기록이 쌓이면 사분면 위에 "나만의 취향 지도" 자동 생성

---

## 3. Phase별 범위

### Phase 1: 개인 기록 (MVP)
- 온보딩 (30초: 동네 선택 + 가본 곳 체크)
- 식당/와인 검색 및 등록 (텍스트 + 사진)
- 사분면 평가 (식당: 가격×분위기, 와인: 산미×바디)
- 와인 향 팔레트
- 상세 페이지 (식당/와인)
- 홈 대시보드 (내 기록 + 추천)
- 위시리스트
- 규칙 기반 추천 (재방문, 상황별, 권위)
- 경험치/레벨 (지역별, 와인 타입별)
- 넛지 시스템 (위치, 사진, 미평가)

### Phase 2: 소셜
- 버블 (폐쇄형 그룹) 생성/가입/공유
- 버블 피드, 댓글, 리액션
- 팔로우
- 프로필 (레벨 뱃지, 사분면 분포)
- 소셜 뱃지

### Phase 3: 수익화
- 프리미엄 통계/대시보드
- Wrapped (연간 결산)
- 크리에이터 채널 (유료 버블)
- 로컬 파트너십

---

## 4. 타겟 유저

**Primary**: 25-40세, 주 2회 이상 외식, 기록 습관 있거나 만들고 싶은 사람
**Secondary**: 와인 입문~중급, 와인 모임 참여자

---

## 5. 기술 스택

- **Frontend**: Next.js (App Router) + TypeScript + Tailwind + shadcn/ui
- **Backend**: Supabase (PostgreSQL + Auth + RLS + Edge Functions + Storage)
- **AI**: Gemini Vision (사진 분석, OCR)
- **외부 API**: 카카오맵, 네이버 지도, 구글 Places (식당), 와인 DB API (TBD)
- **모바일**: PWA 우선, 네이티브 전환 검토 (Phase 2+)

---
