# RATING_ENGINE — 사분면 평가 시스템

> affects: RECORD_FLOW, SEARCH_REGISTER, RESTAURANT_DETAIL, WINE_DETAIL, HOME, RECOMMENDATION, PROFILE
> 비주얼 레퍼런스: `prototype/01_home.html` (screen-rest-record, screen-wine-record), `prototype/00_design_system.html` (§15, §15b)

## 1. 핵심 개념

### 식당 평가 (Quick)
**하나의 평가 = (X%, Y%, 만족도, 상황태그)**
- ~3초 안에 완료되는 4차원 데이터
- 사분면 위치(음식 퀄리티×경험 가치) + 만족도((x+y)/2 자동 산출) + 상황 태그

### 와인 평가 (Deep)
**하나의 평가 = (X%, Y%, 만족도, 아로마, 구조, 페어링)**
- WSET 체계적 시음 기반
- 사분면(구조·완성도×즐거움·감성) + 아로마 팔레트 + 구조평가(복합성·여운·균형) + 페어링 카테고리

## 2. 사분면 축 정의

### 식당
| 축 | 0% | 100% | 의미 |
|----|-----|------|------|
| X축: 음식 퀄리티 | 아쉬움 | 훌륭함 | 맛 중심의 음식 품질 |
| Y축: 경험 가치 | 낮음 | 높음 | 서비스 + 분위기 + 가성비 종합 |
| 만족도 | 1 | 100 | (axis_x + axis_y) / 2 자동 산출 |

```
           경험 가치 높음
              ↑
맛은 아쉽지만  |  맛도 좋고
경험은 좋은    |  경험도 좋은
              |
음식 아쉬움 ──┼── 음식 훌륭함
              |
맛도 아쉽고   |  맛은 좋지만
경험도 아쉬운  |  경험은 아쉬운
              ↓
           경험 가치 낮음
```

### 와인
| 축 | 0% | 100% | 의미 |
|----|-----|------|------|
| X축: 구조·완성도 | 낮음 | 높음 | "이 와인은 객관적으로 얼마나 잘 만들어졌나" |
| Y축: 즐거움·감성 | 낮음 | 높음 | "내가 실제로 마시면서 얼마나 만족했나, 가격 포함" |
| 만족도 | 1 | 100 | (axis_x + axis_y) / 2 자동 산출 |
| **색상: 향** | — | — | 향 팔레트 중심 색상 (`aroma_color`, 와인만) |

```
         즐거움·감성 높음
              ↑
구조는 단순하지만|  잘 만들어졌고
마시기 즐거운   |  마시기도 즐거운
              |
구조 낮음 ────┼──── 구조 높음
              |
구조도 단순하고 |  잘 만들어졌지만
즐겁지도 않은  |  즐겁진 않은
              ↓
         즐거움·감성 낮음
```

## 3. 조작 플로우

사분면 위치와 만족도를 **동시적 제스처**로 조작한다. 모드 전환 없이 드래그 방향으로 구분.

| 단계 | 행동 | 제스처 | 시간 |
|------|------|--------|------|
| 1. 탭/드래그 | 사분면에서 위치 선택·이동 | 점을 자유롭게 드래그 (X축+Y축) | ~1초 |
| 2. 수치 확인 | 축 값 + 만족도((x+y)/2) 실시간 표시 | — | — |
| 3. 완료 | 저장 버튼 | 1-finger tap | ~0.3초 |

> 만족도는 (axis_x + axis_y) / 2로 자동 산출. 별도 만족도 조절 제스처 없음.

### 제스처 구분
- **위치 이동**: 점을 사분면 내에서 자유롭게 드래그 → X%, Y% 실시간 갱신
- **만족도**: 위치 이동 시 (x+y)/2로 자동 계산, 실시간 갱신
- 점 바깥 영역 탭 → 점이 해당 위치로 이동 (위치 재설정)

## 4. 점(Dot) 비주얼 스펙

### 만족도 → 게이지 색상 (5단계)

| 만족도 | 변수 | hex | 이름 |
|--------|------|-----|------|

| 21–40 | `--gauge-2` | `#B0ADA4` | 쿨그레이 |
| 41–60 | `--gauge-3` | `#9FA5A3` | 세이지 |
| 61–80 | `--gauge-4` | `#889DAB` | 스틸블루 |
| 81–100 | `--gauge-5` | `#7A9BAE` | 슬레이트 |

### 만족도 → 점 크기 매핑 (사분면 표시용)

DESIGN_SYSTEM §15 정의.

| 만족도 | 지름 | 게이지 |
|--------|------|--------|
| 1–20 | 14px | gauge-1 |
| 21–40 | 20px | gauge-2 |
| 41–60 | 26px | gauge-3 |
| 61–80 | 36px | gauge-4 |
| 81–100 | 48px | gauge-5 |

### Circle Rating (기록 플로우 인터랙티브)

DESIGN_SYSTEM §15b 정의. 기록 화면에서 직접 조작하는 점.

| 속성 | 값 |
|------|-----|
| 크기 범위 | 28px (점수 0) → 120px (점수 100) |
| 색상 | 만족도 기반 5단계 게이지 색상 (fill = glow) |
| Glow | `box-shadow: 0 0 {size×0.3}px {fillColor}80` |
| 숫자 | weight 800, color #fff, font-size = 원 지름의 30% |
| 인터랙션 | 상하 드래그로 점수 조절 (위=증가, 아래=감소) |

### Glow 레벨 (CSS)

| 레벨 | box-shadow | 효과 |
|------|-----------|------|
| glow-1 | `0 0 16px 4px {color}50%` | 기본 글로우 |
| glow-2 | `0 0 24px 6px {color}65%` | + brightness(1.1) |
| glow-3 | `0 0 32px 10px {color}80%` | + brightness(1.2) |

- 식당 glow 색상: `rgba(122,155,174,*)` (슬레이트 계열)
- 와인 glow 색상: `rgba(139,115,150,*)` (와인 계열)

### 참조 점 (기존 기록)

| 속성 | 값 |
|------|-----|
| opacity | 0.3 |
| 숫자 | 9px, weight 700, color white |
| 이름 라벨 | 9px, `--text-hint`, 점 아래 배치 |
| 포인터 이벤트 | 없음 (pointer-events: none) |

## 6. 배경 레퍼런스

사분면 배경에 반투명 점으로 이전 기록/대표 식당·와인 표시:

**기록 플로우에서 (screen-rest-record, screen-wine-record)**:
- 내 과거 기록을 반투명 참조 점으로 표시
- 점 크기 = 만족도 기반 (§4 매핑)
- 점수 숫자 + 이름 라벨 표시
- opacity 0.3, pointer-events 없음

**상세 페이지에서 (detail_restaurant, detail_wine)**:
- "내 식당/와인 지도" 섹션
- 현재 대상: 불투명 (current), 다른 기록: 반투명 (ref)
- 같은 가격대·지역(식당) 또는 유사 와인과의 상대적 위치

**데이터 소스 우선순위**:
1. 자신의 과거 기록 (primary — 기록 플로우·상세 공통)
2. 같은 지역 사용자 평균 좌표 (데이터 충분 시)
3. 미슐랭/블루리본 사전 배치 (콜드스타트)

**표시 규칙**:
- 최대 8~12개
- 투명도: 0.3 (30%)
- 확대 시에만 이름 표시 (가까운 3~4개)

## 7. 상황 태그 (식당 전용)

식당 기록에서 사분면 평가 후 상황 태그를 선택한다.

| DB 값 | 표시 이름 | 설명 |
|--------|----------|------|
| `solo` | 혼밥 | 1인 식사 |
| `romantic` | 데이트 | 연인 |
| `friends` | 친구 | 친구 모임 |
| `family` | 가족 | 가족 식사 |
| `business` | 회식 | 업무/비즈니스 |
| `drinks` | 술자리 | 음주 중심 |

- **단일 선택** (`scene VARCHAR(20)`)
- **AI 추천**: 사진 분석, 시간대, 위치 기반으로 pre-select (`ai-tag` 뱃지 표시)
- **와인 기록**: 와인 기록 생성 플로우에는 상황 태그 입력 UI가 없음 (페어링이 대체). 단, DB 스키마상 `scene` 필드는 존재하며, 와인 상세 타임라인에서는 "혼술" 등으로 표시될 수 있음

## 8. 와인 심화 평가

식당은 사분면 + 상황태그로 끝. 와인은 WSET 체계적 시음 기반 추가 단계:

| Step | 내용 | 필수 |
|------|------|------|
| 1 | 사분면 (산미×바디) + 만족도 | 필수 |
| 2 | 아로마 팔레트 (WSET 3링 휠) | 필수 |
| 3 | 구조 평가 — 복합성 · 여운 · 균형 (슬라이더 3개) | 선택 |
| 4 | 페어링 (WSET 8카테고리 그리드) | 필수 |
| 5 | 만족도 확정 (자동산출 + 조정) | 필수 |

### 아로마 팔레트 상세 스펙

**레이아웃**: 원형 (시계 방향 12시부터)
- 15섹터, 3개 링 (1차향/2차향/3차향)
- AI가 품종/라벨 기반으로 pre-select (`AI 감지` 뱃지 표시)

**1차향 (과일/꽃) — 바깥 링 (8섹터)**:
| 위치 | 향 | 색상 | hex |
|------|-----|------|-----|
| 12시 | 시트러스 (레몬/라임) | 밝은 노랑 | `#fde047` |
| 1시 | 사과/배 | 연두 | `#a3e635` |
| 2시 | 열대과일 (망고/파인) | 주황 | `#fb923c` |
| 3시 | 핵과 (복숭아/살구) | 연분홍 | `#fda4af` |
| 4시 | 붉은 베리 (딸기/라즈베리) | 빨강 | `#f87171` |
| 5시 | 검은 베리 (블랙베리/카시스) | 보라 | `#a855f7` |
| 6시 | 꽃 (장미/제비꽃) | 핑크 | `#f472b6` |
| 7시 | 흰 꽃 (아카시아/재스민) | 크림 | `#fef3c7` |

**2차향 (발효/숙성) — 중간 링 (4섹터)**:
| 위치 | 향 | 색상 | hex |
|------|-----|------|-----|
| 8시 | 버터/브리오슈 | 베이지 | `#fde68a` |
| 9시 | 바닐라/토스트 | 갈색 | `#d97706` |
| 10시 | 향신료 (후추/정향) | 다크레드 | `#991b1b` |
| 11시 | 허브 (민트/타임) | 초록 | `#4ade80` |

**3차향 (숙성) — 안쪽 링 (3섹터)**:
| 위치 | 향 | 색상 | hex |
|------|-----|------|-----|
| 중앙-좌 | 가죽/담배 | 다크브라운 | `#78350f` |
| 중앙-우 | 흙/미네랄 | 회색 | `#78716c` |
| 중앙-하 | 견과/초콜릿 | 밤색 | `#92400e` |

**인터랙션**:
- 터치: 섹터 탭 → 해당 영역 활성화 (`active` 토글)
- 드래그: 인접 섹터로 드래그하면 연속 영역 칠하기
- 멀티터치: 비연속 영역도 선택 가능
- 셰이크: 마지막 선택 취소 (undo)
- 선택 영역의 가중 평균 색상 = 사분면 점 대표 색상 (`aroma_color`)

**자동 라벨**:
- 선택한 섹터의 향 이름을 자동으로 `aroma_labels[]`에 저장
- 예: `[시트러스, 열대과일, 바닐라]` → "시트러스, 열대과일, 바닐라 느낌의 와인"

**aroma_color 계산**:
- 선택된 모든 섹터 hex의 가중 평균 (활성화된 섹터 면적 비례)
- 결과 hex를 `records.aroma_color`에 저장

### 구조 평가 (Structure) — 선택

3개 슬라이더로 구성. 모두 0~100 범위.

**복합성**:
| 속성 | 값 |
|------|-----|
| 범위 | 0~100 |
| 라벨 | `1차향(과일/꽃)` ← → `2차향(발효)` ← → `3차향(숙성)` |
| AI 초기값 | 아로마 휠 선택 링 수 기반: 1링만 → ~30, 2링 → ~60, 3링 → ~85 |

**여운**:
| 속성 | 값 |
|------|-----|
| 범위 | 0~100 (내부값) |
| 표시 | **초 환산**: 슬라이더 값을 초 단위로 변환하여 `7초+` 형태로 표시 |
| 라벨 | `짧음(<3초)` ← → `보통(5~8초)` ← → `긴(10초+)` |

**균형**:
| 속성 | 값 |
|------|-----|
| 범위 | 0~100 |
| 라벨 | `불균형` ← → `보통` ← → `완벽한 조화` |
| 0 | 불균형 (산미·타닌·알코올 등이 돌출) |
| 100 | 완벽한 조화 (모든 요소가 어우러짐) |

### 페어링 (WSET 8카테고리 그리드)

AI가 와인 특성 기반으로 pre-select (`AI 추천` 뱃지 표시). 복수 선택 가능.

| 카테고리 | 아이콘 | 예시 음식 |
|---------|--------|----------|
| 적색육 | 🥩 | 스테이크 · 양갈비 · 오리 · 사슴 |
| 백색육 | 🍗 | 닭 · 돼지 · 송아지 · 토끼 |
| 어패류 | 🦐 | 생선 · 갑각류 · 조개 · 굴 · 초밥 |
| 치즈·유제품 | 🧀 | 숙성 · 블루 · 브리 · 크림소스 |
| 채소·곡물 | 🌿 | 버섯 · 트러플 · 리조또 · 파스타 |
| 매운·발효 | 🌶️ | 커리 · 마라 · 김치 · 된장 |
| 디저트·과일 | 🍫 | 다크초콜릿 · 타르트 · 건과일 |
| 샤퀴트리·견과 | 🥜 | 하몽 · 살라미 · 아몬드 · 올리브 |

- 직접 입력 필드 추가 (예: "트러플 리조또")
- `pairing_categories TEXT[]`에 저장

### 만족도 자동 산출 (구조평가 입력 시)

```
기본 50 + 복합성[0|+7|+15] + 여운%×0.1 + 균형%×0.15
→ clamp(1, 100) → 사용자 미세 조정
```

- 구조평가 슬라이더를 조작하면 만족도 자동 재계산
- 사용자가 직접 만족도를 조정하면 자동산출값 무시 (수동 우선)
- `records.auto_score`에 자동산출값 별도 저장

## 9. 부가 입력 (통합 기록 화면)

사분면 + 핵심 평가 이후 같은 화면에서 입력하는 부가 정보.
사진은 카메라 촬영 단계에서 이미 확보 (record_photos 테이블에 별도 저장).

### 식당

| 항목 | 입력 방식 | AI 지원 |
|------|----------|---------|
| 한줄 코멘트 | 텍스트 200자 | — |
| 동반자 | 사용자 태그 + 추가 버튼 | 사진 인식 (`AI` 뱃지) |
