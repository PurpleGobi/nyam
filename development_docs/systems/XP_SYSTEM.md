# XP_SYSTEM — 경험치 & 레벨

> affects: RECORD_FLOW, PROFILE, BUBBLE, HOME, ONBOARDING
> simulation: `xp_simulation_v2.py` (검증용, 구현 코드 아님)

---

## 1. 설계 원칙

- **기록 = 메인 보상**, XP = 서브. 외적 보상이 내적 즐거움을 침범하지 않음
- **영역별 독립** — 강남 100회 ≠ 을지로 전문가. 통합 레벨 없음
- **탐험 보상** — 새로운 곳을 발견할수록 효율적. 같은 곳 반복은 체감
- **스트릭 없음** — 식당은 매일 가는 게 아님
- **리더보드 없음** — 식도락 ≠ 경쟁
- **레벨 감소 없음** — 경험은 사라지지 않음

## 2. XP 축

### 식당
- **지역별**: 기록한 식당의 위치 기반 (을지로 Lv.7, 강남 Lv.3)
- 지역 단위 = 생활권 동네 (행정동 아닌 카카오맵/네이버 동네 분류)

### 와인
- **품종별**: 카베르네 소비뇽 Lv.5, 피노 누아 Lv.3, 샤르도네 Lv.2 (Phase 1)
  - 블렌드 와인: grape_varieties의 각 품종에 비율 가중 XP 적립 (예: CS 60% + 메를로 40% → CS에 비율가중, 메를로에 비율가중)
  - 품종 미상(NV 블렌드 등): 와인 type 기반 폴백 (레드/화이트 등) — XP 미적립보다 나음
- **산지별**: 프랑스 Lv.3, 이탈리아 Lv.2 (Phase 2+ — DB 스키마는 미리 준비)

### 식당 장르별 (Phase 2+)
- 일식 Lv.5, 한식 Lv.3 등 — DB 스키마의 `axis_type='genre'`로 지원
- Phase 1에서는 지역별만 활성화

## 6. XP 적립 로직

```
기록 저장 시:
1. 식당 → 지역 식별 → 방문 횟수 조회 → 기본 XP 결정 (신규/재방문 체감)
2. 와인 → 품종/산지 식별 → user_experience에 XP 적립
3. 품질 보너스 계산 (한줄평, 사진, 메뉴태그, 팁, 풀입력)
4. 일일 축별 상한 체크 (40 XP/축/일)
5. 고유 장소 마일스톤 체크 → 보너스 적립
6. XP 적립 후 → 레벨 체크 → 레벨업이면 알림
```

한 번의 기록으로 복수 축에 동시 적용:
- 을지로 일식당 기록 → 을지로 +XP, 일식 +XP (Phase 2+)
- 보르도 카베르네 소비뇽 기록 → 카베르네 소비뇽 +XP, 프랑스 +XP (산지는 Phase 2+)
- 블렌드 와인 (CS 60% + 메를로 40%) → 각 품종에 비율 가중 XP
