# XP_SYSTEM — 경험치 & 레벨

> affects: RECORD_FLOW, PROFILE, BUBBLE, HOME, ONBOARDING

---

## 1. 설계 원칙

- **기록 = 메인 보상**, XP = 서브. 외적 보상이 내적 즐거움을 침범하지 않음
- **영역별 독립** — 강남 100회 ≠ 을지로 전문가. 통합 레벨 없음
- **스트릭 없음** — 식당은 매일 가는 게 아님
- **리더보드 없음** — 식도락 ≠ 경쟁
- **레벨 감소 없음** — 경험은 사라지지 않음

---

## 2. XP 축

### 식당
- **지역별**: 기록한 식당의 위치 기반 (을지로 Lv.7, 강남 Lv.3)
- 지역 단위 = 생활권 동네 (행정동 아닌 카카오맵/네이버 동네 분류)

### 와인
- **타입별**: 레드 Lv.5, 화이트 Lv.3, 스파클링 Lv.1 (Phase 1)
- **산지별**: 프랑스 Lv.3, 이탈리아 Lv.2 (Phase 2+ — DB 스키마는 미리 준비)

### 식당 장르별 (Phase 2+)
- 일식 Lv.5, 한식 Lv.3 등 — DB 스키마의 `axis_type='genre'`로 지원
- Phase 1에서는 지역별만 활성화

## 3. XP 획득 테이블

| 활동 | XP | 비고 |
|------|-----|------|
| 기록 추가 (사분면+만족도+상황) | 10 | 필수 입력만으로 |
| + 한줄평 | +3 | |
| + 사진 1장 이상 | +3 | |
| + 추천 메뉴/페어링 메모 | +2 | |
| + 사용팁 | +2 | |
| 재방문/재음용 기록 | 12 | 재방문 = 깊이 보너스 |
| 와인 향 팔레트 입력 | +5 | 와인 축만 |
| 와인 복합성+여운+균형 | +3 | 와인 축만 |
| 버블에 공유 (P2) | +1 | 레벨 무관 |
| 좋아요 받음 (P2) | +1 (기록당 1회) | |

**최대 단건 (공통 항목만 적용 기준)**:
- 식당: 기록10 + 한줄평3 + 사진3 + 메뉴2 + 팁2 = **20 XP**
- 와인: 기록10 + 한줄평3 + 사진3 + 향팔레트5 + 심화3 = **24 XP**
- 메뉴태그/사용팁은 식당 전용. 와인은 향팔레트/심화 평가로 대체.

**스팸 방지**: 같은 식당 같은 날 2회 이상 = XP 1회만

### 온보딩 기록 XP 규칙
- 온보딩에서 생성된 기록 (axis_x/axis_y/scene이 NULL)은 **XP 미적립**
- 본 기록 플로우에서 사분면/상황태그 보충 시 정상 XP 적립

## 5. XP 적립 로직

```
기록 저장 시:
1. 식당 → 지역 식별 → user_experience에 XP 적립
2. 와인 → 타입/산지 식별 → user_experience에 XP 적립
3. XP 적립 후 → 레벨 체크 → 레벨업이면 알림
4. 뱃지 조건 체크 → 달성이면 알림
```

한 번의 기록으로 복수 축에 동시 적용:
- 을지로 일식당 기록 → 을지로 +10, 일식 +10 (Phase 2+)
- 보르도 레드 기록 → 레드 +10, 프랑스 +10

## 7. 뱃지 (주요)

### Phase 1
| 뱃지 | 조건 |
|------|------|
| 첫 걸음 | 첫 번째 기록 |
| 열 번째 기록 | 총 10개 |
| 백 번째 기록 | 총 100개 |
| 동네 탐험가 | N개 동네 기록 (5→10→20→30→50) |
| 장르 컬렉터 | N개 장르 기록 (5→10→15) |
| 단골 | 같은 식당 N회 (3→5→10) |

### Phase 2+
| 뱃지 | 조건 |
|------|------|
| 와인 탐험가 | N종 와인 기록 (10→25→50→100) |
| 공유의 기쁨 | 버블에 N개 공유 (5→20→50) |
| 기록의 달인 | 사진+한줄평+메뉴 모두 포함 N개 (10→30→100) |

**Retroactive**: 새 뱃지 추가 시 과거 기록 소급 적용

## 9. 소셜 활용 (Phase 2)

- 경험치는 자동 가중치가 **아님** → 필터/정렬 옵션으로만
- "경험자 순" 정렬: 사용자가 선택할 때만
- 초보자 기록이 묻히면 안 됨 → 공정성
