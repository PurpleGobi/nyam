## 1. 디자인 원칙

Nyam의 모든 디자인 의사결정은 다음 5가지 원칙에 근거한다.

### 원칙 1: 투명함을 통한 신뢰 (Trust through Transparency)

**근거**: 기존 맛집 앱의 가장 큰 문제는 광고와 추천의 경계가 모호하다는 것이다. Nyam은 모든 정보의 출처와 근거를 명확히 보여줌으로써 신뢰를 구축한다.

| 구분 | 예시 |
|------|------|
| **Do** | 검증 횟수, 검증일, 프롬프트 출처를 식당 카드에 명시한다 |
| **Do** | "광고 없는 추천"임을 UI 상단에 명시한다 |
| **Don't** | 검증 데이터 없이 "추천 맛집"이라 표시하지 않는다 |
| **Don't** | 검증 과정을 블랙박스로 숨기지 않는다 |

### 원칙 2: 능동적 발견 (Empowered Discovery)

**근거**: 사용자는 수동적 소비자가 아닌 능동적 검증자다. UI는 "내가 직접 확인한다"는 행동을 격려하고 보상해야 한다.

| 구분 | 예시 |
|------|------|
| **Do** | "AI 검증하기" 버튼을 눈에 띄게 배치하고, 검증 후 달성감을 제공한다 |
| **Do** | 검증 진행률, 배지, 마스터 칭호를 시각적으로 보여준다 |
| **Don't** | 사용자 개입 없이 자동으로 결과를 표시하지 않는다 |
| **Don't** | 검증 행위를 복잡하거나 번거롭게 만들지 않는다 |

### 원칙 3: 명쾌한 정보 위계 (Clear Information Hierarchy)

**근거**: 정보 과부하는 선택 장애를 유발한다. 가장 중요한 정보가 먼저 보이고, 세부 사항은 점진적으로 노출되어야 한다.

| 구분 | 예시 |
|------|------|
| **Do** | 식당 카드에서 사진 → 이름 → 검증 배지 → 핵심 지표 순서로 배치한다 |
| **Do** | 추천은 최대 3곳으로 제한하여 선택 피로를 줄인다 |
| **Don't** | 한 화면에 10개 이상의 메트릭을 나열하지 않는다 |
| **Don't** | 동일한 시각적 가중치로 모든 정보를 표시하지 않는다 |

### 원칙 4: 따뜻한 기술 (Warm Technology)

**근거**: Nyam은 AI 기술을 활용하지만, 음식이 주는 따뜻함과 즐거움을 잃지 않아야 한다. 차가운 기술 느낌이 아닌, 음식을 발견하는 설렘을 전달한다.

| 구분 | 예시 |
|------|------|
| **Do** | 음식 사진을 크고 선명하게 보여주고, 따뜻한 색감을 사용한다 |
| **Do** | 검증 완료 시 축하 마이크로 인터랙션을 제공한다 |
| **Don't** | 차트와 숫자로만 식당을 평가하지 않는다 |
| **Don't** | 기계적이고 무미건조한 톤의 카피를 사용하지 않는다 |

### 원칙 5: 커뮤니티 연결 (Community Connection)

**근거**: 검증 데이터의 가치는 참여자가 많을수록 커진다. 사용자 간의 연결과 공유를 자연스럽게 유도한다.

| 구분 | 예시 |
|------|------|
| **Do** | 다른 검증자의 활동을 자연스럽게 노출한다 |
| **Do** | 그룹 의사결정(함께 검증)을 지원한다 |
| **Don't** | 경쟁을 과도하게 강조하여 부담을 주지 않는다 |
| **Don't** | 개인 검증 데이터를 동의 없이 공개하지 않는다 |

## 2. 컬러 시스템

### 2-1. 디자인 의도

Nyam의 컬러 팔레트는 세 가지 감성을 조화시킨다:
- **음식의 따뜻함**: 코랄/오렌지 계열 (식욕, 에너지, 발견의 즐거움)
- **기술의 신뢰감**: 딥 네이비/슬레이트 계열 (안정감, 전문성)
- **검증의 확신**: 그린 계열 (안전, 신뢰, 검증 완료)

### 2-6. 컬러 사용 가이드라인

#### 60-30-10 비율

```
60% — Neutral (배경, 카드 표면)
     neutral-0, neutral-50, neutral-100

30% — Secondary (텍스트, 네비게이션)
     secondary-700, secondary-800, secondary-900

10% — Primary + Accent (CTA, 배지, 강조)
     primary-500, accent-500, 검증 컬러
```

#### 접근성 규칙

- 본문 텍스트: 최소 WCAG AA (4.5:1 대비비)
- 대형 텍스트 (18px+): 최소 3:1 대비비
- 아이콘/그래픽: 최소 3:1 대비비
- 인터랙티브 요소: 포커스 링 필수 (`primary-500`, 2px)
- 검증 상태는 색상만으로 구분하지 않고 아이콘/텍스트 병행

## 3. 타이포그래피

### 3-1. 폰트 스택

| 용도 | 1순위 | 2순위 | 3순위 | Fallback |
|------|-------|-------|-------|----------|
| **한국어 (주)** | Pretendard | SUIT | Noto Sans KR | system-ui, sans-serif |
| **영어/숫자** | Inter | -apple-system | Segoe UI | sans-serif |
| **모노스페이스** | JetBrains Mono | Fira Code | Menlo | monospace |

```css
/* CSS 변수 */
--font-sans: 'Pretendard Variable', 'Inter', -apple-system, BlinkMacSystemFont,
             'Segoe UI', 'Noto Sans KR', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Menlo', monospace;
```

**선정 이유**:
- **Pretendard**: 한국어 가독성 최적화, 가변 폰트 지원, Apple SF Pro와 조화
- **Inter**: 숫자/영문 가독성 우수, UI 특화, Pretendard와 x-height 유사
- **JetBrains Mono**: 프롬프트 코드 블록에 사용

### 3-2. 타입 스케일

Base size: `16px` (1rem), Scale ratio: `1.25` (Major Third)

| 토큰 | 크기 (rem/px) | 줄높이 | 자간 | 굵기 | 용도 |
|------|--------------|--------|------|------|------|
| `display` | 2.488rem / 40px | 1.2 | -0.02em | 700 | 히어로 타이틀 |
| `h1` | 2rem / 32px | 1.25 | -0.015em | 700 | 페이지 타이틀 |
| `h2` | 1.602rem / 26px | 1.3 | -0.01em | 700 | 섹션 타이틀 |
| `h3` | 1.266rem / 20px | 1.35 | -0.005em | 600 | 카드 타이틀, 식당명 |
| `h4` | 1.125rem / 18px | 1.4 | 0 | 600 | 서브 헤딩 |
| `h5` | 1rem / 16px | 1.4 | 0 | 600 | 작은 헤딩 |
| `h6` | 0.875rem / 14px | 1.4 | 0.01em | 600 | 라벨 헤딩 |
| `body-lg` | 1.125rem / 18px | 1.6 | 0 | 400 | 긴 본문 (설명) |
| `body` | 1rem / 16px | 1.6 | 0 | 400 | 기본 본문 |
| `body-sm` | 0.875rem / 14px | 1.5 | 0.005em | 400 | 보조 텍스트, 메타 |
| `caption` | 0.75rem / 12px | 1.5 | 0.01em | 400 | 캡션, 타임스탬프 |
| `overline` | 0.688rem / 11px | 1.5 | 0.08em | 600 | 오버라인, 태그 라벨 |

### 3-3. 폰트 굵기

| 토큰 | 값 | 용도 |
|------|-----|------|
| `regular` | 400 | 본문 텍스트, 설명 |
| `medium` | 500 | 버튼 라벨, 네비게이션 항목 |
| `semibold` | 600 | 카드 타이틀, 강조 라벨, 배지 텍스트 |
| `bold` | 700 | 페이지/섹션 타이틀, 숫자 강조 (검증 횟수) |

### 3-4. 가독성 권장사항

- **본문 최대 너비**: 65자 (~520px 한국어 기준) — 읽기 편안한 줄 길이 유지
- **한국어 줄높이**: 영문 대비 10% 높게 (1.6 이상) — 한글 자모 복잡도 고려
- **숫자 표기**: 태뷸러 피겨 (`font-variant-numeric: tabular-nums`) — 정렬 통일
- **긴 식당명**: 2줄까지 허용, 이후 `...` 트렁케이션
- **프롬프트 텍스트**: 모노스페이스 폰트, `body-sm` 크기, `neutral-700` 색상

## 5. 보더 라디우스 및 엘리베이션

### 5-1. 보더 라디우스 스케일

| 토큰 | 값 | 용도 |
|------|-----|------|
| `radius-none` | 0px | 전체 너비 이미지, 디바이더 |
| `radius-sm` | 4px | 태그, 인라인 배지 |
| `radius-md` | 8px | 입력 필드, 작은 버튼 |
| `radius-lg` | 12px | **카드 기본**, 드롭다운 |
| `radius-xl` | 16px | 모달, 바텀 시트 |
| `radius-2xl` | 20px | 프로필 카드, 히어로 카드 |
| `radius-full` | 9999px | 필(pill) 버튼, 아바타, 태그 |

**디자인 방향**: Nyam은 부드럽고 친근한 느낌을 위해 `radius-lg` (12px)를 카드 기본값으로 사용한다. 날카로운 직각은 피하되, 과도하게 둥글지도 않게 한다.

### 5-2. 엘리베이션 (그림자) 레벨

#### 라이트 모드

| 토큰 | CSS box-shadow | 용도 |
|------|---------------|------|
| `shadow-none` | none | 플랫 요소, 배경 위 구분 불필요 시 |
| `shadow-xs` | `0 1px 2px rgba(16,42,67,0.06)` | 보더 대안, 미세한 구분 |
| `shadow-sm` | `0 1px 3px rgba(16,42,67,0.08), 0 1px 2px rgba(16,42,67,0.04)` | 카드 기본, 리스트 항목 |
| `shadow-md` | `0 4px 6px rgba(16,42,67,0.08), 0 2px 4px rgba(16,42,67,0.04)` | 카드 호버, 드롭다운 |
| `shadow-lg` | `0 10px 15px rgba(16,42,67,0.08), 0 4px 6px rgba(16,42,67,0.04)` | 모달, 팝오버 |
| `shadow-xl` | `0 20px 25px rgba(16,42,67,0.08), 0 8px 10px rgba(16,42,67,0.04)` | 바텀 시트, 플로팅 버튼 |

#### 사용 기준

| 엘리베이션 | 사용 시점 |
|-----------|----------|
| `shadow-none` | 페이지 배경과 같은 레벨의 요소 |
| `shadow-xs` ~ `shadow-sm` | 일반적인 카드, 컨텐츠 영역 구분 |
| `shadow-md` | 사용자 상호작용(호버/포커스)으로 떠오르는 요소 |
| `shadow-lg` | 현재 컨텍스트 위에 오버레이되는 요소 |
| `shadow-xl` | 최상위 레이어 (바텀 시트, FAB) |

### 7-2. 검증 배지 (Verification Badge)

별점을 대체하는 Nyam의 핵심 시각 요소.

#### 검증 레벨

| 레벨 | 조건 | 아이콘 | 컬러 | 라벨 |
|------|------|--------|------|------|
| **미검증** | 검증 0회 | `shield` (아웃라인) | `neutral-400` | "미검증" |
| **검증 시작** | 1~4회 | `shield-check` (아웃라인) | `accent-500` | "검증 중 (N회)" |
| **검증 완료** | 5~19회 | `shield-check` (채움) | `#38A169` (success) | "검증 완료" |
| **신뢰 맛집** | 20회 이상 | `shield-check` (채움) + 별 | `#276749` (진한 그린) | "신뢰 맛집" |
| **의심** | 커뮤니티 플래그 | `shield-alert` (채움) | `#E53E3E` (error) | "검증 주의" |

#### 크기 변형

| 사이즈 | 아이콘 크기 | 텍스트 크기 | 높이 | 용도 |
|--------|-----------|-----------|------|------|
| `sm` | 14px | 11px (overline) | 22px | 리스트 뷰, 인라인 |
| `md` | 18px | 12px (caption) | 28px | 카드, 기본 |
| `lg` | 24px | 14px (body-sm) | 36px | 상세 페이지 헤더 |

#### 획득 애니메이션

검증 레벨이 올라갈 때:
1. 배지가 `scale(0)` → `scale(1.2)` → `scale(1.0)` (300ms, spring ease)
2. 배경에서 동심원 파동(ripple) 효과 (`primary-500` → 투명, 600ms)
3. 파티클 이펙트: 작은 별/체크 아이콘 6~8개가 방사형으로 퍼짐 (400ms)
4. 햅틱 피드백 (success 패턴)

## 8. 모션 및 애니메이션

### 8-1. 타이밍 곡선 및 지속 시간

| 토큰 | 지속 시간 | 이징 | 용도 |
|------|----------|------|------|
| `duration-fast` | 100ms | `ease-out` | 버튼 프레스, 토글 |
| `duration-normal` | 200ms | `ease-in-out` | 호버, 포커스, 페이드 |
| `duration-slow` | 300ms | `ease-in-out` | 카드 확장, 드롭다운 |
| `duration-slower` | 500ms | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 배지 획득, 축하 |
| `duration-slowest` | 800ms | `cubic-bezier(0.22, 1, 0.36, 1)` | 페이지 전환, 차트 로드 |

```css
/* CSS 변수 */
--ease-default: cubic-bezier(0.4, 0, 0.2, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-out: cubic-bezier(0, 0, 0.2, 1);
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);
--ease-smooth: cubic-bezier(0.22, 1, 0.36, 1);
```

### 8-2. 핵심 마이크로 인터랙션

#### 프롬프트 복사

```
1. 버튼 프레스       → scale(0.95), 100ms
2. 아이콘 전환       → clipboard → check, 200ms (fade-crossover)
3. 텍스트 변경       → "복사하기" → "복사됨!", 200ms
4. 색상 전환         → secondary → primary-500 텍스트, 200ms
5. 리플 효과         → 버튼 중심에서 원형 파장, primary-100, 400ms
6. 원복              → 2000ms 후 원래 상태, 300ms fade
```

#### AI 검증 시작

```
1. CTA 버튼 프레스    → scale(0.97), 100ms
2. sparkles 아이콘    → 회전 360deg, 500ms
3. 프롬프트 슬라이드업 → 바텀 시트 올라옴, 300ms ease-smooth
4. 프롬프트 텍스트     → 타이핑 효과로 나타남 (선택적), 50ms/char
```

#### 배지 획득

```
1. 배지 진입          → scale(0→1.2→1.0), 500ms spring
2. 배경 파동          → 동심원 ripple, primary-200 → 투명, 600ms
3. 파티클 분출        → 6-8개 미니 아이콘 방사, 400ms
4. 텍스트 페이드인     → "강남 맛집 탐험가 달성!", 300ms delay
5. 진행 바 업데이트    → 부드럽게 채움 이동, 600ms ease-out
```

#### 검증 카운트 증가

```
1. 숫자 롤링          → 이전 숫자 위로 올라가고 새 숫자 아래서 올라옴, 300ms
2. 배지 바운스        → scale(1→1.1→1.0), 200ms
3. 색상 글로우         → 텍스트에 primary-500 glow, 400ms fade-out
```

| 레벨 | 라이트 (shadow) | 다크 (표면 밝기) |
|------|----------------|----------------|
| Level 0 | `shadow-none` | `#0F1419` (배경) |
| Level 1 | `shadow-sm` | `#1A2332` (표면) |
| Level 2 | `shadow-md` | `#243141` |
| Level 3 | `shadow-lg` | `#2A3A4C` |
| Level 4 | `shadow-xl` | `#3D5068` |

다크 모드에서도 미세한 그림자(`0 1px 2px rgba(0,0,0,0.3)`)를 추가하면 요소 간 구분이 명확해진다.

## 11. 반응형 브레이크포인트

### 11-1. 브레이크포인트 정의

모바일 퍼스트 접근. `min-width` 기반.

| 토큰 | 값 | 대상 디바이스 |
|------|-----|-------------|
| `xs` | 0px ~ | 소형 모바일 (SE, Galaxy A) |
| `sm` | 640px ~ | 대형 모바일 (iPhone Pro Max, 가로 모드) |
| `md` | 768px ~ | 태블릿 세로 |
| `lg` | 1024px ~ | 태블릿 가로, 소형 랩톱 |
| `xl` | 1280px ~ | 데스크톱 |
| `2xl` | 1536px ~ | 대형 데스크톱 |

```css
/* Tailwind CSS 기본값과 동일 */
@media (min-width: 640px)  { /* sm */ }
@media (min-width: 768px)  { /* md */ }
@media (min-width: 1024px) { /* lg */ }
@media (min-width: 1280px) { /* xl */ }
@media (min-width: 1536px) { /* 2xl */ }
```

### 11-2. 레이아웃 변화

| 요소 | xs~sm (모바일) | md (태블릿) | lg+ (데스크톱) |
|------|--------------|------------|--------------|
| **식당 카드** | 1열 스택 | 2열 그리드 | 3열 그리드 |
| **네비게이션** | 바텀 탭 바 | 바텀 탭 바 | 사이드 네비게이션 |
| **프롬프트 카드** | 1열 풀 너비 | 2열 | 3열 |
| **Trust Meter** | 수평 바 | 수평 바 | 레이더 차트 (상세) |
| **배지 그리드** | 4열 | 6열 | 8열 |
| **지도 높이** | 300px | 350px | 400px |
| **바텀 시트** | 전체 높이 가능 | 최대 60% 높이 | 사이드 패널 전환 |
| **검색 바** | 상단 고정, 풀 너비 | 상단 고정, 풀 너비 | 상단 중앙, 최대 600px |

### 11-3. 터치 vs 포인터 대응

| 인터랙션 | 터치 (모바일) | 포인터 (데스크톱) |
|----------|-------------|-----------------|
| 카드 | 탭 → 상세 | 호버 → 미리보기, 클릭 → 상세 |
| 버튼 호버 | 없음 | `translateY(-1px)` + 밝기 변화 |
| 스크롤 | 관성 스크롤 | 스크롤바 표시 |
| 프롬프트 복사 | 롱프레스로 선택 영역 보기 가능 | 클릭으로 전문 복사 |
| 지도 | 핀치 줌 | 스크롤 줌 |

### 11-4. Safe Area 대응

```css
/* iOS 노치/다이나믹 아일랜드/홈 인디케이터 대응 */
padding-top: env(safe-area-inset-top);
padding-bottom: env(safe-area-inset-bottom);
padding-left: env(safe-area-inset-left);
padding-right: env(safe-area-inset-right);
```

- 바텀 탭 바: `56px + safe-area-inset-bottom`
- 상단 네비게이션: `56px + safe-area-inset-top`
- 전체 화면 모달: 4방향 모두 safe area 적용
