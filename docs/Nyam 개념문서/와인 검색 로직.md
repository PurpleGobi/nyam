# 와인 검색 로직

## 진입점 3가지 (모두 /add?type=wine)

| 경로 | 트리거 | 동작 |
|------|--------|------|
| 카메라 촬영 | 라벨 촬영 | Phase 1 → Phase 2 |
| 앨범에서 추가 | 갤러리 선택 | Phase 1 → Phase 2 |
| 이름으로 검색 | 검색창 입력 | 텍스트 검색만 (사진 없음) |

---

## Phase 1: 리사이즈 + 즉시 업로드 (카메라/앨범 공통)

```
유저가 사진 촬영 또는 앨범 선택
  │
  ├─ base64 → File 변환
  │
  ├─ imageService.resizeImage() — 중앙화된 리사이즈 로직
  │   (식당 사진과 동일: 800px width, WebP, quality 0.7)
  │
  ├─ imageService.uploadImage(userId, 'pending', blob, uuid)
  │   → Storage: record-photos/{userId}/pending/{uuid}.webp
  │   → 사진 즉시 업로드 완료 (검색 결과와 무관)
  │
  ├─ 리사이즈된 blob → base64 변환 (Phase 2 검색용)
  │
  └─ 기록 미완료 시 cleanup:
      컴포넌트 unmount → imageService.deleteImage(tempUrl)
```

### Phase 1 관련 파일

| 파일 | 역할 |
|------|------|
| `src/infrastructure/storage/image-upload.ts` | resizeImage, uploadImage, deleteImage (중앙화) |
| `src/domain/entities/record-photo.ts` | PHOTO_CONSTANTS (MAX_WIDTH, QUALITY, FORMAT) |
| `src/presentation/containers/add-flow-container.tsx` | resizeAndUpload, cleanup |

## Phase 2: AI 검색 (리사이즈된 이미지 사용)

```
리사이즈된 base64 (WebP, ~100-300KB)
  │
  ├─ POST /api/records/identify
  │
  ├─ Gemini Vision (gemini-2.5-flash) 호출
  │   mime_type 자동 감지 (WebP/PNG/JPEG)
  │   프롬프트: WINE_LABEL_PROMPT
  │   출력: wine_name, producer, vintage, region, country,
  │         wine_type, variety, grape_varieties, abv,
  │         classification, body/acidity/sweetness level,
  │         food_pairings, serving_temp, decanting,
  │         reference_price, drinking_window, vivino_rating,
  │         tasting_notes, confidence
  │
  ├─ upsertWineFromAI() — 3단계 퍼지 매칭으로 중복 체크
  │   │
  │   ├─ 1단계: 정확한 이름 매칭
  │   │   wines.name ILIKE 'AI가 읽은 이름' (+ vintage 일치)
  │   │
  │   ├─ 2단계: 핵심 단어 포함 매칭 (1단계 실패 시)
  │   │   이름에서 Château/Domaine 등 접두사 제거
  │   │   → 3글자 이상 단어 최대 3개 추출
  │   │   → wines.name에 각 단어 ILIKE %word% 검색
  │   │   → 2개 이상 단어 매칭 시 동일 와인 판정
  │   │   예: AI="Terre de Pique-Sègue Anima Vitis"
  │   │       DB="Château Terre de Pique Montravel Rouge"
  │   │       → "Terre"(O) + "Pique"(O) = 2개 매칭 → 동일 와인
  │   │
  │   ├─ 3단계: 생산자 + 빈티지 매칭 (2단계 실패 시)
  │   │   wines.producer ILIKE '%생산자%' AND vintage = 빈티지
  │   │
  │   └─ 모두 실패 → 새 와인 INSERT (AI 추정 데이터 전체 저장)
  │
  ├─ confidence >= 0.5 → isConfidentMatch = true
  │   → 빠른 추가: 기록 자동 INSERT + Phase 1 사진 연결 + 성공 화면
  │
  ├─ 후보 있지만 확신 부족 → wine_confirm 화면 (유저 확인)
  │
  └─ 인식 실패 → search 화면으로 폴백
```

### Phase 2 관련 파일

| 파일 | 역할 |
|------|------|
| `src/app/api/records/identify/route.ts` | API 엔드포인트 + upsertWineFromAI |
| `src/infrastructure/api/gemini.ts` | Gemini Vision 호출 (recognizeWineLabel) |
| `src/application/hooks/use-camera-capture.ts` | identify 훅 |
| `src/presentation/containers/add-flow-container.tsx` | 전체 플로우 관리 |

## 이름 검색 (텍스트, 사진 없음)

```
유저가 검색창에 와인 이름 입력
  │
  ├─ [1차] DB 검색 — 즉시 실행
  │   GET /api/wines/search?q=검색어
  │   wines.name ILIKE '%검색어%' OR wines.producer ILIKE '%검색어%'
  │   → 기록 있는 와인 우선 정렬
  │
  ├─ [2차] AI 검색 — DB 결과 부족 시 병렬 실행
  │   POST /api/wines/search-ai
  │   Gemini Text (gemini-2.5-flash) 호출
  │   프롬프트: WINE_SEARCH_PROMPT (영문 프롬프트)
  │   → 검색어 단어가 직접 포함된 와인만 추천 (최대 5개)
  │   → 생산자 검색 시 기본/엔트리 라인 우선
  │   → name 필드는 반드시 영문, name_ko에 한국어
  │   → DB 저장 없음 (메모리 후보 리스트만)
  │
  ├─ 유저가 DB 결과 선택 → 바로 기록 폼으로 이동
  │
  └─ 유저가 AI 후보 선택
      │
      ├─ [3차] 상세 정보 조회 + DB 저장
      │   POST /api/wines/detail-ai
      │   Gemini Text 호출 (WINE_DETAIL_PROMPT)
      │   → 19필드 상세 정보 생성
      │   → upsertWineFromAI()로 DB 저장 (이름 매칭)
      │
      └─ 기록 폼으로 이동
```

### 이름 검색 관련 파일

| 파일 | 역할 |
|------|------|
| `src/app/api/wines/search/route.ts` | DB 검색 API |
| `src/app/api/wines/search-ai/route.ts` | AI 후보 검색 API + 라벨 이미지 검색 |
| `src/app/api/wines/detail-ai/route.ts` | AI 상세 조회 + DB 저장 |
| `src/infrastructure/api/gemini.ts` | searchWineByName, getWineDetailByName |
| `src/application/hooks/use-wine-search.ts` | 검색 훅 (DB + AI 통합) |

## 사진 연결 흐름

```
Phase 1 업로드 (pending 경로)
  │
  ├─ 기록 완료 시:
  │   photoRepo.savePhotos(recordId, [{ url: tempUrl }])
  │   → record_photos 테이블에 URL 저장
  │   → 기존 pending 경로 파일은 그대로 유지 (이동 불필요)
  │
  └─ 기록 미완료 (이탈) 시:
      컴포넌트 unmount → imageService.deleteImage(tempUrl)
      → Storage에서 pending 파일 삭제
```

## 전체 흐름 요약

```
카메라/앨범 ──→ Phase 1 (리사이즈+업로드) ──→ Phase 2 (Gemini Vision) ──→ 기록
                    ↑ 중앙화 로직                      ↑ 퍼지 매칭
                                                       │
이름 입력 ──→ DB ILIKE 검색 ──→ 유저 선택 ──→ 기록
         └──→ Gemini Text ──→ 유저 선택 ──→ detail-ai ──→ upsert ──→ 기록
```

## 모델 및 설정

- 모델: `gemini-2.5-flash`
- temperature: 0 (결정론적 응답)
- Vision maxOutputTokens: 2048
- Text maxOutputTokens: 4096
- 이미지 리사이즈: 800px width, WebP, quality 0.7 (PHOTO_CONSTANTS 중앙 관리)
- mime_type: base64 헤더에서 자동 감지 (WebP/PNG/JPEG)
