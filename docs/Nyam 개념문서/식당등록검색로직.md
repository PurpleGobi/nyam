카카오 API 를 이용해 식당을 불러옴.
카카오 API 응답 전체 필드

  ┌─────────────────────┬───────────────────────────────┬───────────────────┐
  │        필드         │             설명              │    우리가 사용    │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ id                  │ 카카오 장소 ID                │ O → kakaoId       │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ place_name          │ 식당 이름                     │ O → name          │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ road_address_name   │ 도로명 주소                   │ O → address       │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ address_name        │ 지번 주소                     │ O (폴백)          │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ x, y                │ 경도, 위도                    │ O → lng, lat      │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ phone               │ 전화번호                      │ O → phone         │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ category_group_name │ 대분류 (음식점)               │ O → category      │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ category_name       │ 세분류 (음식점 > 한식 > 냉면) │ nearby에서만 사용 │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ distance            │ 거리(m)                       │ nearby에서만 사용 │
  ├─────────────────────┼───────────────────────────────┼───────────────────┤
  │ place_url           │ 카카오맵 링크                 │ O → kakaoMapUrl   │
  └─────────────────────┴───────────────────────────────┴───────────────────┘

  카카오 API → 모든 정보 저장 (phone, place_url, category 등)
            → 기록 폼에 genre 자동 매핑 + 최대한 프리필
            → 다음 선택 시 DB 데이터 사용 (+ next_refresh_at로 갱신 판단)

  restaurants 테이블에 cached_at과 next_refresh_at 컬럼이 있습니다. 로직은:

  1. 첫 선택: 카카오 데이터 → INSERT + cached_at = now(), next_refresh_at = now() + 30일
  2. 재선택 시 DB에 있음: 이름 중복 체크로 기존 id 반환 → next_refresh_at 확인
    - 아직 유효 → DB 데이터 그대로 사용
    - 만료됨 → 카카오 API 재호출 → UPDATE + cached_at 갱신

## Google Places API 보강 (영업시간·전화번호)

카카오 로컬 검색 API는 영업시간, 메뉴 정보를 제공하지 않음.
→ Google Places API (New)의 Place Detail을 통해 추가 정보를 보강한다.

### 카카오 API가 제공하지 않는 필드

| 정보     | 카카오 API | Google Places API |
|----------|:----------:|:-----------------:|
| 영업시간 | X          | O (regularOpeningHours) |
| 전화번호 | O (간헐적 누락) | O (nationalPhoneNumber) |
| 메뉴     | X          | X (API 미제공)    |
| 평점     | X          | O (rating)        |

### 보강 흐름

```
사용자가 검색 결과에서 식당 선택
       ↓
POST /api/restaurants (식당 등록)
       ↓
externalIds에 google placeId가 있는가?
       ├─ YES → Google Place Detail API 호출
       │         ├─ 영업시간 (hours) → BusinessHours 형태로 파싱 후 저장
       │         ├─ 전화번호 (phone) → 카카오에서 누락된 경우 보강
       │         └─ 평점 (google_rating) → 저장
       └─ NO → 카카오 데이터만으로 저장
```

### Google Place Detail API 호출 상세

```
GET https://places.googleapis.com/v1/places/{placeId}?languageCode=ko

Headers:
  X-Goog-Api-Key: {GOOGLE_PLACES_API_KEY}
  X-Goog-FieldMask: nationalPhoneNumber,regularOpeningHours,rating

응답에서 추출:
  - nationalPhoneNumber → phone (카카오 phone이 없을 때 폴백)
  - regularOpeningHours.periods → hours (BusinessHours JSONB)
  - regularOpeningHours.weekdayDescriptions → hours (periods 없을 때 폴백)
  - rating → google_rating
```

### 영업시간 파싱 로직

```
Google periods 배열:
  [{ open: {day: 1, hour: 9, minute: 0}, close: {day: 1, hour: 21, minute: 0} }, ...]
       ↓
BusinessHours 변환:
  { mon: "09:00~21:00", tue: "09:00~21:00", ... }

폴백 (periods 없을 때):
  weekdayDescriptions: ["월요일: 09:00~21:00", "화요일: 09:00~21:00", ...]
       ↓
  같은 BusinessHours 형태로 파싱
```

### 적용 시점

| 시점 | 동작 |
|------|------|
| 신규 등록 (INSERT) | google placeId 있으면 Detail 호출 → hours, phone, google_rating 저장 |
| stale 갱신 (UPDATE) | 기존 또는 새 google placeId로 Detail 재호출 → 최신 정보로 갱신 |

### 환경 변수

```
GOOGLE_PLACES_API_KEY=AIzaSy...  # .env.local (서버 전용, 클라이언트 노출 금지)
```

### 검색 API 3종 동시 호출 구조

```
사용자 검색어 입력
       ↓
Nyam DB 검색 (restaurants 테이블)
       ↓ (5개 미만이면)
3개 API 동시 호출:
  ├─ 카카오 로컬 (/v2/local/search/keyword) → name, address, phone, category, kakaoId
  ├─ 네이버 로컬 → name, address, phone, category, naverId
  └─ 구글 Places (searchText) → name, address, lat/lng, rating, googlePlaceId
       ↓
중복 제거 (이름+주소 정규화)
       ↓
사용자에게 통합 결과 제공
       ↓
사용자가 선택 → POST /api/restaurants
       ↓
externalIds: { kakao?: "...", naver?: "...", google?: "..." }
       ↓
google placeId 있으면 → Detail API로 영업시간·전화 보강
       ↓
restaurants 테이블에 INSERT/UPDATE
```
