## 1. 디자인 철학

### 1-1. 브랜드 퍼스낼리티

Nyam은 **따뜻하고 똑똑한 친구**다. 맛집 정보를 떠들썩하게 자랑하는 SNS가 아니라, 조용히 내 취향을 알아가는 개인 노트.

- **따뜻함**: 음식 사진이 주인공. UI는 사진을 돋보이게 하는 배경. Coral Orange는 식욕을 자극하면서도 부담 없는 온도감.
- **똑똑함**: AI가 일하지만 티내지 않는다. "AI가 분석했습니다" 대신 결과만 조용히 보여준다.
- **개인적**: 내 기록, 내 DNA, 내 버블. 대중을 위한 랭킹이 아니라 나를 위한 데이터.
- **성장감**: 기록할수록 풍성해지는 시각적 변화. 빈 캘린더 → 사진으로 가득 찬 캘린더. 회색 레이더 → 선명한 DNA.

### 1-2. 비주얼 랭귀지

- **사진 중심**: 음식 사진은 항상 크고 선명하게. UI 요소는 사진 위에 최소한으로.
- **유기적 곡선**: 모든 모서리는 둥글게 (rounded-xl ~ rounded-2xl). 직각은 사용하지 않는다.
- **여백 활용**: 빽빽하게 채우지 않는다. 카드 사이, 섹션 사이에 호흡 공간.
- **레이어링**: 카드가 배경 위에 떠있는 느낌 (shadow-sm). 깊이감을 주되 과도하지 않게.
- **색상 절제**: 60% Neutral · 30% Text · 10% Accent. Primary는 CTA와 점수에만 집중.

### 1-3. 톤 & 보이스 (UI 텍스트)

- **짧고 따뜻하게**: "저장하기", "기록이 저장되었습니다" (O) / "저장 완료!" (X, 과장)
- **존댓말 기본**: "기록을 남겨보세요", "추천을 받아보세요"
- **AI 존재감 최소화**: "AI가 분석 중입니다" (X) → 스피너만 표시 (O)
- **빈 상태는 격려**: "아직 기록이 없어요" + 행동 유도 ("첫 기록을 남겨보세요")
- **에러는 부드럽게**: "오류가 발생했습니다" (X) → "불러오지 못했어요. 다시 시도해주세요" (O)

### 1-4. 하지 말아야 할 것

- **과도한 장식**: 그라데이션 배경, 패턴, 불필요한 일러스트 금지
- **정보 과부하**: 한 화면에 3개 이상의 CTA 금지. 핵심 행동 하나에 집중
- **차가운 톤**: 회색 일변도, 각진 모서리, 기계적 문구 금지
- **UI 이모지**: 버튼/아이콘/내비게이션에 이모지 사용 금지 (데이터 라벨만 허용)
- **로딩 방치**: 0.5초 이상 걸리면 반드시 스켈레톤 또는 스피너 표시

### 1-5. 디자인 원칙

| 원칙 | 설명 | 적용 페이지 |
|------|------|------------|
| 5초 기록 | 마찰 최소화. 사진 → 평가 → 저장이 한 화면에서 끝남 | 빠른 기록 (5-2) |
| 즉각적 피드백 | 모든 액션에 시각적 반응. 기다림은 스켈레톤으로 | 모든 페이지: 로딩 상태 (6-1) |
| 따뜻한 기술 | 음식의 온기 + AI의 편안함. 차갑지 않은 톤 | 컬러 (2-1), 라운딩 (2-4), 톤&보이스 (1-3) |
| 정보 위계 | 기록 → 사람 → 버블 순서로 깊어지는 탐색 | 홈 (5-1), 발견 (5-7), 버블 (5-9) |
| 데이터가 곧 보상 | 기록이 쌓일수록 시각적으로 풍성해짐 | 홈 캘린더/DNA (5-1), 프로필 (5-12), Wrapped (5-13) |

깜빡임 방지를 위해 `<head>` 인라인 스크립트로 초기 클래스 적용.

```css
/* 다크모드 주요 변수 오버라이드 */

  --color-neutral-50: #1A1F2E;
  --color-neutral-800: #F9FAFB;
  --color-background: #111318;
  --color-foreground: #F9FAFB;
  --color-card: #1A1F2E;
}
```

## 3. 레이아웃 시스템

### 3-1. AppShell

```
┌─────────────────────────────────────────┐
│  Header (sticky top-0 z-50)             │
│  bg-background/95 backdrop-blur         │
│  ┌─ max-w-lg mx-auto h-14 px-5 ──────┐ │
│  │ [nyam]                  [🔍] [🔔]  │ │
│  └────────────────────────────────────┘ │
├─────────────────────────────────────────┤
│                                         │
│  <main> max-w-lg mx-auto pb-20          │
│  (하단 내비 높이만큼 패딩)                │
│                                         │
├─────────────────────────────────────────┤
│  BottomNav (fixed bottom-0 z-50)        │
│  rounded-t-2xl                          │
│  pb-[env(safe-area-inset-bottom)]       │
│  ┌─ max-w-lg mx-auto ────────────────┐ │
│  │ [홈] [발견] [＋FAB] [버블] [프로필] │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 헤더 배경: 전체 폭, 내부 콘텐츠만 `max-w-lg`
- 로고: `nyam` — Comfortaa Bold, `text-[#FF6038]`
- 알림 아이콘: 미읽은 알림 시 우상단에 `w-2 h-2 bg-[#EF4444] rounded-full` 도트

### 3-2. FAB (기록 버튼)

- 위치: 하단 내비 중앙, `-mt-5`로 돌출
- 크기: `h-14 w-14 rounded-full`
- 색상: `bg-[#FF6038]`, 아이콘: `Plus` (white, `h-6 w-6`)
- 그림자: `shadow-lg`
- 프레스: `active:scale-95 transition-transform`

### 3-3. 반응형

- 모든 콘텐츠: `max-w-lg` (512px) + `mx-auto`
- 데스크톱에서도 모바일 레이아웃 유지 (PWA 대상)
- 브레이크포인트 없음 (단일 뷰포트)

### 3-4. 터치 타겟

- 최소: 44×44px (Apple HIG 기준)
- 버튼: `h-9 w-9` 이상 + padding
- 슬라이더 터치 영역: `h-10`

## 4. 컴포넌트 라이브러리

#### Primary

#### Secondary


#### Ghost

```
rounded-lg text-neutral-500 text-sm

```

#### 필터 칩 (토글)

```
rounded-full px-2.5 py-0.5 text-[10px] font-medium transition-colors
활성: bg-[#FF6038] text-white
비활성: bg-neutral-100 text-neutral-500
```

### 4-2. 카드

#### 기본 카드

```
rounded-2xl bg-white shadow-[var(--shadow-sm)] overflow-hidden
```

내부 패딩: `p-4`. 섹션 구분: `border-t border-neutral-100`.

#### 기록 카드 (리스트 아이템)

```
rounded-2xl bg-white shadow-[var(--shadow-sm)] overflow-hidden
├── 사진 영역 (상단, 전폭)
└── 정보 영역 (p-4)
    ├── 메뉴명 (Small, font-semibold, neutral-800)
    ├── 장르 · 날짜 (Caption, neutral-400)
    └── 점수 (Small, font-bold, primary-500)
```

### 4-3. 100점 슬라이더 (RatingScales)

PRD 8-3-4의 종합 경험 평가 UI. 유형별 항목 자동 전환.

```
┌─────────────────────────────────────┐
│ 라벨(w-14) ━━━━━━━━━━●───── 값(w-8) │
│   Caption    트랙(h-3)  썸(w-6 h-6)  Small │
└─────────────────────────────────────┘
```

- 트랙 배경: `bg-neutral-100`, 채움: `bg-[#FF6038]`
- 썸: `w-6 h-6 rounded-full bg-white border-2 border-[#FF6038] shadow-sm`
- 터치: `touch-none`, `onTouchStart/Move/End` 직접 처리
- 값 표시: 썸 우측에 숫자 (0-100)
- 미입력 상태: 값 `0`, 트랙 비어있음, 숫자 회색

### 4-4. 레이더 차트 (RadarChart)

Taste DNA 시각화. SVG 기반.

```
크기: 128×128 (viewBox 0 0 128 128)
중심: (64, 64), 반지름: 46
```

- 그리드: 3단계 동심 다각형 (`neutral-200`, strokeWidth 0.5)
- 축 선: 중심→꼭지점 (`neutral-200`, strokeWidth 0.4)
- 데이터 영역: fill opacity 15%, stroke width 1.4
- 데이터 점: `r=2.5`, 채우기 = 선 색상
- 축 라벨: 다각형 바깥 12px, `neutral-500`, fontSize 7.5

**Food DNA**: 6축 다각형 (매운맛/단맛/짠맛/신맛/감칠맛/풍미), 색상 = `primary-500`
**Wine DNA**: 7축 다각형 (산미/바디감/타닌/당도/균형/여운/향), 색상 = `#6B4C8A`
**Cooking DNA**: 6축 다각형 (식당과 동일 축), 색상 = `primary-500` (프로필 페이지에서만)

> **Note**: 홈 화면 코드의 `WINE_AXES`는 현재 6축(레거시)이며, TECH_SPEC 기준 7축(WSET)으로 마이그레이션 필요. SSOT는 PRD/TECH_SPEC의 7축이다.

### 4-5. 사진 캐러셀 (PhotoCarousel)

```
flex gap-2 overflow-x-auto snap-x snap-mandatory
scrollbar-hide (webkit-scrollbar: none)
```

- 1장: 100% 너비
- 2장+: 85% 너비, `snap-center`
- 높이: `h-64`
- 이미지: `rounded-2xl object-cover`, Next.js Image `fill` 모드
- 인디케이터: 하단 중앙 도트 (현재: `bg-white`, 나머지: `bg-white/50`)

### 4-6. 사진 선택 (PhotoCaptureSheet)

- **구현 패턴**: `<label>` + hidden `<input type="file">` (ref.click() 사용 금지)
- 카메라 버튼: `capture="environment"`
- 앨범 버튼: `multiple`
- 선택 후 그리드: 3열, 각 셀 `aspect-square rounded-xl overflow-hidden`
- 삭제: 우상단 X 버튼 (`w-5 h-5 rounded-full bg-black/50 text-white`)
- 카운터: 마지막 셀에 `N/8` 표시
- 재선택 보장: `e.target.value = ''`
