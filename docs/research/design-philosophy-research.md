## 1. 음식/레스토랑 앱 디자인 트렌드 (2024-2026)

### 1-1. 주요 앱 분석

#### Yelp

| 항목 | 분석 |
|------|------|
| **강점** | 커뮤니티 기반 리뷰 생태계, 상세한 필터 시스템, 사진 중심 리뷰 |
| **약점** | UI가 정보 과부하, 광고와 유기 결과 구분 어려움, 시각적으로 노후화 |
| **디자인 특징** | 레드 브랜드 컬러(`#D32323`), 별점 중심 카드, 사진 갤러리 강조 |
| **Nyam 시사점** | 커뮤니티 요소는 차용하되, 별점 중심 UI는 피할 것 |

#### Google Maps

| 항목 | 분석 |
|------|------|
| **강점** | 지도-검색 통합, 실시간 혼잡도, 방대한 데이터 |
| **약점** | 맛집 탐색에 특화되지 않음, 리뷰 큐레이션 부재 |
| **디자인 특징** | 미니멀한 카드, 지도 중심 인터랙션, Material Design 3 적용 |
| **Nyam 시사점** | 지도 연동은 보조적으로만, 검증 중심 UI가 메인 |

#### 배달의민족

| 항목 | 분석 |
|------|------|
| **강점** | 한국 시장 최적화 UX, 강력한 브랜드 아이덴티티, 유머러스한 톤 |
| **약점** | 배달 중심이라 방문 맛집 추천에 약함, 리뷰 신뢰도 문제 |
| **디자인 특징** | 민트 컬러(`#2AC1BC`), 배민체 커스텀 폰트, 일러스트 중심 |
| **Nyam 시사점** | 한국적 브랜드 톤 참고, 커스텀 폰트의 브랜드 파워 |

#### 캐치테이블

| 항목 | 분석 |
|------|------|
| **강점** | 프리미엄 포지셔닝, 깔끔한 UI, 예약 UX 완성도 |
| **약점** | 예약 가능 식당만 커버, 대중 식당 배제 |

| **Nyam 시사점** | 프리미엄 감성은 참고하되, Nyam은 보다 대중적이고 접근성 높게 |

#### 망고플레이트

| 항목 | 분석 |
|------|------|

| **Nyam 시사점** | 초기 신뢰 구축이 핵심이며, 상업화 시 디자인으로도 투명성 유지 |

#### DoorDash / Uber Eats

| 항목 | 분석 |

### 1-2. 2024-2026 푸드테크 UI/UX 트렌드

#### 트렌드 1: 몰입형 비주얼 (Immersive Visuals)

- 음식 사진이 화면의 50-70%를 차지하는 풀블리드 이미지
- 세로 스크롤 피드에서 각 식당이 "스토리카드" 형태로 표현
- 블러 배경 + 오버레이 텍스트로 정보 계층 구분
- **Nyam 적용**: 검증 데이터를 사진 위에 오버레이하는 형태 고려

#### 트렌드 2: 컨텍스트 어웨어 UI (Context-Aware UI)

- 시간대별 UI 변화 (아침/점심/저녁에 따라 추천 카테고리 자동 전환)
- 날씨, 계절에 따른 음식 추천 UI 변경
- 위치 기반 자동 필터링
- **Nyam 적용**: 시간대별 프롬프트 추천 + UI 톤 변화

#### 트렌드 3: AI 통합 UX

- 자연어 검색 ("비 오는 날 혼자 먹기 좋은 따뜻한 국물 요리")
- 대화형 추천 인터페이스 (챗봇 형태)
- AI 요약 리뷰
- **Nyam 적용**: 프롬프트 브릿지가 이 트렌드와 정확히 부합 -- Nyam의 핵심 차별점

#### 트렌드 4: 소셜 디스커버리 (Social Discovery)

- 인스타/틱톡 스타일의 숏폼 리뷰
- 친구 기반 추천 ("네 친구 3명이 방문한 곳")
- 공유 가능한 카드 형태의 리뷰 스니펫
- **Nyam 적용**: 검증 결과를 SNS 공유용 카드로 자동 생성

#### 트렌드 5: 미니멀리즘 + 의미 있는 디테일

- 전체적으로 미니멀하되, 핵심 인터랙션에서 디테일한 애니메이션
- "Less but better" 접근
- 과도한 장식 대신 타이포그래피와 여백으로 고급감
- **Nyam 적용**: 검증 관련 인터랙션에만 풍부한 모션, 나머지는 미니멀

#### 트렌드 6: 네오브루탈리즘 & 레트로 퓨처리즘

- 2024-2025년 일부 푸드 브랜드에서 네오브루탈리즘 채택 (굵은 테두리, 강한 그림자, 생생한 색상)
- 복고적 타이포그래피 + 현대적 레이아웃 조합
- **Nyam 적용**: 전체 적용은 과하나, 배지/카드 디자인에서 포인트로 활용 가능

## 2. 음식 앱을 위한 색채 심리학

### 2-1. 식욕 자극 색상 vs 신뢰 색상

#### 식욕 자극 색상

| 색상 | Hex | 심리적 효과 | 사용 예시 |
|------|-----|------------|----------|
| **레드** | `#E53935`, `#D32323` | 가장 강한 식욕 자극, 긴급감, 에너지 | Yelp, McDonald's, KFC |
| **오렌지** | `#FF6D00`, `#FF9800` | 따뜻함, 친근함, 식욕 자극 (레드보다 부드러움) | 망고플레이트, Fanta, 당근마켓 |
| **옐로우** | `#FFC107`, `#FFD600` | 행복감, 따뜻함, 주의 집중 | McDonald's, Subway |
| **웜 브라운** | `#795548`, `#8D6E63` | 커피, 초콜릿 연상, 자연스러움, 안정감 | 스타벅스 계열 |
| **웜 그린** | `#66BB6A`, `#43A047` | 신선함, 건강, 자연 | Whole Foods, 비건 앱 |

#### 신뢰 구축 색상

| 색상 | Hex | 심리적 효과 | 사용 예시 |
|------|-----|------------|----------|
| **블루** | `#1976D2`, `#2196F3` | 신뢰, 안정, 전문성, 보안 | 은행, 핀테크 (토스 `#0064FF`) |
| **딥 네이비** | `#1A237E`, `#283593` | 권위, 프리미엄, 전문성 | 금융앱, 법률 서비스 |
| **틸/시안** | `#00897B`, `#009688` | 신뢰 + 현대적, 테크 느낌 | 의료앱, 핀테크 |
| **퍼플** | `#7B1FA2`, `#9C27B0` | 혁신, 프리미엄, 창의성 | Nubank, Revolut |

#### 식욕 억제 색상 (피해야 할 것)

| 색상 | 이유 |
|------|------|
| **순수 블루** (`#0000FF`) | 자연에서 파란 음식이 거의 없어 식욕 억제 |
| **네온 그린** (`#00FF00`) | 부패, 독소 연상 |
| **그레이** (`#9E9E9E`) | 식욕 감소, 우울감 |
| **블랙** (과도 사용) | 무거움, 식욕 저하 (포인트로는 OK) |

### 2-2. Nyam을 위한 색상 전략: "음식의 따뜻함 + 검증의 신뢰"

Nyam의 과제는 **식욕 자극(음식 앱)**과 **신뢰감(검증 앱)**이라는 두 축을 동시에 만족시키는 것이다.

#### 추천 전략: 웜 코럴/앰버 + 틸/딥 블루 조합

**"음식의 따뜻함은 웜톤으로, 검증의 신뢰는 쿨톤으로 분리"**

```
Primary (브랜드/음식):
  - Warm Coral:    #FF6B6B  (식욕 자극 + 친근함, 레드의 공격성 완화)
  - Amber:         #FFB74D  (따뜻함, 긍정적 에너지)

Secondary (검증/신뢰):
  - Deep Teal:     #00796B  (신뢰 + 현대적, 블루의 차가움 완화)
  - Slate Blue:    #3D5A80  (안정감, 전문성)

Accent (강조/알림):
  - Verified Gold:  #F9A825  (검증 완료 배지, 성취감)
  - Fresh Green:    #66BB6A  (긍정적 결과, 안전)
  - Warning Red:    #EF5350  (경고, 의심 플래그)

Neutral:
  - Dark Text:     #212121
  - Body Text:     #424242
  - Muted Text:    #757575
  - Light BG:      #FAFAFA
  - Card BG:       #FFFFFF
  - Warm Gray:     #F5F0EB  (순수 그레이 대신 따뜻한 그레이)
```

#### 왜 이 조합인가

1. **Warm Coral (`#FF6B6B`)**: 전통적 "맛집 앱 레드"보다 부드럽고 현대적. 식욕을 자극하면서도 공격적이지 않아 신뢰감과 양립 가능
2. **Deep Teal (`#00796B`)**: 순수 블루의 차가움 없이 신뢰감을 줌. 의료/헬스케어 앱에서 검증된 "신뢰+케어" 색상. 코럴과 보색 관계로 시각적 대비 우수
3. **Warm Gray 배경 (`#F5F0EB`)**: 순수 화이트/그레이 대신 따뜻한 뉴트럴을 사용해 "음식 앱"의 온기를 배경에서부터 전달

### 2-3. 성공적인 음식 브랜드 색상 참고

| 브랜드 | 메인 색상 | 전략 | 결과 |
|--------|----------|------|------|
| **배달의민족** | 민트 `#2AC1BC` | 경쟁사(빨강) 대비 차별화, 신선함 | 강력한 브랜드 인지도 |
| **당근마켓** | 오렌지 `#FF6F0F` | 따뜻함, 동네 커뮤니티 느낌 | 친근한 로컬 이미지 |
| **토스** | 블루 `#0064FF` | 금융 신뢰, 깔끔함 | "안전한 금융 서비스" 인식 |
| **Sweetgreen** | 그린 `#2E7D32` + 크림 | 건강, 자연, 프리미엄 | 건강한 음식 포지셔닝 |
| **Shake Shack** | 그린 `#5BA829` + 블랙 | 자연+도시적 세련됨 | 프리미엄 버거 이미지 |

### 2-4. 색상별 사용 영역 가이드

```
Navigation / Header:     White + Deep Teal 텍스트
CTA 버튼:               Warm Coral (음식 탐색) / Deep Teal (검증 액션)
검증 배지:               Verified Gold + Deep Teal 텍스트
식당 카드:               White 카드 + Warm Gray 배경
진행 바:                 Deep Teal → Verified Gold 그라데이션
경고/의심:               Warning Red 아웃라인
텍스트:                  Dark on Light (명도 대비 7:1 이상 확보)
```

## 3. 신뢰 기반 디자인 패턴

### 3-1. 핀테크/헬스케어에서 배우는 시각적 신뢰 구축

Nyam은 "검증"이 핵심이므로, 높은 신뢰가 요구되는 핀테크/헬스케어 앱의 디자인 패턴을 적극 참고해야 한다.

#### 패턴 1: 투명한 정보 계층 (Transparent Information Hierarchy)

**토스(Toss)의 사례**:
- 모든 거래에 "왜 이 금액인지" 분해 표시
- 숨겨진 수수료 없음을 시각적으로 증명
- 정보를 감추지 않되, 단계적으로 공개 (Progressive Disclosure)

**Nyam 적용**:
```
식당 카드 기본 노출:
  "47명 AI 검증 | 맛 ■■■■□ | 가성비 ■■■□□"

탭하면 펼쳐지는 상세:
  "검증 소스: ChatGPT 32회, Claude 10회, Gemini 5회"
  "최근 검증일: 2026.03.14"
  "검증자 평균 프로필: 강남구 거주, 30대, 한식 선호"
  "검증 트렌드: 최근 1개월 검증 수 ↑ 23%"
```

#### 패턴 2: 출처 명시 (Source Attribution)

**의료 앱 사례 (Ada Health, WebMD)**:
- 모든 건강 정보에 "출처: WHO 가이드라인 2024" 등 명시
- 정보의 최종 업데이트 날짜 표시
- 신뢰도 등급 (의학적 근거 수준) 시각화

**Nyam 적용**:
- 검증 결과마다 "이 검증은 [사용자명]이 [날짜]에 [AI모델]로 수행" 명시
- 프롬프트 원문 열람 가능 ("이 결과를 만든 프롬프트 보기")
- 데이터 수집 방법론 설명 페이지 제공

#### 패턴 3: 실시간 상태 표시 (Real-time Status)

**뱅킹 앱 사례**:
- 거래 처리 단계를 실시간으로 시각화
- "처리 중 → 확인됨 → 완료"의 명확한 상태 전환
- 불확실성 제거를 통한 안심감

**Nyam 적용**:
- 검증 과정의 시각적 타임라인
- "프롬프트 생성됨 → AI 전달됨 → 결과 수신 대기 중 → 검증 완료"
- 각 단계에 체크마크 애니메이션

#### 패턴 4: 사회적 증거 시각화 (Social Proof Visualization)

**Wise(구 TransferWise) 사례**:
- "지금까지 N억원 전송됨" 실시간 카운터

### 5-3. 가독성 고려사항

#### 한글 특유의 타이포 이슈

| 이슈 | 해결 |
|------|------|
| **한글은 영문보다 시각적으로 큼** | 한글 15-16px = 영문 14px 체감 크기. 혼용 시 한글 기준으로 사이즈 결정 |
| **줄간격(Line Height)** | 한글은 1.6-1.8 권장 (영문 1.4-1.5보다 넓게). 자소 조합 특성상 좁으면 답답 |
| **자간(Letter Spacing)** | 한글은 -0.01em ~ -0.02em 정도 좁히면 응집력 향상. 과도한 자간 좁히기는 금지 |
| **한/영 혼용 사이즈 불일치** | Pretendard/SUIT는 이 문제를 폰트 레벨에서 해결 (한/영 크기 밸런싱) |
| **모바일 최소 크기** | 본문 최소 14px, 보조 텍스트 12px 이상 유지 |

```
base: 16px / 1.7              — 본문 텍스트
lg:   18px / 1.6              — 카드 제목
```

## 6. 마이크로 인터랙션 & 애니메이션

### 6-1. 음식 관련 딜라이트풀 애니메이션

#### 검증 완료 애니메이션 (핵심)

```
시나리오: 사용자가 AI 검증 결과를 Nyam에 공유했을 때

1단계 (0-300ms): 카드가 살짝 흔들림 (anticipation)
2단계 (300-600ms): 검증 배지가 0에서 풀 사이즈로 scale up
                   + circular reveal 효과로 Deep Teal 컬러 확산
3단계 (600-900ms): 체크마크 아이콘이 path animation으로 그려짐
                   ✓ 획이 왼쪽에서 오른쪽으로 그려지는 모션
4단계 (900-1200ms): 작은 파티클 이펙트 (confetti 대신 작은 별/반짝이)
                    + 햅틱 피드백 (success)
5단계 (1200ms+): 배지가 최종 위치로 settle (약간의 bounce)

Easing: cubic-bezier(0.34, 1.56, 0.64, 1) — 탄력 있는 오버슈트
```

#### 프롬프트 복사 애니메이션

```
시나리오: "AI로 검증하기" 버튼 탭 → 프롬프트 클립보드 복사

1단계: 버튼 ripple 효과
2단계: 버튼 텍스트 "AI로 검증하기" → "복사됨!" 전환 (crossfade)
3단계: 버튼 위에 말풍선 형태의 토스트 ("ChatGPT에 붙여넣기 하세요")
4단계: 3초 후 말풍선 fade out, 버튼 원래 상태로

Duration: 전체 1.5-2초
햅틱: 복사 완료 시 light impact
```

#### 식당 카드 로딩 애니메이션

```
시나리오: 식당 리스트 로딩 중

방법 A: Skeleton UI (추천)
  - 카드 형태의 회색 Skeleton
  - 사진 영역: 큰 회색 사각형 + shimmer animation
  - 텍스트 영역: 3줄의 둥근 회색 바 + shimmer
  - Shimmer: left-to-right linear gradient sweep
  - 색상: #F5F0EB → #FFFFFE → #F5F0EB

방법 B: 음식 아이콘 로더 (가벼운 데이터용)
  - 밥그릇 아이콘이 김이 모락모락 나는 애니메이션
  - 또는 젓가락이 음식을 집는 루프 애니메이션
  - 부드러운 bounce/float 모션
  - Duration: 루프, 800ms per cycle
```

#### Pull-to-Refresh

```
시나리오: 리스트를 아래로 당겨 새로고침

일반적인 스피너 대신:
  - 당기는 거리에 따라 돋보기 아이콘이 점점 커짐
  - 트리거 포인트 도달 시 돋보기가 회전 → 검색 시작
  - 완료 시 체크마크로 모핑
  - "N개의 새 검증 결과" 텍스트 노출
```

### 6-2. 마이크로 인터랙션 가이드라인

#### 타이밍 원칙

| 인터랙션 유형 | 권장 Duration | Easing |
|-------------|--------------|--------|
| 버튼 피드백 | 100-200ms | ease-out |
| 카드 전환 | 200-300ms | ease-in-out |
| 모달 열기/닫기 | 250-350ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 페이지 전환 | 300-400ms | ease-in-out |
| 성공 애니메이션 | 800-1200ms | spring (bounce) |
| 배지 획득 | 1000-1500ms | spring with overshoot |
| 토스트 알림 | 표시 200ms, 유지 3s, 사라짐 200ms | ease-out / ease-in |

#### 모션 원칙

1. **목적이 있는 모션만**: 장식적 애니메이션은 최소화. 사용자 행동의 결과를 명확히 전달하는 모션만 사용
2. **일관된 방향성**: 새 콘텐츠는 아래/오른쪽에서, 이전 콘텐츠는 위/왼쪽으로
3. **감속 우선**: 등장 시 deceleration (빠르게 나타나 천천히 정착), 사라질 때 acceleration (천천히 시작해 빠르게 사라짐)
4. **reduce-motion 존중**: `prefers-reduced-motion: reduce` 미디어 쿼리 필수 지원

### 6-3. Lottie 애니메이션 활용 포인트

| 포인트 | 애니메이션 설명 | 파일 크기 목표 |
|--------|---------------|---------------|
| 검증 완료 | 방패 + 체크마크 + 반짝이 | < 30KB |
| 배지 획득 | 메달 회전 + 빛나는 효과 | < 50KB |
| 빈 상태 | 돋보기로 찾는 캐릭터 | < 40KB |
| 에러 상태 | 당황한 표정의 아이콘 | < 20KB |
| 온보딩 | 프롬프트 → AI → 결과 흐름 | < 100KB |

## 8. 다크 모드 고려사항

### 8-1. 음식 앱에서 다크 모드의 딜레마

#### 장점

- 야간 사용 시 눈 편안함 (식사 시간은 주로 저녁)
- OLED 디스플레이 배터리 절약
- 음식 사진이 어두운 배경에서 더 "팝"하게 보임 (대비 효과)
- 프리미엄/고급 느낌 연출 가능

#### 단점/주의점

- **식욕 억제 위험**: 과도한 어둠은 식욕을 떨어뜨릴 수 있음
- **따뜻함 손실**: 라이트 모드의 Warm Gray 배경이 주는 음식적 따뜻함이 사라짐
- **색상 인식 변화**: 브랜드 컬러가 다크 배경에서 다르게 인식됨
- **접근성**: 텍스트 대비 비율 재검증 필요

### 8-2. 다크 모드 색상 시스템

```
다크 모드 Neutral:
  - Background:      #1A1612  (순수 블랙 대신 웜 다크 브라운)
  - Surface:         #2D2520  (카드 배경)
  - Surface Variant:  #3D3530  (강조 영역)
  - Elevated:        #4D4540  (모달/시트)

다크 모드 Text:
  - Primary:   #F5F0EB  (라이트 모드의 배경색을 텍스트로)
  - Secondary: #B8AFA5
  - Disabled:  #756E65

다크 모드 Brand Colors (밝기 조정):
  - Warm Coral:    #FF8A80  (라이트 대비 밝게)
  - Deep Teal:     #4DB6AC  (라이트 대비 밝게)
  - Verified Gold:  #FFD54F  (라이트 대비 밝게)
  - Warning Red:    #FF8A80
  - Fresh Green:    #81C784

핵심: 순수 블랙(#000000) 대신 웜 다크(#1A1612)를 사용하여
      "어두운 레스토랑" 분위기를 연출. 식욕 억제 방지.
```

### 8-3. 다크 모드 구현 원칙

| 원칙 | 설명 |
|------|------|
| **웜 다크 배경** | 순수 블랙 금지. `#1A1612` 같은 웜 다크 브라운으로 레스토랑 무드 유지 |
| **elevation = 밝기** | 카드, 모달 등 높은 elevation일수록 약간 밝게 (Material Design 3 방식) |
| **사진 보정 불필요** | 음식 사진은 다크 배경에서 자연스럽게 돋보이므로 별도 처리 불필요 |
| **아이콘 컬러 조정** | 모든 아이콘 색상을 다크 모드용으로 밝기 +20-30% 조정 |
| **그림자 제거** | 다크 모드에서 그림자 대신 미묘한 border (1px `rgba(255,255,255,0.08)`)로 구분 |
| **시스템 설정 따르기** | 기본적으로 OS 다크모드 설정 존중, 앱 내 수동 전환도 제공 |

### 8-4. 음식 사진과 다크 모드

```
라이트 모드:
  - 사진 모서리: 8-12px border-radius
  - 사진 하단: gradient overlay (transparent → card-bg)
  - 효과: 자연스럽고 깔끔한 전환

다크 모드:
  - 사진 모서리: 동일
  - 사진 하단: gradient overlay (transparent → #2D2520)
  - 추가: 사진 주변에 미묘한 glow 효과 (rgba(255,180,130,0.05))
  - 효과: 사진이 "빛나는" 느낌, 어두운 레스토랑에서 음식이 조명받는 분위기

주의: 사진 brightness/contrast 자동 조정 금지.
      사용자가 올린 실제 사진의 색감을 왜곡하면 신뢰 훼손.
```

```
   - 삼겹살, 치킨, 떡볶이, 냉면, 김밥 등
   - 통일된 스타일 (outlined, rounded, 2px stroke)
```

## 10. 음식 앱 접근성

### 10-1. WCAG 2.2 핵심 준수 사항

#### 색상 대비 (Level AA 필수, AAA 권장)

```
텍스트 대비 비율 (배경 대비):
  - 일반 텍스트 (16px 이하): 최소 4.5:1
  - 큰 텍스트 (18px Bold 또는 24px 이상): 최소 3:1
  - 아이콘/UI 컴포넌트: 최소 3:1

Nyam 색상 대비 검증:

  라이트 모드:
  - #212121 on #FAFAFA → 16.75:1 ✅ (AAA)
  - #424242 on #FFFFFF → 10.42:1 ✅ (AAA)
  - #00796B on #FFFFFF → 4.91:1  ✅ (AA)
  - #FF6B6B on #FFFFFF → 3.46:1  ⚠️ (큰 텍스트만 AA)
    → 소형 텍스트에 사용 금지, CTA 버튼 등 큰 요소에만 사용
  - #F9A825 on #FFFFFF → 2.15:1  ❌
    → 배경에 단독 사용 금지, 항상 다크 텍스트와 함께

  다크 모드:
  - #F5F0EB on #1A1612 → 14.3:1  ✅ (AAA)
  - #B8AFA5 on #2D2520 → 5.8:1   ✅ (AA)
  - #4DB6AC on #1A1612 → 6.2:1   ✅ (AA)
```

#### 터치 타겟 크기

```
최소 터치 영역: 44x44px (iOS HIG) / 48x48dp (Material Design)

적용 포인트:
  - 식당 카드의 "AI로 검증하기" 버튼: 최소 44px 높이
  - 탭바 아이콘: 48x48px 터치 영역 (아이콘 자체는 24px)
  - 검증 배지 탭 (상세 보기): 최소 44x44px
  - 필터 칩: 최소 36px 높이 (간격으로 44px 보충)
  - 닫기(X) 버튼: 최소 44x44px

주의: 검증 데이터 바(맛/가성비/분위기)는 정보 표시용이므로
      터치 타겟이 아닌 경우 작아도 됨
```

### 10-2. 스크린 리더 접근성

#### 식당 카드 시맨틱 구조

```html
<article aria-label="을지로 골목식당">
  <img src="..." alt="을지로 골목식당 대표 사진: 된장찌개와 반찬" />

  <h3>을지로 골목식당</h3>
  <p>한식 · 을지로3가역 도보 5분</p>

  <!-- 검증 배지 -->
  <div role="status" aria-label="AI 검증 47회 완료, 검증 등급: 다수 검증">
    <span aria-hidden="true">🛡️</span>
    <span>47명 AI 검증</span>
  </div>

  <!-- 검증 데이터 바 -->
  <div role="group" aria-label="검증 결과 요약">
    <div role="meter" aria-label="맛 평가" aria-valuenow="92" aria-valuemin="0" aria-valuemax="100">
      맛 92%
    </div>
    <div role="meter" aria-label="가성비 평가" aria-valuenow="85" aria-valuemin="0" aria-valuemax="100">
      가성비 85%
    </div>
  </div>

  <button aria-label="을지로 골목식당 AI로 검증하기">AI로 검증하기</button>
  <button aria-label="을지로 골목식당 저장하기">❤️</button>
</article>
```

#### 검증 애니메이션 접근성

```
- 검증 완료 애니메이션 재생 시:
  aria-live="polite" 영역에 "검증 완료! 47번째 검증이 기록되었습니다" 텍스트 추가
  → 스크린 리더가 결과를 읽어줌

- 배지 획득 시:
  aria-live="assertive"로 "축하합니다! '맛집 분석가' 배지를 획득했습니다" 알림

- 모든 장식적 아이콘에 aria-hidden="true"
- 의미 있는 아이콘에는 aria-label 필수
```

### 10-3. 음식 앱 특화 접근성

#### 식이 제한/알러지 정보

```
시각적 표시:
  - 알러지 정보: 아이콘 + 텍스트 병행 (색상만으로 구분 금지)
  - 채식/비건 마크: 초록 잎 아이콘 + "비건" 텍스트 라벨
  - 할랄/코셔: 해당 인증 마크 + 텍스트

스크린 리더:
  - "이 식당은 땅콩 알러지 주의가 필요합니다"
  - "비건 메뉴 제공 식당입니다"

색각 이상 (색맹) 대응:
  - 검증 등급을 색상만으로 구분하지 않음
  - 색상 + 아이콘 형태 + 텍스트 라벨 3중 구분
  - 예: 미검증(빈 방패), 검증됨(찬 방패+체크), 다수 검증(방패+별)
```

#### 모터 접근성

```
- 스와이프 액션(저장/검증)에는 항상 대체 버튼 제공
- "길게 누르기" 액션에는 항상 대체 경로 제공 (예: ... 더보기 메뉴)
- 복잡한 제스처 대신 단일 탭으로 모든 핵심 기능 접근 가능
- 프롬프트 복사 후 "AI 앱으로 이동" 버튼 제공 (클립보드 붙여넣기만 필요)
```

### 10-4. 성능 접근성

```
- 저사양 기기/느린 네트워크 대응:
  - 이미지 lazy loading 필수
  - 음식 사진 WebP/AVIF 포맷 + srcset 반응형
  - Skeleton UI로 CLS(Cumulative Layout Shift) 방지
  - 핵심 기능(프롬프트 복사)은 이미지 로딩 전에도 작동

- 데이터 절약 모드:
  - prefers-reduced-data 미디어 쿼리 지원
  - 썸네일 우선 로드, 상세 사진은 요청 시
  - 애니메이션 최소화 옵션

- 오프라인 지원:
  - 저장한 맛집 목록은 오프라인에서도 열람 가능
  - 프롬프트 템플릿 로컬 캐싱
  - 검증 결과 로컬 저장 후 온라인 시 동기화
```
