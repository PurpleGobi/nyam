기록 피로(Logging Fatigue)의 메커니즘

  • CHI 2015 설문: 141명 기록자 중 가장 많은 불만 — "too much effort"(31명), "time-consuming"(27명), "tedious"(16명)
  • Streak fatigue: 긴 연속 기록이 한 번 끊기면 동기가 급격히 하락 (MyFitnessPal 커뮤니티)
  • 사진 촬영의 식사 방해: "음식 사진 촬영은 식사 절차를 중단시키고 음식을 식게 만든다" (Frontiers in Psychology 2021)

위치 추적 프라이버시 거부감

  • 다이어트 앱 사용자 60% 이상이 데이터 프라이버시 우려 (market.us)
  • iOS ATT 도입 이후 위치 권한 허용률 급감
  • Android 알림 옵트인율: 2024년 85% → 2025년 67%로 1년 만에 18%p 하락
  • "건강 앱도 거부감인데, 맛집 기록용 상시 위치 추적이라면?"
판정: "AI로 기록을 편하게 만들어도, 기록 행위 자체의 심리적 마찰을 완전히 제거할 수 없다." 15년간의 앱 시장 역사가 반복적으로 증명한 사실이다.

한국 음식 인식의 특수한 난제

  • 반찬 5~7개 + 국/찌개 + 밥이 한 사진에 — 개별 분류 극도로 어려움
  • 찌개류 외관 유사성: 된장찌개, 김치찌개, 순두부찌개 → 같은 갈색/붉은 국물
  • 떡볶이+튀김+순대 한 접시 — 복수 음식 분할 문제
  • PMC 종합 리뷰: "CNN 기반 모델은 조리되고 섞인 음식에서 여전히 성능 문제"
판정: 혼합 요리 정확도 73%는 "10번 중 3번 틀린다"는 뜻. 사용자가 매번 수정해야 한다면 "자동 인식"의 가치가 없다.

한국어 OCR의 현실

KORIE 벤치마크 (MDPI, 2026.1) — 한국 영수증 OCR 전문 연구:

  • 작은 폰트의 한글 인식 어려움
  • 자모 분리 현상(jamo fragmentation): 블러 시 초성/중성/종성이 분리
  • 한글 음절 수 ~11,000개 (영어 26자 대비 압도적으로 복잡)
  • 메뉴판 포맷 비표준: 가게마다 완전히 다른 레이아웃, 디자인 폰트

배터리: 사용자가 가장 싫어하는 것

"공격적인 온디바이스 학습은 눈에 띄는 기기 발열과 급격한 배터리 소모를 유발하여 사용자 불만과 앱 삭제로 이어진다" — ML Systems Book

음식 인식 + 위치 추적 + OCR 동시 상시 구동 → 배터리 15~20% 추가 소모. 사용자에게 "맛집 기록 앱 때문에 배터리가 닳는다"는 인식이 생기는 순간 삭제.

3. AI 리뷰 자동생성 — "누가 읽고 싶어하는가?"

AI 생성 콘텐츠에 대한 소비자 불신

55%
소비자가 AI에 불편함을 느끼는 비율 (Forbes 2025)
  • NIM 연구 (2025): AI 생성 라벨링 시 소비자는 "덜 자연스럽고 덜 유용하다"고 인식, 구매 의향 하락
  • KPMG 2024: AI를 통해 수집된 데이터를 신뢰하는 소비자 29%에 불과
  • KO Insights (2025): "소비자들은 AI 생성 콘텐츠를 거부(rejecting)하고 있다" — Authenticity Premium 형성 중

"진정성"의 훼손

리뷰의 가치는 실제 경험한 사람의 주관적 감정과 세부 묘사에 있다. AI가 "객관식 5개를 조합해 만든 문단"은 이 본질을 빗나간다.

  • 객관식 5개 탭 → AI 문단 = 노력 최소화 = 소유감 최소화 (IKEA Effect 역적용)
  • 소유감이 없으면 공유 동기도 없고, 수정/관리 동기도 없다
  • "내 리뷰"가 아니라 "앱이 써준 것"이 된다

가짜 리뷰 생태계 악화

"AI 지원 리뷰가 현재 속도로 증가하면, 2026년 말에는 AI 리뷰가 인간 작성 리뷰를 초과할 수 있다" — Whitespark 2025
  • 온라인 리뷰의 약 30%가 이미 가짜 (ReviewDriver 2025)
  • Yelp: AI 지원 리뷰 공식 금지
  • "AI가 리뷰를 대신 써주는 기능"은 가짜 리뷰 생태계를 정당화하는 것과 다를 바 없다
판정: AI 리뷰 자동생성은 "편의성"과 "진정성"이 정면 충돌한다. 편하게 만들수록 가치가 떨어지는 역설.
4. "모든 것을 하는 앱"의 실패 역사

Foursquare/Swarm: 체크인 + 추천의 몰락

  • 밸류에이션: 2013년 $6.5억 → 2016년 $3.25억으로 반토막
  • 체크인(Swarm) + 추천(Foursquare) 분리 → 양 앱 모두 앱스토어 순위 급락
  • 통합 상태에서도 성장 정체 → 분리해도 해결 안 됨 → 다기능 전략 자체의 문제

Path: 라이프로그 소셜의 완벽한 실패

  • "음식, 운동, 기분, 음악, 수면 기록 + 소셜" 통합 앱
  • $6,620만 투자 → 2018년 종료
  • 각 카테고리별 전문 앱(Instagram, Spotify, 수면 앱)에 모두 패배

슈퍼앱은 왜 실패하는가

  • Forbes: "미국 주요 산업은 과점 구조 — 슈퍼앱이 진입할 틈이 없다"
  • 카카오톡: 2025년 슈퍼앱 개편 시도 → 사용자 집단 반발 → 사실상 후퇴
  • CB Insights: 스타트업 실패 1위 = "시장 수요 부재". Feature creep이 이를 악화
"2%의 사용자만 혜택을 받는 기능이 남은 개발 시간의 30%를 잡아먹는다" — Pragmatic Coders
판정: "와인+음식+카페+집밥" 통합은 각 영역 전문 앱(Vivino, Beli, Savor)과 동시에 싸우겠다는 의미. 역사적으로 이 전략은 실패한다.
5. 네트워크 효과 부재 — "개인 도구는 해자가 없다"

도구 → 소셜 전환은 거의 불가능

"사용자를 도구에서 네트워크로 전환시키는 것은 어렵다. 행동 변화를 요구하기 때문이다." — Andrew Chen, a16z

Evernote의 교훈

  • "당신의 외부 두뇌" → 단일 사용자 도구에서 멀티플레이어 전환 실패
  • 2011년 $10억 밸류에이션 → Bending Spoons에 헐값 인수
  • Notion, Obsidian, Apple Notes에 지속적으로 침식
  • 네트워크 효과 없는 도구는 언제든 대체될 수 있다

"나만의 기록"은 공유될 이유가 없다

  • "오늘 점심에 김치찌개 먹었다"는 정보의 공유 가치가 극히 낮다
  • 인스타에서 음식 사진이 공유되는 이유는 시각적 매력이지 기록 자체가 아니다
  • 맛집 추천이 공유되는 이유는 발견의 가치이지 일상 식사 로그가 아니다
  • Obsidian/Notion도 소셜 기능 추가에 실패 — 개인 도구 → 소셜 성공 사례 거의 없음
판정: 개인 기록 도구는 네트워크 효과가 없어 해자가 없다. 네이버/카카오가 같은 기능을 추가하면 한 분기면 따라잡는다.

CAC vs LTV가 성립하지 않는다

  • CAC: 한국 모바일 앱 CPI $2-5, 유료 전환 시 $30-50+
  • LTV: 월 $4 구독, 평균 3-6개월 유지 → $12-24
  • CAC > LTV: 유료 전환율 5% 미만이면 단위 경제 성립 불가

한국 시장의 특수한 포화

  • "먹스타그램"이라는 용어가 보편화 — 이미 인스타가 식음료 기록 역할
  • 맛집 공유 = 카카오톡으로 네이버 링크 보내기 — "왜 새 앱을 깔아야 하나?"
  • 네이버 블로그 맛집 리뷰 ~1,050만 블로그 — 확립된 행동 패턴을 바꿔야 함
  • 캐치테이블, 다이닝코드, 식신, 테이블링, 포잉 — 이미 니치별로 포화
판정: 네이버/카카오에게 이 기능을 추가하는 건 "신규 기능 하나 정도의 개발 리소스"에 불과하다. 거인이 한 분기만 투자하면 따라잡는다.
