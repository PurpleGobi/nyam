사용자가 인지하는 구조:

  • 종합 레벨: 냠 Lv.23 미식가 — "나의 전체 수준"
  • 영역 레벨: 을지로 Lv.8, 일식 Lv.7 — "내가 잘 아는 분야"
  • 대표 프로필에는 Lv.5 이상 달성한 영역만 노출 (최대 3개, 사용자 선택)
  • 상세 페이지에서 모든 영역의 진행 상황 확인 가능
핵심: "냠 Lv.23 미식가 · 을지로 Lv.8" — 종합 수준 + 전문 분야가 한눈에. 친구에게 "나 을지로 8레벨이야"라고 말할 수 있다.

3. 뒷단: 영역 레벨 산출 알고리즘

영역 XP 산출

기록 1건이 생성되면, 해당 기록이 속하는 모든 영역에 XP가 적립된다.

영역 XP = base_xp × quality_multiplier × recency_bonus base_xp: 사진 포함 기록 = 8 사진 없는 기록 = 3 quality_multiplier (1.0 ~ 2.0): 제스처 평가 입력 → ×1.3 코멘트 작성 → ×1.2 일기 작성 → ×1.2 (곱연산: 평가+코멘트+일기 = 1.3 × 1.2 × 1.2 = 1.87) recency_bonus: 해당 영역에 최근 7일 내 기록이 있으면 ×1.1 (연속 활동 보너스) 예시: 사진+평가+코멘트 기록 → 8 × 1.3 × 1.2 × 1.0 = 12.5 → 12 XP

영역 간 시너지

같은 기록이 여러 영역에 동시 적립되므로, 다양한 활동이 여러 영역을 동시에 키운다. 하지만 특정 영역에 집중하면 그 영역이 빠르게 성장.

전략 A: "을지로 전문가" 을지로에서만 계속 기록 → 📍 을지로 Lv.빠르게 상승 장르는 다양하면 → 🍴 각 장르 Lv.느리게 분산 상승 전략 B: "일식 전문가" 여러 지역의 일식집 → 🍣 일식 Lv.빠르게 상승 지역은 분산 → 📍 각 지역 Lv.느리게 분산 상승 전략 C: "다재다능 탐험가" 다양한 지역+장르+씬 → 모든 영역 골고루 상승 특정 영역 Lv.5+ 도달은 느리지만, 종합 냠 Lv.는 빠르게 상승

종합 냠 Lv.에 영역 레벨이 미치는 영향

종합 냠 Lv. 산출의 5대 요소 중 Activity에 영역 레벨의 다양성과 깊이가 반영된다:

Activity 요소 내부: volume_score × 0.4 (총 기록 수) recency_score × 0.3 (최근 활동) domain_score × 0.3 (영역 레벨 다양성+깊이) domain_score 산출: = (Lv.5+ 달성 영역 수 × 0.5 + 최고 영역 Lv. × 0.3 + 평균 영역 Lv. × 0.2) / 정규화 → 영역 레벨이 높을수록 종합 냠 Lv.도 빠르게 상승
설계 원칙: 영역 레벨은 "보이는 즉각 보상" (XP가 바로 올라간다), 종합 냠 Lv.은 "종합적 성장 지표" (뒷단에서 복합 산출). 사용자는 "기록 → 영역 XP 올랐다 → 레벨업!" 체감.
4. 뒷단: 종합 냠 레벨 산출 알고리즘

사용자는 "냠 Lv."만 보지만, 뒷단에서는 5개 요소의 복합 점수

냠 Lv. = f(Activity, Quality, Consistency, Community, Calibration) 각 요소는 0.0 ~ 1.0 사이 정규화 점수 가중 합산 → 0~100 Raw Score → 비선형 매핑 → Lv. 1~99

5대 요소 (Internal Factors)

요소코드명가중치측정 대상
활동량Activity30%총 기록 수, 최근 4주 기록 수

각 요소 세부 산출

Activity (활동량) — 30%

Activity = w1 × volume_score + w2 × recency_score volume_score: 기록 수 0-9 → 0.0 ~ 0.2 기록 수 10-49 → 0.2 ~ 0.5 기록 수 50-199 → 0.5 ~ 0.8 기록 수 200+ → 0.8 ~ 1.0 (로그 스케일: log2(records + 1) / log2(max_records + 1)) recency_score: 최근 4주 기록 수 기반 (시간 감쇠) decay_factor = 0.95^(days_since_last_record) 최근 활동이 없으면 점수가 서서히 하락 w1 = 0.6, w2 = 0.4

Quality (기록 품질) — 25%

Quality = 기록별 completeness 가중 평균 기록별 completeness (0~100): 사진 있음 +25 메뉴+식당 확인됨 +15 제스처 평가 입력 +20 코멘트 있음 +15 태그 있음 +5 가격 입력 +5 일기 작성 +10 그룹 공유 +5 Quality = avg(최근 50건의 completeness) / 100 가중치: 최신 기록일수록 가중치 높음 (지수 감쇠)

Consistency (꾸준함) — 20%

Consistency = 1 - normalized_std_dev(weekly_records) 최근 12주간 주별 기록 수의 표준편차를 정규화 편차가 작을수록 (꾸준할수록) 점수 높음 예시: 매주 3,3,2,3,3,2,3,3,3,2,3,3 → std=0.45 → Consistency ≈ 0.85 매주 0,0,10,0,0,8,0,0,12,0,0,0 → std=4.2 → Consistency ≈ 0.15 최소 활동 기준: 4주 중 2주 이상 기록이 있어야 산출 미달 시 Consistency = 0.0

Community (커뮤니티) — 15%

Community = w1 × share_ratio + w2 × social_impact + w3 × group_activity share_ratio: 그룹에 공유한 기록 수 / 전체 기록 수 0% → 0.0, 20%+ → 0.7, 50%+ → 1.0 social_impact: (받은 저장 + 받은 좋아요) / 총 공개 기록 수 Bayesian average로 기록 수 적을 때 보정: impact = (C × M + 실제합) / (C + 기록수) C = 전체 사용자 평균 기록 수, M = 전체 평균 반응률 group_activity: 소속 그룹 내 활동 (공유 + 반응) 빈도 주 1회+ 활동 → 0.5, 주 3회+ → 0.8, 매일 → 1.0 w1 = 0.3, w2 = 0.4, w3 = 0.3

Calibration (평가 정밀도) — 10%

Calibration = w1 × participation + w2 × coverage participation: 최근 30일 Pairwise 비교 횟수 0회 → 0.0, 10회 → 0.5, 30회+ → 1.0 coverage: Elo 랭킹이 완성된 카테고리 수 (최소 5건 이상 기록된 카테고리) 0개 → 0.0, 1개 → 0.3, 3개 → 0.7, 5개+ → 1.0 w1 = 0.6, w2 = 0.4

Raw Score → 냠 Lv. 매핑

Raw Score = Activity×0.30 + Quality×0.25 + Consistency×0.20 + Community×0.15 + Calibration×0.10 Raw Score 범위: 0.0 ~ 1.0 Lv. 매핑 (비선형 — 초반은 빠르게, 후반은 느리게): Lv. = floor(50 × Raw^0.7 + 1) Raw 0.05 → Lv. 2 (새내기) Raw 0.10 → Lv. 4 (새내기) Raw 0.15 → Lv. 6 (단골) Raw 0.25 → Lv. 9 (단골) Raw 0.35 → Lv. 13 (단골) Raw 0.45 → Lv. 16 (미식가) Raw 0.55 → Lv. 19 (미식가) Raw 0.65 → Lv. 23 (미식가) Raw 0.75 → Lv. 27 (미식가) Raw 0.82 → Lv. 31 (감별사) Raw 0.90 → Lv. 37 (감별사) Raw 0.95 → Lv. 43 (감별사) Raw 0.98 → Lv. 49 (감별사) Raw 0.99 → Lv. 51 (마스터) → 새내기→단골: 빠르게 (첫 경험 보상) → 미식가→감별사: 꾸준히 (중간 도전) → 마스터: 매우 어렵게 (희소성 유지)
설계 원칙: 초반 5레벨은 기록 3-5건이면 달성 → "성장감 즉시 체감". 마스터(Lv.51+)는 전체 사용자의 1-3%만 도달 → "희소성".

시간 감쇠 (Time Decay)

오래 활동하지 않으면 레벨이 서서히 하락:

비활동 기간에 따른 Raw Score 감쇠: 1주 비활동 → 감쇠 없음 2주 비활동 → Recency score × 0.95 4주 비활동 → Recency score × 0.85 8주 비활동 → Recency score × 0.70 12주+ 비활동 → Recency score × 0.50 단, Volume/Quality/Community는 감쇠 없음 (축적된 가치) Consistency만 0으로 리셋 복귀 시: 기록 1건 작성하면 즉시 Recency 복구 시작
원칙: 레벨은 "한번 달성하면 영원한 것"이 아니라 "유지하려면 활동이 필요한 것". 단, 하락 속도는 느리게 (12주 비활동해도 반만 감쇠). 복귀 시 빠르게 회복 가능.
6. Elo 랭킹 시스템 (Layer 3 상세)

Elo 알고리즘

초기 Elo: 모든 기록 = 1500 비교 결과 반영: 예상 승률 E_a = 1 / (1 + 10^((Elo_b - Elo_a) / 400)) 승자 갱신 Elo_a += K × (1 - E_a) 패자 갱신 Elo_b -= K × E_a 무승부 Elo_a += K × (0.5 - E_a) K값 (동적): 해당 기록의 비교 횟수 < 5 → K = 40 (초기 빠른 수렴) 비교 횟수 5-15 → K = 24 (안정화) 비교 횟수 15+ → K = 16 (미세 조정) 최소 비교 횟수: 랭킹 표시 = 3회 이상 비교된 기록만 신뢰 랭킹 = 8회 이상 비교된 기록 (별도 표시)

7. 레벨 포인트 상세 — 사용자가 보는 즉각 보상

포인트 ≠ 레벨

포인트는 즉각적 보상감을 주는 도구이고, 레벨은 5개 요소의 복합 산출 결과다. 포인트가 쌓이면 레벨이 올라가지만, 포인트만으로는 레벨업이 안 된다 (Quality, Consistency 등이 뒷받침되어야).

포인트 획득 전체표

포인트는 "느낌"이고, 실제 레벨은 뒷단 5대 요소로 결정. 포인트를 많이 모아도 Quality나 Consistency가 낮으면 레벨이 안 오른다. 단, 사용자에게는 "포인트를 열심히 모으면 레벨이 오른다"고 느끼게 한다 — 실제로 대부분의 경우 포인트 획득 행동이 5대 요소를 자연스럽게 충족시키기 때문.

앞단 UX: "기록하면 +5, 평가하면 +2, 일기 쓰면 +3" → 즉각 보상. 뒷단 로직: 5대 요소 복합 산출 → 실제 레벨 결정. 괴리는 거의 없지만, 어뷰징 시에만 차이 발생.

핵심 원칙

  1. 사용자가 외울 건 하나: "냠 Lv.23 미식가"
  2. 즉각 보상: 모든 행동에 +N 포인트 피드백
  3. 초반은 빠르게: 기록 3-5건이면 새내기 졸업 → "나도 성장한다" 체감
  4. 정상은 희소하게: 마스터는 전체 1-3%. 가치 있는 목표
  5. 강제 없음: Layer 2, 3은 선택. 넛지와 보상으로 유도
  6. 뒷단은 정교하게: 5대 요소 + 어뷰징 방지 + 시간 감쇠
