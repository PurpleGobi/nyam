## 핵심 철학

**"앱이 AI를 호출하지 않는다. 사용자가 AI를 무기로 쓸 수 있게 돕는다."**

Nyam은 LLM API를 직접 호출하는 대신, 사용자가 이미 사용하는 AI(ChatGPT, Claude, Gemini 등)에 **최적화된 프롬프트를 원클릭으로 전달**할 수 있는 브릿지 역할을 한다. 이 구조는 다음을 달성한다:

| 이점 | 설명 |
|------|------|
| **비용 제로** | LLM API 비용이 사용자 측에서 발생 (무료 티어 활용 가능) |
| **모델 중립** | 특정 AI에 종속되지 않음, 사용자가 선호하는 AI 사용 |
| **신뢰 이전** | "앱이 말한 것"이 아닌 "내가 AI로 확인한 것" → 능동적 검증의 심리적 만족 |
| **책임 분산** | 추천 결과에 대한 앱의 법적/도덕적 책임 경감 |
| **사용자 역량 강화** | 단순 소비자 → 능동적 검증자로 역할 전환 |

## 1. 프롬프트 브릿지 시스템

### 1-1. 작동 방식

```
사용자가 식당 정보를 본다
  → "AI로 검증하기" 버튼 탭
  → 상황에 맞는 프롬프트가 자동 생성됨
  → 클립보드 복사 / ChatGPT 앱 딥링크 / 공유 시트
  → 사용자가 AI에서 결과 확인
  → (선택) 결과 요약을 Nyam에 다시 공유
```

### 1-2. 프롬프트 템플릿 카테고리

#### A. 리뷰 신뢰도 검증 프롬프트

> 페인포인트 #1, #4, #11 해결

```
"[식당명] [지역]의 리뷰를 분석해줘.
네이버, 구글맵, 인스타그램 등 여러 소스의 평판을 종합해서:
1. 체험단/협찬 리뷰 비율이 높은지
2. 최근 6개월 리뷰 트렌드 (개선/악화)
3. 반복 언급되는 긍정/부정 키워드
4. 가격 대비 만족도
5. 실제 방문 가치가 있는지 (10점 만점)
솔직하게 평가해줘."
```

**동적 변수**: 식당명, 지역, 메뉴 카테고리가 자동 삽입됨

#### B. 상황 맞춤 추천 프롬프트

> 페인포인트 #3 해결

```
"[지역]에서 [상황]에 맞는 식당 3곳만 추천해줘.
조건:
- 인원: [N]명
- 예산: 1인당 [금액]원
- 분위기: [키워드]
- 제외: [알러지/비선호]
추천 이유와 대표 메뉴, 예상 가격을 포함해줘.
광고/협찬이 아닌 실제 평판 기반으로만."
```

**상황 프리셋 예시**:
- 비즈니스 점심 → 조용함, 코스 가능, 빠른 서빙
- 데이트 → 분위기, 사진 잘 나오는, 예약 가능
- 혼밥 → 카운터석, 부담 없는, 1인 메뉴
- 가족 모임 → 넓은 공간, 다양한 메뉴, 주차

#### C. 비교 분석 프롬프트

> 페인포인트 #5, #10 해결

```
"다음 식당들을 비교해줘: [식당A], [식당B], [식당C]
비교 기준: 맛, 가성비, 서비스, 분위기, 대기시간
표 형식으로 정리하고 최종 추천 1곳과 이유를 알려줘."
```

#### D. 정보 최신성 확인 프롬프트

> 페인포인트 #4 해결

```
"[식당명] [지역] 현재 영업 중인지 확인해줘.
최근 메뉴, 가격, 영업시간, 휴무일, 예약 필요 여부 등
가장 최신 정보를 알려줘."
```

#### E. 숨은 맛집 발굴 프롬프트

> 페인포인트 #2, #8 해결

```
"[지역]에서 광고나 인플루언서 없이 입소문으로만 유명한
숨은 맛집을 찾아줘. 조건:
- 네이버 블로그 체험단 리뷰가 적을 것
- 동네 주민/단골이 추천하는 곳
- 오래된 곳 우선
- [음식 종류] 중심으로"
```

### 1-3. 프롬프트 전달 경로

| 경로 | 구현 | UX |
|------|------|-----|
| **클립보드 복사** | `navigator.clipboard.writeText()` | 가장 단순, 모든 AI 앱에서 붙여넣기 |
| **ChatGPT 딥링크** | `https://chat.openai.com/?q={prompt}` | 원탭으로 ChatGPT에서 바로 대화 시작 |
| **공유 시트** | Web Share API / 네이티브 공유 | Claude 앱, Gemini 앱 등으로 공유 |
| **URL 스킴** | 각 AI 앱의 URL scheme 활용 | 설치된 AI 앱으로 직접 이동 |

## 2. 심리적 참여 설계

### 2-1. IKEA 효과 (노력 정당화)

> **"내가 직접 검증한 맛집"은 남이 추천한 것보다 더 가치 있게 느낀다.**

- 사용자가 AI 프롬프트를 실행하고 결과를 확인하는 **능동적 과정** 자체가 식당 선택에 대한 확신을 높임
- 수동적 별점 소비 → 능동적 AI 검증자로 역할 전환
- 검증 결과를 Nyam에 공유하면 **"내가 발견한 맛집"**이라는 소유감 형성

**구현**:
- "AI 검증 완료" 배지를 사용자 프로필에 누적
- "검증한 맛집" 개인 컬렉션 자동 생성
- 검증 과정의 스크린샷/결과 공유 기능

### 2-2. 사회적 증거 (Social Proof) 재설계

> **기존 별점 대신, 검증 행동 기반의 새로운 신뢰 지표**

기존 플랫폼의 별점은 조작 가능하지만, **"몇 명이 AI로 검증했는가"**는 조작 동기가 없다.

| 기존 지표 | Nyam 대안 지표 | 조작 가능성 |
|-----------|---------------|------------|
| ★ 4.5 (1,200리뷰) | "47명이 AI로 검증" | 낮음 (검증 행위 자체에 비용) |
| "맛집 TOP 10" | "이번 주 가장 많이 검증된 식당" | 낮음 |
| 파워블로거 추천 | "검증왕 @user의 인증 맛집" | 낮음 |

**구현**:
- 식당 카드에 "AI 검증 N회" 표시
- 검증 히트맵 (어떤 항목이 많이 검증되었는지)
- "검증된 맛집만 보기" 필터

### 2-3. 가변 보상 (Variable Reward)

> **슬롯머신 원리: 예측 불가능한 보상이 반복 행동을 유발**

검증할 때마다 동일한 경험이 아니라, 때때로 **예상치 못한 발견**이 있도록 설계.

- **숨은 맛집 알림**: "당신이 검증한 [지역] 근처에 아직 아무도 검증하지 않은 식당이 있어요"
- **검증 보상 랜덤**: 검증 시 랜덤으로 "맛집 카드" 획득 (수집 요소)
- **트렌드 발견**: "이 식당 최근 검증 급증 중! 먼저 발견한 N명 중 한 명이에요"

### 2-4. 목표 그래디언트 효과 (Goal Gradient)

> **목표에 가까울수록 행동이 가속된다.**

- **"이 지역 맛집 마스터"** 칭호 시스템
  - 강남구 맛집 10곳 검증 → 브론즈
  - 30곳 → 실버
  - 100곳 → 골드 + "강남 맛집 마스터" 배지
- 진행 바를 보여줘서 "3곳만 더 검증하면 실버!" 동기 부여
- 지역별/카테고리별 검증 맵 (빈 칸 채우기 욕구 자극)

### 2-5. 자기결정성 이론 (Self-Determination Theory)

> **자율성, 유능감, 관계성 — 세 가지 기본 심리 욕구 충족**

| 욕구 | 기존 앱 | Nyam |
|------|---------|------|
| **자율성** | 앱이 추천 → 수동 소비 | 사용자가 AI로 직접 검증 → 능동적 선택 |
| **유능감** | 별점만 보고 판단 → 실패 시 무력감 | AI 검증으로 정확한 판단 → "나는 똑똑한 소비자" |
| **관계성** | 익명 리뷰 나열 | 검증 커뮤니티 소속감, 같은 취향 그룹 |

### 2-6. 손실 회피 (Loss Aversion) 프레이밍

> **"좋은 곳을 찾는 것"보다 "나쁜 곳을 피하는 것"이 더 강한 동기.**

- 프롬프트를 "추천해줘"가 아닌 **"피해야 할 곳이 있는지 알려줘"** 톤으로도 제공
- "검증 없이 방문한 사용자의 30%가 후회했어요" 같은 넛지
- **"AI 검증 안 한 식당"** 경고 표시 (과하지 않게)

### 2-7. 커밋먼트 & 일관성 (Commitment & Consistency)

> **작은 행동에서 시작해 점점 깊은 참여로.**

```
1단계: 프롬프트 복사 (1탭, 부담 없음)
2단계: 검증 결과에 "맞아요/아니에요" 반응 (1탭)
3단계: 한 줄 코멘트 추가 ("김치찌개 진짜 맛있었음")
4단계: 자기 프롬프트 커스터마이징
5단계: 다른 사용자 검증 결과에 피드백
```

각 단계의 마찰을 극도로 낮춰서 자연스럽게 다음 단계로 이동.

## 3. 페인포인트별 솔루션 매핑

### PP#1. 리뷰 신뢰도 붕괴

**전략**: 리뷰를 "읽는 것"에서 "검증하는 것"으로 행동 전환

| 기능 | 설명 | 심리적 효과 |
|------|------|------------|
| **리뷰 검증 프롬프트** | 식당의 리뷰 패턴을 AI에게 분석 요청 | 유능감 (내가 직접 팩트체크) |
| **검증 스코어** | 사용자 검증 결과 기반 신뢰도 점수 | 사회적 증거 (다수가 확인한 곳) |
| **의심 플래그** | 커뮤니티가 "리뷰 의심" 표시 가능 | 집단 지성 참여, 정의감 충족 |

### PP#2. 광고와 추천의 구분 불가

**전략**: 추천 근거를 투명하게 + 사용자가 직접 확인 가능

| 기능 | 설명 |
|------|------|
| **"왜 이 식당?" 프롬프트** | 특정 식당이 왜 추천되는지 AI에게 질문 |
| **광고 없는 추천** | Nyam 자체 추천에 광고 개입 없음을 명시 |
| **추천 근거 표시** | 각 추천에 데이터 소스와 이유 표시 |

### PP#3. 개인화/추천 알고리즘의 한계

**전략**: 알고리즘 대신 사용자가 직접 AI에게 상황을 설명

- 상황별 프롬프트 프리셋 (데이트, 비즈니스, 혼밥 등)
- 사용자가 프롬프트에 자기 조건을 자유롭게 추가 가능
- **알고리즘이 맞추려 하는 것 vs 사용자가 직접 말하는 것** → 후자가 항상 정확

### PP#5. 평점 시스템의 구조적 문제

**전략**: 별점을 대체하는 검증 기반 평가 체계

```
기존:  ★★★★☆ (4.2) - 1,234개 리뷰
Nyam:  🔍 47명 검증 | 맛 ■■■■□ | 가성비 ■■■□□ | 분위기 ■■■■■
       최근 검증: "2026.03.14 - 점심 방문, 된장찌개 추천"
```

- 다차원 평가는 검증 결과에서 자동 추출 (사용자가 AI 결과 공유 시)
- 별점 없이도 직관적으로 판단 가능한 UI

### PP#9. MZ/Z세대 행동 변화

**전략**: AI 네이티브 세대의 행동 패턴에 맞춤

- **"AI한테 물어봐"**가 이미 익숙한 세대 → 프롬프트 브릿지가 자연스러움
- 검증 결과를 인스타 스토리/틱톡으로 공유 가능한 카드 형태로 생성
- 숏폼 시대에 맞는 **한 줄 검증 결과** ("AI가 분석한 이 식당: 맛 9/10, 웨이팅 평균 20분")

### PP#10. 정보 과부하 / 선택 장애

**전략**: "딱 3곳" 프롬프트 + 그룹 의사결정 지원

- 추천 프롬프트가 기본적으로 **3곳**만 추천하도록 설계
- **그룹 검증**: 모임 멤버가 각자 AI 검증 → 결과를 Nyam 그룹에 공유 → 투표
- 비교 프롬프트로 최종 후보 정리

### PP#12. 사용자 리텐션

**전략**: 검증 활동 자체를 습관 루프로 설계

```
트리거 → "점심 뭐 먹지?" (시간 기반 알림)
행동   → 프롬프트 1탭 복사 → AI 검증
보상   → 검증 배지, 새로운 발견, 맛집 카드 수집
투자   → 검증 이력, 맛집 맵, 커뮤니티 평판
```

**리텐션 루프**:
- 매일: 점심/저녁 시간대 추천 프롬프트 서제스천
- 매주: "이번 주 검증 리포트" (내가 검증한 곳, 트렌드)
- 매월: "이달의 검증왕", 지역별 검증 랭킹

## 4. 사용자 참여 퍼널

### Stage 1: 탐색자 (Passive)

- 프롬프트 복사만 하는 단계
- **목표**: 첫 AI 검증 성공 경험
- **넛지**: "이 식당, AI로 30초만에 검증해보세요" CTA

### Stage 2: 검증자 (Active)

- 정기적으로 AI 검증 후 결과를 공유
- **목표**: 검증 습관 형성
- **넛지**: 진행 바 + 지역 마스터 칭호 진전 표시

### Stage 3: 큐레이터 (Creator)

- 자기만의 프롬프트를 만들어 공유
- 다른 사용자의 검증에 피드백
- **목표**: 커뮤니티 기여자로 정체성 형성
- **넛지**: "당신의 프롬프트를 N명이 사용했어요"

### Stage 4: 가이드 (Leader)

- 특정 지역/카테고리의 검증 전문가
- 다른 사용자들이 이 사람의 검증을 참고
- **목표**: 커뮤니티 내 사회적 자본 축적
- **넛지**: "강남 맛집 가이드" 공식 타이틀

## 5. 프롬프트 커뮤니티

### 5-1. 프롬프트 마켓플레이스

사용자가 효과 좋은 커스텀 프롬프트를 공유하는 공간.

```
🔥 인기 프롬프트 Top 3
1. "진짜 현지인 맛집 찾기" by @foodie_kim (1,247회 사용)
2. "가성비 최강 점심 세트" by @lunch_master (892회 사용)
```

- **적중률 높은 프롬프트 → 자연스럽게 상위 노출**

## 6. 수익 모델 (광고 없이)

| 모델 | 설명 | 사용자 이익 충돌 |
|------|------|----------------|
| **프리미엄 프롬프트** | 고급 검증 프롬프트 팩 (무료 기본 제공) | 없음 (기본 기능 무료) |
| **식당 검증 리포트** | 식당에게 AI 검증 데이터 기반 리포트 판매 | 없음 (노출 순위에 영향 없음) |
| **지역 가이드북** | 검증 데이터 기반 오프라인/PDF 가이드 | 없음 |
| **B2B API** | 검증 데이터를 다른 서비스에 제공 | 낮음 (익명화 처리) |

**핵심 원칙**: 수익 모델이 추천 결과를 왜곡하지 않는 구조.

## 7. 기술적 고려사항

### 앱이 하지 않는 것

- LLM API 직접 호출 (비용, 의존성, 책임 회피)
- AI 응답 결과 저장/캐싱 (저작권, 정확성 이슈)
- 사용자의 AI 계정 정보 수집

### 앱이 하는 것

- 식당 데이터 + 사용자 컨텍스트 → 최적 프롬프트 생성
- 클립보드/딥링크/공유를 통한 프롬프트 전달
- 사용자가 자발적으로 공유한 검증 결과 수집/집계
- 프롬프트 템플릿 관리 및 커뮤니티 기능

### 딥링크 우선순위

```
1. ChatGPT (시장 점유율 1위, 딥링크 지원)
2. 클립보드 복사 (범용, 모든 AI 앱 대응)
3. 시스템 공유 시트 (Claude, Gemini 등)
```
