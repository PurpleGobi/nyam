2026-03-16 딥리서치 기반 참고: Telegram · Discord · Strava · Michelin · Patreon

핵심 컨셉: "버블" 네트워킹이란

냠의 네트워킹은 오픈 소셜이 아니라 소규모 신뢰 그룹(버블) 기반이다. 모든 사람에게 공개되는 피드가 아니라, 내가 선택한 사람들과만 경험을 나눈다.

개인 기록 → 버블(그룹) 공유 → 버블 간 발견 → 네트워크 확장

Evernote·Path처럼 "도구 → 오픈 소셜"은 실패한다. 냠은 처음부터 "도구 + 버블 네트워킹"을 병행한다.

1. 그룹 유형 체계

유형 전환 규칙

그룹 운영자는 언제든 유형을 변경할 수 있다. 단, 아래 규칙을 따른다:

전환허용조건
비공개 → 공개✅즉시. 기존 멤버에게 알림
비공개 → 보기전용✅즉시. 기존 멤버는 참여 권한 유지
공개 → 비공개✅기존 멤버 유지. 신규 참여 차단
공개 → 유료✅운영자 Lv.마스터 이상. 기존 멤버 무료 유지 or 유예 기간 설정
비공개 → 유료✅동일 조건
유료 → 무료✅즉시. 기존 구독자에게 알림 + 환불 정책 안내
원칙: 유형 변경 시 기존 멤버의 권한은 최소 30일간 유지 (기득권 보호). 갑작스러운 유료 전환으로 기존 멤버를 쫓아내지 않는다.
2. 참여 조건 시스템 (Gate)

운영자가 설정 가능한 참여 문턱

그룹 운영자는 아래 조건을 조합하여 참여 문턱을 설정할 수 있다. 조건을 충족하지 못하면 "참여 요청" 버튼이 비활성화되고, 부족한 조건이 안내된다.

3. 그룹 내 역할과 권한

3-Tier 역할 구조

Discord의 다층 권한 시스템에서 영감을 받되, 냠의 컨텍스트에 맞게 단순화.

역할인원권한
운영자 👑1-2명모든 권한: 그룹 설정, 유형 전환, 참여 조건, 멤버 관리, 기록 삭제, 역할 지정
모더레이터 🛡️운영자 지정멤버 승인/제거, 부적절 기록 숨기기, 공지 작성. 그룹 설정 변경 불가
멤버 👤제한 없음기록 공유, 반응(좋아요/댓글), 그룹 피드 열람
참고: Discord 역할 시스템

Discord는 "Admin은 극도의 신뢰를 바탕으로 1-2명에게만 부여, Moderator가 일상적 관리의 핵심"이라는 원칙을 따른다. 냠도 운영자는 최소한으로, 모더레이터가 실질적 운영을 담당하는 구조.

자동 모더레이터 승격 조건 (제안)

운영자가 직접 지정하는 것이 기본이지만, 아래 조건을 충족한 멤버를 "모더레이터 추천" 알림으로 제안:

  • 그룹 내 기록 공유 50건+ (상위 10%)
  • 멤버 가입 후 3개월+
  • 기록 품질(완성도) 평균 80%+
  • 신고 이력 0건
4. 그룹 발견과 성장 엔진

Bottom-up 발견 플로우

냠에서는 그룹을 직접 검색하지 않는다. 좋은 기록 → 좋은 사람 → 좋은 그룹 순서로 자연스럽게 발견한다.

공개 기록 발견 → ❤️ 저장 → 👤 작성자 프로필 → 🧩 궁합 확인 → 👥 소속 그룹 확인 → 참여 요청

이 플로우의 핵심은 "콘텐츠(기록)의 품질이 그룹의 마케팅"이라는 것이다. 좋은 기록을 많이 쓰는 그룹일수록 자연스럽게 발견된다.

미슐랭은 1900년 타이어 마케팅 도구로 시작, 익명 인스펙터 + 광고 제거 + 유료화로 신뢰를 구축했다. 핵심 교훈: "사람은 자기가 돈을 낸 것만 진지하게 대한다" (앙드레 미슐랭). 냠의 유료 그룹도 이 원리를 따른다 — 유료 그룹의 멤버는 더 열심히 기록하고 정보를 정제한다.

Strava는 개인 운동 기록 도구로 시작했지만, 클럽 기능이 성장의 핵심 엔진이 되었다. 2025년 러닝 클럽 3.5배 증가, 클럽 이벤트 1.5배 증가. 비공개 그룹 챌린지(최대 200명)는 유료 구독자 전용 — 무료/유료 차별화에 그룹 기능을 활용한 좋은 사례.

5. 최적 그룹 크기 — Dunbar's Number 적용

Dunbar의 사회적 관계 계층

Robin Dunbar(1992)의 연구에 따르면, 인간의 사회적 관계는 프랙탈 구조로 5 → 15 → 50 → 150으로 확장된다:

규모관계 유형접촉 빈도냠 적용
5명핵심 관계 (intimates)평균 12.6일 간격"절친 맛집" — 가족, 베프
15명좋은 친구 (good friends)평균 23.9일 간격"직장 동료 맛집" — 팀 단위
50명친구 (friends)평균 25.0일 간격"동호회 맛집" — 와인 모임, 러닝 크루
150명의미 있는 관계 (meaningful contacts)평균 46.3일 간격"지역 커뮤니티" — 을지로 맛집 탐험대

원칙: 시스템이 강제하지 않되, 권장 크기를 넘으면 "그룹 분할을 고려해보세요" 넛지. 소규모일수록 참여율이 높다는 데이터를 안내.

7. 유료 그룹 운영 모델

유료 그룹이란

그룹 운영자가 열람에 월정액을 설정할 수 있는 프리미엄 그룹. Discord의 Server Subscription + Patreon의 Tier 모델에서 영감.

유료 그룹 개설 조건

  • 운영자 레벨: 마스터 이상 (기록 200건+, 높은 기여도)
  • 기존 그룹 운영 실적: 무료 그룹 3개월+ 운영 & 활동 지표 상위 20%
  • 플랫폼 심사 통과 (자동 기준 + 수동 검토)

가격 정책

항목규격근거

참고: Patreon 가격 전략 (2025)

Patreon은 3-Tier 전략을 권장: Starter ($6-9), Growth ($25-120), Pro ($180-900). 무료 티어로 팬을 유입하고, 디지털 제품으로 충동 구매를 유도하고, 유료 티어로 반복 수익을 구축. 냠의 유료 그룹은 Starter 레벨(₩2,900-4,900)이 가장 적합할 것.

유료 그룹 구독자에게 제공되는 가치

  • 큐레이션된 기록: 마스터급 운영자의 엄선된 식당 리뷰
  • 독점 정보: 비공개 신규 오픈, 메뉴 변경, 예약 팁 등
  • 그룹 내 소통: 운영자/멤버에게 질문, 추천 요청
  • 월간 큐레이션 리포트: "이달의 추천 5곳" 같은 정기 콘텐츠

유료 그룹 운영자의 의무

  • 최소 주 2회 이상 기록 공유
  • 4주간 기록 0건 시 구독자에게 경고 알림 + 환불 옵션
  • 8주간 비활성 시 유료 기능 자동 정지
원칙: "돈을 내면 더 좋은 정보를 받는다"가 아니라 "돈을 내면 더 열심히 정보를 정제하는 사람들의 그룹에 들어간다." 미슐랭의 교훈: 유료화가 품질을 끌어올린다.
8. 그룹 활성화 도구

그룹 챌린지

Strava의 Group Challenges에서 영감. 그룹 운영자가 미션을 설정하고 멤버가 함께 참여.

챌린지 유형예시기간
카테고리 탐험"이번 주 동남아 음식 3곳 기록하기"1주
지역 정복"성수동 맛집 팀 합산 20곳 달성"2주
신규 발견"처음 가보는 식당 5곳 기록하기"1주
공유 미션"이번 달 기록 10건 이상 그룹에 공유"1개월

그룹 리더보드

주간 기준, 그룹 내 활동 랭킹. Dunbar 규모(5-15명)에서 리더보드 참여율 30-50%.

마케팅팀 맛집 — 이번 주 랭킹 🥇 민지 12건 기록 +85 XP 📍 을지로 마스터 🥈 준호 8건 기록 +62 XP 🍷 와인 전문가 🥉 소영 6건 기록 +45 XP 🆕 새 발견 3곳! 4. 현우 4건 기록 +30 XP 5. 나 3건 기록 +22 XP ← 여기!

그룹 통계 대시보드

그룹의 종합 데이터를 시각화. 그룹의 가치를 멤버에게 체감시키는 핵심 도구.

  • 그룹 TOP 식당: 멤버 N명 중 M명이 추천한 곳 (과반수 기준)
  • 카테고리 분포: 그룹 전체의 식음료 취향 비율
  • 활동 히트맵: 그룹 기록의 시간대/요일 패턴
  • 멤버 기여도: 기록 수, 반응 수, 저장 수 기준
  • 월간 리포트: "이번 달 우리 그룹의 새 발견 N곳"
9. 지역 전문가 영입 시스템

지역 특화 → 인재 영입의 연결

사용자의 기록이 특정 지역에 밀집되면 자동으로 "지역 특화"가 프로필에 노출. 이것이 그룹 운영자의 영입 도구가 된다.

그룹 운영자 대시보드 — "멤버 추천" 이 그룹에 어울릴 수 있는 사용자: 👤 민지 · Lv.분석가 📍 을지로 38건 (마스터) 🧩 정보 보완 궁합 91% "을지로 기록이 풍부해 그룹에 도움이 됩니다" [초대하기] 👤 준호 · Lv.큐레이터 📍 을지로 12건 · 🍷 와인 42건 🧩 정보 보완 궁합 87% "와인 기록이 그룹에 새로운 영역을 추가합니다" [초대하기]

영입 흐름

운영자: 추천 확인 → 초대 메시지 전송 → 대상자: 초대 수신 → 그룹 프리뷰 확인 → 참여 수락

초대 메시지에 포함되는 정보:

  • 그룹 이름, 설명, 멤버 수
  • "당신의 [을지로] 기록이 이 그룹에 도움이 됩니다"
  • 그룹 TOP 3 식당 미리보기
  • 정보 보완 궁합 수치
원칙: 영입은 "당신이 필요합니다"가 아니라 "서로에게 도움이 됩니다". 상호 가치를 명확히 보여줘야 수락율이 높다.
10. 다크소셜 기반 성장 전략

카카오톡이 핵심 성장 채널

한국에서 맛집 공유는 "카톡으로 네이버 링크 보내기"가 지배적. 냠은 이 행동을 가로채야 한다.

공유 유형카카오톡 프리뷰비설치자 경험
기록 공유사진 + 메뉴 + 평가 + 식당명. 매력적인 OG 카드웹 프리뷰에서 기록 열람 가능 → 앱 설치 유도
그룹 초대"[민지]님이 [직장동료 맛집]에 초대합니다" + 그룹 미리보기웹에서 그룹 프리뷰 → 가입 유도
맛 궁합"우리 맛 궁합: 87%!" 비주얼 카드"나도 해보기" → 앱 설치
Year in Review개인 맛 리포트 카드 (Wrapped 스타일)바이럴 최대. 인스타 스토리 공유도 가능
다크소셜 규모

DM 기반 인터랙션이 전체 소셜 커뮤니케이션의 41%를 차지. 인스턴트 메시징 앱 시장 2024년 $286억 → 2033년 $637억 (CAGR 9.3%). 한국에서 카카오톡 메시징 시장 점유율 94% — 카톡 공유 최적화가 곧 성장 전략.

바이럴 루프 설계

바이럴 루프: 사용자 A: 맛집 기록 → 카카오톡에 기록 카드 공유 ↓ 사용자 B: 카톡에서 프리뷰 확인 → "이거 뭐야?" → 앱 설치 ↓ 사용자 B: 기록 시작 → A의 그룹에 초대 수락 ↓ 사용자 B: 자기 기록 공유 → 사용자 C가 발견... ↓ (반복)

바이럴 계수 목표: K-factor > 1.0 (각 사용자가 평균 1명 이상 초대)

커뮤니티 버블 운영 핵심 원칙 요약

원칙설명
버블 우선오픈 소셜이 아닌 소규모 신뢰 그룹. 5-15명이 최적
Bottom-up 발견기록 → 사람 → 그룹 순서. 그룹을 직접 검색하지 않는다
참여 조건 = 동기 부여거부가 아닌 넛지. "N건 더 기록하면 참여할 수 있어요"
분산 모더레이션그룹 내 자치 우선. 플랫폼은 최소 개입
유료 = 품질돈이 품질을 끌어올림. 미슐랭 원리
콘텐츠가 마케팅좋은 기록이 그룹을 자연스럽게 발견시킴
기득권 보호유형 변경 시 기존 멤버 최소 30일 권한 유지
상호 가치영입은 "당신이 필요"가 아니라 "서로에게 도움"
