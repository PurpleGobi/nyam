# Nyam - Product Requirements Document (PRD)

> 버전: 1.0.0 | 작성일: 2026-03-15
> 상태: Draft

---

## 1. 제품 개요

### 1-1. 한 줄 정의

**Nyam은 사용자가 AI를 활용해 맛집을 직접 검증하고, 그 결과를 커뮤니티와 공유하는 앱이다.**

### 1-2. 문제 정의

기존 맛집 추천 앱은 구조적으로 실패하고 있다:

| 문제 | 심각도 | 현 상황 |
|------|--------|--------|
| 리뷰 신뢰도 붕괴 | 매우 높음 | Google이 2023년 한 해 1.7억 개 가짜 리뷰 삭제. "별점 4.5인데 가보면 별로"라는 인식 고착화 |
| 광고-추천 이해충돌 | 매우 높음 | 광고비 지불 식당이 상위 노출. "상위 노출 = 돈 많은 곳"이라는 불신 |
| 개인화 부족 | 높음 | 모든 사용자에게 동일한 위치 기반 인기순 결과. 상황별 추천 부재 |
| AI 가짜 리뷰 급증 | 높음 | ChatGPT 활용한 가짜 리뷰 대량 생산. 기존 탐지 시스템 우회 |
| 별점 시스템 무의미화 | 높음 | 대부분 식당이 4.0-4.5점대 집중. 변별력 소멸 |
| 리뷰 피로 | 높음 | Yelp 활성 리뷰어 수 2019년 대비 20-30% 감소 추정 |
| 낮은 리텐션 | 중간 | 맛집 앱 30일 리텐션율 평균 10-15% |

### 1-3. 솔루션

**프롬프트 브릿지 + 커뮤니티 검증 모델**

Nyam은 LLM API를 직접 호출하지 않는다. 대신:

1. 식당 데이터 + 사용자 상황 → **최적화된 프롬프트를 자동 생성**
2. 사용자가 ChatGPT, Claude, Gemini 등 **자신의 AI에 원클릭 전달**
3. 검증 결과를 Nyam 커뮤니티에 공유 → **집단 검증 데이터 축적**
4. 검증 행동 자체를 **심리적 보상 체계**로 습관화

```
사용자 → "AI로 검증하기" 탭 → 프롬프트 자동 생성
→ 클립보드 복사 / ChatGPT 딥링크 → AI에서 결과 확인
→ (선택) 결과 요약을 Nyam에 공유 → 커뮤니티 검증 데이터 축적
```

### 1-4. 핵심 가치 제안

> **"별점 말고, 직접 확인하세요."**
>
> 광고가 아닌 데이터로, 별점이 아닌 검증으로,
> 지금 당신의 상황에 딱 맞는 식당을 찾습니다.

### 1-5. 왜 이 접근인가

| 장점 | 설명 |
|------|------|
| **비용 제로** | LLM API 비용이 사용자 측에서 발생 (무료 티어 활용 가능) |
| **모델 중립** | 특정 AI에 종속되지 않음 |
| **신뢰 이전** | "앱이 말한 것" → "내가 AI로 확인한 것" (IKEA 효과) |
| **책임 분산** | 추천 결과에 대한 법적/도덕적 책임 경감 |
| **조작 내성** | "AI 검증 횟수"는 별점보다 조작 동기/수단이 적음 |
| **확장성** | AI 모델 발전에 자동으로 편승 |

## 2. 타겟 사용자

### 2-1. 주요 페르소나

#### 페르소나 A: "똑똑한 소비자" 민지 (28세, 직장인)

- **행동**: 식당 가기 전 네이버/인스타 리뷰를 30분 이상 검색하지만 결국 확신 없이 결정
- **불만**: "체험단 리뷰밖에 안 나와서 진짜 맛집을 모르겠다"
- **AI 사용**: ChatGPT 유료 구독, 일상적으로 AI에 질문
- **Nyam 가치**: 프롬프트 1탭으로 30분 리서치를 3분으로 단축. "내가 직접 검증한 맛집"이라는 확신

#### 페르소나 B: "맛집 탐험가" 준호 (33세, 프리랜서)

- **행동**: 새로운 동네의 숨은 맛집 찾기가 취미. SNS에 자주 공유
- **불만**: "인스타 맛집은 비싸고 양 적고 광고뿐"
- **AI 사용**: Claude 무료 사용, 맛집 검색에 AI 활용 시도 경험
- **Nyam 가치**: 지역 마스터 칭호, 검증 커뮤니티에서의 인정. 검증 결과 공유 카드

#### 페르소나 C: "결정 장애" 소영 (26세, 대학원생)

- **행동**: 친구들과 식사 장소 정할 때 항상 30분 이상 고민
- **불만**: "너무 많은 선택지, 다 비슷해 보여서 못 고르겠다"
- **AI 사용**: ChatGPT 무료 가끔 사용
- **Nyam 가치**: "딱 3곳" 추천 프롬프트, 그룹 검증으로 함께 결정

### 2-2. 타겟 시장

- **1차**: 25-39세, AI 앱을 이미 사용하는 도시 거주 직장인/프리랜서
- **2차**: 20-24세, AI 네이티브 세대 (Z세대)
- **지역**: 서울/수도권 → 광역시 → 전국 순차 확대

## 4. 프롬프트 시스템 상세

### 4-1. 기본 프롬프트 템플릿 5종

#### A. 리뷰 검증 프롬프트

**페인포인트**: 리뷰 신뢰도 붕괴, AI 가짜 리뷰

```
"[식당명] [지역]의 리뷰를 분석해줘.
네이버, 구글맵, 인스타그램 등 여러 소스의 평판을 종합해서:
1. 체험단/협찬 리뷰 비율이 높은지
2. 최근 6개월 리뷰 트렌드 (개선/악화)
3. 반복 언급되는 긍정/부정 키워드
4. 가격 대비 만족도
5. 실제 방문 가치가 있는지 (10점 만점)
솔직하게 평가해줘."
```

#### B. 상황 맞춤 추천 프롬프트

**페인포인트**: 개인화 부족, 정보 과부하

```
"[지역]에서 [상황]에 맞는 식당 3곳만 추천해줘.
조건:
- 인원: [N]명
- 예산: 1인당 [금액]원
- 분위기: [키워드]
- 제외: [알러지/비선호]
추천 이유와 대표 메뉴, 예상 가격을 포함해줘.
광고/협찬이 아닌 실제 평판 기반으로만."
```

**상황 프리셋**:

| 상황 | 자동 삽입 키워드 |
|------|----------------|
| 비즈니스 점심 | 조용함, 코스 가능, 빠른 서빙 |
| 데이트 | 분위기, 사진 잘 나오는, 예약 가능 |
| 혼밥 | 카운터석, 부담 없는, 1인 메뉴 |
| 가족 모임 | 넓은 공간, 다양한 메뉴, 주차 |
| 친구 모임 | 시끌벅적 OK, 공유 메뉴, 술 가능 |

#### C. 비교 분석 프롬프트

**페인포인트**: 별점 무의미화, 선택 장애

```
"다음 식당들을 비교해줘: [식당A], [식당B], [식당C]
비교 기준: 맛, 가성비, 서비스, 분위기, 대기시간
표 형식으로 정리하고 최종 추천 1곳과 이유를 알려줘."
```

#### D. 정보 최신성 확인 프롬프트

**페인포인트**: 폐업/가격/영업시간 부정확

```
"[식당명] [지역] 현재 영업 중인지 확인해줘.
최근 메뉴, 가격, 영업시간, 휴무일, 예약 필요 여부 등
가장 최신 정보를 알려줘."
```

#### E. 숨은 맛집 발굴 프롬프트

**페인포인트**: 광고-추천 이해충돌, 소상공인 불리

```
"[지역]에서 광고나 인플루언서 없이 입소문으로만 유명한
숨은 맛집을 찾아줘. 조건:
- 네이버 블로그 체험단 리뷰가 적을 것
- 동네 주민/단골이 추천하는 곳
- 오래된 곳 우선
- [음식 종류] 중심으로"
```

### 4-2. 프롬프트 전달 경로

| 우선순위 | 경로 | 구현 | UX |

### 4-3. 동적 변수 처리

프롬프트 내 `[식당명]`, `[지역]` 등은 현재 컨텍스트에서 자동 삽입:

| 변수 | 소스 | 예시 |
|------|------|------|
| `[식당명]` | 현재 보고 있는 식당 | "을지로 골목식당" |
| `[지역]` | 식당 주소 또는 사용자 위치 | "을지로3가" |

## 5. 심리적 참여 설계

### 5-1. 적용 심리 원리

| 원리 | 적용 방식 | 기대 효과 |
|------|----------|----------|
| **IKEA 효과** | 사용자가 직접 AI 검증 수행 → "내가 발견한 맛집" 소유감 | 검증 결과 신뢰도 ↑, 앱 충성도 ↑ |
| **사회적 증거** | "47명이 AI로 검증" (별점 대체) | 조작 내성 있는 신뢰 지표 |
| **가변 보상** | 검증 시 랜덤 맛집 카드 획득, 숨은 맛집 알림 | 반복 검증 행동 유도 |
| **목표 그래디언트** | 지역 마스터 진행 바 "3곳만 더!" | 검증 완주율 ↑ |
| **자기결정성** | 능동적 검증 (자율성), 정확한 판단 (유능감), 커뮤니티 (관계성) | 내재적 동기 |
| **손실 회피** | "검증 없이 방문한 사용자의 30%가 후회" | 검증 행동 촉발 |
| **커밋먼트 에스컬레이션** | 1탭 복사 → 반응 → 코멘트 → 프롬프트 작성 | 점진적 참여 심화 |

### 5-2. 사용자 참여 퍼널

```
Stage 1: 탐색자 (Passive)
  행동: 프롬프트 복사만
  목표: 첫 AI 검증 성공 경험
  넛지: "이 식당, AI로 30초면 검증 끝"

    ↓

Stage 2: 검증자 (Active)
  행동: 정기적 검증 + 결과 공유
  목표: 검증 습관 형성
  넛지: 진행 바 + "실버까지 3곳!"

    ↓

Stage 3: 큐레이터 (Creator)
  행동: 커스텀 프롬프트 제작 + 공유
  목표: 커뮤니티 기여자 정체성
  넛지: "당신의 프롬프트를 N명이 사용"

    ↓

Stage 4: 가이드 (Leader)
  행동: 지역/카테고리 전문가 활동
  목표: 사회적 자본 축적
  넛지: "강남 맛집 가이드" 공식 타이틀
```

### 5-3. 리텐션 루프

```
트리거 → "점심 뭐 먹지?" (시간 기반 / 위치 기반)
행동   → 프롬프트 1탭 복사 → AI 검증
보상   → 검증 배지, 맛집 카드 수집, 새로운 발견
투자   → 검증 이력, 맛집 맵, 커뮤니티 평판 (이탈 비용)
```

| 주기 | 넛지 |
|------|------|
| 매일 | 점심/저녁 시간대 프롬프트 서제스천 |
| 매주 | "이번 주 검증 리포트" (내가 검증한 곳, 트렌드) |
| 매월 | "이달의 검증왕", 지역별 검증 랭킹 |

### 5-4. 검증 등급 체계

| 등급 | 조건 | 배지 시각 | 컬러 |

분위기  ██████████░░░░░  75%

- 바 높이 6px, 풀 라디우스
- 색상: `primary-500` fill, `neutral-200` track
- 상세 페이지에서는 4축 레이더 차트로 확장

| W7 | 인증, 프로필, 찜하기, 배지 시스템 기본 |
| W8 | QA, 성능 최적화, 베타 출시 |

### Phase 2: 커뮤니티 (4주)

**목표**: 참여 심화 및 네트워크 효과

- 프롬프트 마켓플레이스
- 그룹 검증
- 검증 결과 공유 카드
- 리더보드
- 시간대별 추천

### Phase 3: 수익화 (4주)

**목표**: 지속 가능한 수익 모델

- 프리미엄 프롬프트 팩
- 소상공인 대시보드
- B2B API

## 12. 리스크 및 대응

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| AI 딥링크 정책 변경 | 높음 | 클립보드 복사를 항상 1순위 대안으로 유지 |
| 사용자가 프롬프트 실행을 귀찮아함 | 높음 | 1탭 UX, 가치 체감 온보딩, 최소 행동 단위 설계 |
| 검증 결과의 정확성 편차 | 중간 | 프롬프트별 적중률 추적, 커뮤니티 보정 |
| 검증 참여율 저조 | 중간 | 심리적 보상 체계 강화, 초기 시드 데이터 확보 |
| 경쟁사 모방 | 중간 | 프롬프트 커뮤니티 + 검증 데이터 네트워크 효과가 해자 |
| 식당 데이터 부족/부정확 | 중간 | Kakao/Google API 병행, 사용자 제보 시스템 |
| LLM 환각(hallucination)으로 잘못된 정보 | 중간 | 검증 결과는 참고용임을 명시, 다수 검증으로 보정 |

## 13. 경쟁 우위 요약

| 기존 맛집 앱 | Nyam |
|-------------|------|
| 앱이 추천한다 | 사용자가 AI로 검증한다 |
| 별점을 믿어라 | 직접 확인해라 |
| 광고비 = 노출 | 검증 횟수 = 신뢰 |
| 리뷰 써라 (피로) | 프롬프트 1탭 (간편) |
| 수동적 소비자 | 능동적 검증자 |
| AI 비용 = 앱 부담 | AI 비용 = 제로 |
| 블랙박스 알고리즘 | 투명한 검증 과정 |
| 앱 내 완결 | AI 생태계 활용 |
