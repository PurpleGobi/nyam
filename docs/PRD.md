## 1. 제품 정의

### 1-1. 한 줄 정의

**Nyam은 식당, 와인, 직접 만든 요리 — 세 가지 식음료 경험을 기록하고, 즉시 활용하며, 신뢰할 수 있는 그룹과 공유하는 "맛의 Second Brain"이다.**

### 1-2. 핵심 가치

#### 가치 1: 취향 기록 & 분석 — "기록할수록 나를 알아간다"

기록이 쌓이면 Nyam이 사용자의 취향을 두 가지 축으로 자동 분석한다.

**Taste DNA — 맛 그 자체의 취향**
- 기록할 때 붙는 맛 태그(매운, 짭짤한, 감칠맛 등)와 평가 점수가 누적된다.
- 예: "매운" 태그가 붙은 기록에서 맛 점수가 높을수록 → 매운맛 선호 게이지 상승.
- 6축 레이더 차트로 시각화: 매운맛, 단맛, 짠맛, 신맛, 감칠맛, 풍미.

**Style DNA — 맛 너머의 취향 패턴**
- 음식의 본질(맛)뿐 아니라, 내가 즐겨 찾는 경험의 외적 특성도 데이터로 쌓인다.
- **장르(WHAT)**: 어떤 종류의 음식을 주로 먹는가 — 한식, 일식, 와인, 디저트 등
- **지역(WHERE)**: 어디서 주로 먹는가 — 강남, 이태원, 성수 등
- **씬(WHY)**: 어떤 상황에서 먹는가 — 혼밥, 데이트, 비즈니스, 친구모임 등
- 각 축에서 기록 수 + 다양성 + 최근성 + 일관성으로 전문성 레벨이 올라간다.

**활용**: 기록이 충분히 쌓이면(콜드 스타트 해소) Taste DNA + Style DNA를 결합해 맞춤 추천을 제공한다. 기록이 적을 때는 순수한 개인 기록 도구로서의 가치에 집중한다.

#### 가치 2: 버블 네트워킹 — "신뢰할 수 있는 사람들의 진짜 정보"

버블(폐쇄형 그룹)은 Nyam의 소셜 단위이다. 개방형 리뷰 플랫폼과 근본적으로 다르다.

**공유 방식의 다양성**
- **양방향 버블**: 모든 멤버가 기록을 공유하고 서로의 기록에 리액션. 친구 모임, 동호회에 적합.
- **열람 전용 버블**: 운영자의 기록만 공개. 구독자는 보기만 가능. 전문가/인플루언서 채널에 적합.

**정보 품질 관리**
- 개인 또는 버블 단위로 팔로우/언팔로우 가능.
- 정보 오염이 의심되면 해당 소스만 언팔로우하면 끝 — 플랫폼 전체를 버릴 필요가 없다.
- 내 취향과 비슷한 사용자를 팔로우해 정확한 추천을 받을 수도 있고, 전혀 다른 전문 분야의 사용자를 팔로우해 새로운 경험의 폭을 넓힐 수도 있다.

**다양성 = 재미**
- 수많은 버블이 각자의 관점으로 존재한다는 것 자체가 정보의 다양성이고, 곧 재미다.
- 하나의 식당도 "혼밥러 버블"과 "와인 동호회 버블"에서는 완전히 다른 평가를 받는다.

#### 가치 3 (보조): AI — 곳곳의 윤활유

AI는 핵심 가치라기보다 **모든 가치를 매끄럽게 만드는 도구**이다.

- **기록을 빠르게**: 사진 → 식당/메뉴/카테고리/태그 자동 완성 (5초 기록)
- **기록을 풍부하게**: 질문에 답하면 AI가 매거진 스타일 블로그를 자동 생성 (Phase 2)
- **데이터를 활용하게**: Taste DNA + Style DNA 데이터를 조합해 맞춤 추천, 궁합 매칭, Today's Pick 등 다양한 발견 기능을 만들어낸다.

### 1-3. 정체성 우선순위

```
1순위: 개인 기록 도구 — "나만의 맛 데이터베이스" (콜드 스타트에서도 가치)
2순위: 취향 분석 — "기록이 쌓이면 나의 Taste DNA가 선명해진다"
3순위: 신뢰 공유 — "내가 고른 사람들의 진짜 평가"
4순위: 발견 — "취향이 맞는 새로운 곳"
5순위: 크리에이터 경제 — "진짜 맛잘알이 인정받고 수익을 얻는 생태계"
```

5순위는 장기 비전이다. 꾸준히 양질의 기록을 쌓은 사용자가 유료 버블을 운영하거나, 전문 분야 큐레이터로서 팔로워를 모아 수익을 만들 수 있는 구조. 광고비를 쓰는 식당이 아니라, 진정성 있는 기록을 쌓은 개인이 보상받는 생태계를 만든다.

Nyam은 **리뷰 플랫폼이 아니다**. 다이어트 앱도 아니다. "기록할수록 똑똑해지는 경험 지식 도구"이다.

## 2. 문제 정의

### 2-1. 리뷰를 믿을 수 없다

| 문제 | 심각도 | 현황 |
|------|--------|------|
| 리뷰 신뢰도 붕괴 | 매우 높음 | Google이 2023년 한 해 1.7억 개 가짜 리뷰 삭제 |
| 광고-추천 이해충돌 | 매우 높음 | 상위 노출 = 광고비 많은 곳 |
| AI 가짜 리뷰 급증 | 높음 | 2026년 말 AI 리뷰가 인간 리뷰 초과 전망 |
| 별점 무의미화 | 높음 | 대부분 식당 4.0-4.5점대 집중, 변별력 소멸 |

### 2-2. 나에게 맞는 추천이 없다

| 문제 | 심각도 | 현황 |
|------|--------|------|
| 일반화된 추천 | 매우 높음 | 모든 사용자에게 동일한 "인기 맛집" 추천 |
| 취향 반영 불가 | 높음 | 과거 기록을 활용하지 않는 추천 시스템 |
| 상황 무시 | 중간 | 혼밥/데이트/비즈니스 구분 없는 추천 |

### 2-3. Nyam의 해법

```
문제: 리뷰를 믿을 수 없다
→ 해법: 버블(폐쇄형 그룹) 안에서만 공유. 가짜 리뷰 구조적 차단.

문제: 나에게 맞는 추천이 없다
→ 해법: 기록 → Taste DNA 자동 구축 → 개인화 추천. 기록할수록 정확해진다.

문제: 기록이 귀찮다
→ 해법: 사진 1장 → AI가 5초 안에 식당/메뉴/카테고리/태그 자동 완성.
```

## 3. 타겟 사용자

### 3-1. 핵심 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
|----------|------|-----------|
| 🍜 기록광 | 맛있는 경험을 체계적으로 기록하고 싶은 사람 | "내 맛집 데이터베이스를 만들고 싶다" |
| 🧭 탐험가 | 새로운 맛집을 찾아다니는 사람 | "믿을 수 있는 추천을 받고 싶다" |
| 🍷 전문가 | 와인/요리 등 특정 분야에 깊은 관심 | "내 전문성을 기록으로 증명하고 싶다" |
| 👥 모임장 | 맛집 정보를 그룹과 공유하는 사람 | "우리 모임만의 맛집 리스트가 필요하다" |

### 3-2. 타겟 시장

- **1차**: 25-39세 수도권 직장인 (맛집 탐방 활발, 기록 의지 있음)
- **2차**: 와인/요리 동호회 커뮤니티
- **3차**: 해외 거주 한인 커뮤니티 → 글로벌 확장

## 4. 핵심 컨셉

### 4-1. 3-Phase 기록 시스템

기록의 깊이를 3단계로 나누어, 각 단계가 독립적으로 가치를 제공한다.

```
Phase 1 — 빠른 캡처 (사용자, 5초, +5 XP)
  사진 촬영/선택 + 만족도 평가(100점 슬라이더)
  → AI 개입 없음. 즉시 저장.

  ↓ (백그라운드, 사용자 대기 없음)

AI 비동기 처리
  1) 사진/위치 분석 → 식당·메뉴 특정
  2) 식당 기본정보 조회 (카카오/네이버)
  3) 웹 리뷰 검색 → 맛 특성 교차 검증
  4) 맛 프로필 데이터 완성 (이 음식의 객관적 맛 특성 점수화)
  5) DNA 도출: AI 수집 맛 특성 × 사용자 만족도 = Taste DNA 반영

Phase 2 — AI 블로그 리뷰 (사용자, 2-5분, +15 XP)
  AI가 이미 분석을 마친 상태.
  사용자는 AI 질문에 답변 → 매거진 스타일 블로그 자동 생성.

Phase 3 — 비교 게임 (사용자, 3분, +5 XP)
  유사 기록끼리 토너먼트 → Elo 기반 정밀 순위 산출.
```

**핵심 원칙**: Phase 1만으로도 완전한 기록. Phase 2, 3은 선택적 심화. AI는 Phase 1 저장 직후 백그라운드에서 모든 분석을 완료한다.

### 4-2. 기록 유형과 AI 분석

#### 기록 유형별 사용자 평가 항목 (100점 만점 슬라이더)

| 유형 | 사용자가 평가하는 항목 |
|------|----------------------|
| 🍽️ 식당 | 맛, 가성비, 서비스, 분위기, 청결, 양 |
| 🍷 와인 | 향, 바디감, 산미, 여운, 밸런스, 가성비 |
| 🍳 요리 | 맛, 난이도, 소요시간, 재현성, 플레이팅, 재료비 |

#### AI 백그라운드 분석 흐름

```
사진 + GPS
  ↓
1. 식당·메뉴 특정 (사진 OCR + 위치 매칭)
  ↓
2. 웹 리뷰 교차 검증
   - 특정된 "식당 + 메뉴"로 네이버 블로그/카카오 리뷰 검색
   - 리뷰 N건에서 맛 관련 키워드 추출 + 빈도 분석
   - 예: "마라공방 마라탕" 리뷰 → "얼얼하고 매워요"(5건), "국물 진하고 감칠맛"(8건)
  ↓
3. 맛 프로필 점수화 (0-100)
   - 리뷰 키워드 빈도 + 맥락 → 6축 객관적 맛 특성 산출
   - 같은 "마라탕"이라도 A식당은 매운맛 85, B식당은 매운맛 55
   - LLM 단독 추정이 아닌 실제 리뷰 데이터 기반 → 식당별 차이 반영
  ↓
4. DNA 도출
   - AI 맛 특성(객관): "이 음식은 매운맛 75"
   - × 사용자 만족도(주관): "이 음식에 맛 90점을 줬다"
   - = 이 사용자는 매운맛 75짜리 음식을 좋아한다 → 매운맛 선호 ↑
```

**핵심**: AI는 "이 음식이 얼마나 매운가"를 판단하고, 사용자는 "이 음식이 얼마나 좋았는가"를 판단한다. 둘의 교차가 Taste DNA를 만든다.

### 4-3. Taste DNA

사용자의 **맛 그 자체**에 대한 선호를 다차원 벡터로 자동 구축하는 시스템.

**카테고리별 Taste DNA 축**

🍽️ 식당 (6축):

| 축 | 스펙트럼 | 도출 방식 |
|---|---|---|
| 매운맛 | 순한 ↔ 매운 | AI 리뷰 분석 맛 특성 × 사용자 만족도 |
| 단맛 | 담백 ↔ 달콤 | 〃 |
| 짠맛 | 싱거운 ↔ 짠 | 〃 |
| 신맛 | — | 〃 |
| 감칠맛 | 깔끔 ↔ 감칠맛 | 〃 |
| 기름진맛 | 담백 ↔ 기름진 | 〃 |

🍷 와인 (6축):

| 축 | 스펙트럼 |
|---|---|
| 바디감 | 라이트 ↔ 풀바디 |
| 산미 | 낮음 ↔ 높음 |
| 타닌 | 부드러운 ↔ 강한 |
| 당도 | 드라이 ↔ 스위트 |
| 향 | 심플 ↔ 복합 |
| 여운 | 짧은 ↔ 긴 |

🍳 요리 (6축):

| 축 | 스펙트럼 |
|---|---|
| 맛 완성도 | — (사용자 자체 평가) |
| 난이도 | 쉬운 ↔ 어려운 |
| 시간투자 | 빠른 ↔ 오래걸리는 |
| 재현성 | 쉬움 ↔ 어려움 |
| 건강도 | 간편식 ↔ 건강식 |
| 가성비 | 비싼 ↔ 저렴 |

**보조 태그** (기록 시 선택적): 식감(바삭, 쫄깃, 아삭, 부드러운), 온도(뜨거운/따뜻한/차가운)

**시각화**: 6축 레이더 차트 + 취향 코드 (예: "bold-savory")

### 4-4. Style DNA

사용자의 **맛 너머의 경험 패턴**을 기록으로 자동 구축하는 시스템.

**3축 (카테고리 공통 구조, 내용은 카테고리별 상이)**

🍽️ 식당:
- **WHAT** (장르): 한식, 고기/구이, 치킨, 일식, 중식, 양식, 분식, 해산물, 카페/디저트, 피자, 버거, 아시안 등
- **WHERE** (지역): 상권/동네 단위 — 성수, 홍대, 강남, 이태원 등 (GPS 자동)
- **WHEN** (상황): 혼밥, 데이트, 비즈니스, 가족, 친구, 회식, 야식, 브런치 등

🍷 와인:
- **WHAT**: 레드, 화이트, 스파클링, 로제, 오렌지, 내추럴
- **WHERE**: 프랑스, 이탈리아, 스페인, 칠레, 호주, 미국, 한국 등 (산지)
- **WHEN**: 혼술, 데이트, 페어링, 파티, 테이스팅

🍳 요리:
- **WHAT**: 한식요리, 양식요리, 중식요리, 베이킹, 디저트, 음료
- **WHERE**: 해당 없음
- **WHEN**: 일상식사, 밀프렙, 손님초대, 도시락, 다이어트

각 축에서 **기록 수 + 다양성 + 최근성 + 일관성**으로 전문성 레벨이 산출된다.
**활용**: 프로필 전문성 표시, 그룹 참여 조건, 상호보완 매칭

### 4-5. 버블 네트워킹

신뢰 기반 폐쇄형 그룹 시스템.

| 유형 | 설명 | 진입 방식 |
|------|------|-----------|
| Private | 초대 전용 | 초대 링크 |
| Public | 누구나 참여 | 조건 충족 시 자동 승인 |
| View-only | 열람 전용 | 기록 공유 불가, 보기만 가능 |
| Paid | 유료 구독 | 결제 후 참여 |

**발견 흐름**: 기록 → 작성자 프로필 → 소속 그룹 발견 (Bottom-up)

## 6. 심리학적 참여 설계

### 6-1. 핵심 원칙

| 원칙 | 적용 |
|------|------|
| IKEA 효과 | 내가 쌓은 Taste DNA → 애착, 이탈 비용 증가 |
| 가변 보상 | Today's Pick 매일 다른 추천, 비교 게임 결과 |
| 목표 기울기 | "다음 레벨까지 15 XP!" 진행률 시각화 |
| 던바의 수 | 버블 규모 5-30명 권장, 신뢰 유지 |
| 축적 효과 | 기록이 쌓일수록 추천 정확도 상승 (체감 가능) |

### 6-2. 리텐션 루프

```
일간: 기록 알림 → Today's Pick 확인 → 기록
주간: 주간 통계 리포트 → 목표 달성 확인
월간: 월간 요약 → Style DNA 성장 확인
연간: Wrapped (연간 리뷰) → 소셜 공유
```
