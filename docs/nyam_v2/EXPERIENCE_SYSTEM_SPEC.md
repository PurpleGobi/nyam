## 1. 경험치의 존재 이유

### 1-1. 왜 경험치가 필요한가

Letterboxd에는 경험치 시스템이 없다. 그런데도 잘 된다. 왜?
→ **영화는 공유 문화가 이미 존재**하기 때문. "무슨 영화 봤어?"는 자연스러운 대화.

식도락은 다르다:
- "어제 뭐 먹었어?" → 대화는 하지만 **기록하지는 않는다**
- 기록 습관이 없는 사람에게 **기록을 시작하게 만드는 동기**가 필요
- 기록이 쌓인 사람에게 **계속 기록하게 만드는 이유**가 필요

경험치는 이 두 가지를 해결한다:
1. **초기 동기**: "을지로 Lv.2가 되었어요!" → 다음 레벨이 보임 → 또 기록
2. **장기 정체성**: "을지로 Lv.7" = 나의 식도락 정체성. 버블에서 내 기록의 무게가 달라짐

### 1-2. 경험치가 아닌 것

```
경험치 ≠ 경쟁 도구       → 리더보드/랭킹 없음
경험치 ≠ 매일 해야 하는 것 → 스트릭 없음
경험치 ≠ 기록의 전부      → 기록 자체의 만족감이 메인, 경험치는 서브
경험치 ≠ 자동 가중치      → 소셜에서 자동으로 기록을 밀어올리지 않음
```

## 2. 경험치 축

### 2-1. 핵심 원칙: 영역별 독립

전체 통합 레벨(Google Local Guide 방식)은 안 쓴다. 강남에서 100번 기록해도 을지로 전문가는 아니다.

**경험치는 "어디서" + "무엇을" 기록했느냐에 따라 독립적으로 쌓인다.**

### 2-2. 식당 경험치 축

```
지역 경험치 (자동 부여):
  기록한 식당의 위치 기반으로 해당 지역 XP 자동 적립

  레벨 예시:
  광화문 Lv.5 · 을지로 Lv.7 · 강남 Lv.3 · 성수 Lv.2 · 연남 Lv.1
```

**지역 단위**: 사용자가 인식하는 "동네" 단위. 행정동이 아닌 생활권.
- 1차: 카카오맵/네이버의 동네 분류 활용 (광화문, 을지로, 성수, 연남, 이태원...)
- 2차: 상위 묶음으로 구 단위도 집계 (종로구 = 광화문 + 을지로 + 삼청동...)

### 2-3. 와인 경험치 축

```
타입 경험치 (자동 부여):
  기록한 와인의 타입/산지 기반

  레벨 예시:
  레드 와인 Lv.5 · 화이트 Lv.3 · 스파클링 Lv.1
  보르도 Lv.3 · 부르고뉴 Lv.2 · 나파밸리 Lv.1
```

**와인 축 구조**:
- 1차: 타입 (레드/화이트/로제/스파클링/주정강화/디저트)
- 2차: 산지 국가 (프랑스/이탈리아/스페인/호주/미국/칠레/뉴질랜드...)
- 3차 (Phase 3): 세부 산지 + 품종 (보르도/부르고뉴, 카베르네/피노 누아)

## 3. XP 획득 기준

### 3-1. 기본 원칙

- **기록 추가 = 핵심 XP 원천** (전체 XP의 70% 이상)
- 부가 행동(사진, 한줄평 등)은 **보너스**
- 소셜 행동(공유, 좋아요)은 **아주 약한 보너스**
- 스팸 방지: 같은 식당 같은 날 2회 이상 = XP 1회만

### 3-2. XP 테이블

| 활동 | XP | 적용 축 | 비고 |
|------|-----|---------|------|
| **기록 추가** (사분면+만족도+상황) | 10 | 해당 지역 + 해당 장르(식당) 또는 타입+산지(와인) | 필수 입력만으로 |
| + 한줄평 포함 | +3 | 동일 | |
| + 사진 1장 이상 | +3 | 동일 | |
| + 추천 메뉴/페어링 메모 | +2 | 동일 | |
| + 사용팁 | +2 | 동일 | |
| **재방문/재음용 기록** | 12 | 동일 | 재방문은 깊이 = 약간 더 |
| **와인 향 팔레트 입력** | +5 | 와인 축 | 심화 평가 보상 |
| **와인 복합성+여운+균형** | +3 | 와인 축 | 심화 평가 보상 |
| 버블에 공유 | +1 | — (레벨 무관) | 소셜 참여 보너스 |
| 기록에 좋아요 받음 | +1 (기록당 1회) | — | 커뮤니티 기여 |
| 동네 인증 | +15 (1회) | 해당 지역 | 일회성 보너스 |

**최대 단건 XP**: 기록 10 + 한줄평 3 + 사진 3 + 메뉴 2 + 팁 2 = **20 XP**
**와인 최대**: 기록 10 + 한줄평 3 + 사진 3 + 향팔레트 5 + 심화 3 = **24 XP**

### 3-3. XP가 적용되는 곳

한 번의 기록으로 **복수 축에 XP가 동시 적용**:

```
예: 을지로 일식당에서 기록
  → 을지로 지역 경험치 +10
  → 일식 장르 경험치 +10 (Phase 2+, 선택적)

예: 보르도 레드 와인 기록
  → 레드 와인 타입 경험치 +10
  → 프랑스 산지 경험치 +10
  → 보르도 세부 산지 경험치 +10 (Phase 3)
```

## 4. 레벨 시스템

### 4-1. 레벨 테이블

| 레벨 | 누적 XP | 대략 기록 수 | 의미 | 색상 |
|------|---------|------------|------|------|
| Lv.1 | 0 | — | 시작 | — |
| Lv.2 | 30 | ~3회 | 입문 | 초록 |
| Lv.3 | 80 | ~8회 | 관심 | 초록 |
| Lv.4 | 160 | ~16회 | 익숙 | 파랑 |
| Lv.5 | 280 | ~28회 | 탐험가 | 파랑 |
| Lv.6 | 450 | ~45회 | 전문가 | 보라 |
| Lv.7 | 700 | ~70회 | 달인 | 보라 |
| Lv.8 | 1050 | ~105회 | 맛잘알 | 주황 |
| Lv.9 | 1500 | ~150회 | 구루 | 주황 |
| Lv.10 | 2100 | ~210회 | 레전드 | 금색 |

**설계 의도:**
- Lv.2까지 3회 → 온보딩 직후 달성 가능 (즉각 보상)
- Lv.5까지 28회 → 한 달에 2번 기록하면 ~1년. "관심 있는 사용자"
- Lv.7이상 → 진짜 그 지역/종류에 많이 기록한 사람. 신뢰할 만한 레벨
- Lv.10 → 210회+. 2~3년 꾸준히 기록한 사람. 매우 드뭄

### 4-2. 레벨업 시 경험

```
레벨업 순간:
  1. 축하 알림 (앱 내 + 푸시)
     "을지로 Lv.5 달성! 🎉"
     "을지로에서 28번째 기록을 남겼습니다"
  
  2. 프로필 뱃지 업데이트
     을지로 뱃지 색상이 초록 → 파랑으로 변경
  
  3. 사분면에 표시 (선택적)
     내 사분면 차트에 "Lv.5 달성" 마커
  
  4. 버블에서 표시
     내 기록 옆 뱃지가 새 레벨로 갱신
```

### 4-3. 레벨 표시 규칙

- **프로필**: 모든 레벨 표시 (메인 3개 강조 + 나머지 접이식)
- **버블 기록 옆**: 해당 기록과 **관련된** 레벨만 표시
  - 을지로 식당 기록 → "을지로 Lv.7" 뱃지만
  - 보르도 와인 기록 → "보르도 Lv.3" 뱃지만
- **식당/와인 상세페이지**: 기록자의 관련 레벨 표시
- **비표시 옵션**: 프로필 설정에서 레벨 숨김 가능

## 5. 뱃지 시스템

레벨과 별개로, **특정 조건 달성 시 해금되는 성취 뱃지**.

### 5-1. 설계 원칙

- 뱃지는 **탐색 동기** — "이 뱃지 따려면 새로운 동네를 가봐야겠다"
- 뱃지는 **자기만족** — 컬렉션 채우는 재미
- 뱃지로 **비교/경쟁하지 않음** — 다른 사용자의 뱃지 수를 비교하는 UI 없음

### 5-2. 뱃지 목록

#### 탐험 뱃지 (지역/장소)

| 뱃지 | 조건 | 단계 |
|------|------|------|
| **동네 탐험가** | N개 다른 동네에서 기록 | 5 → 10 → 20 → 30 → 50 |
| **서울 식도락** | 서울 N개 구에서 기록 | 5 → 10 → 20구 |
| **전국구** | N개 다른 시/도에서 기록 | 3 → 5 → 10 |
| **해외 미식가** | N개 다른 나라에서 기록 | 2 → 5 → 10 |

#### 다양성 뱃지 (장르/스타일)

| 뱃지 | 조건 | 단계 |
|------|------|------|
| **장르 컬렉터** | N개 다른 장르(한식/일식/양식...) 기록 | 5 → 10 → 15 |
| **와인 탐험가** | N종 와인 기록 | 10 → 25 → 50 → 100 |
| **세계의 와인** | N개 다른 산지 국가 와인 | 3 → 5 → 10 |
| **품종 마스터** | N개 다른 품종 와인 | 5 → 10 → 20 |

#### 깊이 뱃지 (재방문/충실도)

| 뱃지 | 조건 | 단계 |
|------|------|------|
| **단골** | 같은 식당 N회 재방문 기록 | 3 → 5 → 10 |
| **기록의 달인** | 사진+한줄평+메뉴 모두 포함한 기록 N개 | 10 → 30 → 100 |
| **와인 소믈리에** | 향 팔레트+심화 평가 포함 와인 기록 N개 | 5 → 15 → 50 |

#### 소셜 뱃지 (Phase 2)

| 뱃지 | 조건 | 단계 |
|------|------|------|
| **공유의 기쁨** | 버블에 기록 N개 공유 | 5 → 20 → 50 |
| **공감 받기** | 좋아요 총 N개 받기 | 10 → 50 → 200 |
| **버블 리더** | 버블 생성 후 멤버 N명 | 5 → 10 → 30 |

#### 마일스톤 뱃지

| 뱃지 | 조건 | 단계 |
|------|------|------|
| **첫 걸음** | 첫 번째 기록 | — (일회성) |
| **열 번째 기록** | 총 10개 기록 | — |
| **백 번째 기록** | 총 100개 기록 | — |
| **오백 번째 기록** | 총 500개 기록 | — |

### 5-3. 뱃지 UI

- **프로필 뱃지 컬렉션**: 달성한 뱃지 그리드 + 미달성은 실루엣(잠금)
- **달성 알림**: 인앱 + 푸시 + 축하 애니메이션
- **Retroactive**: 새 뱃지 추가 시 과거 기록 소급 적용
- **뱃지 상세**: 탭하면 달성 조건 + 현재 진행도 + 달성일

## 6. 프로필 정체성

경험치/레벨/뱃지의 최종 목적지는 **프로필**.

### 6-1. 프로필에서 보이는 것

```
┌──────────────────────────────────────┐
│ [아바타]                              │
│ 닉네임                               │
│                                       │
│ ── 숫자 요약 ──                       │
│ 기록 127 · 식당 98 · 와인 32          │
│                                       │
│ ── 메인 레벨 (상위 3개) ──             │
│ 을지로 Lv.7 🟣  광화문 Lv.5 🔵  강남 Lv.3 🟢 │
│ [전체 레벨 보기]                       │
│                                       │
│ ── 나의 사분면 ──                     │
│ [사분면 차트: 내 전체 기록 분포]        │
│ "저렴+캐주얼에서 맛있는 곳을 잘 찾는 사람" │ 자동 생성 한줄 프로필
│                                       │
│ ── 뱃지 컬렉션 ──                     │
│ [달성 뱃지 8개] [미달성 실루엣 4개]     │
│                                       │
│ ── 최근 기록 ──                       │
│ (기록 타임라인)                        │
└──────────────────────────────────────┘
```

### 6-2. 자동 생성 프로필 한줄

사분면 분포 + 레벨 데이터 기반으로 **자동 생성되는 한줄 설명**:

```
분석 로직:
  1. 가장 높은 레벨 지역 → "을지로 전문가"
  2. 높은 만족도 기록의 사분면 집중 영역 → "동네 맛집 발굴형" or "프리미엄 다이닝형"
  3. 와인 취향 → "풀바디 레드 애호가"

생성 예시:
  "을지로 골목 맛집을 꿰뚫는 사람. 풀바디 레드 와인도 좋아함."
  "강남 파인다이닝과 보르도 와인의 조합을 즐기는 미식가."
  "전국 각지의 저렴한 로컬 맛집을 찾아다니는 탐험가."
```

이건 Phase 3 기능이지만, 사분면 데이터 + 레벨 데이터가 충분하면 간단한 규칙 기반으로도 가능.

### 6-3. 프로필이 소셜에서 하는 역할

Letterboxd에서 프로필은 "이 사람이 어떤 영화를 좋아하는지"를 한눈에 보여준다.
Nyam 프로필은 "이 사람이 어떤 식도락을 즐기는지"를 보여준다.

```
사용 시나리오:
  1. 버블에서 김영수의 기록을 봄
  2. 김영수 프로필 탭 → "을지로 Lv.7, 일식 Lv.5"
  3. "이 사람 을지로 진짜 잘 아는구나" → 기록 신뢰도 ↑
  4. 김영수를 팔로우 → 김영수의 공개 기록이 내 피드에
```

## 7. Wrapped / 연간 결산 (F21)

### 7-1. 구성

```
── Nyam Wrapped 2026 ──────────────

📍 올해의 숫자
   기록 87개 · 식당 62곳 · 와인 25종
   방문한 동네 14곳

📊 나의 사분면 히트맵
   올해 기록의 분포. 밀도 높은 영역 강조.
   "당신은 저렴+캐주얼 영역에서 보물을 찾는 타입"

⭐ 올해 최고의 식당
   1. 도쿄등심 — 92점 (회식)
   2. 미진 — 88점 (혼밥)
   3. 스시코우지 — 87점 (데이트)

🍷 올해 최고의 와인
   1. Ch. Margaux 2018 — 94점 (블랙베리)
   2. Penfolds Bin 389 — 87점 (자두)

🗺 가장 많이 간 동네
   1. 을지로 (18회)
   2. 광화문 (12회)
   3. 강남 (9회)

🎭 상황별 비율
   혼밥 38% · 데이트 25% · 회식 20% · 친구 12% · 가족 5%

📈 성장
   을지로 Lv.3 → Lv.5
   레드 와인 Lv.2 → Lv.4
   새로운 뱃지 3개 획득

👥 버블 활동 (P2)
   가장 활발한 버블: 직장 맛집
   가장 많은 ♡ 받은 기록: 미진 (12개)
   
[이미지로 공유하기] → SNS 바이럴
```

| 기능 | 무료 | 프리미엄 |
|------|------|---------|

| 상시 통계 대시보드 | — | 언제든 접근 |
| 사분면 트렌드 (월별 변화) | — | 프리미엄 전용 |

## 8. 소셜 성장 플라이휠

경험치 시스템이 Nyam의 소셜을 성장시키는 구조:

```
┌─────────────────────────────────────────┐
│                                         │
│  기록한다 ──→ XP가 쌓인다               │
│    ↑              │                     │
│    │              ▼                     │
│    │         레벨이 오른다               │
│    │              │                     │
│    │              ▼                     │
│    │     프로필에 정체성이 생긴다         │
│    │         (을지로 Lv.7)              │
│    │              │                     │
│    │              ▼                     │
│    │     버블에서 내 기록이 신뢰받는다    │
│    │              │                     │
│    │              ▼                     │
│    │     다른 멤버가 내 기록을 찜/팔로우  │
│    │              │                     │
│    │              ▼                     │
│    │     좋아요/댓글 받음 = 보상감        │
│    │              │                     │
│    └──────────────┘                     │
│         다시 기록하고 싶어진다            │
│                                         │
└─────────────────────────────────────────┘
```

**Letterboxd와의 비교:**
```
Letterboxd: 기록 → 리스트/리뷰 공유 → 팔로워 → 리뷰 문화 형성 → 더 기록
Nyam:       기록 → 레벨/정체성 → 버블 신뢰 → 공유/반응 → 더 기록
```

차이점: Letterboxd는 **공개 리뷰 문화**가 플라이휠. Nyam은 **폐쇄형 신뢰**가 플라이휠.
경험치/레벨은 이 신뢰의 **시각적 근거**를 만들어준다.

## 9. 데이터 구조

```sql
-- 사용자 경험치 (축별)
CREATE TABLE user_experience (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  axis_type VARCHAR(20) NOT NULL,  -- 'area', 'genre', 'wine_type', 'wine_region'
  axis_value VARCHAR(50) NOT NULL, -- '을지로', '일식', 'red', '프랑스'
  total_xp INT DEFAULT 0,
  level INT DEFAULT 1,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (user_id, axis_type, axis_value)
);

-- XP 이력 (디버깅/감사용)
CREATE TABLE xp_history (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES users(id),
  record_id UUID REFERENCES records(id),
  axis_type VARCHAR(20),
  axis_value VARCHAR(50),
  xp_amount INT,
  reason VARCHAR(50),  -- 'record', 'photo', 'comment_bonus', 'area_verify'
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 뱃지 정의
CREATE TABLE badge_definitions (
  id VARCHAR(50) PRIMARY KEY,  -- 'area_explorer_1', 'genre_collector_2'
  category VARCHAR(30),        -- 'exploration', 'diversity', 'depth', 'social', 'milestone'
  name VARCHAR(50),
  description TEXT,
  condition_type VARCHAR(30),  -- 'count_areas', 'count_genres', 'revisit_count'
  condition_value INT,         -- 5, 10, 20...
  badge_group VARCHAR(30),     -- 그룹 내 단계 식별
  stage INT,                   -- 1, 2, 3...
  icon_url TEXT
);

-- 사용자 달성 뱃지
CREATE TABLE user_badges (
  user_id UUID REFERENCES users(id),
  badge_id VARCHAR(50) REFERENCES badge_definitions(id),
  achieved_at TIMESTAMPTZ DEFAULT NOW(),
  PRIMARY KEY (user_id, badge_id)
);

-- 레벨 테이블 (상수)
```

### XP 적립 로직 (Edge Function / API)

```
기록 저장 시:
1. 기록의 식당 → 지역(동네) 식별 → user_experience에 XP 적립
2. 기록의 식당 → 장르 식별 → user_experience에 XP 적립 (Phase 2+)
3. 기록의 와인 → 타입/산지 식별 → user_experience에 XP 적립
4. XP 적립 후 → 현재 레벨 체크 → 레벨업이면 알림
5. 뱃지 조건 체크 → 달성이면 알림
```

## 11. 핵심 결정 요약

| 결정 | 선택 | 이유 |
|------|------|------|
| 전체 방향 | 기록=메인보상, XP=서브 | 식도락은 이미 즐거운 활동. 외적 보상이 침범하면 안 됨 |
| 레벨 축 | 지역별 + 와인 종류별 독립 | 통합 레벨은 전문성을 반영 못함 |
| XP 핵심 원천 | 기록 추가 (70%+) | 기록이 핵심 행동. 다른 건 보너스 |
| 스트릭 | 없음 | 식당은 매일 가는 게 아님 |
| 리더보드 | 없음 | 식도락≠경쟁 |
| 뱃지 | 탐색/다양성 동기용 | 새로운 곳/종류를 시도하게 만드는 장치 |
| Wrapped | 무료 기본 + 프리미엄 상세 | 바이럴 + 수익화 |
| 레벨 감소 | 없음 | 경험은 사라지지 않음 |
| 소셜 활용 | 필터/정렬 옵션만 | 자동 가중치는 초보자 기록을 묻음 |
| 플라이휠 | 기록→레벨→정체성→신뢰→반응→기록 | Letterboxd 공개리뷰문화 대신 폐쇄형 신뢰 |
