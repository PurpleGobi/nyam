# 경험치 시스템 딥리서치

> 작성일: 2026-03-19
> 목적: Nyam 경험치 시스템 기획을 위한 레퍼런스 분석 및 설계 방향 도출
> 참조: PRD v2.0 §8, BUBBLE_SOCIAL_SPEC §10

---

## 1. 레퍼런스 앱 분석

### 1-1. Untappd — 뱃지 시스템 (음료 기록 앱, 가장 직접적 레퍼런스)

**구조:**
- 체크인(기록)할 때마다 조건에 맞는 **뱃지 자동 해금**
- 뱃지는 **레벨제** — 같은 뱃지를 반복 달성하면 레벨업 (최대 Lv.100)
- 뱃지 종류: 빈도 기반(N번 체크인), 스타일 기반(IPA 10종), 장소 기반(N개 브루어리), 이벤트 기반(시즌 한정)

**핵심 인사이트:**
- 뱃지가 **탐색 동기**를 만든다: "Land of the Free 뱃지 따려면 미국 맥주 20종 더 마셔야 해" → 새로운 맥주 시도
- 뱃지가 **기록 동기**를 만든다: 체크인 안 하면 뱃지가 안 쌓임 → 기록하게 됨
- 뱃지 자체가 **광고 채널**: 브루어리가 자체 뱃지를 만들어 사용자 유입 (Nyam은 이걸 피해야)
- **Retroactive 뱃지**: 새 뱃지가 추가되면 과거 체크인도 소급 적용 → 사용자 불만 방지

**Nyam에 적용할 것:**
- 지역별/종류별 레벨제 뱃지 (PRD v2와 일치)
- 새로운 식당/와인 탐색을 유도하는 뱃지 설계
- Retroactive 정책 (나중에 뱃지 추가해도 기존 기록 반영)

**Nyam이 피할 것:**
- 광고성 뱃지 (신뢰 오염)
- 뱃지 수집이 목적이 되어 기록 품질 하락 (양 > 질 문제)

### 1-2. Strava — 세그먼트 & 트로피 (운동 기록 앱)

**구조:**
- **세그먼트**: 특정 구간에서의 기록 경쟁. 최고 기록자에게 Crown/Cup
- **트로피**: 마일스톤 달성 (100km 달성, 연간 목표 등)
- **클럽 내 랭킹**: 주간 활동량 리더보드
- **개인 기록(PR)**: 자기 자신과의 경쟁. "이번 달 최다 활동" 등

**핵심 인사이트:**
- **경쟁 vs 자기 성장 분리**: 세그먼트(타인 경쟁)와 트로피(자기 성장)가 별개 시스템
- **과도한 경쟁은 이탈 유발**: 리더보드 상위권만 혜택 → 하위권 사용자 의욕 저하
- **프리미엄 연계**: 고급 통계/분석은 유료 구독 (Nyam Phase 3 수익화 연결 가능)
- **Wrapped/Year in Review**: 연간 결산이 바이럴 효과 (SNS 공유)

**Nyam에 적용할 것:**
- 자기 성장 중심 시스템 (타인 경쟁보다 내 기록 축적)
- 연간 Wrapped (PRD v2 F21과 일치)
- 프리미엄 통계 연계 (Phase 3)

**Nyam이 피할 것:**
- 공개 리더보드 (소셜 압박, 프라이버시)
- 과도한 경쟁 요소 (식도락은 운동과 다르다 — 경쟁하면 재미 없어짐)

### 1-3. Duolingo — XP & 리그 시스템 (학습 앱)

**구조:**
- **XP (경험치)**: 레슨 완료, 연습, 퀴즈 등에서 획득
- **리그**: XP 기반 주간 리더보드. 10개 리그, 상위 N명 승격/하위 강등
- **스트릭**: 연속 학습 일수. 놓치면 리셋 (프리미엄 구독으로 복구 가능)
- **레벨**: 코스별 진행도. XP 누적 → 레벨업

**핵심 인사이트:**
- **스트릭 = 가장 강력한 리텐션 도구**: 잃는 것에 대한 두려움(Loss Aversion) 활용
- **리그(경쟁) = DAU 25% 증가**: 경쟁은 단기 참여를 강력하게 끌어올림
- **Over-justification 위험**: 외적 보상(XP)이 내적 동기(학습)를 대체하면 보상 없을 때 이탈
- **스트릭의 어두운 면**: 의무감 → 부담 → 번아웃 → 이탈. 특히 식도락은 매일 기록할 대상이 아님

**Nyam에 적용할 것:**
- 레벨 시스템 (지역별/종류별 독립)
- 마일스톤 축하 (10번째 기록! 등)

**Nyam이 절대 피할 것:**
- 스트릭 시스템: 식당은 매일 가는 게 아니다. 스트릭 압박은 부적절
- 리그/랭킹: 식도락에 순위 경쟁은 철학에 반함
- XP 인플레이션: 모든 행동에 점수 부여하면 의미 희석

### 1-4. Google Local Guides — 기여 포인트 (지도 기여 앱)

**구조:**
- 리뷰/사진/답변/장소 추가 등 **기여 유형별 차등 포인트**
  - 리뷰: 10점, 200자+ 리뷰: 추가 10점
  - 사진: 5점, 동영상: 7점
  - 답변: 1점, 장소 추가: 15점
  - 사실 확인: 1점
- **10개 레벨** (Lv.1: 0점 ~ Lv.10: 100,000점+)
- Lv.4부터 뱃지 표시, Lv.6부터 조기 접근 혜택
- 포인트는 **전체 통합** (지역별 분리 아님)

**핵심 인사이트:**
- **기여 품질에 차등 보상**: 단순 별점 < 상세 리뷰+사진. 품질 높은 기록을 유도
- **레벨이 "전문가" 인상을 줌**: Local Guide Lv.8 = 신뢰감
- **문제점**: 포인트 수집 자체가 목적이 되어 저품질 리뷰 양산 (1줄짜리 리뷰 스팸)

**Nyam에 적용할 것:**
- 기록 품질에 따른 차등 경험치 (한줄평/사진/사분면 세부 입력에 보너스)
- 레벨이 소셜에서 신뢰 시그널로 작동

**Nyam이 피할 것:**
- 전체 통합 포인트 (지역별 분리가 Nyam의 차별점)
- 포인트 스팸 유발 (최소 품질 기준 필요)

### 1-5. Foursquare/Swarm — 시장(Mayor) & 뱃지 (위치 기반 앱)

**구조:**
- **시장(Mayor)**: 특정 장소에 가장 많이 체크인한 사람. 60일 기준 갱신
- **뱃지**: 다양한 조건 달성 시 해금 (장르별, 횟수별, 지역별)
- **코인/스티커**: 소소한 보상 (시각적 아이템)

**핵심 인사이트:**
- **Mayor = 강력한 로컬 정체성**: "이 식당의 단골" 인증. 경쟁+자부심
- **Mayor 시스템의 몰락**: 체크인 피로감 + 의미 없는 경쟁 → 사용자 이탈 → 결국 축소
- **교훈**: 위치 기반 경쟁은 초기에는 재미있지만 장기 리텐션에 불리

**Nyam에 적용할 것:**
- "단골" 개념 (재방문 횟수로 자연스럽게 표현, 명시적 "시장" 타이틀은 불필요)
- 지역별 활동 밀도 시각화

**Nyam이 피할 것:**
- 특정 식당의 "시장" 경쟁 (의미 없는 반복 방문 유도)
- 체크인 피로감

---

### 1-6. Letterboxd — 통계 & Year in Review (영화 기록 앱)

**구조:**
- **뱃지/레벨 시스템 없음**. Letterboxd는 게이미피케이션을 의도적으로 최소화
- **통계(Stats)**: Pro/Patron 구독자에게 연간/전체 통계 제공 (장르 분포, 평균 평점, 시청 패턴 등)
- **Year in Review**: 무료 사용자에게도 연말 제공 → **바이럴 효과** (SNS 공유)
- **리스트/다이어리**: 기록 자체가 보상. 채워나가는 재미

**핵심 인사이트:**
- **게이미피케이션 없이도 리텐션 가능**: 기록 + 소셜 + 통계만으로 충분
- **통계 = 프리미엄 가치**: 사용자는 "내가 올해 뭘 했나" 보는 걸 좋아함
- **Year in Review = 최고의 마케팅**: 사용자가 자발적으로 SNS에 공유

**Nyam에 적용할 것:**
- 통계를 프리미엄 기능으로 (Phase 3 수익화)
- Year in Review / Wrapped (PRD v2 F21)
- 기록 자체의 보상감 (사분면 채워지는 시각적 만족)

## 2. 게이미피케이션의 빛과 그림자

### 2-1. 효과가 검증된 요소

| 요소 | 효과 | 근거 |
|------|------|------|
| **마일스톤 축하** | 기록 지속 동기 | "10번째 기록!" → 성취감 |
| **진행도 시각화** | 완성 욕구 | 프로그레스 바, 사분면 채우기 |
| **차등 보상** | 품질 높은 기록 유도 | 상세 기록 > 간단 기록 |
| **Year in Review** | 바이럴 + 리텐션 | Letterboxd/Spotify 검증 |
| **뱃지/레벨** | 초기 참여 + 정체성 | 22% 리텐션 향상 (ClearTap 2025) |

### 2-2. 주의해야 할 요소

| 요소 | 리스크 | Nyam에서의 판단 |
|------|--------|---------------|
| **스트릭** | 매일 식당 갈 수 없음. 압박 → 번아웃 | **사용 안 함** |
| **리더보드/랭킹** | 소셜 압박, 경쟁 피로, 프라이버시 | **사용 안 함** (버블 내 통계만) |
| **무차별 포인트** | Over-justification. 보상 없으면 이탈 | **품질 차등만** |
| **랜덤 보상** | 중독성 설계. 윤리적 문제 | **사용 안 함** |
| **공개 레벨 비교** | 초보자 위축 | **레벨은 개인+버블 내에서만** |

### 2-3. Nyam의 방향: "기록 자체가 보상"

```
Duolingo 방향: 외적 보상(XP/리그)이 행동을 이끈다
Letterboxd 방향: 기록 자체의 만족감이 행동을 이끈다

Nyam = Letterboxd 방향 + 약간의 Untappd 뱃지

이유:
- 식도락은 이미 즐거운 활동이다 (외적 보상이 내적 동기를 대체하면 안 됨)
- 사분면에 점이 쌓이는 것 자체가 시각적 보상
- 경험치/레벨은 "신뢰의 근거"이지 "경쟁의 도구"가 아님
```

## 3. Nyam 경험치 시스템 설계 방향

### 3-1. 핵심 원칙

1. **지역별/종류별 독립 레벨** — "을지로 Lv.7" / "보르도 와인 Lv.3"
   - 전체 통합 레벨은 의미 없다 (강남에서 100번 기록해도 을지로 전문가는 아님)
   - Google Local Guide의 통합 포인트 문제를 피한다

2. **기록 품질에 차등** — 사분면만 찍으면 기본, 한줄평+사진+추천메뉴까지 하면 추가
   - 하지만 차등이 너무 크면 "의무감" 발생 → 가볍게

3. **경쟁 아닌 성장** — 타인과 비교하는 리더보드 없음. 내 기록이 쌓이는 걸 보는 재미
   - 버블 내 통계는 "재미"로만 (정렬 옵션에 "경험자 순" 있지만 기본값 아님)

4. **레벨이 신뢰를 만든다** — 버블에서 리뷰 옆에 관련 레벨 표시
   - 하지만 자동 가중치 아님 (BUBBLE_SOCIAL_SPEC §10-3에서 결정 완료)

5. **스트릭/랭킹 없음** — 식당은 매일 가는 게 아님. 압박은 철학에 반함

### 3-2. 경험치 축 설계

**식당 경험치 축:**
```
지역별 (동 단위 or 구 단위):
  광화문 Lv.3 · 을지로 Lv.7 · 강남 Lv.5 · 성수 Lv.2

장르별 (선택적):
  일식 Lv.4 · 이탈리안 Lv.3 · 한식 Lv.6
```

**와인 경험치 축:**
```
타입별:
  레드 와인 Lv.5 · 화이트 와인 Lv.3 · 스파클링 Lv.1

산지별 (선택적):
  보르도 Lv.3 · 부르고뉴 Lv.2 · 바로사 밸리 Lv.1

품종별 (선택적):
  카베르네 소비뇽 Lv.4 · 피노 누아 Lv.2
```

### 3-3. 경험치 획득 기준 (초안)

| 활동 | 기본 XP | 조건 |
|------|---------|------|
| 기록 추가 (사분면+만족도+상황) | 10 | 필수 입력만으로 |
| 한줄평 포함 | +3 | 한줄평 있을 때 |
| 사진 포함 | +3 | 사진 1장 이상 |
| 추천 메뉴 태그 | +2 | 메뉴 태그 있을 때 |
| 사용팁 포함 | +2 | 팁 텍스트 있을 때 |
| 재방문 기록 | +5 | 같은 식당/와인 재방문 |
| 버블 공유 | +2 | 기록을 버블에 공유 |
| 리뷰에 좋아요 받음 | +1 (1회/기록) | 커뮤니티 가치 |
| 동네 인증 | +20 (1회) | 해당 지역 일회성 보너스 |
| 와인: 향 팔레트 입력 | +5 | 심화 평가 |
| 와인: 복합성+여운+균형 입력 | +5 | 심화 평가 |

### 3-4. 레벨 테이블 (초안)

| 레벨 | 필요 누적 XP | 의미 | 뱃지 |
|------|------------|------|------|
| Lv.1 | 0 | 시작 | — |
| Lv.2 | 30 | 3~4회 방문 | 🟢 |
| Lv.3 | 80 | ~8회 방문 | 🟢 |
| Lv.4 | 150 | ~15회 | 🔵 |
| Lv.5 | 250 | ~25회 | 🔵 |
| Lv.6 | 400 | ~40회 | 🟣 |
| Lv.7 | 600 | ~60회 | 🟣 |
| Lv.8 | 900 | ~90회 | 🟠 |
| Lv.9 | 1300 | ~130회 | 🟠 |
| Lv.10 | 2000 | ~200회+ | ⭐ |

**설계 의도:**
- 초반 레벨업은 빠르게 (3~4회 방문이면 Lv.2) → 초기 보상감
- 후반은 점점 느리게 → 높은 레벨이 진짜 의미 있음
- Lv.10 달성은 매우 어려움 → 특정 지역/종류의 진짜 전문가
- "을지로 Lv.7" = 을지로에서 60번 이상 기록 = 진짜 을지로 맛잘알

### 3-5. 뱃지/성취 시스템 (Untappd 참고)

레벨과 별도로, **특정 조건 달성 시 해금되는 성취 뱃지**:

| 카테고리 | 뱃지 이름 (안) | 조건 | 레벨제 |
|----------|--------------|------|--------|
| **탐험가** | 새 동네 개척자 | N개 다른 동네에서 기록 | Lv.1~5 (5/10/20/30/50동네) |
| **다양성** | 장르 탐험가 | N개 다른 장르 기록 | Lv.1~3 (5/10/20장르) |
| **단골** | 단골손님 | 같은 식당 N회 재방문 | Lv.1~3 (3/5/10회) |
| **소믈리에** | 와인 탐험가 | N종 와인 기록 | Lv.1~5 (5/10/25/50/100종) |
| **세계일주** | 와인 산지 탐험 | N개 다른 산지 와인 | Lv.1~3 (3/5/10산지) |
| **공유자** | 버블 기여자 | 버블에 N개 기록 공유 | Lv.1~3 (5/20/50개) |
| **기록광** | 꾸준한 기록자 | 총 N개 기록 달성 | Lv.1~5 (10/50/100/200/500개) |
| **미식가** | 미슐랭 탐방 | 미슐랭 선정 식당 N곳 방문 | Lv.1~3 (3/5/10곳) |
| **사진작가** | 사진 기록가 | 사진 포함 기록 N개 | Lv.1~3 (10/30/100개) |

**뱃지 표시:**
- 프로필에 뱃지 컬렉션 (달성한 것 + 미달성 실루엣)
- 달성 시 알림 + 애니메이션
- Retroactive: 새 뱃지 추가 시 과거 기록도 소급 적용

### 3-6. 시각적 보상 — 사분면이 채워지는 재미

경험치와 별도로, **사분면 자체가 시각적 보상**:

```
기록 5개:  사분면에 점이 듬성듬성
기록 20개: 패턴이 보이기 시작 ("나는 좌하단=동네 맛집을 좋아하네")
기록 50개: 사분면이 풍성. 내 취향 지도가 완성됨
기록 100+: 사분면이 꽉 참. 프로필에서 자부심
```

와인은 향 팔레트 색상까지:
```
"내 사분면에 어두운 자주(블랙베리) 점이 많네 → 나는 진한 과일향 풀바디를 좋아해"
```

**이것이 Letterboxd의 "기록 자체가 보상" 철학과 동일.** 뱃지/XP는 보조, 사분면 채우기가 메인 보상.

## 5. 설계 시 주의할 결정 사항

### 5-1. 지역 단위 결정

| 옵션 | 장점 | 단점 |
|------|------|------|
| 동 단위 (광화문동) | 세밀한 로컬 전문성 | 너무 잘게 쪼개짐, 레벨업 어려움 |
| 구 단위 (종로구) | 적절한 범위 | 종로구 안에서도 광화문/을지로가 다름 |
| 커스텀 동네 (을지로, 성수) | 실제 인식과 일치 | 경계 정의 어려움 |
| **혼합** | 유연함 | 구현 복잡도 |

**제안**: "구 + 대표 동네" 혼합. 예: "종로구" 레벨 + 하위에 "광화문" "을지로" 세부 레벨.
또는 사용자가 느끼는 "동네" 개념에 맞춰 카카오맵/네이버의 동네 분류 활용.

### 5-2. 와인 축 설계

| 옵션 | 설명 |
|------|------|
| 타입만 (레드/화이트/로제/스파클링) | 심플하지만 너무 넓음 |
| 타입 + 산지 (국가 수준) | 적절한 깊이 |
| 타입 + 산지 + 품종 | 깊지만 복잡 |

**제안**: Phase 2에서는 타입 + 국가 수준 산지. Phase 3에서 품종별 확장.

### 5-3. 경험치 인플레이션 방지

- 모든 행동에 XP를 주면 의미 희석 → **기록 추가가 핵심 XP 원천** (전체의 60%+)
- 버블 공유, 좋아요 받기 등은 **보너스** 수준
- 일일 XP 상한은 두지 않음 (여행 중 하루 10곳 기록하는 건 합당한 활동)
- 대신 **스팸 방지**: 같은 식당 같은 날 2회 이상 기록 = XP 1회만

### 5-4. 경험치 리셋/감소

- **리셋 없음**: 한번 쌓은 경험치는 사라지지 않음
- **비활동 감소 없음**: 6개월 안 써도 레벨 유지 (Duolingo 스트릭과 반대)
- 이유: 식도락 경험은 사라지지 않음. "을지로 60번 갔던 사람"은 여전히 을지로 전문가
