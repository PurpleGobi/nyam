## 1. 시스템 개요 & 철학

> **한 번의 터치에 3차원 데이터.**  
> 사분면 위치(특성 2축) + 점 크기(만족도) + 상황 태그 = 약 3초에 완전한 평가.

### 설계 원칙

| 원칙 | 설명 |
|------|------|
| **3초 완료** | 필수 평가는 3초 안에 끝나야 한다. 나머지는 전부 선택. |
| **찍는 재미** | 평가 자체가 즐거운 경험. 게임적 느낌. "또 찍고 싶다" |
| **비교의 즐거움** | 배경에 대표 식당/와인이 보여서 상대적 위치를 즐길 수 있다 |
| **점진적 확장** | Quick(필수) → Detail(선택) → Story(선택). 원하는 깊이까지 |
| **통일된 문법** | 식당과 와인이 같은 조작 방식. 한 번 배우면 둘 다 쓸 수 있다 |

### 무엇이 독창적인가

- 별점 5개도, 100점 슬라이더도 **1차원**. Nyam은 **3차원**(위치 2축 + 크기 1축)
- 한 번 찍기로 "이 식당은 어떤 곳이야(특성)" + "얼마나 좋았어(만족도)"를 동시에 표현
- 기록이 쌓이면 사분면 위에 나만의 **취향 지도**가 자동 생성
- 와인은 여기에 **향 색상**이 더해져 4차원. 다른 앱에 이런 평가 UI가 없음 → **Nyam의 시그니처**

## 2. 사분면 축 정의

> 축 선정 기준: 양 끝이 명확한 반대 개념 + 서로 독립적 + 사분면 4칸 골고루 분포

### 식당 사분면

| 축 | 0% | 100% | 의미 |
|----|-----|------|------|
| **X축: 가격** | 저렴 | 고가 | 체감 가격. 단순 단가가 아닌 양/조합/가치 종합 느낌 |
| **Y축: 분위기** | 캐주얼 | 포멀 | 격식/분위기. 편한 동네 맛집부터 파인다이닝까지 |
| **크기: 맛** | 1 (별로) | 100 (최고) | 맛 만족도. 위치와 완전히 독립. **어디서든 맛있을 수 있다** |

**식당 사분면 4칸:**

```
                 포멀
                  ↑
  합리적 격식집    |    프리미엄 다이닝
  (저렴한 코스)   |    (오마카세, 파인다이닝)
                  |
저렴 ─────────────┼───────────── 고가
                  |
  동네 맛집, 분식  |    트렌디 핫플
  (백반, 칼국수)   |    (성수 브런치, 루프탑)
                  ↓
                 캐주얼
```

### 와인 사분면

| 축 | 0% | 100% | 의미 |
|----|-----|------|------|
| **X축: 산미** | 낮음 | 높음 | 신맛의 강도 |
| **Y축: 바디** | 가벼움 (Light) | 무거움 (Full) | 입 안의 무게감 |
| **크기: 만족도** | 1 (별로) | 100 (최고) | 전체 만족도. 특성과 독립 |

**와인 사분면 4칸:**

```
              Full Body
                ↑
  까쇼, 메를로    |    바롤로, 산지오베제
  아마로네        |    시라
                |
산미 낮음 ───────┼──────── 산미 높음
                |
  피노 그리지오   |    소비뇽 블랑, 리슬링
  모스카토        |    샴페인
                ↓
              Light Body
```

## 3. 3차원 데이터 모델

> 하나의 평가 = **(X%, Y%, Score, Scene)**

| 차원 | 입력 방식 | 범위 | 표시 |
|------|----------|------|------|
| **X축 (특성 1)** | 사분면 위치 (좌우) | 0~100% | 백분위 % |
| **Y축 (특성 2)** | 사분면 위치 (상하) | 0~100% | 백분위 % |
| **만족도** | 위아래 드래그 (점 크기) | 1~100점 | 숫자 + 크기 |
| **상황** | 태그 탭 | 6개 중 1개 | 라벨 |

### 만족도 ↔ 점 크기 매핑

| 만족도 | 점 지름(px) | 시각적 느낌 |
|--------|------------|------------|
| 1~20 | 12~16 | 작은 점 · 별로 |
| 21~40 | 17~22 | 보통 이하 |
| 41~60 | 23~30 | 보통 |
| 61~80 | 31~40 | 좋음 |
| 81~100 | 41~54 | 큰 점 · 최고 |

### 만족도 ↔ 색상 매핑

| 만족도 | 색상 | 의미 |
|--------|------|------|
| 1~30 | 레드 (#ef4444) | 별로 |
| 31~50 | 앰버 (#f59e0b) | 보통 이하 |
| 51~70 | 옐로우 (#fbbf24) | 보통 |
| 71~85 | 그린 (#22c55e) | 좋음 |
| 86~100 | 블루 + 글로우 (#3b82f6) | 최고 |

> 와인은 이 색상 대신 **향 팔레트 색상**이 점 색상이 됨 (10장 참조)

## 4. 조작 플로우 & 제스처

### 전체 플로우

| 단계 | 행동 | 제스처 | 화면 변화 | 시간 |
|------|------|--------|----------|------|
| **1. 탭** | 사분면에서 대략 위치 선택 | 1-finger tap | 탭한 사분면이 확대 (줌인) | ~0.5초 |
| **2. 드래그** | 정밀 위치 조정 | 1-finger drag | 점이 손가락 따라감. X%, Y% 실시간 표시 | ~1초 |
| **3. 릴리즈** | 위치 확정 | 손 떼기 | 점 확정. 만족도 모드로 전환 | 자동 |
| **4. 상하 드래그** | 만족도(크기) 조절 | 점을 상하로 드래그 | 위=커짐(고점수), 아래=작아짐(저점수). 숫자 표시 | ~1초 |
| **5. 태그** | 상황 선택 | 1-finger tap | 하단 태그 중 하나 선택 | ~0.5초 |
| **6. 완료** | 기록 버튼 | 1-finger tap | 저장 + 성공 애니메이션 | ~0.3초 |

### 제스처 충돌 방지

| 상태 | 상하 드래그 | 좌우 드래그 |
|------|-----------|-----------|
| **위치 모드** (Step 2) | Y축 조정 | X축 조정 |
| **크기 모드** (Step 4) | 만족도 조절 | 비활성 (무시) |

- 위치 확정 후(손 떼기) 점을 다시 터치 → 크기 모드 진입
- 모드 전환은 시각적으로 명확히 표시 (테두리 실선→점선, 안내 텍스트)

### 위치 재조정

- 크기 모드에서 **점 바깥 영역** 탭 → 위치 모드 복귀
- 또는 **"위치 다시"** 버튼
- 핀치 아웃 or 뒤로가기 → 사분면 전체 보기로 복귀

## 5. 애니메이션 & 햅틱

| 이벤트 | 애니메이션 | 햅틱 | 사운드(선택) |
|--------|----------|------|------------|
| **사분면 탭** | 물결(ripple) 확산 → 해당 사분면 확대 (0.3s ease-out) | light tap | - |
| **확대(줌인)** | 해당 사분면 화면 가득. 나머지 3칸 가장자리로. spring 애니메이션 | - | - |
| **위치 드래그** | 점이 손가락 따라감. 그림자 확대(lift). X%, Y% 실시간 갱신 | 축 중심선(50%) 지날 때 딸깍 (light) | - |
| **위치 확정** | 점 살짝 바운스 후 안착. 테두리 실선→점선 변화 | medium tap | 작은 "톡" |
| **크기 드래그 (위)** | 점 커짐. 숫자 올라감. 색상 밝아짐 | 10점 단위 딸깍 | - |
| **크기 드래그 (아래)** | 점 작아짐. 숫자 내려감. 색상 어두워짐 | 10점 단위 딸깍 | - |
| **상황 태그 선택** | 선택 태그 커지며 강조색 전환 (0.2s) | light tap | - |
| **기록 완료** | 점이 사분면에 "박히는" 느낌. 전체 축소. 체크마크 오버레이 (0.5s → fade) | success notification | "딩" |
| **점 크기 90+ 도달** | 미세 글로우(glow) 이펙트 | heavy tap | - |
| **배경 레퍼런스 등장** | 확대 시 대표 식당/와인 fade-in (0.4s). 가까운 것은 이름 표시 | - | - |

### 점(Dot) 비주얼 스펙

| 속성 | 위치 모드 | 크기 모드 | 확정 후 |
|------|----------|----------|---------|
| 테두리 | 2px solid white | 2px dashed accent | 없음 (그림자만) |
| 그림자 | 8px blur, 40% | 12px blur, 60% | 6px blur, 30% |
| 색상 | accent (반투명) | 만족도 기반 변화 | 최종 색상 고정 |
| 숫자 | X%, Y% (점 옆) | 만족도 점수 (점 위) | 만족도 점수 (점 안/옆) |

## 6. 배경 레퍼런스 시스템

> 사분면 배경에 대표 식당/와인이 반투명으로 표시 → 상대평가 용이

### 데이터 소스 (우선순위)

1. **같은 지역 내 다른 사용자들의 평균 좌표** — 충분한 데이터 있을 때
2. **미슐랭/블루리본 식당 사전 배치** — 콜드스타트/데이터 부족 시
3. **자신의 과거 기록** — 개인 기록 쌓이면 자신의 점도 배경에

### 표시 규칙

| 규칙 | 설명 |
|------|------|
| 최대 표시 수 | 한 화면 8~12개. 너무 많으면 복잡 |
| 투명도 | 15~25% opacity |
| 이름 표시 | 확대 시에만. 가까운 3~4개만 |
| 내 기록 구분 | accent 계열 / 타인·레퍼런스는 회색 |
| 터치 반응 | 배경 점 탭 → 간단 정보 툴팁 (가게명, 평균 점수) |

## 8. 확장 평가 (선택)

### 식당 확장

| 항목 | 입력 방식 |
|------|----------|
| 한줄평 | 자유 텍스트 (최대 200자) |

| 함께 간 사람 | Nyam 사용자 태그 or 닉네임 |

|------|----------|
| 한줄평 | 자유 텍스트 |
| 사진 | 라벨/잔 사진 |
| 페어링 메모 | "스테이크와 잘 맞았다" |

| 구매처/장소 | 검색 (식당 연결 가능) |

## 10. 와인 심화 평가 시스템

> 와인은 기호식품·취미. 식당의 Quick Rate(3초)와 달리, 와인 애호가는 더 깊은 평가를 즐긴다.
>
> **점 하나에 담기는 4차원:** 위치(산미×바디) + 색상(향) + 크기(만족도) + 테두리(상황)

### 10-1. 전체 플로우

| Step | 이름 | 수집 항목 | 소요 | 필수 |
|------|------|----------|------|------|
| 1 | 사분면 | 산미% × 바디% (위치) | ~2초 | 필수 |
| 2 | 향 팔레트 | 주요 향 영역 선택 → 점 색상 결정 | ~3초 | 필수 |
| 3 | 복합성 | 1차/2차/3차 향 다양성 | ~2초 | 선택 |
| 4 | 여운 & 균형 | 여운 길이 + 균형도 | ~2초 | 선택 |
| 5 | 만족도 확정 | Step 3,4 기반 자동 산출 → 사용자 미세 조정 | ~1초 | 필수 |
| 6 | 상황 태그 | 6개 중 1개 | ~1초 | 필수 |

> Step 3,4 건너뛰면 만족도 직접 입력 (식당과 동일한 Quick 모드).  
> 최소 Step 1+2+5+6 = ~7초.

### 10-2. 향 팔레트 시스템

> **핵심 원리:** 같이 느낄 수 있는 향은 가까이, 같이 느낄 수 없는 향은 멀리 배치.
> 사용자는 점 하나를 찍는 게 아니라, **영역을 그려서** 복수의 향을 동시에 표현할 수 있다.

#### 설계 원칙

1. **근접성 = 공존 가능성** — 블랙베리와 자두는 함께 느낄 수 있으므로 인접 배치
2. **거리 = 비공존** — 블랙커런트와 딸기는 동시에 느끼기 어려우므로 먼 위치
3. **색상 = 실제 과일색** — 복숭아는 복숭아색, 라임은 녹색, 블랙베리는 어두운 자주
4. **영역 그리기** — 점 하나가 아니라 **드래그로 영역을 칠하는** UI. 복합적 향을 자연스럽게 표현

#### 레드 와인 향 팔레트 배치

```
향 배치 (색상환 / 그라데이션):

      블랙커런트          블랙베리
     (거의 검정)        (어두운 자주)
         ·───────────────·
        /                 \
카시스 ·                   · 자두
(짙은보라)                (자주-적갈색)
       \                  /
        ·────────────────·
     블루베리            레드체리
    (남보라)            (선명한 빨강)
        \               /
         ·─────────────·
       라즈베리       딸기
      (핑크레드)    (밝은 핑크)

※ 인접한 것끼리 = 함께 느낄 수 있는 향
※ 먼 것끼리 = 동시에 느끼기 어려운 향
```

#### UI: 영역 그리기 방식

> 점 하나를 찍는 게 아니라, **손가락으로 영역을 칠하는** 방식.
> "이 와인에서 블랙베리 + 자두 + 약간의 체리를 느꼈어" → 그 영역을 쓱 칠함.

**조작법:**

1. 팔레트 위에서 **손가락으로 드래그** → 지나간 영역이 칠해짐 (하이라이트)
2. 칠한 영역의 **중심 색상**이 사분면 점의 색이 됨
3. 영역의 **넓이** = 향의 다양성 힌트 (좁으면 단일 향, 넓으면 복합 향)
4. 잘못 칠하면 **흔들어서 리셋** (shake to undo)

**장점:**
- "블랙베리만" → 작은 점 하나 → 단일 향
- "블랙베리~자두~체리" → 넓은 영역 → 복합 향이 자연스럽게 표현
- 같이 느낄 수 있는 향은 인접해 있으니 **한 번의 드래그로** 여러 향 선택
- 먼 영역을 칠하려면 손가락을 많이 움직여야 → 자연스럽게 비정상적 입력 방지

**시각적 결과:**

칠한 영역의 모양이 **이 와인의 향 프로필**이 됨.
기록 상세에서 "이 와인은 블랙베리-자두 계열 (어두운 자주)" 같은 요약이 자동 생성.

#### 향 색상 → 사분면 점 색상

팔레트에서 칠한 영역의 **가중 평균 색상** = 사분면 점 색상

```
예시:
- 블랙베리만 칠함 → 점 색상: 어두운 자주 (#3d0a2e)
- 블랙베리+자두 넓게 칠함 → 중간색: (#540f2f)
- 라임+레몬 칠함 → 녹-노랑 중간 (#6ab030)
```

### 10-3. 복합성 (Complexity)

> 향의 다양성 = 와인의 깊이. 1차/2차/3차 향을 얼마나 느꼈는지.

| 레벨 | 이름 | 향의 종류 | 유래 |
|------|------|----------|------|
| **1차** | 과일·꽃 | 과일, 꽃, 허브 | 포도 자체 |
| **2차** | 효모·버터 | 빵, 버터, 비스킷, 크림 | 양조 과정 |
| **3차** | 바닐라·가죽 | 바닐라, 가죽, 흙, 담배, 시가 | 숙성 |

**UI:** 토글 스위치 3개. ON/OFF.

| 복합성 | 만족도 기여 |
|--------|-----------|
| 1차만 | 기본 (보정 없음) |
| 1차 + 2차 | +5~10점 가산 |
| 1차 + 2차 + 3차 | +10~20점 가산 |

### 10-4. 여운 & 균형

| 항목 | UI | 범위 | 만족도 기여 |
|------|-----|------|-----------|
| **여운 (Finish)** | 수평 슬라이더: 짧음 ↔ 긺 | 0~100% | 여운% × 0.1 |
| **균형 (Balance)** | 수평 슬라이더: 불균형 ↔ 조화 | 0~100% | 균형% × 0.15 |

### 10-5. 만족도 자동 산출

> Step 3,4를 입력하면 만족도 시작점이 자동 계산되고, 사용자가 미세 조정.

**산출 공식 (초안):**

```
기본점수 = 50 (중립)
복합성 보너스 = [0 | +7 | +15]
여운 보너스 = 여운% × 0.1
균형 보너스 = 균형% × 0.15
──────────────────────
자동 산출 = clamp(기본 + 복합성 + 여운 + 균형, 1, 100)
```

**예시:**
- 복합성 2차(+7) + 여운 72%(+7.2) + 균형 85%(+12.8) = 50 + 27 = **77점 시작**
- 사용자가 위아래 드래그로 조정 가능

### 10-6. 사분면에 쌓인 와인 기록 — 향 색상 시각화

기록이 쌓이면 사분면 위에 **다양한 색상의 점**들이 분포:

```
              Full Body
                ↑
   ◉(어두운자주)  |   ◉(자주적갈)
   91점          |   88점
                |
산미↓ ───────────┼──────── 산미↑
                |
   ·(핑크)       |   ●(진한녹색)
   65점          |   78점
                ↓
              Light Body
```

**한눈에 보이는 것:**
- "나는 어두운 자주(블랙베리) 점이 많네" → 진한 과일향 선호
- "큰 점이 우상단에 몰려있네" → 산미 높고 풀바디인 와인 좋아해
- "녹색 점(시트러스)은 작아" → 화이트/시트러스는 별로

### 10-7. 향 팔레트 애니메이션 & 햅틱

| 이벤트 | 애니메이션 | 햅틱 |
|--------|----------|------|
| 팔레트 진입 | 원형 팔레트 중앙에서 확대 등장 (0.3s spring) | - |
| 영역 드래그 | 손가락 지나간 곳에 반투명 하이라이트 칠해짐. 색상 실시간 반영 | 향 계열 경계 지날 때 가벼운 딸깍 |
| 영역 확정 | 칠한 영역 가장자리 부드럽게 정리(smooth). 대표 향 라벨 표시 | medium tap |
| 복합성 토글 | ON 시 점의 광택 증가 (1차:매트 → 2차:새틴 → 3차:글로시) | light tap |
| 여운/균형 슬라이더 | 드래그 시 점 크기 실시간 미세 변화 (만족도 예측) | 25% 단위 딸깍 |
| 만족도 자동 산출 | 숫자 카운트업 애니메이션 (0.5s) | success |
| shake to undo | 팔레트 칠한 영역 페이드아웃 | error notification |
