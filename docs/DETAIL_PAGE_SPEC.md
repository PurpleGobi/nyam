> 작성일: 2026-03-19
> 대상: 식당 상세 (`/restaurants/[id]`) + 와인 상세 (`/wines/[id]`)
> 참조: PRD v2.0 (섹션 4-4, F10), DESIGN_SPEC v3.0 (섹션 5-17)

## 1. PRD v2 요구사항 정리

### 1-1. 식당 상세 페이지 (PRD v2 §4-4)

| 영역 | 내용 | Phase |
|------|------|-------|
| **기본 정보** | 가게명, 주소, 장르, 영업시간, 가격대, 대표 사진 — 카카오/네이버/구글 교차 취합 | P1 |
| **외부 평가** | 미슐랭/블루리본 선정 여부, 네이버/다음/구글맵 평점 (각각 표시) | P1 |
| **나의 기록** | 방문 횟수, 날짜별 기록, 평가 점수, 한줄평, 메뉴별 별표, 사용팁 등 | P1 |
| **재방문 기록** | 방문마다 새 기록 추가 가능. 상황/경험/메뉴가 매번 다를 수 있음 | P1 |
| **버블 리뷰** | 내가 속한 버블에서 이 식당에 대한 다른 멤버들의 리뷰 | P2 |

### 1-2. 와인 상세 페이지 (PRD v2 §4-4)

| 영역 | 내용 | Phase |
|------|------|-------|
| **기본 정보** | 와인명, 품종, 산지, 빈티지, 타입(레드/화이트/스파클링 등), 알코올 도수 | P1 |
| **나의 기록** | 마신 날짜, 장소(식당과 연결 가능), 평가, 페어링 메모, 한줄평 | P1 |
| **재음용 기록** | 같은 와인도 빈티지/상황에 따라 경험이 다름. 매번 새 기록 | P1 |

### 1-3. 현재 DESIGN_SPEC 상태

현재 DESIGN_SPEC v3의 5-17 식당 상세는 **기본 정보 + 메뉴판 + 관련 기록** 3섹션만 정의되어 있으며, PRD v2에서 요구하는 외부 평가, 재방문 기록 구조, 버블 리뷰 등이 반영되지 않은 상태. 와인 상세 페이지는 **미정의**.

## 2. 레퍼런스 앱 분석

### 2-1. Letterboxd (영화 → 가장 직접적 레퍼런스)

PRD v2에서 "레터박스의 영화 상세 페이지와 같은 구조"를 명시. 핵심 패턴:

**영화 상세 페이지 구조:**
```
① 히어로 (포스터 + 백드롭 이미지)
② 제목 + 연도 + 감독 + 러닝타임
③ 커뮤니티 평균 별점 + 평가 분포 히스토그램
④ 내 평가/로그/위시리스트 버튼 (CTA 바)
⑤ 시놉시스 (접이식)
⑥ 캐스트 & 크루 (가로 스크롤 아바타)
⑦ 친구들의 활동 (로그/리뷰)
⑧ 인기 리뷰
⑨ 관련 리스트
```

**Nyam에 적용할 핵심 인사이트:**
- **"친구들의 활동"이 전체 리뷰보다 먼저** → 버블 기록 > 공개 기록 순서
- **내 평가 CTA가 항상 접근 가능** → 기록 남기기 버튼 고정
- **평가 분포 히스토그램** → 점수 분포 시각화
- **리스트 연결** → Phase 2 큐레이션 리스트 (F24) 연결 포인트

### 2-2. Vivino (와인)

**와인 상세 페이지 구조:**
```
① 와인 라벨 이미지 + 와인명/생산자 (배경색 = 와인 타입별)
② 평균 평점(4.2) + 평가 수 (큰 숫자)
③ Action 버튼 (위시, 리뷰, 공유)
④ Taste 프로필 (Light—Bold, Smooth—Tannic 등 수평 슬라이더)
⑤ 음식 페어링 (아이콘 + 텍스트)
⑥ 가격 비교 / 구매 CTA
⑦ 커뮤니티 리뷰
⑧ 와인 Facts (포도 품종, 산지, 알코올 등 메타데이터)
```

**Vivino의 UX 문제점 (Nyam이 개선해야 할 것):**
- Taste 프로필과 음식 페어링이 **분리**되어 있어 사용자가 연관성을 놓침
- Action 메뉴의 시인성이 낮음 (작고 회색)
- 가치 있는 Taste 리뷰가 일반 Reviews 섹션 **밖에** 배치되어 놓침
- 구매 CTA에 과도한 비중 → Nyam은 거래 플랫폼이 아니므로 해당 없음

**Nyam에 적용할 핵심 인사이트:**
- **와인 타입별 배경색 변화** → Wine Purple 계열 활용
- **Taste 프로필 + 음식 페어링 연속 배치** (Vivino의 분리 문제 해결)
- **라벨 이미지의 세로 배치** → 와인 병의 세로 특성에 맞는 레이아웃

### 2-3. Google Maps (식당)

**식당 상세 페이지 구조:**
```
① 사진 캐러셀 (카테고리별: 음식/내부/외관)
② 식당명 + 카테고리 + 별점 + 리뷰 수 + 가격대
③ Action 칩 (길찾기 / 저장 / 공유 / 전화)
④ 영업 정보 (현재 상태, 시간표, 혼잡도)
⑤ 주소 + 지도 미니맵
⑥ 인기 메뉴 (사진 + 언급 횟수)
⑦ 리뷰 하이라이트 (AI 요약)
⑧ 전체 리뷰
```

**2025년 리디자인 트렌드:**
- 수평 구분선 → **카드형 컨테이너**(약간 어두운 배경)로 전환
- 정보 그룹핑 강화
- 지도가 배경에 살짝 보이는 레이어 구조

**Nyam에 적용할 핵심 인사이트:**
- **외부 평점 각각 표시** (PRD v2 요구사항과 일치)
- **영업 상태 실시간 강조** (영업 중 = 초록, 영업 종료 = 회색)
- **인기 메뉴 = 기록에서 자주 언급된 메뉴 자동 추출**로 전환 가능
- 사진 카테고리 분류는 Phase 2+

### 2-4. Untappd (맥주 → 음료 기록)

**맥주 상세 페이지 구조:**
```
① 맥주 라벨 + 이름 + 브루어리
② 평균 점수 + 체크인(기록) 수
③ 스타일 태그 + ABV + IBU
④ Taste 프로필 (레이더/바)
⑤ 내 체크인 이력 (타임라인)
⑥ 친구 체크인
⑦ 배지/뱃지
```

**Nyam에 적용할 핵심 인사이트:**
- **내 체크인(기록) 이력을 타임라인으로** → 재방문/재음용 기록 시각화
- **스타일 태그 + 수치 데이터(ABV/IBU) 조합** → 와인의 품종/산지/알코올 조합에 적용
- **배지 시스템과 상세페이지 연동** → 경험치/레벨 표시 (PRD v2 §8)

### 2-5. Yelp (식당 리뷰)

**핵심 패턴:**
- 메뉴 항목별 개별 리뷰 → Nyam의 "메뉴별 별표" 기능과 유사
- 리뷰 필터 (최신/높은점수/낮은점수/사진있음)
- "유용한 리뷰" 투표 → 버블 내 좋아요와 유사

## 3. 공통 디자인 패턴 추출

### 3-1. 상세페이지 정보 위계 (8개 레이어)

모든 레퍼런스 앱에서 공통적으로 나타나는 정보 배치 순서:

```
Layer 1: 비주얼 히어로 — 첫인상, 감성적 진입
Layer 2: 아이덴티티 — 이름 + 핵심 메타정보
Layer 3: 평가 요약 — 한눈에 보이는 점수/등급
Layer 4: 액션 — CTA 버튼 (기록/저장/공유/전화)
Layer 5: 특성 데이터 — 사분면 분포, 향 팔레트, 페어링
Layer 6: 실용 정보 — 위치/영업/메뉴/가격
Layer 7: 소셜 — 신뢰순: 버블 → 팔로우 → 전체
Layer 8: 메타데이터 — 세부 사실, 부가 정보
```

### 3-2. 식당 vs 와인 차이점

| 요소 | 식당 | 와인 |
|------|------|------|
| 히어로 레이아웃 | 사진 캐러셀 (가로, 여러 장) | 라벨 이미지 (세로, 단일) + 텍스트 좌우 배치 |
| 핵심 메타 | 장르, 지역, 가격대 | 품종, 산지, 빈티지, 타입 |
| 외부 평가 | 미슐랭/블루리본 + 네이버/다음/구글 평점 | 해당 없음 (Nyam 자체 데이터만) |
| 사분면 | X=가격(저렴↔고가) × Y=분위기(캐주얼↔포멀), 점 크기=맛 만족도 | X=산미(낮음↔높음) × Y=바디(가벼움↔무거움), 점 크기=만족도, 점 색상=향 |
| 실용 정보 | 주소/지도/전화/영업시간/메뉴판 | 알코올도수/생산자/음식 페어링/적정 온도 |
| 연결 | 이 식당에서 마신 와인 | 이 와인을 마신 식당 |
| 재방문 구조 | 방문마다 상황(혼밥/데이트/회식)이 다름 | 빈티지/상황에 따라 경험이 다름 |
| 컬러 코드 | Coral Orange (`#FF6038`) | Plum Purple (`#6B4C8A`) |

### 4-2. 섹션별 상세 스펙

#### Layer 1: 히어로 — 사진 캐러셀

- DESIGN_SPEC 4-5 PhotoCarousel 컴포넌트 재사용
- 높이: `h-56` (224px)
- 사진 없을 때: `bg-neutral-100` + 식당 장르 아이콘 (UtensilsCrossed)
- 뒤로가기/하트/더보기 버튼: 사진 위 투명 오버레이
- 하트: 찜(Wishlist) 토글

#### Layer 2: 아이덴티티

```
식당명: H2 (1.563rem, font-semibold, neutral-800)
장르 · 지역: Small (0.875rem, neutral-500)
가격대: Small (0.875rem, neutral-400) — ₩ ~ ₩₩₩₩ 표시
```

- 장르는 PRD v2의 상황 태그와 별도 — 음식 카테고리 (한식/일식/양식/중식 등)
- 지역은 "구 + 동" 수준 (예: "종로구 광화문")

#### Layer 3: 평가 요약

**Nyam 점수 (좌측)**
```
큰 숫자: Display (2rem) font-bold primary-500
부제: Caption "14명이 기록" neutral-400
```
- 모든 기록의 총점 평균
- 기록이 1개일 때도 표시 (내 점수만)
- 기록 0개: 이 영역 숨김

**외부 평점 (우측)**
```
N 4.2 — 네이버 (초록)
K 4.0 — 카카오 (노랑)
G 4.3 — 구글 (파랑)
```
- 각 플랫폼 로고 대신 알파벳 약자 + 색상으로 구분
- 데이터 없는 플랫폼은 숨김
- Caption 사이즈 (0.75rem)

**권위 뱃지 (하단)**
- 미슐랭: Star 아이콘 + "미슐랭 1스타" (있을 경우만)
- 블루리본: Ribbon 아이콘 + "블루리본 선정" (있을 경우만)
- `rounded-lg bg-amber-50 text-amber-700 px-2 py-0.5`

#### Layer 4: 액션 바

```
[기록 남기기] — Primary 버튼 (Coral Orange, flex-1)
[찜]         — Ghost 아이콘 버튼 (Heart, 토글)
[공유]       — Ghost 아이콘 버튼 (Share2)
```

- 기록 남기기: 탭 → 기록 생성 플로우 진입 (식당 자동 선택 상태)
- 이미 기록한 경우: "재방문 기록" 텍스트 변경
- 찜: Wishlist 토글 (하트 fill/outline)

#### Layer 5: 나의 기록

**상태 A — 기록 있음:**
- 헤더: "나의 기록" + 방문 N회 · 최근 YYYY.MM.DD
- 기록 타임라인 (세로, 최신순)
- 각 기록 카드:
  ```
  ● 날짜 — 상황 태그 (칩)
    총점 (Bold, primary) · 한줄평 (Small, neutral-600)
    [사진 썸네일 h-12 w-12 가로 스크롤] (있을 때)
    메뉴별 별표: "A세트 ★" (있을 때)
  ```
- 기록 카드 탭 → 기록 상세 페이지 이동
- 타임라인 왼쪽에 세로 라인 + 도트 (journey 스타일)

**상태 B — 기록 없음:**
- 빈 상태 카드
- "아직 방문 기록이 없어요"
- [첫 기록 남기기] Primary CTA

**상태 C — 찜만 한 상태:**
- "찜한 식당이에요. 방문하셨나요?"
- [기록 남기기] Primary CTA

#### Layer 6: 사분면 분포

- 기록 2개 이상일 때만 표시 (데이터 충분해야 의미 있음)
- **사분면 차트**: 이 식당에 대한 모든 기록을 2D 좌표 위에 점으로 표시
  - X축: 가격 (저렴 0% ↔ 고가 100%)
  - Y축: 분위기 (캐주얼 0% ↔ 포멀 100%)
  - 점 크기: 맛 만족도 (1~100)
  - 점 색상/테두리: 상황 태그 색상 (혼밥=블루, 데이트=핑크, 회식=퍼플 등)
  - 내 기록: 불투명, 타인 기록: 반투명
- **점 위치 = 이 식당의 성격**: 점들이 모여있는 위치가 "이 식당이 어떤 곳인가"를 말해줌
  - 점들이 우상단에 모임 → 고가+포멀 식당
  - 점들이 좌하단에 모임 → 저렴+캐주얼 식당
  - 점 크기가 전반적으로 크면 → 맛 만족도가 높은 곳
- **인기 메뉴**: 기록에서 태그된 추천 메뉴 빈도 순 표시
  - "A세트 ★4" = 4명이 추천

#### Layer 7: 실용 정보

- **주소**: MapPin 아이콘 + 주소 텍스트 + 미니 지도 (h-32, rounded-xl)
  - 미니 지도 탭 → 외부 지도 앱 열기 (카카오맵/네이버맵 선택)
- **영업시간**: Clock 아이콘 + 현재 영업 상태
  - 영업 중: `text-green-600 font-semibold` "영업 중 · 22:00에 마감"
  - 영업 종료: `text-neutral-400` "영업 종료 · 내일 11:00 오픈"
  - 탭 → 요일별 시간표 확장
- **전화**: Phone 아이콘 + 번호 + [전화하기] 버튼 → `tel:` 링크
- **메뉴판**: 접이식 (Collapsible)
  - 헤더: "메뉴" + ChevronDown 아이콘
  - 내용: 메뉴명 + 가격 리스트
  - 데이터 없으면 이 항목 숨김

#### Layer 8: 식당-와인 연결 (NEW)

- 이 식당에서 와인을 기록한 이력이 있을 때만 표시
- 가로 스크롤 카드: 와인 라벨 썸네일 + 와인명 + 내 점수
- 탭 → 해당 와인 상세 페이지

#### Layer 9: 소셜 — 버블 기록 (P2)

- Phase 1에서는 숨김
- Phase 2에서 활성화:
  - 버블별 기록 그룹핑
  - 각 기록: 아바타 + 닉네임 + 버블명 뱃지 + 총점 + 한줄평 + 상황 태그
  - 리뷰에 좋아요/댓글 가능

### 5-2. 식당 상세와의 차이점

| 요소 | 식당 | 와인 |
|------|------|------|
| 히어로 | 사진 캐러셀 (전폭, 가로) | 라벨 이미지 + 텍스트 (좌우 배치) |
| 컬러 톤 | Coral Orange 강조 | Plum Purple 강조 (`#6B4C8A`) |
| 배경 hint | 기본 (white) | Wine Light (`#F8F5FA`) 은은한 배경 |
| 외부 평가 | 미슐랭/블루리본 + 3사 평점 | 없음 |
| 사분면 | X=가격, Y=분위기, 크기=맛, 색=상황 | X=산미, Y=바디, 크기=만족도, 색=향 팔레트 |
| 실용 정보 | 주소/지도/전화/영업시간/메뉴판 | 품종/산지/알코올/온도 (테이블) |
| 연결 | 여기서 마신 와인 | 이 와인을 마신 식당 |
| 내 기록 | 방문 N회 · 상황별 | 음용 N회 · 식당 연결 |

### 5-3. 와인 전용 디자인 토큰

```css
/* 와인 상세 페이지 진입 시 적용 */
--wine-primary: #6B4C8A;        /* Plum Purple */
--wine-primary-light: #F8F5FA;  /* Wine Light (배경) */
--wine-tag-bg: #EDE6F5;         /* 태그 배경 */
--wine-accent: #8B5CF6;         /* 보라 계열 accent */
```

- 만족도 숫자: `text-[#6B4C8A]` (Coral Orange 대신 Plum Purple)
- 사분면 점: 향 팔레트 색상 (블랙베리=어두운 자주, 라임=진한 녹색 등)
- 타입 칩 배경: 와인 타입별 미세 변화
  - 레드: `bg-red-50 text-red-700`
  - 화이트: `bg-amber-50 text-amber-700`
  - 로제: `bg-pink-50 text-pink-700`
  - 스파클링: `bg-sky-50 text-sky-700`

## 6. 공통 인터랙션 패턴

### 6-1. 스크롤 시 헤더 전환

```
스크롤 < 히어로 높이: 투명 헤더 (아이콘만)
스크롤 ≥ 히어로 높이: 고정 헤더 (bg-background/95 backdrop-blur)
                      + 식당명/와인명 표시
```

### 6-2. 기록 남기기 CTA 고정

- 스크롤이 Layer 4(액션 바)를 넘어가면, "기록 남기기" 버튼을 **하단 고정** (sticky bottom)
- 하단 내비 위에 floating
- 이미 기록한 경우: "재방문 기록" / "재음용 기록"

### 6-3. 섹션 접이식 (Collapsible)

- 메뉴판, 와인 정보, 영업시간 상세: 기본 접힌 상태
- 헤더 탭 → 확장 (ChevronDown → ChevronUp 전환)
- 애니메이션: `normal` (200ms ease-in-out)

### 6-4. 빈 상태 패턴

| 섹션 | 빈 상태 메시지 | CTA |
|------|---------------|-----|
| 나의 기록 | "아직 방문 기록이 없어요" | [첫 기록 남기기] |
| 사분면 분포 | (섹션 자체 숨김) | — |
| 버블 기록 | "아직 버블 멤버의 기록이 없어요" | [버블 초대하기] |
| 사진 | 장르 아이콘 + neutral-100 배경 | — |
| 식당-와인 연결 | (섹션 자체 숨김) | — |

### 6-5. 로딩 상태

- 전체 페이지: 스켈레톤 (히어로 + 텍스트 블록 3개 + 카드 2개)
- 외부 평점: 개별 로딩 (각 플랫폼 API 응답 시간 다를 수 있음)
- 지도: `bg-neutral-100` placeholder → 로드 완료 시 전환

## 9. 핵심 디자인 원칙 (상세페이지)

1. **내 기록이 주인공** — 외부 정보보다 나의 경험이 먼저 보인다. 식당/와인 DB 정보는 맥락을 제공하는 배경이다.

2. **신뢰 순서** — 버블 기록 > 팔로우 기록 > 공개 기록. Letterboxd의 "친구 활동 우선" 패턴.

3. **사분면이 식당/와인의 성격을 말해준다** — 점들의 분포 위치와 크기를 보면 "이 식당이 어떤 곳인가", "이 와인이 어떤 스타일인가"를 한눈에 알 수 있다. 식당은 점 색상=상황 태그, 와인은 점 색상=향 팔레트.

4. **기록 CTA는 항상 접근 가능** — 스크롤 어디에서든 "기록 남기기"가 보인다 (sticky/floating).

5. **빈 상태도 가치 있게** — 기록이 없어도 외부 정보(영업시간, 위치, 기본 정보)는 즉시 유용. 기록 유도 CTA와 함께.

6. **재방문은 깊이** — 같은 식당/와인이라도 매 경험은 다름. 타임라인 구조로 경험의 층위를 보여준다.

7. **연결이 가치** — 식당↔와인, 기록↔상황, 사람↔버블. 데이터 간 연결이 Nyam만의 차별점.
